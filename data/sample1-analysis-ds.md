# 程序基本功能

该程序是一个简单的堆管理器，允许用户通过菜单交互进行内存操作。主要功能包括：
- **malloc**（选项1）：分配指定大小的内存块（大小限制在128-256字节），并允许用户输入内容。分配后还会打印内存地址作为“gift”。
- **free**（选项2）：释放指定索引的内存块。
- **edit**（选项3）：编辑指定索引的内存块内容，但最多只能使用两次（受`key1`全局变量限制）。
- **show**（选项4）：显示指定索引的内存块内容，但需要管理员权限（`key2`全局变量非零），默认可能无法使用。

主要函数作用：
- `main`：程序入口，循环处理用户菜单选择，调用相应功能函数。
- `menu`：打印菜单选项。
- `read_int`：从标准输入读取整数。
- `ma`：处理内存分配逻辑。
- `fr`：处理内存释放逻辑。
- `ed`：处理内存编辑逻辑。
- `sh`：处理内存显示逻辑。

# 关键结构体

程序中没有显式定义结构体，但使用了以下关键全局数据结构：
- `heap`：一个全局指针数组（类型为`_QWORD*`），用于存储分配的内存块地址。索引范围0-0x20（即最多32个块）。
- `len`：一个全局整数数组，用于存储每个内存块的大小，与`heap`数组索引对应。
- `key1`：全局整数，用于限制`edit`操作的次数（初始值未显示，但当`key1 == 2`时，`edit`会退出程序）。
- `key2`：全局整数，用于控制`show`操作的访问权限（非零时允许显示）。

# 漏洞成因

漏洞位于`ed`函数（编辑功能）中的以下代码行：
```c
v2[read(0, v2, (int)len[v1])] = 0;
```
**漏洞原因**：  
- `read`函数返回实际读取的字节数，然后该值被直接用作索引在`v2`（指向内存块的指针）处写入一个零字节（null终止符）。  
- 如果用户输入的数据长度恰好等于`len[v1]`（即内存块的大小），则写入操作将发生在`v2[len[v1]]`位置，这超出了分配的内存块边界（因为有效索引范围为`0`到`len[v1]-1`），导致**堆缓冲区溢出（off-by-one错误）**。  
- 这种溢出可能覆盖相邻堆块的关键元数据（如size字段或指针），从而可能被利用来实现任意内存读写、代码执行或权限提升。此外，结合堆分配策略，攻击者可能通过精心构造的输入实现堆布局操纵，进一步利用如Use-After-Free（UAF）或双重释放等漏洞。

**补充说明**：  
- 程序在`fr`函数中释放内存后会将指针置零，并重置长度，这减少了直接UAF的风险，但通过off-by-one溢出可能破坏堆结构，间接导致UAF。  
- `key1`限制编辑次数为两次，这可能增加利用难度，但off-by-one漏洞在第一次编辑时即可触发。
