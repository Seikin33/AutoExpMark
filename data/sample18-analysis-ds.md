# 程序基本功能

这是一个模拟步枪电子商务系统的程序，主要功能包括添加步枪信息、显示已添加的步枪、提交订单、添加订单留言、查看系统状态等。

主要函数作用：
- `sub_80485EC`: 去除字符串末尾的换行符
- `sub_8048644`: 添加新的步枪信息（名称和描述）
- `sub_8048729`: 显示所有已添加的步枪信息
- `sub_80487B4`: 输入订单留言
- `sub_8048810`: 提交订单并释放所有步枪内存
- `sub_8048896`: 读取用户输入的数字选项
- `sub_8048906`: 显示系统状态统计
- `sub_804898D`: 主菜单循环，处理用户选择
- `main`: 程序入口，初始化全局变量并启动菜单

# 关键结构体

程序使用链表结构来存储步枪信息，每个节点结构大致如下：
```c
struct rifle_node {
    char description[56];      // 步枪描述
    char name[56];            // 步枪名称（实际从偏移25开始）
    struct rifle_node *next;  // 指向下一个节点（在偏移52字节处）
};
```

全局变量：
- `dword_804A288`: 链表头指针
- `dword_804A2A4`: 新建步枪计数
- `dword_804A2A0`: 订单提交计数
- `dword_804A2A8`: 订单留言缓冲区指针

# 漏洞成因

**漏洞位置**: `sub_8048644` 函数中的堆缓冲区溢出漏洞

**漏洞代码**:
```c
fgets(dword_804A288 + 25, 56, stdin);  // 写入步枪名称
fgets(dword_804A288, 56, stdin);       // 写入步枪描述
```

**漏洞原因**:
1. 每个步枪节点只分配了 `0x38` (56) 字节的内存：
   ```c
   dword_804A288 = (char *)malloc(0x38u);
   ```

2. 但是数据写入存在重叠和溢出：
   - 第一行 `fgets` 从偏移25开始写入56字节名称，这会导致写入到分配的内存块之外
   - 第二行 `fgets` 从偏移0开始写入56字节描述
   - 两个写入操作总共可能写入超过分配大小的数据

3. 具体计算：
   - 分配大小：56字节
   - 名称写入：从偏移25开始写56字节 → 需要81字节空间
   - 描述写入：从偏移0开始写56字节
   - 实际需要空间：max(81, 56) = 81字节，但只分配了56字节

这导致堆缓冲区溢出，可能破坏堆内存结构，造成内存破坏，可能被利用来执行任意代码。
