# 程序基本功能

这是一个管理女孩信息的交互式程序。用户可以通过菜单进行添加、显示、编辑和呼叫女孩信息等操作。程序使用一个全局数组来存储女孩信息的指针，每个女孩信息包括动态分配的名字字符串和固定的呼叫字符串。编辑功能实际上并未实现，只是提示用户删除并添加新信息。呼叫功能会随机决定是否成功，但在此过程中会释放女孩名字的内存，导致潜在漏洞。

主要函数作用：
- `sub_ADD()`: 显示菜单选项。
- `sub_B49()`: 添加女孩信息，包括分配内存存储名字和呼叫信息。
- `sub_CE9()`: 伪编辑功能，仅打印消息。
- `sub_CFC()`: 显示指定索引的女孩信息。
- `sub_DD6()`: 呼叫女孩功能，释放名字内存并随机返回结果。
- `sub_F08()`: 主循环函数，处理用户输入并调用相应功能。
- `main()`: 程序入口，调用初始化函数和主循环。

# 关键结构体

程序使用一个全局数组 `unk_202060`（类型为 `girl_info*[100]`）存储最多100个女孩信息的指针。每个女孩信息结构体如下（根据内存布局推断）：

```c
struct girl_info {
    char *name;       // 指向动态分配的名字字符串（偏移0）
    int name_size;    // 名字字符串的大小（偏移8）
    char call[12];    // 呼叫字符串，最多11字符加空终止（偏移12）
};
```

全局变量 `dword_20204C` 记录当前添加的女孩数量。

# 漏洞成因

1. **Use-After-Free (UAF) 漏洞**：
   - 位置：在函数 `sub_DD6()`（对应菜单选项4）中，代码 `free(**((void ***)&unk_202060 + v2));` 释放了女孩名字指针，但未将结构体中的指针置空。
   - 原因：释放后，指针仍保留在全局数组中。后续在 `sub_CFC()`（菜单选项2）中显示信息时，通过 `puts(**((const char ***)&unk_202060 + v1));` 使用已释放的指针，导致 Use-After-Free。攻击者可通过控制释放后的内存内容实现信息泄漏或代码执行。

2. **数组越界读漏洞**：
   - 位置：在函数 `sub_CFC()`（对应菜单选项2）中，用户提供的索引 `v1` 通过 `__isoc99_scanf("%d", &v1);` 读取，但未检查范围。
   - 原因：直接使用 `v1` 访问全局数组 `unk_202060`，如果 `v1` 为负数或大于等于100，会访问数组之外的内存地址，导致信息泄漏或程序崩溃。
