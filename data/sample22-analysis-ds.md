# 程序基本功能

这是一个名为"tinypad"的简易记事本程序，具有以下主要功能：

1. **显示功能**：显示欢迎界面和4个记事本条目
2. **添加笔记(A)**：用户可以添加新笔记，最大大小为256字节
3. **删除笔记(D)**：删除指定索引的笔记
4. **编辑笔记(E)**：编辑已存在的笔记内容
5. **退出程序(Q)**：退出程序

主要函数作用：
- `main`：主程序循环，处理用户命令
- `read_n`/`write_n`：基础的读写函数
- `read_until`：读取输入直到遇到特定分隔符
- `writeln`/`writerrln`：带换行的输出函数
- `read_int`：读取整数输入

# 关键结构体

程序使用一个全局数组`tinypad`来管理笔记，结构如下：

```c
// 每个笔记条目占用16字节
struct note_entry {
    long size;        // 偏移240+16*i: 分配的大小
    char *content;    // 偏移248+16*i: 指向实际内容的指针
};

// tinypad内存布局：
// - 前256字节(0-255): 编辑缓冲区
// - 后64字节(256-319): 4个note_entry结构(每个16字节)
```

# 漏洞成因

**漏洞位置**：在编辑功能(E命令)中存在**堆溢出漏洞**

**漏洞代码**：
```c
// 在编辑功能中
strcpy(tinypad, *(const char **)&tinypad[16 * v20 + 248]);  // 复制原内容到缓冲区
// ...
v14 = strlen(*(const char **)&tinypad[16 * v20 + 248]);    // 获取原内容长度
read_until(tinypad, v14, 10);                             // 读取新内容
// ...
strcpy(*(char **)&tinypad[16 * v20 + 248], tinypad);       // 复制回堆中
```

**漏洞原因**：

1. **长度计算问题**：程序使用`strlen`获取原内容长度作为`read_until`的读取限制，但`strlen`遇到空字节就停止，如果原内容中间有空字节，计算的长度会小于实际分配的大小。

2. **堆块边界跨越**：当原内容中间有空字节时，`v14`（strlen结果）会小于实际分配的堆块大小。但在调用`read_until`时，可以读取到原分配大小的数据，导致写入超过原字符串长度的数据。

3. **无边界检查**：在将编辑后的内容复制回堆时，使用`strcpy`而没有检查目标缓冲区大小，如果编辑后的内容比原内容长，就会发生堆溢出。

**利用场景**：
- 用户可以创建一个包含空字节的笔记
- 编辑时输入超过原字符串长度（但不超过分配大小）的数据
- `strcpy`将超长数据复制回堆中，覆盖相邻堆块元数据

这是一个典型的堆溢出漏洞，攻击者可以通过精心构造的输入覆盖堆管理结构，最终可能实现代码执行。
