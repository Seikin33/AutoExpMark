# 程序基本功能

该程序是一个简单的写作管理应用，允许用户管理多个页面（page）并设置姓名和消息。主要功能包括：

- **init()**: 初始化程序，设置输入缓冲区和输出提示信息。
- **init_name_message()**: 初始化用户姓名和消息，读取姓名（最多0x20字节）并为消息分配0xB0字节内存。
- **read_int()**: 从输入读取字符串并转换为整数。
- **menu()**: 显示菜单选项，包括添加页面、编辑页面、删除页面、更改姓名、更改消息和完成操作。
- **add()**: 添加一个页面，要求页面大小在128到239字节之间，分配内存并存储页面指针和大小。
- **edit()**: 编辑指定页面的内容，读取新内容到已分配的内存中。
- **delete()**: 删除指定页面，释放页面内存并重置指针和大小。
- **change_name()**: 更改用户姓名，读取新姓名到全局变量`name`。
- **change_message()**: 更改消息，释放旧消息内存，分配新内存，但存在漏洞（见下文）。
- **main()**: 主循环，处理用户选择并调用相应函数。

# 关键结构体

程序中没有明确定义结构体，但使用了以下全局变量和数组：

- `page_list`: 一个指针数组（可能大小为8），存储每个页面的内存地址。
- `size_list`: 一个整数数组（可能大小为8），存储每个页面的大小。
- `name`: 全局字符数组，存储用户姓名，大小为0x20字节。
- `message`: 全局指针，指向动态分配的消息缓冲区。

# 漏洞成因

漏洞位于`change_message()`函数中，具体代码段如下：

```c
void *change_message()
{
  // ...
  printf("Your message is saved at %p\n", message);
  free(message);  // 释放旧消息内存
  // ...
  buf = malloc(v1); // 分配新内存
  // ...
  read(0, message, 0x10u); // 漏洞：在释放后使用message指针
  // ...
  message = buf; // 更新message指向新缓冲区
}
```

**漏洞原因**：  
在释放`message`指向的内存后，程序直接使用`read(0, message, 0x10u)`向已释放的内存写入0x10字节数据。这构成了**使用后释放（Use-After-Free, UAF）**漏洞。由于`message`指针在释放后未被立即置空，攻击者可以通过写入数据破坏堆内存布局，可能导致堆元数据损坏或控制相邻内存块，进而实现任意代码执行。

**利用场景**：  
攻击者可以结合堆分配机制，通过多次操作（如添加、删除页面）来重用已释放的内存，从而控制程序流。例如，在释放`message`后，如果其他分配操作重用相同内存，写入操作可能覆盖关键数据或函数指针。
