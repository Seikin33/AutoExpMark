# 程序基本功能

这是一个简单的物品管理程序（xxxx），用户可以通过菜单进行以下操作：

1. **show_item** - 显示盒子中的所有物品
2. **add_item** - 添加新物品（包括设置物品名称长度和名称）
3. **change_item** - 修改已有物品的信息
4. **remove_item** - 删除物品
5. **exit** - 退出程序

程序开始时会在堆上分配一个16字节的内存块，存储两个函数指针（hello_message和goodbye_message）。

# 关键结构体

程序使用两个全局数组来管理物品：
- `unk_6020C8`：存储物品名称指针的数组（每个元素8字节）
- `itemlist`：存储物品名称长度的数组（每个元素4字节）

每个物品占用两个数组中的相同索引位置，最多支持100个物品。

# 漏洞成因

**漏洞位置**：`change_item()`函数中的堆缓冲区溢出漏洞

**漏洞代码**：
```c
printf("Please enter the length of item name:");
read(0, nptr, 8u);
v2 = atoi(nptr);
printf("Please enter the new name of the item:");
*(_BYTE *)(*((_QWORD *)&unk_6020C8 + 2 * v1) + (int)read(0, *((void **)&unk_6020C8 + 2 * v1), v2)) = 0;
```

**漏洞原因**：
1. 在修改物品时，程序允许用户重新指定物品名称的长度（v2）
2. 但是程序**没有验证新长度是否超过原始分配的内存大小**
3. 直接使用用户控制的长度v2来读取数据到原始缓冲区中
4. 如果用户输入的长度大于原始malloc分配的大小，就会导致堆缓冲区溢出

**利用后果**：
- 可能覆盖相邻的堆块元数据
- 可能破坏堆管理结构
- 可能导致任意代码执行（通过覆盖函数指针等）

这个漏洞是典型的堆缓冲区溢出，攻击者可以通过精心构造的输入来利用此漏洞实现代码执行。
