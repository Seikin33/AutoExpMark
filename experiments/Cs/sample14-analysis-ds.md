# 程序基本功能

该程序是一个简单的堆内存管理程序，允许用户通过菜单选项分配、释放和写入内存块。程序启动时显示一个横幅，并进入一个交互式循环，等待用户输入选择。主要功能包括：
- **选项1（sub_400916）**：分配一个内存块。用户指定大小（1-120字节），程序将分配内存并将指针存储在全局数组`buf`中。最多允许分配4个块（索引0-3）。
- **选项2（sub_4009D7）**：释放一个内存块。用户指定索引，程序释放对应指针的内存。
- **选项3（sub_400A4D）**：向一个内存块写入数据。用户指定索引，程序从标准输入读取8字节数据写入该指针指向的内存。
- **选项4（sub_400896）**：检查全局变量`qword_602090`，如果为0则执行`system("/bin/sh")`启动shell，否则打印"Not yet"。
- **选项5**：退出程序。

主要函数作用：
- `main`：程序入口，调用初始化函数和主循环函数。
- `sub_4008BF`：初始化函数，设置超时（32秒）和标准输入/输出缓冲，打印横幅。
- `sub_400ACD`：主循环函数，处理用户输入并调用相应功能。
- `sub_400916`：内存分配函数。
- `sub_4009D7`：内存释放函数。
- `sub_400A4D`：内存写入函数。
- `sub_400896`：shell启动函数。

# 关键结构体

程序中没有显式定义结构体，但使用以下关键全局变量：
- `dword_6020BC`（类型：`int`）：用于跟踪当前已分配的内存块数量，也作为分配时的索引。初始值可能为0，最大值为3（允许最多4个块）。
- `buf`（类型：`void*`数组）：一个全局指针数组，存储分配的内存块指针。数组大小基于`dword_6020BC`，最多4个元素。
- `qword_602090`（类型：`unsigned __int64`）：一个控制变量，决定是否启动shell。如果为0，则启动shell。

# 漏洞成因

程序存在多个内存破坏型漏洞，主要源于内存管理操作中的安全检查缺失：

1. **Use-After-Free (UAF) 漏洞**：
   - 位置：`sub_4009D7`（释放函数）和`sub_400A4D`（写入函数）。
   - 成因：在`sub_4009D7`中释放内存块后，没有将`buf`数组中的对应指针设置为`NULL`。随后在`sub_400A4D`中，允许用户向已释放的内存块写入数据（通过索引访问）。这导致UAF漏洞，攻击者可以操作已释放的内存，可能破坏堆结构或执行任意代码。

2. **双重释放 (Double-Free) 漏洞**：
   - 位置：`sub_4009D7`（释放函数）。
   - 成因：释放内存时没有检查该索引对应的指针是否已被释放。用户可以通过多次选择选项2并指定同一索引，导致同一内存块被多次释放，引发双重释放漏洞。这可能造成堆管理器崩溃或允许任意内存写入。

3. **索引越界漏洞**：
   - 位置：`sub_4009D7`和`sub_400A4D`中读取用户输入索引的部分。
   - 成因：程序使用`atoi(s)`将用户输入转换为索引，但没有验证索引是否在有效范围内（0-3）或是否已分配。如果用户提供无效索引（如负数或大于3），可能导致越界访问`buf`数组，造成内存损坏。

这些漏洞可被利用来劫持程序控制流，例如通过UAF覆盖函数指针或堆元数据，最终触发`sub_400896`中的shell获取系统权限。
