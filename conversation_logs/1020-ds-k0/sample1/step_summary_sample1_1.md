### 2. 关键内存变化发现

#### `len`数组状态 (`0x602060`):
```
索引0: 0xf8 (248字节)
索引1: 0xf8 (248字节) 
索引31: 0xf8 (248字节)
索引32: 越界写入0xf8到0x6020e0
```

**关键发现**: exp代码中`malloc(32,0xf8,'bbbb')`使用了索引32，这超出了`len`数组的边界（0-31），导致**数组越界写入**，将值`0xf8`写入了`heap`数组的起始位置。

#### `heap`数组状态 (`0x6020e0`):
```
索引0: 0xf8 (被len数组越界写入覆盖，原应指向0x21aaf010)
索引1: 0x21aaf210 (指向索引1的堆块)
索引31: 0x21aaf310 (指向索引31的堆块)
```

**关键发现**: 由于`len`数组的越界写入，`heap[0]`被错误地设置为`0xf8`而不是正确的堆块地址`0x21aaf010`。这破坏了索引0的内存块管理。

#### `pro`数组状态 (`0x6021e0`):
```
索引0: 0x21aaf110 (指向索引32的堆块)
```

**关键发现**: `pro`数组存储了索引32的堆块地址，这表明程序可能使用`pro`数组来处理超出正常索引范围的分配。
