## 内存状态总结

### 1. 堆布局（Heap Layout）
```
0x25c21000: 初始堆块 (0x21字节)
0x25c21020: 堆块0 (0x91字节) - 存储"aaaa"
0x25c210b0: 堆块1 (0x91字节) - 存储"bbbb"  
0x25c21140: Top Chunk
```

### 2. 关键数据结构

#### 全局数组 `itemlist` (地址: 0x6020c0)
```
0x6020c0: [0x00000080, 0x00000000, 0x25c21030, 0x00000000]  // 索引0
0x6020d0: [0x00000080, 0x00000000, 0x25c210c0, 0x00000000]  // 索引1
0x6020e0: [0x00000000, 0x00000000, 0x00000000, 0x00000000]  // 索引2-9
```

**结构说明**：
- 每16字节对应一个物品
- 前4字节：物品名称长度 (0x80 = 128字节)
- 中间4字节：填充
- 后8字节：指向堆块用户数据的指针

#### 初始堆块 (0x25c21000)
- 大小：0x21字节
- 内容：存储两个函数指针
  - `fd = 0x400896 <hello_message>`
  - `bk = 0x4008b1 <goodbye_message>`
- 状态：完好，可作为利用目标

### 3. 堆块详细信息

#### 堆块0 (0x25c21020)
- 用户数据地址：0x25c21030
- 大小：0x91 (144字节，包含16字节元数据 + 128字节用户数据)
- 用户数据：`0x61616161` ("aaaa")
- 标志：PREV_INUSE

#### 堆块1 (0x25c210b0)  
- 用户数据地址：0x25c210c0
- 大小：0x91 (144字节，包含16字节元数据 + 128字节用户数据)
- 用户数据：`0x62626262` ("bbbb")
- 标志：PREV_INUSE

### 4. 漏洞利用相关观察

**堆溢出方向**：
- 如果通过`change_item()`修改堆块0，指定长度大于0x80，可以：
  - 覆盖堆块0的元数据（大小字段、标志位）
  - 覆盖堆块1的元数据和用户数据
  - 可能覆盖Top Chunk

**潜在利用目标**：
- 初始堆块中的函数指针（`hello_message`和`goodbye_message`）
- 堆块元数据（伪造chunk、house of force等）
- GOT表（如果程序有Partial RELRO）

**内存保护状态**：
- No PIE：代码段固定地址0x400000
- Partial RELRO：GOT表可写
- NX enabled：堆栈不可执行
- Stack Canary：栈保护开启

当前内存状态为后续的堆溢出利用提供了清晰的基础信息，特别是函数指针的存在为代码执行提供了直接目标。
