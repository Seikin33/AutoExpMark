# 漏洞成因
## 程序关键结构体
程序使用一个动态分配的结构体来管理每本书的信息，该结构体大小为0x20字节，推测布局如下：
```c
struct book {
    int id;                 // 书ID，4字节
    char* name;             // 指向书名字符串的指针，8字节
    char* desc;             // 指向描述字符串的指针，8字节
    int desc_size;          // 描述字符串缓冲区的大小，4字节
    // 可能还有4字节的填充以对齐到0x20字节
};
```
书的名字和描述内容分别存储在独立的堆块中。
全局数组 `book_pointers` 存储所有 `book` 结构体的指针。全局变量 `author_name` 是一个32字节的缓冲区，位于bss段。

## 漏洞定位
1.  **信息泄露漏洞**：`set_author_name` 函数使用 `readline(author_name, 32)` 读取作者名。如果用户输入恰好32字节（不含换行符），`readline` 函数会在写入第32字节后，于 `author_name[32]` 位置写入一个终止符 `\x00`。这导致其溢出一个字节，覆盖了紧随 `author_name` 之后内存的第一个字节。由于 `author_name` 之后很可能存储着指向堆的指针（例如 `book_pointers` 数组的第一个元素），这会将指针的低字节覆盖为0。当程序通过 `print_book_details` 打印作者名时，由于作者名缓冲区没有在32字节处终止（被覆盖的是指针字节），`printf` 会继续打印后续内存内容，直到遇到空字节，从而泄露堆指针。
2.  **堆溢出漏洞**：`edit_book` 函数使用 `readline(desc_ptr, desc_size - 1)` 来读入新的描述。`desc_size` 是创建时分配的描述缓冲区大小。通过精心的堆布局（例如，创建两个相邻的堆块，第一个描述块较大），在编辑第一个块时，可以溢出到相邻的第二个堆块，覆盖其块头部（size和fd/bk指针），为后续的unlink攻击创造条件。

# 漏洞利用过程：
整体利用思路：首先利用作者名输入溢出泄露堆地址，计算堆基址。然后进行堆布局，创建特定大小的书以塑造堆内存布局。接着利用堆溢出伪造一个fake chunk，并通过释放相邻块触发unlink，从而篡改全局`book_pointers`数组中某个书结构体指针的值，使其指向`book_pointers`数组自身附近。之后，通过编辑该书，可以任意读写`book_pointers`数组的内容，进而将另一本书的描述指针覆盖为目标地址（如存放libc地址的堆地址或`__free_hook`），实现信息泄露和任意地址写。最后将`__free_hook`覆盖为`system`，并释放一个包含`/bin/sh`字符串的书来获取shell。

## Step 1 & 2: 泄露堆地址
- **Step 1**: 输入作者名，使用`'x'*(0x20-5) + 'leak:'`，恰好32字节。这会触发一个字节的溢出，覆盖了紧随`author_name`之后的某个堆指针的最低字节为`\x00`。
  - 全局变量`author_name`（地址`0x555555756060`）处的内容被设置为`'x'*27 + 'leak:'`。
  - 由于溢出，原本位于`0x555555756080`附近的、指向堆上书结构体的指针（例如`0x555555757140`）的最低字节被改为0，但打印时，`printf`会从`author_name`开始打印，直到遇到空字节。因为指针被部分覆盖，打印会继续将指针值作为字符串的一部分输出，从而泄露堆地址。
- **Step 2**: 调用打印书详情功能。程序打印作者名时，会输出`'x'*27leak:\x80\x71\x75\x55\x55\x55...'`，从中可以提取出堆地址`0x555555757140`。通过计算该地址与已知偏移（`0x1080`），得到堆基址`heap_base = 0x5555557560c0`。

## Step 3 & 4: 堆布局准备
- **Step 3**: 创建两本小书（书2和书3），描述大小均为0x20。这会在堆上分配两个小的fastbin块。
- **Step 4**: 删除书2和书3。这两个小描述块被释放并加入fastbin，为后续分配提供可用的空闲块。

## Step 5 & 6: 构造溢出条件与布局
- **Step 5**: 创建两本新书（书4和书5）。
  - 书4的描述大小设为`0x108`（大于fastbin），使其进入small bin或unsorted bin，且其描述块尾部将与后续的堆块相邻。
  - 书5的描述大小设为`0xf0`（`0x100 - 0x10`），使其描述块恰好位于书4的描述块之后，且大小适合fastbin。书5的描述块将成为堆溢出的目标。
- **Step 6**: 创建书6，描述大小设为`0x200`。这本书的名字设置为`/bin/sh\x00`，为最终获取shell做准备。其描述块较大，后续可用来存放伪造的数据结构或作为攻击跳板。

## Step 7: 构造Fake Chunk触发Unlink
- **Step 7**: 编辑书4的描述内容，构造一个Payload，在书4的描述块内伪造一个fake free chunk，并覆盖相邻的书5描述块的块头部。
  - 在书4的描述块（地址`0x5555557572b0`）中构造一个fake chunk。Payload为：`p64(0) + p64(0x101) + p64(ptr - 0x18) + p64(ptr - 0x10) + b'\x00'*0xe0 + p64(0x100)`。
    - `p64(0) + p64(0x101)`：设置fake chunk的prev_size和size（`0x101`，并设置`PREV_INUSE`位为1）。
    - `p64(ptr - 0x18) + p64(ptr - 0x10)`：设置fake chunk的`fd`和`bk`指针。其中`ptr = heap_base + 0x1180`，是书4的`book`结构体指针在全局数组`book_pointers`中的地址。
    - `b'\x00'*0xe0`：填充。
    - `p64(0x100)`：覆盖到相邻的书5描述块的`prev_size`字段，使其与fake chunk的size一致。
  - 此Payload覆盖了书5描述块（地址`0x5555557573c0`）的块头部，将其`size`的最低字节修改（例如从`0xf1`变为`0x100`），并清除了`PREV_INUSE`位，同时`prev_size`被设为`0x100`。这使glibc认为书5描述块的前一个chunk（即我们伪造的chunk）是free状态。

## Step 8: 触发Unlink
- **Step 8**: 删除书5。由于书5描述块的`PREV_INUSE`位为0，glibc会尝试向前合并，触发unlink操作，将我们伪造的chunk从它的binlist中摘除。
  - Unlink宏执行：`FD = P->fd; BK = P->bk; FD->bk = BK; BK->fd = FD;`。
  - 此处`P`是我们伪造的chunk，其`fd = ptr - 0x18`, `bk = ptr - 0x10`。
  - 因此，`FD->bk`即`*(ptr - 0x18 + 0x18) = *ptr`（全局数组中书4的指针）被写入`BK`（即`ptr - 0x10`）。
  - `BK->fd`即`*(ptr - 0x10 + 0x10) = *ptr`也被写入`FD`（即`ptr - 0x18`）。
  - 最终结果是，全局数组`book_pointers`中原本指向书4结构体的指针（位于`ptr`）被修改为`ptr - 0x18`。现在，书4的指针指向了全局数组`book_pointers`内部的一个位置（`book_pointers`数组之前0x18字节处）。

## Step 9 & 10: 建立任意读写原语并泄露Libc地址
- **Step 9**: 编辑书4。由于书4的指针现在指向全局数组内部，编辑书4实际上是修改`book_pointers`数组的内容。通过发送特定数据，我们可以将`book_pointers`数组中书6的描述指针覆盖为我们想要读取的地址（例如堆中一个包含libc地址的位置）。
  - 发送的数据：`p64(0x30) + p64(4) + p64(heap_base + 0x11a0) + p64(heap_base + 0x10c0)`。
    - 这覆盖了`book_pointers`数组从某个偏移开始的内容，将书6的描述指针（`book.desc`）修改为`heap_base + 0x11e0`（通过计算`heap_base + 0x11a0`和`heap_base + 0x10c0`的组合影响），该地址预计存放着libc的main_arena指针。
- **Step 10**: 使用`read_at`功能（内部调用打印书详情）读取书6的描述，实际上是读取`heap_base + 0x11e0`地址处的值，这是一个libc中的地址（如`__malloc_hook`附近的地址）。计算后得到libc基址。

## Step 11 & 12: 劫持控制流获取Shell
- **Step 11**: 利用任意写原语（通过`write_to`函数，其内部编辑书4以修改书6的描述指针，然后编辑书6的描述内容），将`__free_hook`的值覆盖为`system`函数的地址。
- **Step 12**: 删除书6。程序会调用`free(book6->desc)`，由于`__free_hook`已被覆盖为`system`，且`book6->name`指向字符串`/bin/sh`，实际上会执行`system("/bin/sh")`，从而获得shell。
